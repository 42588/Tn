# CLAUDE.md — Tn terminal (project guide for Claude Code)

Tn is a **Windows-first, Rust, GPU-accelerated terminal for vibe coding**: hosts AI
coding CLIs (Claude Code, Codex) with great UX, flexible tiling, WSL + SSH. Stack:
**GPUI** (Zed's framework, DirectX 11 + DirectWrite on Windows) · **alacritty_terminal**
(VT engine) · **portable-pty** (ConPTY/WSL) + **russh** (SSH). License: GPL-3.0-or-later.

## Workspace (crates)
```
tn-core    alacritty wrapper: Term + VTE parser + TerminalSnapshot + Palette/RGB resolve + row_runs. HEADLESS.
tn-pty     PtyBackend trait + LocalPty (ConPTY). HEADLESS. (WSL/SSH = M2)
tn-config  config + theming. STUB — M1.3 fills it.
tn-ui      GPUI front-end (ONLY gpui-linked lib): terminal_view.rs (pane), workspace.rs (tabs + n-ary panes).
tn-app     the `tn` binary.
tn-cli     headless ConPTY smoke harness.
```
Rule: `gpui` only in `tn-ui`/`tn-app`. `tn-core`/`tn-pty`/`tn-config` must build & test headless. Don't leak alacritty types past `tn-core` (e.g. `CellRun` exposes bold/italic bools, not `Flags`).

## Build / run / test
`cargo` is at `%USERPROFILE%\.cargo\bin\cargo.exe` (NOT on the bash PATH — use full path in PowerShell, or it's on PATH in fresh shells).
```powershell

cargo test  -p tn-core                       # engine unit tests (5)
cargo run   -p tn-cli                         # ConPTY smoke: spawn shell, render grid to stdout, PASS/FAIL

$env:TN_AUTOQUIT="1"; cargo run -p tn-app     # headless self-test: first pane runs a cmd, dumps grid, quits (exit 0)
```

- **M1.5 repaint push+vsync** (`crates/tn-ui/src/terminal_view.rs`): replaced the 8ms `dirty` poll with a **push** model — the reader thread sends a wake down a `futures::channel::mpsc::unbounded` (coalesced by the `dirty` AtomicBool so ≤1 wake is ever queued); a foreground `cx.spawn` task awaits it and calls `cx.notify()`, which GPUI folds onto its vsync frame clock. Idle terminal = zero wakeups (no poll). **DEC 2026 synchronized output is already handled by the alacritty `vte` `Processor` (`StdSyncHandler`)** — the grid only mutates on BSU→ESU completion or the handler's timeout, so every `snapshot()` is a whole frame (no partial-update tearing); no extra code needed in tn-ui. *Edge: the sync timeout is only re-checked on the next `advance`, so a stalled mid-sync stream defers display — acceptable; real apps send ESU promptly.*

## Gotchas (hard-won)
- **ConPTY**: must route alacritty `Event::PtyWrite` replies back to the PTY writer (answers the startup `ESC[6n` DSR) or the child stalls (only 4 bytes read). ConPTY gives no reliable EOF — poll `try_wait`, not `read()==0`. Keep the portable-pty `SlavePty` alive for the session.
- **gpui 0.2.2 on Windows**: builds fine (DX11+DirectWrite, no Vulkan); first build is several minutes. Runtime `HRESULT(0x887A002D)` is just the optional DXGI *debug* layer missing — harmless.
- Debug builds keep a console window; release hides it (`#![cfg_attr(not(debug_assertions), windows_subsystem = "windows")]` in tn-app).
- **GPUI window class on Windows = `Zed::Window`** (not titled "Tn"); for screenshot/key-injection tooling, enumerate top-level windows by class, not `FindWindow(title)`.
- **Workspace shortcuts use `Ctrl+Shift+*`.** On Chinese / multi-layout Windows the system "switch keyboard layout" hotkey is also `Ctrl+Shift`, which can swallow these before they reach the app. Verified (via synthesized SendInput) that the bindings + action dispatch are correct — every action fires. Fix is environmental: disable Windows' Ctrl+Shift layout-switch hotkey (Settings → Time & Language → Typing → Advanced keyboard settings → Input language hot keys → set "Between input languages"/layout switch to *Not Assigned*), or remap once config keybindings are wired to actions (M1.4 — `[[keybindings]]` is parsed by `tn-config` now but `workspace.rs` still uses hardcoded `bind_keys`). `Ctrl+Tab` (next tab) and mouse click-to-focus are unaffected.
- `gpui::Pixels.0` is private → use `f32::from(px)`. GPUI async: `cx.spawn(async move |this: WeakEntity<Self>, cx: &mut AsyncApp| ...)`, `WeakEntity::update(cx, |v, cx| ...)`, `bg_executor.timer(d).await`, `cx.quit()`.
- **Flexbox `min-size: auto` (Taffy)**: a too-tall/wide child inflates its flex item past its `relative()` share and spills out of the window — and because `terminal_view` reads its `canvas`-captured bounds to size the grid, the inflated height feeds back so the grid never shrinks. Cure: `min_w(px(0.))`/`min_h(px(0.))` + `overflow_hidden()` on every flex level (body, split container, pane wrap, terminal root). This was the "down-split overflows the window" bug.
