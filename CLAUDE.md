# CLAUDE.md - Tn 终端 Claude 项目记忆入口

## 定位

本文只作为 Claude Code 的项目记忆入口，负责给出入口层级、必读链路和长期知识索引，不承载长篇实现细节。

Tn 是一个 Windows 优先、Rust 编写、GPU 加速、为 vibe coding 打造的终端：把 Claude Code、Codex 等 AI 编码 CLI 当一等公民托管，提供 GPUI 界面、VT 终端、ConPTY/WSL/SSH 能力。许可证：GPL-3.0-or-later。

- `CLAUDE.md` 是 Claude 的项目记忆入口。
- `AGENTS.md` 是 Codex 的项目记忆入口。
- 二者同级，不互相作为父级；共享规则和项目知识写入 `docs/` 下的权威文档。

## Claude 协作规则

- 默认使用中文回答。
- 每次项目任务按“规划 -> 划分任务 -> 执行 -> 测试 -> 输出”的节奏组织。
- 每次执行会改动仓库内容、产生验证结论或改变长期文档状态的项目任务，先在根目录 `TODO.md` 新增或更新任务索引；完整目标、过程、验证和后续事项写入 `docs/任务/YYYY-MM-DD-<中文主题>.md`。
- 纯只读审查可以不登记；一旦审查结论转为修复、验证或文档状态更新，必须登记。
- `TODO.md` 只做任务看板和定位，不写长流水账。
- 不要恢复已废弃的 `.planning`。

## 新增信息落点

- 新增长期有效信息时，先按 [文档治理索引](docs/文档治理索引.md) 判断权威落点。
- 架构、crate 边界、数据流写入架构子文档；产品、UX、交互写入产品体验子文档；样式和 UI 还原写入界面样式子文档。
- bug 根因、修复和验证写入修复与优化子文档；已知限制写入已知问题子文档；跨 agent 工作规则、环境约束和踩坑规则写入共享记忆子文档。近期要执行的已知问题还要同步到 `TODO.md`。
- 母文档只保留定位、索引、摘要、状态、日期和链接；没有合适子文档时先创建中文命名子文档，再更新母文档索引。

## UI 样式开发必读链路

任何 UI / 颜值改动都先走这条链：

1. 设计真源：[design/mockup.html](design/mockup.html)；逐面板原型：[design/panels/](design/panels/)；共享样式：[design/calm-glass.css](design/calm-glass.css)。
2. 具体数值和译法：[界面样式实现规则](docs/界面样式实现规则.md)、[界面翻译规则](docs/界面样式/界面翻译规则.md) 与 [组件规格与自动生成数值](docs/界面样式/组件规格与自动生成数值.md)。
3. 写 GPUI 时优先使用 `style.rs` 令牌和 `col()` / `cola()`，不要手写主题色裸 hex。
4. 改 `calm-glass`、mockup 或主题后，用 `TN_GEN_SPEC=1 cargo test -p tn-ui spec_gen` 刷新样式规格。
5. 颜值、动画和窗口材质必须真机肉眼检查；headless 只能验证网格和结构收敛。

## 常用验证

常用构建、测试、ConPTY 烟雾和 GUI 自测命令见 [工作区与构建约定](docs/共享记忆/工作区与构建约定.md)。按任务范围选择最窄验证;文档重构任务优先跑链接检查和格式检查。

## 提交约定

- 用户明确要求不要提交时，不提交。
- 里程碑完成需要提交时，提交结尾带 `Co-Authored-By: xqq`。
- 多行提交信息用 Bash 工具的 `git commit -F -` + 单引号 heredoc。
- 改依赖版本走根 `Cargo.toml` 的 `[workspace.dependencies]`。
- `main` 上允许 WIP，里程碑完成时单次提交。
