## 新增信息落点

- 本文只新增或更新索引行:日期、主题、状态、子文档链接和 20-40 字摘要。
- 根因分析、Before/How/After、验证命令、测试名称、真机反馈、待复验项和勘误必须写入对应 `docs/修复与优化/` 子文档。
- 用户可见发布摘要写入 `CHANGELOG.md`;当前任务进度写入 `TODO.md` 和对应 `docs/任务/` 子文档。
- 仍未完成的问题、风险和退出条件写入 `docs/已知问题索引.md` 或 `docs/已知问题/`。
- 新建子文档必须使用中文命名,并按“定位 / 背景 / 具体内容 / 验证或状态 / 反向链接”组织。
