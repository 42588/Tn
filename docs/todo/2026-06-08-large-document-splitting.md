# 大型文档拆分为母文档索引 + 子文档详情

## 状态

待办。

## 目标

把大型文档拆成“母文档精准定位子文档、子文档展示具体信息”的一对多结构,降低单文件膨胀和长期记忆漂移。

## 初始拆分对象

- `CLAUDE.md`: 保留项目主入口和高频规则,把细节型工作记忆拆到 `docs/memory/` 或同类目录。
- `docs/优化日志.md`: 保留日期和主题索引,把具体优化记录拆到 `docs/优化日志/` 下的主题文件。
- `docs/未修复.md`: 保留问题索引和优先级,把具体根因、复现、限制拆到 `docs/未修复/` 下的专题文件。
- `docs/架构蓝图.md`: 保留架构导航,把 crate / 子系统细节拆到架构子文档。
- `docs/样式还原手册.md`: 保留样式入口和守卫规则,把组件级细节拆到样式子文档。

## 拆分原则

- 按主题拆,不按行数拆。
- 母文档只放摘要、状态、日期和链接。
- 子文档是具体信息的权威落点。
- 同一事实不在多个母文档重复铺开。
- 迁移时先保持语义不变,再做必要的标题和索引整理。

## 验收标准

- 每个被拆的大型母文档都能快速定位到对应子文档。
- 子文档标题能清楚说明自己负责的主题。
- 原有有效信息没有丢失。
- 新增信息规则已写入 `AGENTS.md` 和 `CLAUDE.md`。

## 后续

等待用户确认首批拆分对象和目录命名后执行。
