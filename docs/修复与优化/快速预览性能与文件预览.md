# 快速预览性能与文件预览

## 定位

本文承载 Quick Look 文件预览、异步加载、PDF/图片/Office 处理、搜索性能、缓存驱逐和相关勘误。编辑器自绘、Diff、IME 和保存守卫见 [快速预览编辑器近期变更](快速预览编辑器近期变更.md)。

## 背景

2026-06-02 至 2026-06-05 的 Quick Look 优化从“能打开更多文件”推进到“极端切换也不阻塞、不飙内存”。中途存在 Fast-Path、任务防抖和缓存策略的多轮替换,后续审查对旧记录做了勘误。

## 具体内容

### 文本识别与只读守卫

- UTF-16 文件曾因前几百字节包含 `0x00` 被误判为二进制。修复为通过 `content_inspector::inspect` 识别格式,避免 Unicode 文本误杀。
- 只读文件误显“Enter 编辑”曾允许 PDF、图片、Office 和普通二进制进入编辑模式。修复为 `is_editable()` 同时检查数据类型和扩展名黑名单,UI 提示与键盘行为保持一致。
- 审查发现截断大文本仍可编辑会导致保存时只写回前 4000 行,永久丢失尾部内容。修复为 `Text { truncated: true, .. }` 直接只读。

### 长文档、PDF 与图片

- PDF、Docx、Xlsx 早期从全量阻塞读取改为后台执行和渐进式显示。Office/长文本通过 `take(MAX_LINES)` 限制装载。
- PDF 内存泄漏路径:旧逻辑可能预渲染大量高清页并保留无压缩位图。后续改为 JPEG 压缩底包、`UniformList` 虚拟列表按需光栅化可见页,并用 `remove_asset` 主动驱逐 GPUI 纹理缓存。
- 图片预览新增 `png|jpg|jpeg|webp|bmp|gif` 路由,文件 I/O 与 `gpui::Image::from_bytes` 解码进入后台执行器。dev 构建下 `image` / `zune-jpeg` / `png` 等依赖启用 opt3,避免 debug 逐像素解析慢到卡 UI。
- 2026-06-05 修正 PDF / 图片预览空白:PDF 页去掉白边框和大段白背景,未解码占位改为暗色;图片使用 `size_full + object_fit(Contain) + 10px padding` 铺满预览区。

### 搜索和表格

- `find_in_chars` 的朴素 O(N^2) 字符匹配改为 `str::match_indices`,通过字节边界到宽字符偏移映射定位高亮。
- 审查后进一步优化 `all_matches`:同一行多命中时 byte -> char 偏移从上个匹配处续数,避免每个命中从行首重算。
- Excel / CSV 预览曾声称 two-pass 对齐但代码未落地。后续补 `align_table`:第一趟统计每列显示宽度,第二趟按列左对齐补空格;避免行尾单元格补尾随空格,覆盖 CJK 和列数不齐。

### 异步加载、取消与缓存驱逐

- Fast-Path 0 帧同步加载曾用于小文件避免 skeleton 闪烁,但阈值过高时会把图片、Office 解析重新带回 UI 线程,导致快速切文件事件堆积和假死。
- 后续 `load_task` / `diff_task` + 50ms 防抖也已被更直接的取消令牌模型覆盖。
- 当前主策略:切文件即生成新的 cancellation token,所有重解析、图片解码、PDF 渲染均在后台线程执行,耗时操作前后检查 token;过期结果直接丢弃。
- GPUI 隐式纹理缓存会保留图片/PDF 资产。Quick Look 切换、关闭或任务取消时主动调用 `remove_asset`,避免疯狂切图后内存长期高位。

### 高亮缓存与编辑态勘误

- 预览态 `file_highlight_cache` 保留:只读内容稳定,行号 key 安全。
- 编辑态 `edit_highlight_cache` 曾按行号 key 缓存且只在输入时清空;退格、删除、撤销、重做、粘贴会显示旧行内容。后续彻底移除 edit cache,编辑态可见行数量有限,直接计算更安全。
- Quick Look 关闭记录勘误:组件不是从状态树 `take()` 强制析构,而是 `close()` 清空状态并配合 `remove_asset` 达成纹理驱逐。

## 验证 / 状态

- 相关阶段曾通过 `cargo test -p tn-ui --lib` 和 `cargo test --workspace --lib`。
- `align_table` 有列宽、尾随空格、CJK、列数不齐和空输入的 headless 单测。
- 待真机关注:取消 Fast-Path 后,快速 ↑↓ 浏览小文件时 skeleton 是否可感知。当前取舍是“0ms 派发 + 可取消”优先于“完全不闪烁”。
