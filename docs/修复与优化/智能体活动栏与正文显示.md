# 智能体活动栏与正文显示

## 定位

本文承载 Agent 活动栏、shell 内启动 Codex/Claude、agent pane 正文显示、Claude 卡顿治理和默认 Agent 启动入口收敛的修复记录。

## 背景

Tn 的 agent 体验同时涉及 PTY、活动栏、Quick Look、文件树上下文和启动器配置。2026-06-03 至 2026-06-08 的问题集中在活动栏可用性、pane CWD 隔离、shell 内命令识别和 agent surface 布局。

## 具体内容

### 活动栏列表滚动与预览导航

- “本次改动”文件卡片列表无法滚动的根因是 scrollable div 没有稳定 `ElementId`,GPUI 无法跨帧维持滚动偏移。
- 修复:给 scrollable 添加 `.id("arail-scrollable")`;`rail_shell` 的 body 类型调整为 `AnyElement`,调用点 `.into_any_element()`。
- 从活动栏卡片打开 QuickLook 后,↑↓ 曾错误穿透整个 Explorer 文件树。修复为在 `TerminalView` 增加 `rail_find_idx` / `rail_nav`,Workspace 记录 `ql_rail_pane` 和 `ql_rail_idx`;QuickLook nav 根据上下文走 rail 循环导航或 explorer 导航。

### 多窗格 agent CWD

- 从欢迎页启动 agent 时,活动栏曾读取进程全局 `std::env::current_dir()` 作为根,导致多窗格不同工作区串台。
- 修复:初始化时优先读取 `TerminalLaunch` 传入的 pane 专属 `cwd`,剥离对进程全局目录的依赖。
- 后续 per-pane 工作区上下文进一步把文件树状态、打开文件夹语义和 agent pane `cd` 隔离沉入 [每窗格工作区上下文](每窗格工作区上下文.md)。

### shell 内敲 Codex / Claude 的 PATH 与正文显示

- PATH 痛点:Windows Terminal 中 `codex` / `claude` 正常,Tn 子 shell 内可能不可解析或进入空白 agent。根因是 Tn 本地 PTY 子进程 PATH 缺少当前进程动态 shim 目录,例如 `%APPDATA%\npm`。
- PATH 修复:`tn-pty::LocalPty::spawn` 在 Windows 下以 `portable-pty::CommandBuilder` 已有 PATH 为基底,去重追加当前 Tn 进程 PATH 里的缺失条目。单测覆盖大小写不敏感去重和写回。
- 正文显示痛点:普通 shell 中直接输入 `claude` 或 `codex` 后,UI 切到 agent 头和活动栏,终端正文只剩外壳。headless 验证说明 ConPTY -> alacritty -> snapshot 不是根因。
- 正文显示修复:shell-detected agent 仍走 activity rail overlay,避免命令运行中改变正文宽度;overlay 宿主补成 `flex_col`,让 `term_area.flex_1()` 在 flex 父级拿到高度。agent surface 统一拥有 viewport,隐藏 shell block bar,滚轮交给 agent/alt-screen 程序。
- 诊断:新增 `tn::agent_surface` 一次性日志,记录 shell 命令识别、rail layout、body container、canvas bounds、grid size、nonblank rows 和 block bar 可见性。

### Claude Code 卡顿与环境剥离

- Claude 双光标:由于 Claude 自带虚拟光标而 ConPTY 对隐藏物理光标的 ANSI 序列不稳定,UI 层在 Claude agent 态强制不画 GPUI 物理光标。
- 周期性输入卡顿:usage poller 曾在 GPUI 后台执行器里做同步磁盘 I/O,造成线程池饥饿。修复为同步 fs 工作进入独立 OS 线程,通过 oneshot channel 回传。
- PTY 阻塞:reader 同步写回光标查询回复曾可能持 writer 锁,用户输入也抢 writer 锁。修复为 reader replies 和用户 send_bytes 都排队到单写线程,保证 UI 解耦和写入顺序。
- 后续勘误:打字真正还会争 terminal 锁,因此 reader 每批 advance 后 yield,并配合 dev profile opt3 降低热路径成本。
- IDE 环境变量:从 VS Code 集成终端启动 Tn 时,Claude 会继承 `TERM_PROGRAM=vscode` 和 `VSCODE_*`,误判自己在 VS Code 中并反复尝试 IDE extension。修复为 `LocalPty::spawn` 移除 VS Code/Cursor 特有变量并设置 `TERM_PROGRAM=Tn`。

### 默认 Agent 启动器

- 2026-06-06 默认配置从 Claude/Codex 双磁贴收敛为通用 `Agent` profile。
- `discover_profiles` 过滤旧配置中的内置 Claude/Codex profile,没有其它 agent profile 时自动补一个通用 Agent。
- 欢迎页、命令面板和 Quick Terminal 原型统一暴露 `Agent/pwsh/WSL/SSH`。
