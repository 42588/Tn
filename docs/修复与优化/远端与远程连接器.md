# 远端与远程连接器

## 定位

本文承载 Quick Terminal SSH 入口、SSH 最近连接管理、WSL/SSH CWD 同步、Remote root 修复和远端文件树相关记录。

## 背景

2026-06-05 的远端工作围绕两个方向展开:让 Quick Terminal 的 SSH 卡片真正接入连接器,以及让左侧文件浏览器能和 WSL/SSH 终端 pane 的当前目录保持一致。后续审查修正了 Remote root 伪装本机路径带来的空文件树、rebuild 竞态和 cd 误投 agent。

## 具体内容

### Quick Terminal SSH 入口

- 痛点:Quick Terminal 启动器已经显示 SSH 卡片,但 `PickerItem::SshPrompt` 仍是 no-op,点击后不会打开 SSH 输入、最近连接或 ssh-config 逻辑。
- 修复:点击 SSH 卡片后切到 `user@host[:port]` 输入面板,合并展示 `[[profiles]] kind="ssh"`、`ssh_recents.json` 最近连接和 `~/.ssh/config` Host alias。
- 启动:Enter 或点击行构造 `LaunchSpec { ssh: Some(SshConfig) }`,复用 `TerminalView` / `SshBackend` 启动远程会话。
- Quick Terminal 订阅 `SshConnected`、`SshRetryRequested`、`SshCloseRequested`、`SshRememberPassword`,补齐最近连接写入、错误重试、关闭和本次会话记住密码。

### 收藏、重命名与输入解析

- 最近 SSH 行补星标按钮和铅笔重命名按钮,直接更新 `ssh_recents.json` 的 `favorite` / `name` 字段。
- 地址输入时禁用 IME,避免中文输入法介入 `user@host[:port]`;进入重命名草稿时启用 IME,并通过 `EntityInputHandler` 承接 marked / commit 文本。
- Quick Terminal 输入区与主窗口连接器对齐:抽出 `workspace::parse_ssh_target_chips`,两处共用 user / host / port / error chip 解析。quick 输入行改为终端字体渲染文本,caret 紧跟真实输入,右侧显示错误或字段 chip。

### WSL / SSH CWD 双向同步

- 痛点:进入 WSL 或 SSH 后,左侧文件浏览器不跟随终端 pane 的 CWD;在左侧选择新文件夹时,WSL/SSH 终端也不会自动 `cd`。
- 路径命名空间:在 `ExplorerRoot` 划分 `Host`、`Wsl`、`Ssh`,通过 `path_for_namespace` 进行路径翻译。
- 状态发布订阅:在 `TerminalView` 实现 `CwdChanged` 事件,io 重绘循环对比 `cwd()` 与 `last_cwd`,改变时 emit;Workspace 订阅后触发 `follow_root`。
- 命令分发:Workspace 组装对应环境的 `cd` 命令,本地 cmd 用 `cd /d`,WSL/SSH 用 `cd`。
- SSH shell hook:连接成功后注入静默脚本,追加 bash `PROMPT_COMMAND` 和 zsh `precmd_functions`,让提示符呈现时输出 `OSC 633;P;Cwd=` 序列。

### Remote root 审查修复

- Remote root 清空 Explorer:早期把 SSH/macOS 远端 cwd 包成 `ExplorerRoot::Remote`,但 Explorer rebuild 只能枚举本机 `PathBuf`,结果 rows 被清空。修复为不把远端路径伪装成本机 path;后续通过 `ExplorerRoot::Ssh` + `tn-pty::remote_fs` 接回远端 FS。
- rebuild 竞态:后台扫描旧 root 完成后可能覆盖新 root。修复为 `rebuild_rev` 递增,前台写回时校验 rev 和当前 root path。
- cd 误投 agent:`cd_panes_to_root` 曾对所有 pane 发 `cd`,包括 Claude/Codex agent。修复为跳过 `spec.agent.is_some()`;Host shell 还要用 `is_host_process_path` 校验,避免把 `\\wsl$` 等跨命名空间路径塞给本地 shell。
- WSL `/root` 看似空白:Explorer 曾无条件过滤 dotfiles。修复为 Host root 继续隐藏 dotfiles,WSL root 显示 dotfiles,但仍过滤 `.git`、`target`、`node_modules` 等噪声。
- 边界:Host -> WSL 翻译只支持盘符路径,不再把普通 UNC 网络路径误翻成 `//server/share`;WSL header 显示为 `Distro:/`;Explorer 空状态区分加载中、读取失败、Host 无可见项和 WSL 真空目录。

## 验证 / 状态

- Quick Terminal SSH 入口、最近连接管理和输入解析已与主窗口连接器对齐。
- 远端文件树首版依赖 `tn-pty::remote_fs`;远端 Quick Look 有界预览与文本 guarded write-back 已记录在相关架构/模块文档中。
- SSH / WSL 实机场景需持续覆盖路径翻译、cwd hook、远端 rebuild 和 `cd` 分发边界。
