# 终端滚动与窗口尺寸

## 定位

本文承载终端窗口尺寸、scrollback、鼠标命中、PTY 读写锁争用、IME 残留和后台 scrollback swap 的根因、修复和验证记录。

## 背景

2026-06-02 至 2026-06-08 的终端问题集中在三类路径:隐藏或恢复帧产生坏 bounds,滚动历史与 agent/TUI 滚轮语义混淆,以及大吞吐或 Claude Code 高频重绘导致锁争用和卡顿。

## 具体内容

### 瞬态 bounds 与 1x1 resize

- 痛点:标签切换、最小化或窗口恢复时,GPUI canvas 可能回报很小但非零的临时 bounds。旧逻辑把不足一个单元格的区域兜成 `1x1`,并写给 alacritty 和 ConPTY,导致光标落到 row 0,随后 Enter 在顶部覆盖历史。
- 修复:抽出 `fit_grid_size_from_bounds`,只有 bounds、cell width、line height 有限且扣除 `BODY_PAD` 后能容纳完整单元格和行时才返回 `GridSize`;否则跳过本帧 resize,保留上一帧尺寸。
- 验证:`terminal_fit_ignores_tiny_hidden_bounds` 锁定隐藏帧不能触发 `1x1` PTY resize。

### scrollback 与 live cursor

- 痛点:滚动查看终端历史时仍显示 live cursor;滚到历史顶部后按键可能因 `display_offset == history_size` 被误判为已经在底部而不回到底部。
- 修复:`Terminal::snapshot()` 在 `display_offset > 0` 时隐藏 live cursor;输入前回到底部的判断改为 `display_offset > 0`。
- 验证:`scrolled_snapshot_hides_live_cursor` 与 `input_scrolls_to_bottom_for_any_scrollback_offset` 覆盖历史视图隐藏光标和任意 offset 输入回底部。

### agent / alt-screen 滚轮语义

- 痛点:Codex 等 agent 是类 TUI 但不一定进入 alt-screen。旧逻辑把它当普通 shell 滚 Tn scrollback,会翻出 Codex 启动或重绘旧主屏帧。
- 修复:agent pane 与 alt-screen pane 的滚轮统一交给程序自身,普通 shell 继续滚终端历史。ConPTY resize 增加 `ResizeAnchoring`,普通 shell 顶锚定防丢历史,agent / alt-screen 底锚定避免把当前 TUI 屏推进 scrollback。
- 验证:`agent_and_alt_screen_scroll_wheel_drives_terminal_app` 覆盖滚轮分发。

### 恢复帧鼠标命中与拖选粘连

- 痛点:恢复帧可能派发 `(0,0)` 或正文 padding 命中。旧 `cell_at` 会 clamp 到 `(0,0)` 或最后一格;若切换期间吞掉 `mouse_up`,旧 selecting 状态会在下一次 move 继续更新历史选区。
- 修复:`cell_at_from_bounds` 只接受 `BODY_PAD` 内侧真实网格矩形,padding、右下空白和恢复帧坏坐标返回 `None`;新增 `selection_drag_move` 状态机,在 `on_mouse_move` 检 `pressed_button != Left` 兜底结束拖选。
- 验证:`mouse_hit_test_rejects_body_padding_not_grid_cells`、`mouse_hit_test_rejects_space_beyond_rendered_grid`、`selection_drag_ends_when_mouse_up_was_lost`。

### PTY 读取、写入与锁争用

- 大吞吐路径:PTY reader 使用复用的 16KiB 缓冲,并在读满或过半时 yield / sleep,减少前台渲染线程抢不到锁导致的滚动撕裂。
- Claude 卡顿路径:41 曾把 reader 回复和用户输入改走 mpsc 单写线程,解决 writer 锁卡 UI 和写入乱序风险;43 进一步指出打字真正争的是 terminal 锁,reader 在 Claude 高频小批量输出下需要每批 `advance` 后 `yield_now()`。
- 结果:配合 dev profile opt3 降低 `advance` 持锁时间,UI 在 agent 高频输出时更容易抢到锁。

### IME 残留与光标形态

- 早期修复支持 Block、Underline、Beam 三种终端光标形态,并处理 IME 退格吞噬。
- 后续勘误指出“偶尔删不掉的符号”更像 `ime_marked` preedit 残留,不是写入乱序。修复为命名键或带修饰键进入 `on_key` 时,若发现残留 preedit 就清掉并 notify;退格优先清残留并消费,失焦时也清理。
- Claude 态强制隐藏物理光标时不再 spawn 无效光标动画。

### Disk-backed scrollback swapping

- 初始目标:后台或闲置终端把大 scrollback grid 序列化到临时文件,降低多标签高 scrollback limit 的常驻内存。
- 确证 bug:旧 `swap_out` 先设置 `swapped_snapshot = Some(snapshot)` 再 `File::create?` / `serialize_into?`;写盘失败会留下 snapshot 但无 swapped path,导致后续 snapshot 永久返回静态帧,restore 又无文件可恢复。
- 修复:快照先放局部变量,只有序列化成功后才提交 snapshot、换空 grid 和设置 path;错误路径零副作用。
- 同步序列化卡顿修复:新增 `swap_out_async`,持锁只 clone grid,序列化和写盘交给独立 OS 线程。`try_finish_swap` 仅当 clone 后无新输出时换空,否则丢弃过时文件并保留 live grid。

## 验证 / 状态

- 已补同步 round-trip、swapped 静态帧、异步 round-trip、stale 丢弃等 headless 单测。
- restore 失败语义保留为合理降级:文件读不回时降级为空 grid 但保持可交互,避免静态帧卡死。
- 远端 agent / SSH 场景仍需随真实使用持续复验滚轮、resize 与 scrollback 行为。
