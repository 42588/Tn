# 快速预览编辑器近期变更

## 定位

本文记录 Tn Editor / Quick Look 自绘编辑器在 2026-06 的近期变更、修复、性能守卫和待验事项。发布摘要见 [CHANGELOG.md](../../CHANGELOG.md)。

## 背景

本轮围绕 Quick Look 编辑器从 `uniform_list` 逐步迁移到自绘 File/Edit/Diff 路径,同时把纯文本编辑核心下沉到 `tn-editor` headless crate。CHANGELOG 只保留面向发布的摘要,实现细节沉到本文。

## 具体内容

### headless 编辑核心

- 新增 `tn-editor` workspace crate,无 GPUI 依赖。
- 迁移 Quick Look 的 `char_to_byte`、`line_chars`、`insert`、`newline`、`backspace`、`delete`、`move`、`page`、`delete-range`、`insert-multiline` 等纯文本编辑函数。
- 新增 `Document`、`Selection`、`CursorSet`、`EditTransaction`、`UndoStack`、`SearchState` headless 模型和独立单测。
- `tn-ui::quick_look` 编辑态通过 `QuickLookEditState` 薄壳把打字、选区、剪切/粘贴、查找/替换、撤销/重做和保存接入 `Document` 主状态。
- 旧 `uniform_list` renderer 暂继续读取适配后的 `Rc<Vec<String>>` 快照,为 LineLayout / EditorElement 上移打基础,用户可见行为保持不变。

### 自绘 File / Edit 路径

- `TN_QL_ELEMENT=1` 时启用自绘只读 File 预览:新增 `paint_file_preview` 与 `QuickLook::file_element`。
- File 只读预览改走 GPUI `canvas` 自绘,包含行号右对齐、语法着色文本和横滚 thumb。
- 使用 `editor::{geometry,prepaint}` 模型按 1/2 列网格逐段 `shape_line` + `paint`,ASCII 连排,CJK 单字 2 列步进,固定单元格避免漂移。
- 纵向滚动由 `el_scroll_y` 驱动并 clamp;横向复用 `hscroll_px`;文本通过 `with_content_mask` 裁到 gutter 右侧。
- 自绘 File 预览新增只读选区、复制、CJK 命中、Shift+滚轮 / 触控板横向滚动和横滚条 thumb 拖拽。
- `file_element` 鼠标 down/move/up 通过 y 反推行号,通过 `caret_col_at_x` / `hover_char_at_x` 计算列,并处理 CJK 双宽与 `pressed_button` 兜底。
- `Ctrl+C` / `Ctrl+A` 复用既有 `place_cursor`、`copy`、`select_all`。
- 自绘 File 预览/编辑器已成为默认路径;`TN_QL_LEGACY=1` 强制回退旧 `uniform_list`,Diff 仍保留旧路作为逃生口。
- TnE-11 让 `file_element` 同时承载预览 + 编辑;编辑态行源走 `edit` 镜像,避免每帧深拷。
- `paint_file_preview` 加 `editing` / `caret` / `ime`:绘制瞬时反相块 caret、选区底色、IME preedit 和 accent 下划线。
- editing 时 `window.handle_input` 注册输入处理器,支持中文合成和 WM_CHAR。
- 新增 `el_follow_caret`,仅在 cursor 变化时跟随并做纵横滚动去抖。

### 查找与 IME parity

- `find_next` 在 `el_render` 下走 `el_center_row`,命中行滚到视口纵向居中,并 pin `last_follow_cursor`,避免 render 顶部的 `el_follow_caret` 把它弹回边缘。
- `file_element` 实时计算 `all_matches(query)`,`paint_file_preview` 逐行绘制全部命中底色。
- 命中高亮从 `cola(accent_alt,0.20)` 纯填充升级为 `gpui::quad`,使用更醒目的填充、描边和圆角。
- `find_next` 新增 `el_reveal_col`,命中列超出视口时将 `hscroll_px` 横向居中到该列。
- 查找框中文输入修复:查找打开时注册 IME handler;`find_key` 只接管 Esc/Enter/Tab/Backspace,可打印键放行给 IME handler。
- `replace_text_in_range` 在 `find_open` 时把提交/合成文本路由进 `find_query` / `replace_query`。
- 查找条 active 字段回显 `ime_marked` preedit,正文不重复画。
- IME 候选框定位修复:新增 `find_field_bounds`,find_bar 激活字段输入框挂占位 `canvas` 每帧写入窗口坐标;`bounds_for_range` 在 `find_open` 时把候选框对齐到输入框左下缘。

### Diff / LineLayout / 配置模型

- 新增 `tn-ui::editor::diff` 装饰/导航模型。
- `DiffRowKind{HunkHeader|Addition|Deletion|Context|Meta}` 提供 `gutter()` / `is_content()`。
- `classify_diff_line` 区分 `+++` / `---` 文件头、增删内容、git meta、mode、rename、二进制和 no-newline。
- 新增 hunk 跳转 `hunk_header_rows`、`next_hunk`、`prev_hunk`。
- 只读 Diff renderer 接入计划为按 `DiffRowKind` 上色并复用 prepaint;本轮不做 accept/reject,真机对照留待后续。
- 新增 `tn-config [editor]` 段与 motion 降级策略。
- `EditorAnimations{Off|Subtle|Full}` 默认 `subtle`。
- `Editor::effective_motion(reduced_motion, high_load)` 在 `off`、OS 减少动态效果或高渲染负载时降级为 `Instant`,否则透传 `subtle` / `full`。
- 默认 `config/config.toml` 加 `[editor] animations` 段。
- 新增 `tn-editor::line_layout` logical -> visual 映射。
- `WrapMode{None | Word{width_cols}}`、`VisualLine`、`LineLayout::build` 支持词换行、硬断、超宽单字独占、CJK=2 列和空行保一条。
- 提供 `logical_to_visual`、`visual_to_logical`、`range_segments`、`hit_test`。
- 本轮不接 GPUI renderer,不改代码文件默认横滚;软换行接入 File/Edit 是后续 TnE-16。

### Prepaint 与几何模型

- 新增 `tn-ui::editor::prepaint` 只读渲染模型,产出 `EditorElement::prepaint` 所需布局纯函数。
- 关键函数包括 `visible_row_indices`、`row_top`、`gutter_label`、`content_origin_x`、`prepaint_readonly`。
- `ReadOnlyPrepaint` 包含 `content_w`、`max_off`、`h_offset`、`rows`、`thumb`、`content_x`。
- 新增 `tn-ui::editor` 几何模块骨架 `crates/tn-ui/src/editor/`。
- 把 Quick Look `uniform_list` renderer 内联几何复刻成纯函数和数据结构:`Metrics`、`disp_width` / `prefix_cols`、`caret_x`、`content_width`、`max_h_offset`、`hover_char_at_x` / `caret_col_at_x`、`visible_rows` / `row_out_of_view`、`follow_h_offset`、`h_scroll_thumb` / `h_offset_from_drag`、`caret_abs_x`。
- 几何模块为后续 `EditorElement` 的 layout/prepaint 提供 Quick Look / Editor Pane / Diff Review 共享可测模型。
- 几何模块不接入任何 render 路径,保留 `#![allow(dead_code)]` 脚手架;Quick Look 默认渲染与旧 `uniform_list` 不变。

## 修复记录

- 退出编辑回预览显示旧内容:预览从 `file_data` 渲染,编辑只改 `Document` / 编辑缓冲,escape 退出编辑时没有把缓冲镜像回 `file_data`。新增 `QuickLook::sync_preview_from_edit`,dirty 时把编辑缓冲写回 `file_data` 并清高亮缓存,不写盘、不动 dirty。
- OS 关窗绕过 dirty-close 守卫:标题栏关闭 / Alt+F4 曾绕过 `request_quit`,有未保存 Quick Look 编辑时会静默丢失。在 workspace reveal 块注册 `window.on_window_should_close`,Quick Look 打开时返回 `quick_look.request_quit()`。
- 旧编辑 renderer 下线推迟到 TnE-13:旧 `uniform_list` 路径仍被 Diff tab 共用,且是 `TN_QL_LEGACY=1` 紧急逃生口,暂不删除。

## 性能守卫

- 复核并钉死编辑核心增量机制:`tn-editor::Document` 的 undo/redo 使用 `EditTransaction` + 行区间 `EditSnapshot`,只拷受影响行,不存整 buffer。
- 连续打字按 `start_row` coalesce 成一条仍行有界的记录,移动光标即断开。
- Quick Look 薄壳 `sync_lines` 仅按 `last_transaction` 的行区间 `splice` 镜像,不再每键 `to_vec()` 整 buffer。
- 新增 `continuous_typing_keeps_undo_records_line_bounded`:4000 行连打 500 字 + 换行,undo 栈合并为 1 条且 before/after 行数 < 8,移动光标后第二条记录仍有界。
- 与既有 `undo_history_does_not_store_full_buffer_for_single_line_edit`、`tn-ui::edit_state_updates_line_mirror_without_replacing_whole_buffer` 合围守卫。
