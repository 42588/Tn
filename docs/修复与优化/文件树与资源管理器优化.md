# 文件树与资源管理器优化

## 定位

本文承载 Explorer / 文件树资源管理器的异步 rebuild、热重载、git 状态刷新、噪声过滤、菜单焦点和相关审查勘误。per-pane 工作区上下文见 [每窗格工作区上下文](每窗格工作区上下文.md)。

## 背景

早期文件树依赖同步扫描和其它组件的文件事件,在大目录、构建产物风暴或多 pane 场景下容易卡 UI 或状态串台。后续优化把 Explorer 调整为独立、后台化和按 pane 隔离的组件。

## 具体内容

### 异步 rebuild

- 痛点:点击展开 Explorer 目录时,旧 `ExplorerView::rebuild` 同步执行 `std::fs::read_dir`,点开 `node_modules` 或慢速挂载盘会阻塞主渲染线程。
- 修复:将深度遍历 `walk` 从 `&self` 抽离为无状态独立函数;`rebuild` 触发时使用 `cx.background_executor().spawn()` 把目录 I/O 放入后台;扫描完成后通过 `cx.update` 回前台更新 `rows`。
- 生命周期闭包签名按 GPUI 0.2 约束修正为引用借用型。

### 文件树热重载

- 痛点:左侧 Explorer 没有独立 watcher,曾依赖右侧 TerminalView 的文件变动监听,导致普通终端命令或外部编辑器改工作区后文件树不刷新。
- 修复:在 `ExplorerView` 注入独立 `notify::RecommendedWatcher`,监听当前 root。事件回调先过滤 `target`、`.git`、`node_modules` 等噪声,再通过异步通道和防抖触发后台 rebuild。
- 后续审查把防抖改为 trailing-edge,长构建期间持续事件会推后 rebuild,停下后只刷一次。

### git 状态后台化与共享噪声过滤

- 审查发现 `explorer.rs::start_git_refresh` 曾在 GPUI 前台 executor 里同步调用 `compute_git_status`;内部 `gitutil::capture_bounded` 可能阻塞到 git 返回,大仓库或 `.git` 锁住时会卡 UI。
- 修复:git status 刷新改为 background executor + 独立 OS 线程 + oneshot 回传,前台只 await。
- `is_noise_path` 从 io / explorer 的重复内联实现抽到共享 `gitutil::is_noise_path`,避免过滤规则漂移。

### 折叠状态与 root 切换

- 文件树折叠状态曾随 CWD 变化长期残留,占内存并拖慢重绘。修复为重新构建可见目录树或刷新 CWD 时,只保留当前 root 和有效展开链路内的状态。
- 异步 rebuild 竞态在远端专题中也出现:后台扫描捕获旧 root 后完成,如果不校验版本就会覆盖新 root。后续通过 `rebuild_rev` 和 root path 双重校验丢弃过时结果。

### 菜单焦点与打开文件夹

- 痛点:从主窗口左上角下拉菜单点击“打开文件夹”偶尔没反应,需要再点一次。
- 根因:菜单项 `on_mouse_down` 未 `cx.stop_propagation()`,点击事件冒泡到全屏 scrim,焦点被重新分配,异步调起的原生 `prompt_for_paths` 可能闪退或被挤到主窗口后方。
- 修复:所有弹出菜单项和菜单容器本体补 `cx.stop_propagation()`,将点击事件限制在菜单区域内。

## 验证 / 状态

- Explorer rebuild 和 snapshot 相关逻辑有 headless 单测覆盖。
- git status 后台化和 trailing-edge watcher 属 UI 卡顿风险修复,需在大仓库、构建期和 `.git` 短暂锁定时持续观察。
- 菜单焦点修复目标是原生文件夹对话框稳定置顶拉起。
