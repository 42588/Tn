# 每窗格工作区上下文

## 定位

本文记录面板解耦中 per-pane 工作区上下文的文件树状态、打开文件夹语义和验证边界。发布摘要见 [CHANGELOG.md](../../CHANGELOG.md)。

## 背景

本轮目标是让每个终端窗格拥有自己的工作区上下文,避免文件树状态被全局单例串台,并确保「打开文件夹」只影响当前焦点 pane。

## 具体内容

### per-pane 文件树状态

- 新增 `ExplorerSnapshot` 位于 `crates/tn-ui/src/explorer.rs`。
- 新增 `ExplorerView::snapshot()` / `switch_pane()`。
- Workspace 按 `PaneId` 保存 `explorer_states` 快照,并用 `explorer_pane` 记录当前展示的 pane。
- 焦点在分屏 pane 间切换时,保存旧 pane 的展开态/选中文件,恢复新 pane 的展开态/选中文件。
- 各 pane 文件树互不串台。
- 同 pane 内 `cd` 仍走 `follow_root`,保留子目录展开态。
- 快照保存时惰性裁掉已关闭 pane,无需逐 `remove` 钩子。
- 纯函数 `snapshot_under_root` 会把恢复内容过滤到新 root 内,并有 headless 单测覆盖。

### 打开文件夹语义

- `cd_panes_to_root` 从“广播给所有非 agent pane”收敛为 `cd_pane_to_root(id, ...)` 单 pane 操作。
- `menu_open_folder` 只 `cd` + `set_rail_root` 当前焦点 pane。
- 其它 pane 保持各自目录。
- agent pane 永不被 `cd`。
- SSH pane 跳过本机 picker,打开当前远端 root 的应用内 SFTP 浏览,并只向该 pane 发送远端 `cd`。
