# 协作入口规则

## 定位

本文负责记录 AGENTS/CLAUDE 两个项目记忆入口的同级关系和共享规则落点。具体任务进度写入 `TODO.md` 和 `docs/任务/`;文档拆分格式遵循 `docs/文档治理/文档盘点与首批拆分方案.md`。

## 背景

大型文档拆分要求母文档精准定位子文档,子文档承载具体信息。`CLAUDE.md` 是 Claude 的项目记忆入口,`AGENTS.md` 是 Codex 的项目记忆入口,二者同级,不互相作为上级文档。

## 具体内容

- 两个入口文件只保留各自 agent 启动所需的短索引和必要工作规则。
- 跨 agent 的长期工作规则、踩坑、环境约束和协作约定写入 `docs/共享记忆/` 下的中文子文档。
- 新增共享信息前,先按 `docs/文档治理/文档盘点与首批拆分方案.md` 判断是否属于共享记忆;若属于,创建或补充职责明确的中文子文档。
- 根目录入口文件避免复制彼此的长篇指导;需要共同引用的信息应通过共享子文档维护。
- `.planning` 已废弃,不要恢复为任务或计划入口。

## 验证 / 状态

- 状态: 已创建共享记忆落点。
- 待主控整合: 若 Claude 入口重构也引用共享规则,应指向本文或同目录下更具体的中文子文档。
