## 新增信息落点

- 新增坑位先判断事实属性:终端/进程、图形渲染、焦点/输入、Windows/工具链分别写入对应子文档。
- 子文档只保留长期有效原则、最小根因摘要和权威专题链接;完整修复过程、测试名和验证日志写入 `docs/修复与优化/`。
- 架构边界写入 [系统架构索引](../系统架构索引.md) 或 `docs/架构/`;仍未完成或待真机复验的问题写入 [已知问题索引](../已知问题索引.md) 或 `docs/已知问题/`。
- 本文只新增或更新索引行,不承载长篇复盘。
