# 图形界面与渲染坑位

## 定位

本文记录 GPUI、Calm Glass、虚拟列表、Quick Look 渲染、滚动条和大文件预览相关的长期坑位原则。具体视觉数值和修复复盘由样式、架构和修复专题承载。

## 坑位原则

### GPUI 与 Calm Glass

- gpui 0.2.2 Windows 使用 DX11 + DirectWrite;DXGI debug layer 缺失提示通常无害。
- gpui `Blurred` 是窗口级 acrylic,不是逐元素 backdrop-filter;Tn 默认保持 Opaque,玻璃感靠内部面板层叠、边框和投影。
- `overflow_hidden` / `ContentMask` 会裁投影,也不替子元素处理圆角;叶子组件自己裁内容,外层 wrapper 不要层层裁。
- 投影需要直接写 `BoxShadow`;祖先裁剪会吃掉面板浮起感。
- 大面积半透明渐变容易在 8-bit 下产生色带;大面积面板优先纯色中点,深度靠窄边框、层次和状态变化。

### 终端字形与网格

- CJK 宽字符要按固定格宽处理,不能依赖回退字体自然步进;终端渲染要跳过 spacer 并按列跨度定宽。
- 反相块光标要在块上重绘光标处字符,不要用半透明叠层糊住字形。
- Quick Look / 编辑器里的 CJK 光标、选区和 hit-test 也要走固定单元格模型,否则误差会随中文字数累积。

### 虚拟列表、Quick Look 与性能

- 长列表必须虚拟化;几百行代码 × 每行多 token div 一次性布局会冻结窗口。
- Quick Look 打开 / 导航热路径不能同步跑 `git diff`;diff 只能惰性、有界、后台化。
- 手写 scanner 每轮必须证明游标前进;不同 Unicode 判定宽窄不一致时要有兜底 `+1`。
- GPU paint 阶段的冻结要用可复现探针定位,不要只依赖 render 计时。
- PDF/图片预览页框和占位用暗 gutter,避免白边和白闪。

### flex、分屏与滚动条

- Taffy flex 子项需要 `min_w(0)` / `min_h(0)`,否则内容会撑破分配份额。
- 分屏间距用 wrap 内侧 padding,不要用 flex gap 或 wrap margin 改 seam。
- Quick Look 长行横滚不走 `overflow.x = Scroll`;自绘横向滚动条 + 内容偏移更可控。
- caret-follow 只能在光标变化时触发,不能每帧覆盖用户手动滚动。
- 自绘滚动条可见条要细,命中区要大;任何拖动都要用 `pressed_button` 在 move 中兜底清理。
