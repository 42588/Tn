# 视窗系统与工具链坑位

## 定位

本文记录 Windows 窗口系统、DPI、cargo、Git 中文路径、外部工具和配置文件操作相关的长期坑位原则。

## 坑位原则

### Windows 窗口与 DPI

- 外部 `SetWindowPos` / `ShowWindow` 不能在 GPUI 更新回调里同步调用,否则可能重入窗口状态借用;放到前台任务中执行。
- Quick Terminal 启动器的窗口几何是物理像素,GPUI 布局是逻辑像素;设置窗口前必须按 DPI 缩放。
- 无边框 Quick Terminal 的圆角不是 GPUI 子元素圆角,需要窗口形状或系统窗口能力配合。
- 自绘标题栏的 OS 控制区由窗口命中测试执行,不要再给 Close / Min / Max 加重复 on_click。

### 构建和运行环境

- `cargo` 可能不在当前 shell PATH 中;Windows 上优先确认 `%USERPROFILE%\.cargo\bin`。
- debug 构建保留控制台窗口,release 隐藏控制台;截图/注入工具不要靠窗口标题找 Tn。
- GPUI 窗口类可能不是产品名;工具自动化需要按真实窗口类或进程定位。
- 从 IDE 集成终端启动 Tn 会继承 IDE 环境变量,可能误导 agent 或外部工具;必要时清理 `VSCODE_*`、`TERM_PROGRAM` 等。

### Git、路径和用户配置

- `git status --porcelain` / `git diff` 对非 ASCII 路径默认 quotePath 转义;需要 `-c core.quotePath=false`,否则中文文件名状态会对不上文件树真实路径。
- 用户手写 `config.toml` 不要整体反序列化重写;新增/删除 profile 或 agent 用片段追加、块级删除,避免丢注释和格式。
- `wsl --list --quiet` 输出是 UTF-16LE,解析时不要按普通 UTF-8 文本处理。
- Git watcher 要过滤 `.git`、`target`、`node_modules` 等噪声,并防抖;不要让自己触发的 git diff 形成刷新循环。
