# 终端与进程坑位

## 定位

本文记录 ConPTY、PTY 生命周期、agent 进程托管、SSH/WSL 命名空间和终端 resize 相关的长期坑位原则。完整修复过程和验证日志写入修复专题。

## 坑位原则

### ConPTY 与 PTY 生命周期

- ConPTY DSR 回复必须把 alacritty `Event::PtyWrite` 写回 PTY writer,否则子进程可能卡在启动期查询。
- ConPTY 无可靠 EOF,用 `try_wait` 轮询子进程状态,不要把 `read()==0` 当作唯一退出信号。
- `LocalPty` drop 必须杀子进程;关闭 pane / tab 要能沿视图 drop 链释放 PTY。
- reader 线程持锁推进 VT 解析时要在锁内捕获 panic,避免 Mutex 中毒后把 GPUI 前台回调拖到 abort。
- pane 构造发生在 GPUI non-unwinding 窗口回调内,spawn 失败必须优雅回退,不要 `expect()`。

### agent 进程与 shell-detected agent

- Windows 上 `claude` / `codex` 常是无扩展名 npm shim;直接 `CreateProcessW` 可能失败,需要经 shell / PATHEXT 路径启动。
- 持久 agent pane 托管在 `pwsh -NoExit` 时,agent 子命令退出不等于 PTY 子进程退出;需要明确的退出哨兵或 block 状态来清理 agent surface。
- shell 内手敲 agent 只能认当前运行命令首 token,不要靠进程树或“最近会话新鲜度”猜测。
- shell-detected agent 的 overlay 宿主必须保持 flex-column,否则正文区域会塌成“只有外壳没有正文”。
- Agent Host 身份走开放 `AgentId` + registry,UI 不重新引入闭合 `AgentKind` 特判。
- sidecar 遥测是 per-pane 资源,不能塞进共享 registry adapter;sidecar 是否联网也不等于 agent 本体是否远程运行。

### SSH / WSL / 远端命名空间

- WSL/SSH 的 `/home/...` 不能直接转成宿主 Windows `PathBuf`;必须经 `LaunchSpec::file_namespace` 显式区分 Host / WSL / SSH。
- SSH cwd 可能只通过裸 `OSC 633;P;Cwd` 更新模型级 cwd,不要只读当前 block / last finished block。
- SSH 覆盖卡存在时要前端阻断输入,后端在 `request_shell` 成功前 drain pending input,避免连接后 replay 到远端 shell。
- SSH 认证不能只尝试 `password`;很多服务器只接受 `keyboard-interactive`,需要按服务器开放方式尝试。
- known_hosts 写入失败不能静默继续;“记住主机”必须真正写入成功。
- SSH 连接器的数据源要分清:最近/收藏来自 Tn 自己的 `ssh_recents.json`,ssh-config Host 是只读 OpenSSH 配置入口。

### resize / scrollback / TUI 归属

- ConPTY 与 alacritty 尺寸不能长期不一致;行锁定会制造空白和光标错位。
- 长高 resize 要按 ConPTY repaint 语义处理 anchoring,避免把历史行拉进视口后又被 ConPTY 覆盖。
- 隐藏/恢复帧的小 bounds 不能被 clamp 成 `1x1` 真实终端尺寸;不足一个完整 cell / row 时跳过本帧 resize。
- scrollback 里不要画 live cursor;任何非 0 `display_offset` 输入前先回到底部。
- agent / alt-screen pane 的滚轮归程序自身,普通 shell 才滚 Tn scrollback。
- 从 VS Code / Cursor 集成终端启动 Tn 时要清理会误导 agent 的 IDE 环境变量。
