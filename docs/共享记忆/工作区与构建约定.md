## 常用命令

```powershell
cargo build --workspace
cargo test --workspace --lib
cargo run -p tn-cli
cargo run -p tn-app
$env:TN_AUTOQUIT="1"; cargo run -p tn-app
```

按任务选择更窄验证:

- UI / 样式守卫: `cargo test -p tn-ui token_drift`, `cargo test -p tn-ui spec_gen`。
- 终端核心: `cargo test -p tn-core --lib`, `cargo bench -p tn-core --bench engine`。
- ConPTY 烟雾: `cargo run -p tn-cli` 或 `cargo run -p tn-cli -- <program> [args]`。
- GUI 肉眼验收: 有窗口环境时运行 `cargo run -p tn-app`。

## 构建注意事项

- `cargo` 位于 `%USERPROFILE%\.cargo\bin\cargo.exe`;若 shell PATH 不含该目录,新开 PowerShell 或显式调用。
- 根 `Cargo.toml` 的 dev profile 已按高频 TUI 重绘场景优化依赖,不要随意撤销。
- 改依赖版本走根 `Cargo.toml` 的 `[workspace.dependencies]`。
- `cargo test --workspace --lib` 只跑库单测;去掉 `--lib` 会额外跑集成测试,可能需要 ConPTY 环境。
- `TN_AUTOQUIT=1` 可用于无交互 GUI 自测;真实视觉、窗口行为和动画仍需真机肉眼确认。
