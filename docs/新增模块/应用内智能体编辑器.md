# 应用内智能体编辑器

## 定位

本文记录应用内智能体编辑器的用户可见能力、config 持久化、与智能体宿主平台的边界、修复项和后续未做事项。发布摘要见 [CHANGELOG.md](../../CHANGELOG.md)。

## 背景

用户反馈手编 `config.toml` `[[agents]]` 太极客。本轮把「加 agent」升级为应用内现代交互。该编辑器建立在智能体宿主平台之上,只产出 config 数据,不要求平台层改动 agent 知识。

## 具体内容

### 添加与编辑体验

- 欢迎页 launchpad 新增「+ 添加 Agent」磁贴。
- 新增居中玻璃浮层编辑器 `workspace::render_agent_form`。
- 表单收集名称、命令、颜色和「由 Agent 自绘光标(Ink TUI)」开关,并实时显示磁贴预览。
- 颜色使用 `tn_config::ACCENT_SWATCHES` 预设。
- 交互支持 `Tab` 切字段、`Enter` 保存、`Esc` 取消、点外关闭。
- 名称字段支持中文 IME,复用 `EntityInputHandler`,与 SSH 重命名同源。
- 命令字段为 ASCII 输入,IME 关闭。
- 自定义磁贴 hover 显示编辑/删除动作,由 `welcome::agent_tile_actions` 产生。
- 编辑动作预填表单并回写;删除会抹掉 `[[agents]]` 和 `[[profiles]]`。
- 事件通过 `EditAgentRequested`、`DeleteAgentRequested`、`AddAgentRequested` 回到 workspace。

### 保存与配置持久化

- 保存后无需重启:`workspace::reload_agents` 重读 config,重建 `AgentRegistry` global,刷新 `launch_profiles`,重建 welcome。
- welcome 重建复用 `subscribe_welcome` 订阅,新磁贴立即出现。
- `tn-config` 新增 `append_agent[_to]` 与 `remove_agent[_from]`,按块追加/删除并保留注释。
- `block_ranges` 被泛化用于块级 config 操作。
- 新增 `agents_toml_fragment` 和 `ACCENT_SWATCHES`。
- `slugify` 用名称或命令首词派生稳定 `AgentId`,并做 dedup。
- 命令首词进入 `aliases`,shell 中敲对应命令即可自动切 Agent 态。
- config-only agent 默认是 generic,无用量遥测;真实遥测需要内置或外部 adapter。

### 内置 agent 识别与高级遥测

- `agent_host::build_registry` 在 startup 和 `reload_agents` 共用。
- 每个 manifest 先尝试 `tn_ai::builtin_adapter_for_manifest`:命令/别名命中 claude/codex 时,使用内置日志解析器和用户自己的颜色、名称、Ink 设置。
- `ClaudeAdapter` / `CodexAdapter::with_descriptor` 会强制开启 usage 能力。
- 用户在「+ 添加 Agent」填命令 `claude` 即自动出现用量环,不需要配置 sidecar。
- 平台 `tn-agent` 保持零 agent 知识,只有 `tn-ai` 认内置名字。
- 编辑器新增「高级 / 用量遥测」区域,文案明确“不懂就留空”。
- `AgentField::Sidecar` 文本字段和「联网 sidecar」开关用于高级遥测。
- 留空保持 generic;设置 sidecar 后写入 manifest `sidecar` 并加 `capabilities=["usage"]`。
- 勾选联网只写 `allow_network=true`,启动时走确认卡。
- 进阶遥测也可通过 GUI 配置,不必手编 `config.toml`。
- 设计真源位于 [design/panels/04-overlays.html](../../design/panels/04-overlays.html),新增编辑器原型和高级遥测区。

## 修复记录

- 修复联网 sidecar 把 agent 开成 shell:GUI 曾把「联网 sidecar」写成 agent 的 `runtime_support=["remote_daemon"]`,导致 PTY 启动器把 `claude` 这类命令型 agent 当非 PTY 拒绝,最终回退 pwsh shell。修复后 sidecar 网络属性只走 `allow_network`;`AgentDescriptor::sidecar_launch` 只看 `network_policy`;命令型 agent 的 `runtime_support` 永远是 PTY。旧版存的 agent 重新保存一次即可修复。
- 修复一个 agent 显示成两张磁贴:旧版默认 config 出厂带 claude/codex profile,用户经编辑器声明同名 `[[agents]]` 后,遗留 profile 与新增 profile 同时显示。`discover_profiles` 增 `dedup_agent_profiles`,按 agent id 去重并保留最新保存的一条;非 agent 的 shell/WSL/SSH 不去重。
- 修复欢迎页「打开文件夹」失效:欢迎 launchpad 的焦点 pane 是 `WELCOME_DUMMY` 且无 `LaunchSpec`,旧逻辑走 SSH 式应用内浏览并提前 return,原生选择器不弹。修复为无 spec 时默认用原生选择器,选目录后重定 explorer root,欢迎页磁贴启动的 agent/shell 继承该 root 为 cwd。
