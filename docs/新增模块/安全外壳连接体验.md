# 安全外壳连接体验

## 定位

本文记录安全外壳快速连接交互面板、连接状态可视化、错误/密码/TOFU 覆盖卡、最近连接与收藏、以及 A/B/C 体验优化路线。安全外壳协议、PTY 后端、安全校验和认证实现见 [安全外壳远程连接](安全外壳远程连接.md)。

## 背景

M2b 阶段把 SSH 从“单行输入 + 终端 ANSI 文本”升级为有记忆、有进度、可操作的 Calm Glass 体验。设计稿真源为 [design/panels/06-ssh.html](../../design/panels/06-ssh.html),落地时遵循 `design -> 组件规格 -> gpui -> 四守卫`;组件数值见 [组件规格与自动生成数值](../界面样式/组件规格与自动生成数值.md)。

## 快速连接面板

### 已实现能力

- **全局动态输入面板(Quick Connect)**:欢迎页、命令面板和分屏操作去除“即将支持”占位,落地真实 `LaunchRow::SshPrompt` 交互条目。
- **GPUI Calm Glass 视觉与工作区感知居中**:面板复用原生命令面板的玻璃态设计。渲染时读取左侧 Explorer 宽度与工作区缝隙,通过弹性 padding 让弹窗对齐当前主编辑区视觉中心,替代固定顶部悬浮。
- **焦点接管与交互闭环**:支持键盘捕获、Esc 退出、Enter 提交、点击遮罩关闭。点击面板本体通过 `stop_propagation` 和 `focus(w)` 保活,解决按键穿透、鼠标点不到面板等焦点问题。
- **Intent 路由**:`SshPromptIntent` 记录来源(Welcome / Palette / Split)。提交 `user@host:port` 后组装 `LaunchSpec`,交给 `spawn_pane_with` 启动 SSH pane。

### 修改的代码区域

- `crates/tn-ui/src/welcome.rs` 与 `quick_terminal.rs`:移除旧 `SshSoon` 占位,新增 `SshPromptRequested` 事件。
- `crates/tn-ui/src/workspace.rs`:新增 `render_ssh_prompt`、`on_ssh_prompt_key`、焦点保活、基于 Explorer 宽度的弹性偏移计算和真实 SSH 连接入口。

## 体验路线总览

### A. 交互方式

- **A1 最近连接 + 收藏**:已实现。成功连过的主机自动进入最近连接,连接器内可一键回连;收藏置顶。
- **A2 收藏承接长期保留 + 昵称**:已实现。取消“保存为连接”入口,长期保留统一为收藏;收藏行支持中文昵称,再次点星标可取消收藏。
- **A3 结构化输入 + 实时解析**:已部分实现。保留快捷单行,连接器已有 user/host/port 实时解析 chips;后续可扩展为分字段和更强非法标红。
- **A4 枚举 `~/.ssh/config` 主机**:已实现。`Host` 别名作为连接器只读来源,点击别名经别名连接,保留 `HostName` / `User` / `Port` / `IdentityFile` 等 OpenSSH 规则。

### B. 界面

- **B1 连接进度卡**:已实现。建立连接、认证、打开 shell 三步反馈,取代只在终端里输出 ANSI 文本。
- **B2 主机指纹确认**:已实现。首连未知主机弹 TOFU 信任面板;host key 不匹配时危险中止并显示本次指纹。“仍信任并更新 known_hosts”未做,当前安全默认中止。
- **B3 密码框增强**:已实现。支持显隐切换、记住密码(仅本会话)、输错原地重试且不杀会话。
- **B4 断线重连横幅 + 连接态信息胶囊**:已实现。SSH pane 头部显示连接中/已连接/重连/断开状态、目标和认证方式;断线时显示非模态横幅。倒计时和立即重连未做。
- **B5 可视件流程**:保留原则,可视件先在 mockup/panels 出稿再落 GPUI。

### C. 提示

- **C1 可操作错误卡**:已实现。认证失败显示原因、服务器开放方式和针对性提示(如 `PermitRootLogin`),提供重试和关闭。
- **C2 拨号前校验**:已实现。空 host、悬空 `@` / `:`、端口越界等输入当场显示红色 chip,Enter 被拦截。
- **C3 首连引导 + 成功反馈**:已实现。连接器空态展示三步上手卡;连接成功后头部胶囊展示真实目标和登录方案(密钥/密码/交互)。

## 实现明细

### 最近连接 + 收藏

- `crates/tn-ui/src/ssh_recents.rs` 新增 `SshRecent` / `SshRecents`,记录 host、user、port、昵称、认证方式、最近使用时间和收藏状态。
- `record` 按 host(不分大小写)+user+port upsert,刷新 `last_used`,保留收藏与昵称。
- 支持 `toggle_favorite`、`rename`、排序、过滤、裁剪和 JSON round-trip;数据落 `%APPDATA%\Tn\ssh_recents.json`。
- 只有 shell 打开后收到 `PtyEvent::Connected { method }` 才记录;失败和超时不记录。
- 连接器输入行包含图标与实时解析 chips,下方列表展示最近/收藏项、认证徽章和相对时间。
- 键盘支持 `↑↓` 选择、Enter 连接选中项或当前输入、Esc 关闭;点击行即连接。
- 顺带修复 `spawn_pane_with` 在 `TN_AUTOQUIT` / `TN_DEMO` 首 pane 早于 tab push 时的 cwd 继承越界 panic,改为 `self.tabs.get(active)` 兜底。

### 收藏即长期保留 + 中文昵称

- `ssh_conn_rows()` 合并最近/收藏与 `~/.ssh/config` Host 别名,按输入过滤并按端点去重。
- 不再把 `config.toml [[profiles]]` 的 SSH profile 显示为“已保存”段;连接器也不再写入或删除 `config.toml`。
- 最近行支持星标收藏;收藏行额外显示重命名入口。
- 重命名态使用 `EntityInputHandler` 和 `window.handle_input` 支持中文 IME;普通 SSH 地址输入仍关闭 IME,避免 host 输入被中文输入法干扰。
- 面板增加 `max_w(relative(.92))`、`max_h(relative(.86))`,列表区域限制高度并裁剪;收藏量很大时可后续升级为 `uniform_list` 虚拟滚动。
- `tn-config` 的 `append_profile` / `remove_profile` / `profiles_toml_fragment` 仍保留给手编配置、测试和旧路径兼容。

### 连接进度卡 + 可操作错误卡

- 后端新增结构化事件:
  - `PtyEvent::SshProgress { phase, detail }`,其中 `SshPhase = Connecting | Authenticating | OpeningShell`。
  - `PtyEvent::SshFailed { kind, detail, offered }`,其中 `SshErrorKind` 覆盖认证失败和 host key mismatch。
- `TerminalView` 新增 `ssh_target`、`ssh_progress`、`ssh_error` 状态;渲染优先级为密码框 > 错误卡 > 进度卡。
- 进度卡诚实展示三步状态:已完成为绿,当前为 accent 点,等待为灰点;无帧时钟时不用假旋转。
- 错误卡显示失败原因、服务器开放方式和后端 detail 提示。认证失败可重试/关闭,host key mismatch 默认只关闭。
- `Workspace` 将 pane 安装逻辑抽成 `install_pane(id, spec, cx)`;重试复用同一 pane id 替换旧视图,关闭按 id prune 掉失败或连接中的 SSH pane。

### 密码卡增强

- 后端密码认证改为最多 3 次原地重试,同一 SSH handle 不重连。每次失败重新发 `NeedPassword` 并带 `error` 文案。
- 密码卡支持遮罩/明文切换、记住密码复选、连接和取消按钮;重试时保留 reveal/remember 状态但清空输入。
- 勾选“记住密码”只写入本会话内存:workspace 缓存在 `pane_specs[id].ssh.password`,后端也补到当前 `SshConfig.password`;永不落盘。
- 后续可考虑接入系统凭据管理器,而不是仅内存保存。

### TOFU 信任卡 + 连接态反馈

- 未知主机不再静默接受。后端发 `NeedHostKeyConfirm { host, fingerprint, reply }`,UI 展示信任卡。
- 信任卡显示 SHA256 指纹,支持“记住写 known_hosts”并连接,或取消。
- `AcceptAndSave` 写盘是可失败路径;如果无法创建父目录或写入 known_hosts,连接中止并提示,避免用户误以为已保存信任。
- `TerminalView` 维护 `SshConnState`,由进度、Connected、Disconnected 事件驱动连接中/已连接/重连/断开状态。
- SSH pane 头部显示连接态信息胶囊,包含状态点、状态词、目标和登录方案。
- 断线/重连时显示顶部非模态横幅,提供取消按钮;重连不弹大进度卡,避免干扰。

### ssh-config 枚举 + 拨号前校验 + 首连引导

- 后端 `list_ssh_config_hosts()` 读取 `~/.ssh/config`,先解析 `Host` 别名,跳过通配与否定项,再用 `ssh2_config::query(alias)` 解析目标。
- 连接器缓存 `ssh_config_hosts`,每次打开连接器刷新;渲染为只读 ssh-config 行。
- 点击别名时经 alias 本身连接,而不是经已解析 host 连接,避免丢失别名级密钥和 OpenSSH 规则。
- `validate_ssh_target()` 纯函数校验空 host、悬空 `@` / `:`、端口越界;非数字 `:后缀`不报错,与 `SshConfig::parse` 保持一致。
- 空态且无最近/config 时显示三步引导:输入地址、自动认证、记住连接。
- `Connected { method }` 驱动成功反馈,头部胶囊展示密钥/密码/交互认证方式。
