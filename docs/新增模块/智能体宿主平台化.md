# 智能体宿主平台化

## 定位

本文记录智能体宿主平台化的设计边界、落地内容、移除项和验证守卫。发布摘要见 [CHANGELOG.md](../../CHANGELOG.md)。

## 背景

本轮把「Claude/Codex 特判」重构为对具体 agent 零知识的智能体宿主平台,按 P0-P6 分阶段落地,每阶段要求独立编译与测试通过。

## 具体内容

### 平台 crate 与身份模型

- 新增 `tn-agent` headless crate。
- `AgentId` 使用开放字符串身份,替代闭合 `AgentKind`。
- `AgentDescriptor`、`AgentCapabilities`、`AgentRuntimeKind` 描述身份、能力插槽和运行位置。
- `AgentEvent` 与 `AgentStatus` 成为 UI 唯一输入契约。
- `AgentAdapter` trait 与 `GenericAdapter` 支持“有身份、无遥测”的外部 agent。
- `AgentRegistry` 按 id/命令解析 agent;空 registry 表示纯 shell 宿主;`register_manifest` 支持注册 config agent。
- `AiUsage` 与 pricing 从 `tn-ai` 上移。

### 配置 manifest 与 runtime 契约

- 新增 config `[[agents]]` manifest,用户写 TOML 即可让新 agent 进入启动器、头部和能力插槽。
- `AgentManifest` 转 `AgentDescriptor::from_manifest`;默认无遥测。
- `runtime_support` 声明 PTY、structured、http、websocket、remote-daemon 支持。
- `allow_network = true` 只进入“需用户确认”策略,不静默放行。
- `[agents.<id>]` 主题色表和 `[general.billing]` 按 id billing 覆盖,对应 `accent_for` / `billing_for`。
- `LaunchSpec::runtime()` 从 ssh/file_namespace 派生 `AgentRuntimeKind` 的 PTY 家族,并开放 `RemoteDaemon`、`Http`、`WebSocket`、`Structured`。
- runtime 与 `FileNamespace` 严格分离。

### 外部事件与 sidecar 遥测

- `ExternalEventAdapter` 接 JSONL 并转换为 `AgentEvent`。
- `ExternalProcessAdapter` 可拉起 stdio 子进程,从 stdout JSONL 实时入队,stderr 转 `ErrorReported`。
- UI 仅对 `has_realtime_events()` 的 adapter 启动轻量事件 poller。
- `[[agents]] sidecar = "cmd"` 映射到 `AgentDescriptor.realtime_command`。
- `AgentDescriptor::sidecar_launch()` 是默认拒绝网络的单点决策,返回 `SidecarLaunch::{None,SpawnNow,Confirm}`。
- launched agent pane 每 pane 拉起自己的 `ExternalProcessAdapter::from_descriptor` 和事件 poller。
- 本地 stdio sidecar 直接启动;networked sidecar 弹 `SidecarConfirm` 确认卡。
- adapter 由 `TerminalView.realtime_adapter` 持有,`clear_agent` 或 drop 即杀子进程。
- 结果是配置声明的 agent 也可获得真实用量、transcript 和权限事件,无需内置 adapter。

### UI 归约与平台接入

- `StatusChanged`、`ModelChanged`、`TranscriptAppended`、`PermissionRequested`、`ErrorReported` 进入同一个 `reduce_agent_event` 漏斗。
- agent 头部显示状态、权限、错误和 transcript 摘要 chip。
- `CwdChanged` 只更新目标 pane 的上下文和活动栏 cwd。
- `tn-ai` 改为内置 Claude/Codex `AgentAdapter` provider,作为平台的两个种子 provider。
- `claude.rs` / `codex.rs` 继续薄包装解析逻辑。
- `detect.rs` 泛化为 `resolve_pane_session(&dyn AgentAdapter)`,第三个 agent 不需要新增 match。
- `builtin_registry()` 保留,但默认 app 不再注册内置 agent,出厂无内置 agent。
- `tn-ui` 身份全面改走 `AgentId` + `AgentRegistry` GPUI Global。
- per-pane 缓存 `agent_accent`、`label`、`short`、`manages_cursor`、`caps`,并经 `resolve_agent_view` 在 agent 变更时解析。
- `force_hide_cursor` 改为描述符 `manages_own_cursor`。
- header 用量环 gate `caps.usage`;活动栏 gate `caps.git_diff`;状态栏按 `AgentId` 聚合。
- 用量轮询和外部实时事件都经 `AgentEvent` 归约器入账;内置日志 adapter 不新增热路径。

### 移除项

- 移除闭合 `AgentKind` 枚举。
- 移除 kind-dispatch API:`resolve_session`、`session_mtimes`、`parse_session`、`update_session`、`detect_subscription`、`agent_kind_for_command` 等。
- agent 身份与解析全部改走 `AgentId` + adapter。
- 移除 `tn-ai` 对 Claude/Codex 原始解析器的 public re-export。外部只拿 `ClaudeAdapter`、`CodexAdapter`、`builtin_registry`,具体 JSONL 解析函数继续留在 `tn-ai` 内部供 adapter 和单测使用。

## 验证 / 状态

- 新增守卫测试 `agent_host::guard::ui_has_no_closed_agent_enum`,扫描 `tn-ui/src`,锁死 UI 零闭合枚举。
- 每阶段按要求保持独立编译与测试通过。
- 应用内智能体编辑器仍只产 config 数据,不把具体 agent 知识写回平台层;高级 sidecar 字段也只落 manifest 配置。
