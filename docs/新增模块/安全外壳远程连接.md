# 安全外壳远程连接

## 定位

本文记录安全外壳远程连接的后端能力、PTY 事件桥、认证与安全边界、断线重连和验证状态。连接器 UI 与体验路线见 [安全外壳连接体验](安全外壳连接体验.md)。

## 背景

M2b 阶段把 SSH pane 从“能启动终端”推进到可按 OpenSSH 配置拨号、校验 host key、处理密码与 keyboard-interactive、向 UI 发结构化事件并支持断线重连的远程终端基础能力。

## 具体内容

### 已实现功能

- **`~/.ssh/config` 解析支持**:使用 `ssh2-config` crate 读取本地 SSH 配置文件,自动提取指定主机的 `HostName`、`Port`、`User`、`IdentityFile` 等配置,并与用户的 Tn 配置融合。
- **TOFU 与公钥安全验证**:通过 `russh::keys::check_known_hosts_path` 验证公钥。基础实现曾在首次连接未知主机时写入 `~/.ssh/known_hosts`;后续 UX 迭代改为首连信任面板确认后才接受并写盘。若密钥不匹配,连接会被主动阻断并向前端发危险提示。
- **动态密码输入交互通道**:当密钥验证失败且配置中未提供密码时,后端通过事件请求 UI 捕获密码,前端用 Calm Glass 覆盖卡拦截终端按键并遮罩输入。
- **自动断线重连与超时控制**:TCP 连接增加 15 秒超时封装,避免死锁挂起。网络层断开时进入 5 秒延时自动重连循环。
- **终端本地化中文彩色反馈**:后端前置状态消息中文化,连接、密钥验证、密码请求等信息以 ANSI 彩色文本流式写入终端,同时关键状态走结构化事件给 UI。
- **认证方式探测 + 密码双通道**:认证前用 `authenticate_none` 探测服务器实际开放的认证方式,等价 `ssh -v` 的认证方式提示。拿到密码后按服务器开放方式尝试 `password` 与 `keyboard-interactive`,覆盖 Linux/PAM 服务器只接受 keyboard-interactive 的情况。若服务器未开放密码类方法,错误信息会提示检查 `PasswordAuthentication` / `PermitRootLogin`。

### 技术细节

- **前后端事件通信桥梁**:
  后端 `tn-pty/src/ssh.rs` 的 `SshBackend` 与前端 `tn-ui/src/terminal_view/mod.rs` 的 `TerminalView` 之间建立基于 `std::sync::mpsc` 和异步 waker 的事件通道。`PtyEvent::NeedPassword` 让异步 SSH 协程可中断并挂起,请求 UI 线程获取用户输入。UI 通过 GPUI `cx.notify()` 和 update 唤醒渲染,再经 reply channel 回传密码。
- **渲染循环与按键拦截**:
  `TerminalView::render` 注入悬浮输入卡。当存在挂起密码请求时,`TerminalView::on_key` 拦截终端按键流转,捕获密码明文,支持退格、回车提交与取消。
- **会话协程重构**:
  `run_session` 改为循环包裹,利用 `tokio::select!` 监听 PTY `out_rx`、`kill_rx` 和 `channel.wait()` 等数据流,区分网络层断开和应用主动关闭/Eof。网络断开进入自动重连,应用主动关闭则顺利结束线程。
- **结构化 SSH 事件**:
  后续体验迭代引入 `PtyEvent::SshProgress`、`PtyEvent::SshFailed`、`PtyEvent::Connected { method }`、`PtyEvent::NeedHostKeyConfirm` 等事件,让 UI 能显示连接进度、错误卡、连接态胶囊和 TOFU 信任面板。
- **输入隔离**:
  SSH 覆盖卡存在时,前端短路 `send_bytes`、粘贴、重跑命令、alt-screen 滚轮等输入;后端在 shell 打开后、发送 `Connected` 前 drain 掉覆盖期间排队的输入,避免 replay 到远端 shell。

### 修改的代码区域

- `crates/tn-pty/Cargo.toml`:引入 `ssh2-config`。
- `crates/tn-pty/src/lib.rs`:新增 SSH 相关 `PtyEvent`,并在 `PtyBackend` trait 加入 `try_recv_event` 和 `set_waker`。
- `crates/tn-pty/src/ssh.rs`:完善 `SshConfig::parse`;实现 known_hosts 校验、TOFU/host key 分支、认证方式探测、password/keyboard-interactive 双通道、连接超时与自动重连。
- `crates/tn-ui/src/terminal_view/mod.rs` 与 `io.rs`:拉取 SSH 后端事件,维护密码/host key/进度/错误/连接态状态,并在覆盖卡存在时接管输入。

## 验证 / 状态

- 2026-06-05:SSH 首次真机端到端验证。Tn 以密钥认证 `id_ed25519` + 交互 shell 成功登录 AlmaLinux 9.7(WSL) sshd。
- `authenticate_none` 打印的服务器认证方式与 `ssh -v` 吻合,并帮助定位 `PermitRootLogin no` 的服务器侧配置问题。
- 真机仍需重点回归:密码 UI、keyboard-interactive、断线重连、TOFU 首连信任与 host key mismatch。
- `ssh-agent` 连接暂未实现。Windows 本地 pipe 集成复杂且 `russh` 无默认支持,可作为后续 M2 进阶需求。
- `~/.ssh/config` 扩展解析仍有限,尚未覆盖 `ProxyJump` 等完整 OpenSSH 配置能力。
