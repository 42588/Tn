# 远端文件服务与改动流

## 定位

本文记录 SFTP 远端文件服务、远端 Quick Look、远端 git 改动流和 SSH/WSL 目录 picker 的具体实现、修复与待验项。发布摘要见 [CHANGELOG.md](../../CHANGELOG.md)。

## 背景

本轮把 SSH pane 从“终端可登录”推进到“远端文件树、远端预览、编辑写回、远端 diff 与 hunk 操作可用”的首版能力。CHANGELOG 只保留发布级摘要,具体服务边界和真机暴露问题沉到本文。

## 具体内容

### 远端文件与命令服务

- `tn-pty::remote_fs` 新增 SFTP v3 后端:`RemoteFileService` + `SftpFileService` 支持 SSH 远端列目录、有界读文件、stat 元数据和写回文件。
- SFTP 后端复用 SSH 配置、密钥和内存密码;后台探测不弹 host-key/password UI。
- `tn-pty::remote_cmd` 新增有界 SSH exec 服务,支持 POSIX shell quoting、stdout/stderr/exit-status 捕获和 stdin 传入,供远端 git / hunk patch 使用。

### SSH Explorer root 与远端 Quick Look

- `ExplorerRoot::Ssh` / `ExplorerPath::Remote` 用 `RemoteId` 表示远端路径。
- 焦点 SSH pane 的 cwd 可驱动左侧远端文件树;展开目录通过 SFTP 枚举,不再把 `/home/...` 伪装成本机 `PathBuf`。
- Explorer 打开远端文件时走 `QuickLook::open_remote`,最多读取 `REMOTE_READ_LIMIT`。
- 文本文件可进入编辑态;保存前用 SFTP stat + 内容 hash guard 检测远端变化,冲突时显示「重新载入 / 取消 / 覆盖远端」,避免静默覆盖。
- 本地文本打开时记录 `TextFormat` 与 `FileGuard`;保存前检测磁盘外部修改/删除,冲突时显示「重新载入 / 取消 / 覆盖保存」,正常写回保留原编码与 LF/CRLF 风格。

### 远端 git 与 hunk 操作

- `remote_git` 通过远端 `git diff --numstat` 渲染 SSH agent 活动栏。
- 改动卡可打开远端 full diff;新增 hunk 解析、单 hunk patch 构造和 `git apply -` stdin helper。
- Quick Look Diff tab 仅在远端 diff(`remote_diff_file` 存在)时,为每个 `@@` 行右侧渲染「接受 / 拒绝」按钮。
- 点击 hunk 按钮后,`QuickLook::apply_hunk` 后台重新拉取当前 `git diff`,经 `remote_git::parse_file_diff` 与 `apply_remote_hunk` 通过 SSH 跑 `git apply --cached -`(接受)或 `--reverse -`(拒绝),patch 走 stdin。
- `DiffLine` 新增 `hunk_index`;`parse_diff` 与 `remote_git::parse_file_diff` 同序计数 `@@`,用单测 `parse_diff_numbers_hunks_in_lockstep_with_remote_file_diff` 保证点击行映射到正确 hunk。
- 应用成功后设置 `diff_dirty` 重拉 diff,发送 `QuickLookEvent::RemoteChangesDirty`,刷新每个 pane 的「本次改动」并 `explorer.mark_stale()`。
- apply 期间 `hunk_busy` 禁用按钮;失败时通过 `hunk_error` 红色横幅显示 `git apply` stderr 并可关闭;开新文件即清。

### 目录 picker 与远端打开文件夹

- SSH pane 的「打开文件夹」不走本机 picker,而是打开当前远端 root 的应用内 SFTP 目录 picker。
- picker 支持父目录、目录过滤、刷新/错误状态和确认/取消;确认后只向目标 SSH pane 发送远端 `cd`。
- `RemoteDirPicker` 泛化为 `PickerSource::{Ssh,Wsl}` + `PickerEntry`;WSL 通过 `\\wsl$\<distro>` 的本地 `std::fs` 列目录。
- `open_folder_should_use_native_picker` 仅对 Host/欢迎页返回 true;SSH 与 WSL 都走应用内导航 picker。
- `fallback_remote_root` 在 cwd 未知时从 `/` 起。

### 与编辑器拆分的边界

- `tn-editor` headless 编辑核心首版随本轮远端编辑写回一起进入发布记录,但编辑器模型、查找/IME、自绘 File/Edit/Diff 的细节统一沉到 [快速预览编辑器近期变更](../修复与优化/快速预览编辑器近期变更.md)。
- SFTP 专题只记录远端读写、远端目录、远端 diff/hunk 和真机 SSH/WSL picker 问题。

## 修复记录

- 终端正文 canvas 在隐藏 / 恢复帧可能回报很小但非零的 bounds,旧 resize 逻辑会把不足一个 cell / row 的区域兜成 `1x1` 并写给 alacritty + ConPTY。新增 `fit_grid_size_from_bounds`,不足一个完整单元格 / 行的临时 bounds 本帧跳过 resize,避免真实 PTY 被缩成 `1x1` 后覆盖历史。
- 鼠标命中收紧为只有 `BODY_PAD` 内侧真实网格矩形才映射成 cell;恢复帧临时 `(0,0)`、padding、右下空白不再 clamp 到左上角或最后一格。
- 文本拖选补 `pressed_button` 兜底,标签切换/最小化吞掉 `mouse_up` 后下一次 move 会结束旧拖选状态。
- `Terminal::snapshot` 在 `display_offset > 0` 时隐藏 live cursor;输入前回到底部改按 `display_offset > 0` 判断,避免滚到历史顶部时仍停在历史视图。
- agent / alt-screen pane 的鼠标滚轮改交给程序自身,普通 shell 才滚 Tn scrollback。
- ConPTY grow resize 增 `ResizeAnchoring`,普通 shell 顶锚定保历史,agent / TUI 底锚定避免把当前主屏帧推进 scrollback。
- `TerminalView::cwd()` 过去只读逐 block cwd,而 SSH 注入脚本只发裸 `OSC 633;P;Cwd` 不建 block,导致 cwd 恒 None。修复为 `cwd()` 兜底 `BlockModel::cwd()`,注入脚本钩好后立即 `__tn_pc` 报当前 cwd。
- 远端目录 picker 顶部增加可点击「`..` 上级目录」行。
- 目录列表改用 `uniform_list` 虚拟化 + `track_scroll`,键盘 `↑↓` 配 `scroll_to_item(Center)`。
- `Workspace::render` 的 `overlay_focused` 曾漏掉 `remote_dir_picker`,导致 picker 打开时焦点每帧被抢回根。修复为 `overlay_focused` 加 `remote_dir_picker.is_some()`。
- `disable_ime` 补 picker/split/layout/palette,避免无 `EntityInputHandler` 的导航浮层被活动 CJK IME 吞导航键。

## 验证 / 状态

- 远端文件树、「打开文件夹」和 picker 键鼠导航已真机确认可用(2026-06-07)。
- SFTP/SSH hunk 仍需端到端回归:hunk 按钮真改远端 working tree、`git apply` 拒绝补丁失败文案、并发/超时。
- 远端目录 picker hunk 头随长行水平滚动仍待处理;极长行需常驻浮起按钮。
