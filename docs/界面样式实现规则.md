## 工作流与守卫规则

1. 图形界面样式改动先看设计真源：`design/panels/`、`design/calm-glass.css`、`design/mockup.html`。
2. 写 GPUI 时优先使用 `col()` / `cola()`、`style.rs` 令牌和 `FontWeight(N.)`；不要硬编码主题色。
3. 组件确切数值以 [组件规格与自动生成数值](界面样式/组件规格与自动生成数值.md) 为准，自动生成区不得手改。
4. UI / GPUI 样式改动至少按范围运行 `cargo test -p tn-ui` 中相关守卫；图形界面改动有 GUI 环境时再用 `cargo run -p tn-app` 与原型并排看。
5. 四道核心守卫是 `token_drift`、`mockup_and_calm_glass_roots_mirror`、`no_hardcoded_theme_colors`、`spec_gen`；细节见 [视觉守卫和生成流程](界面样式/视觉守卫和生成流程.md)。
