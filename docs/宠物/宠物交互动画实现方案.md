## 0 · 渲染基座(所有动画的公共底座)

| 概念 | 值 / 说明 | 出处 |
|---|---|---|
| 像素网格 | **14 列 × 9 行**(y0–y8;y9–11 留空给容器底部) | `BreedSprite.rows` |
| 单格物理边长 | `CELL = 6.0` px | `pet.rs` |
| 容器 | 固定 `100 × 84`(`BOX_W/BOX_H`),恒定占位杜绝布局抖动;欢迎页 `s=2.0` 整体 ×2 | `render` |
| 雪碧图原点 | `SPRITE_X = (100 − 14·6)/2 = 8`、`SPRITE_Y = 84 − 10 − 9·6 = 20`(底部留 10px 给岗台) | const |
| 岗台线 | 雪碧图底缘 `oy + 9·CELL` 处一条 1px `H1` 发丝,左 28px `PH_DIM` 点睛 | `shelf` |
| 绘制方式 | `canvas` 内逐格 `window.paint_quad(fill(...))`,**不进 terminal grid、不写 buffer** | — |

### 0.1 quad 词汇表(帧 = 一串 quad)

`frame_cells()` 返回 `Vec<(x_cell, y_cell, color, dx_px, dy_px, w_scale, h_scale)>`,painter 按下式落点:

```
w  = CELL·s · w_scale
h  = CELL·s · h_scale
qx = ox + x_cell·CELL·s + dx·s + (CELL·s − w)/2     // 居中缩放
qy = oy + y_cell·CELL·s + dy·s + (CELL·s − h)/2
```

**所有动画都只动这几个量**(零新原语):

| 词汇 | 含义 | 典型用途 |
|---|---|---|
| `dx / dy` | 子格像素位移(可负、可超出格) | 跳跃、歪头、尾摆、步态、入场 |
| `w_scale / h_scale` | 格内缩放(<1 收窄,>1 拉高) | 眯眼横缝、呼吸补缝、爱心/zZ 小方块 |
| 水平镜像 | 后处理 `x → 13 − x`、`dx → −dx` | 追尾转圈(翻转即转身) |
| 衬底叠绘 | 同格先铺毛色再叠眼色 | 闭眼方案 A、委屈眼 |
| 岗台线下裁切 | `qy ≥ shelf` 不画、跨线截断 | 探头入场 |

### 0.2 颜色表(全部 `0xRRGGBB`,经 `rgb()`;含 alpha 的走 `rgba(0xRRGGBBAA)`)

| 名 | 值 | 用途 |
|---|---|---|
| 毛色等 | 见 `pixel_color`:`W F4F1E1`/`P FFAAAB`/`K 323F49`/`G F2C867`/`D DAA14A`/`B 303338`/`T C4905E`/`R E36B6B`/`A 7EA0F0`/`C C28F6C`/`N 965F3E`/`U 613922` | 品种像素 |
| `HEART` | `F08C98` | 爱心(Play / 投喂 / 摸头 / 久别 / 提交)|
| `ZZ` | `69748E`(t2 弱文灰) | 打盹 zZ |
| `BISCUIT` | `C99052` | 小饼干 |
| 球 / 骨头 / 毯子 | `E36B6B` / `EDE6D6` / `6A7A95`(暖色**非磷光**) | 玩具摆设 |
| `PH` / `PH_DIM` | 磷光体系 | 岗台点睛、磷光通道 |

### 0.3 帧驱动模型(谁在重绘)

| 驱动 | 周期 | 负责 | 自停条件 |
|---|---|---|---|
| 主 tick | 240ms | 上下文推导、相位翻转、眨眼/微动作调度、气泡消散 | 视图存活 |
| 探头驱动 `spawn_peek_driver` | 30ms | 探头入场上浮 | `peek_until_ms` 过期 |
| 微动作驱动 `spawn_micro_driver` | 80ms | 抓痒抖动 / 追尾镜像(快于 240ms) | `micro` 清空 |
| 入场驱动 `spawn_entrance_driver` | 30ms | 小跑/天降/它先到了 | `entrance` 落定 |
| 投喂驱动 `spawn_feed_driver` | 30ms | 抛接/咀嚼/爱心 + 深夜彩蛋收尾 | `feed_until_ms` 过期 |
| 通道驱动 `spawn_cue_driver` | 30ms | 磷光通道光点飞行 | `cue` 收束 |

- **变化检测**:主 tick 用 `frame_fingerprint(now)`(单 u64 折叠所有窗口/相位),指纹不变就不 `notify` → 空闲零重绘。
- **reduced motion**(`[editor] animations = "off"`):一切动画退化为静帧/终帧,仅姿态与气泡文字保留(逐条见下)。

## 1 · 基础上下文姿态(状态机驱动)

上下文优先级(`tick` 推导,降序):`Drag(需真位移) > Feed > Play > Error > Success > Running > Typing > Hover > Sleep > Idle`。除 Idle/Sleep 外任何状态都刷新「活动时刻」,不计入打盹倒计时。

### 1.1 IDLE — 活着的静止
- **呼吸**:`!lie && breath` 时全身 `body_dy += 1.0`;`breath = (now/1920)%2==0`(~2s 沉浮)。
- **眨眼**:`now ≥ next_blink_ms` → `blink_until = now+160ms`(眯眼一帧,见 §2.1);下次间隔按品种 `blink_min..blink_max` 伪随机(§7)。
- **尾摆**:慢摆,`now % 1440 < 240` 翻一次相位,幅度 = 品种 `tail_amp`(1/2/3px),`dy += phase ? −amp : amp`。
- reduced motion:全静站姿,无呼吸/眨眼/尾摆。

### 1.2 TYPING — 专注
- 触发:`now − LAST_KEY_MS < 1200ms`(键入信号)。
- 立耳:耳格 `dy −= CELL`(竖起 1 格);仅立耳犬(西高地/德牧)有耳格,其余无变化。
- 期间暂停一切主动演出(微动作/庆祝/通道加戏)。

### 1.3 RUNNING — 天然 loading
- 触发:`RUN_COUNT > 0 && now − RUN_START_MS > 1000`(命令 >1s)。
- 小跑步态:腿行(y==8)`dx += (x<7)==phase ? +2 : −2`,每 tick 翻相位,左右脚交替。
- 尾摆幅度 2px。

### 1.4 SUCCESS — 一次性蹦跳
- 触发:`LAST_EXIT_KIND==1 && exit_age < 900ms`。
- 全身 `body_dy = −4.0`(上跳 2 设计像素)。
- 头顶冒像素小心:`(9,−1, OK, dx=2, 0.8×0.8)`;若共情为「提交时刻」(empathy_kind==4)改冒单 `HEART`(§6)。

### 1.5 ERROR — 委屈 3s
- 触发:`LAST_EXIT_KIND==2 && exit_age < 3000ms && !companion_lie`。
- 全身下沉 `body_dy = +2.0`(垂头丧气)。
- 耳朵下垂:耳格 `dy += CELL·0.6`。
- 委屈眼「- -」:见 §2.2。

### 1.6 HOVER — 摸摸眯眼
- 触发:指针悬停本体。
- 眯眼(squint),姿态同 §2.1。

### 1.7 SLEEP — 真·趴下打盹
- 触发:`now − idle_since_ms > 品种打盹阈值`(西施 45s … 德牧 180s);主窗失焦时倒计时本应 ×2(规则,当前未接窗口焦点信号 —— 留作跟进)。
- 趴姿 = **每品种专用 `lie_rows` 网格**(腿收起、肚皮贴岗台、身长 +2、头压低),不是整体下移。
- 闭眼:`squint`(§2.1)。
- 吸气帧(`breath`):上半身(y≤6)`dy −= 1`,y==7 `dy −= 1` 且 `hs=(CELL+1)/CELL` 拉高补缝,肚皮行(y8)不动 —— 背部隆起 1px,无裂缝。
- zZ:`(11,3, ZZ, 0.5×0.5)` + `(12,1, ZZ, 0.65×0.65)`,随 `breath` 上浮 `lift`(−2 / −3px)。
- 唤醒:任何活动直切回站姿(reduced motion 同;趴姿本身是姿态,reduced motion 仍趴)。

### 1.8 DRAG — 拎起换窝
- 触发:`drag.moved == true`(**静止长按不算拖拽**,留给摸头 §5.1)。
- 全身悬空 `body_dy = −10.0`;腿格额外 `dy += 3`(悬空下垂)。
- 实时跟随鼠标更新 `state.right/bottom`(右移→right 减,下移→bottom 减),钳制在窗内。
- 松手裁决:`moved` → 换窝安顿(§5.3)+ 存档;`patted` → 已摸头,什么都不做;否则 → 单击「汪」(§5.0)。

## 2 · 闭眼三件套(审核定稿 · 方案 A)

闭眼被 hover / 眨眼 / 打盹 / 摸头共用;**根因**是旧版把眼格缩成横缝后露出透明洞(深色终端 = 黑窟窿)。

### 2.1 眯眼(方案 A:毛色衬底 + 下缘横缝)
对每个眼格(品种 `eyes` 坐标,落在 `K` 像素上 —— 有结构守卫测试):
1. 先铺**毛色衬底**整格:`(x,y, sp.fur, dx,dy, 1.0×1.0)`。
2. 再叠**眼色 2px 下缘横缝**:`(x,y, 眼色, dx, dy+2.0, 1.0 × 2.0/CELL)`。

每眼 2 个 quad,远看「眯成一条线」。`squint` 触发条件:`(motion && (眨眼窗口 || Hover)) || lie || 摸头窗口`(摸头/趴姿即使 reduced motion 也眯眼)。

### 2.2 委屈眼「- -」(Error 专用,区分眯眼)
1. 毛色衬底整格。
2. 眼色 2px 横条**居中**:`(x,y, 眼色, dx, dy, 1.0 × 2.0/CELL)`(贴下缘 vs 居中 = 眯眼 vs 委屈的差别)。

### 2.3 哈欠口型(深夜问候 / 投喂 / 它先到了察觉段)
鼻下张口:在 `(6, eyes.y+1)` 与 `(7, eyes.y+1)` 各画一块深色 `K`,宽 0.8 格,高 = 张口度:
- 张口 `0.7`(窗口剩余 > 250ms)→ 半闭 `0.3`(剩余 ≤ 250ms);总 850ms(张口 600 + 半闭 250)。
- 站姿时口型再 `dy += 1`(贴鼻下);配合眯眼方案 A。

## 3 · 探头入场(岗台线下升起)

- 窗口:`PEEK_MS = 500ms`;30ms 驱动。
- 升起量:`p = 1 − (peek_until−now)/PEEK_MS`,`ease = 1 − (1−p)³`(ease-out),全身 `body_dy += (1−ease) · 9·CELL`(从线下 9 格处升到 0)。
- **裁切**:`peek_clip` 时,painter 对 `qy ≥ shelf` 的格不画、跨线的格截断高度 → 逐段露出耳尖→头→眼→全身。
- 触发:首次现身、换形象、显隐切换、欢迎页 1×↔2× 切换;也是每日见面「式一」(§4.1)。
- reduced motion:`peek()` 直接返回,直切落定(无升起)。

## 4 · 每日见面(规则 A · `begin_daily_meeting`)

当日(本地自然日)首次现身触发;`last_seen_date != 今天` 即「新的一天」。新的一天:`first_met` 初始化、`days_together += 1`、存档。本地日期/时刻走 Win32 `GetLocalTime`(`local_now`)。

### 4.0 入场池四式(性格加权、连续两次不重样)
权重见 §7 `entrance_weights`;`pick_entrance(full)`:新的一天用四式,日常显隐只用轻量二式(探头/小跑)。

| 式 | 时长 | 画法 |
|---|---|---|
| **式一 · 探头** | 500ms | 走 §3 peek 通道(`peek_until` + peek 驱动)。 |
| **式二 · 小跑** | 600ms | `p=el/600`,`ease=1−(1−p)³`;全身 `body_dx += (1−ease)·44`(起步在右 44px 滑入到位)。 |
| **式三 · 天降** | 520ms | `p=el/520`;前 78%:`q=p/0.78`,`body_dy += (1−ease(q))·(−60)`(从高处落);后 22%:`q2=(p−0.78)/0.22`,`body_dy += 1.5·(1−q2)`(落地三帧律回弹)。 |
| **式四 · 它先到了** | 1200ms | 前 55%:`already_lie` 渲染**趴睡**(lie 网格 + 闭眼);55–78%:`already_yawn` 出哈欠口型(§2.3);其后:起身站姿。= 现身即睡 → 察觉 → 哈欠 → 起身。 |

`Entrance::duration_ms()` 给各式时长;`spawn_entrance_driver`(30ms)到点 `entrance=None` 落定。

### 4.1 落定 + 时段问候(`greet`)
- 时段词(`DayPart`,本地时钟):清晨 5–8「早!」/ 白天 9–17「来啦!」/ 傍晚 18–22「晚上好」/ 深夜「夜深了…」。
- 新的一天:状态栏席位短显「`名字 · 第 N 天`」5s(`day_badge_until = now+5000`)。
- 里程碑(`milestone_word`):第 7/30/100/365 天替换问候为「第 7 天!」「第 30 天!」「100 天啦!」「一周年!」。
- 深夜:`greet` 追加哈欠帧(`yawn_until = now+850`)+ `quiet_hours_seen += 1`(§7 记忆)。
- reduced motion:跳过入场,只出问候气泡(静音路径)。

> 气泡通用规格:`(−18, right −2)` 处 `L4` 底 + `H2` 边 + `R_CARD` 圆角 + mono 11 `T0` 文,2s 自散、新顶旧。**名字永不进气泡**(词表封闭)。

## 5 · 主动互动(规则 G)

优先级:`Drag > 摸头判定 > Feed/Play > 叫名字/换窝安顿 > 上下文姿态`。均不占主动演出预算,但守三帧律 / reduced motion / 工作神圣边界。

### 5.0 单击「汪」(既有)
原地按下松手(未移动、未长按):`bark()`。歪头杀:头部行(y≤5)`dx+=3, dy+=1.5`,600ms;气泡 = 品种口癖。**德牧无口癖** → 不冒泡,改点头:`nod_until=now+500`,头部行 `dy += CELL·0.5`。

### 5.1 摸头(长按 350ms)
- 判定:左键按下即起 350ms 定时器;到点若 `drag` 仍在、`!moved && !patted` → `patted=true` + `pat()`。移动超阈值(`|dx|+|dy|>4`)则转拖拽,不触发。
- 演出(`pat_until = now+1200`):头部行 `dy += CELL`(头低 1 格)+ 眯眼(squint 含摸头窗口)+ 尾巴快摆 + 头顶单 `HEART`(`(9,−1,HEART,0.8)`)。
- 2s 内重复只刷新 `pat_until`(延长开心),不叠气泡(`extend` 时不再记忆/不再额外心)。
- reduced motion:仅切眯眼终帧(头低/爱心走 `motion` 门控,squint 不门控)。

### 5.2 叫名字(菜单,10s 冷却)
- `call_name()`:`now − last_call < 10_000` 则忽略;否则 `call_until = now+900`。
- 演出:耳格 `dy −= CELL`(竖耳)、其余头行 `dy −= 1`(抬头);全身 `body_dx`:`cf=900−(call_until−now)`,`off = cf<300? cf/300·CELL : cf<600? CELL : (1−(cf−600)/300)·CELL`,`body_dx −= off`(朝左=用户方向迈 1 格再回)。无长句气泡(词表无字)。

### 5.3 换窝安顿(拖拽松手且移动)
- `settle(quadrant)`:`settle_until = now+700`;记 `favorite_perch` 象限(§7)。
- 演出:回弹 `sf=700−(settle_until−now)`,`sf<160` 时 `body_dy += 2·(1−sf/160)`;`sf≥200` 时头行 `dy += CELL·0.7`(低头嗅地)。
- 象限由松手位置算:中心相对视口左/上半 → 0 左下 / 1 右下 / 2 左上 / 3 右上。

### 5.4 摆个玩具(菜单:球 / 骨头 / 毯子 / 收起)
`set_toy(Option<Toy>)` 存 `toy_choice`;**贴地渲染,不随身体动,放在镜像之后**(免追尾跳位):
- 球:`(12,8, E36B6B, dy=−1, 0.9×0.9)`。
- 骨头:三块 `EDE6D6` —— `(11,8,dx1,0.45)`、`(12,8,dx0,0.9×0.45)`、`(12,8,dx4,0.45)`。
- 毯子:`x∈1..5` 各 `(x,8, 6A7A95, dy=+3, 1.0×0.5)`(蓝灰软垫贴岗台)。
- 无货币 / 无稀有度 / 无收集;同一时间最多一个,可随时收起。

### 5.5 久别亲近(≥3 自然日)
`note_affection` 返回 `miss`(距 `last_affection_at` ≥3 天)→ `maybe_extra_heart`:`extra_heart_until=now+1300`,在常规演出外**多冒一颗心** `(11,−2,HEART,0.7)`。无「你终于来了」愧疚文案。

## 6 · 工作共情(规则 E · 结构化事件派生,零输出扫描)

信号在 io reader 线程经 `SessionRunGuard` 喂入:`command_line(cmd)` 记结构化命令行、`command_start` 记起点、`command_end(exit)` 分类。`signal_command_end` 维护:

- `SUCCESS_STREAK` / `FAIL_STREAK`;`run_ms = now − start`;`cmd_is_commit` = 首两词 `git commit`/`git push`。
- 一次性事件槽 `EMPATHY_KIND`(1 三连胜 / 2 大功告成 >60s / 3 翻盘 / 4 提交)+ `EMPATHY_MS`,优先级:提交 > 大功告成 > 翻盘 > 三连胜。

`poll_empathy`(主 tick)消费事件槽,受**共享冷却 ≤2 次/10 分钟**(`celebrate_times`):

| 事件 | 演出(复用既有帧) |
|---|---|
| 三连胜 | 走 Play 双爱心蹦跳(`play_until`,不冒泡) |
| 大功告成 | Play 双爱心 + 品种口癖气泡 |
| 翻盘 | 普通 Success 跳(exit0 已触发)+ 气泡「!」 |
| 提交时刻 | Success 跳,小心改冒**单爱心**(`empathy_kind==4` → `(9,−1,HEART)`) |
| 连败陪伴 | `FAIL_STREAK≥3` → `companion_lie`:就近**趴下**(lie 网格,**无 zZ**)+ 气泡「呜…」仅一次;下个 exit0 自然起身。**不演委屈** |
| 深夜陪工 | 深夜 Running:小跑帧减速(规则;按需在步态相位减半实现) |

`empathy_kind` 仅在 Success/Play 窗口内生效,过后清零。

## 7 · 性格系统(规则 D · 表即实现)

`Personality` 常量表逐犬一行,**只调参数与权重,无品种私有代码路径**。`Micro` 池顺序 = `[抓痒, 舔爪, 追尾, 竖耳, 望屏外, 伸懒腰]`;`Entrance` 池顺序 = `[探头, 小跑, 天降, 它先到了]`。

| 品种 | 眨眼(ms) | 尾摆 | 打盹(ms) | 口癖 | 微动作权重 | 入场权重 |
|---|---|---|---|---|---|---|
| 西高地 | 4000–6000 | 1px | 120000 | 汪! | `[1,1,1,3,3,1]` | `[3,1,1,1]` |
| 金毛 | 5000–8000 | 2px | 90000 | 汪~ | `[1,3,1,0,1,3]` | `[1,3,1,1]` |
| 德牧 | 6000–9000 | 1px | 180000 | (点头) | `[1,1,1,4,1,1]` | `[2,2,1,1]` |

## 8 · 活物引擎(规则 C · idle 微动作)

调度(主 tick,仅纯 Idle、无探头、无气泡):`now ≥ next_micro_ms` 且 `micro_budget_ok`(≤2 次/分钟,`micro_times` 环)→ `pick_micro()` 加权选一个,`micro_until = now+1500`,排下一拍 `next_micro = now + 20000 + rand(0..40000)`(20–60s)。离开 Idle 立即弃帧(`micro=None`)。`sub = (now/110)%2` = 快速子相位。

## 10 · 磷光通道(规则 J · `PetSpatialCue` + `channel_overlay`)

> v2.3 标志性动效(**改版二 · 小狗叼开空间裂隙**):关键 UI 不凭空弹出,而是**一只像素小狗从岗台跃出、跨屏奔到真实 UI 接缝,落在接缝上蹲身蓄力、猛地起身把一道磷光锯齿裂隙拉开,UI 就从裂隙里展开**。小狗 = Tn 空间的钥匙,它**亲手**打开空间(不再是抽象光点飞行)。**UI 本体绝不等待此动画**(workspace 发完 cue 立即响应)。

### 10.1 事件与节律(`spatial_cue`)
- `PetSpatialCue`:UI 控制层显式发,只含转场语义,**不读终端内容**。开门类(`is_open`)/ 召回类 / 切换类(`is_switch`)。
- 优先级闸:拖拽/摸头/投喂/逗弄期间直接放弃本次(UI 已自响应,不欠动画)。
- 节律:同一时刻只一个 cue(新替旧);`last_cue` 距今 ≥30s → **完整 1100ms**,否则**短版 640ms**;切换 280ms。`spawn_cue_driver`(30ms)到点 `cue=None`。
- reduced motion:不奔跑、不拉裂隙,锚点 3px 磷光点 `(now/160)%2` 闪一下(静态换岗)。

### 10.2 层级(关键)
宠物 overlay 是 **workspace 最顶层子节点** → 小狗/裂隙盖过 Quick Look / 命令面板的 scrim(否则被压暗 ~38% 看不见)。宠物根穿透,不抢焦点;cue 驱动 `notify` 重绘宠物自身 → 动画不冻结。

### 10.3 全屏画法(`channel_overlay`,纯绘制 canvas)

**起跑点 / 落点。** 起跑点 `(ex,ey)` = 岗台小狗头顶中心:`ex=(vw−right−box_w)+box_w/2`,`ey=(vh−bottom−box_h)+SPRITE_Y·s`。落点在接缝 `(ax,ay)` **正上方**:小狗中心 `base_cy=max(ay−3.5·DCELL, 22)`,使其脚行(gy8)正好踩在裂隙顶端 `ay`。锚点 `(ax,ay)` 按 cue(视口近似,够不到精确控件几何时按方位):

  | cue | 锚点 |
  |---|---|
  | `OverlayOpen`(命令面板) | `(vw/2, TITLEBAR_H+4)` 顶缘 |
  | `QuickLook*` | `(ql_left, vh/2)` —— Quick Look 居中卡**左侧边**(小狗落在侧边,裂隙长在接缝上不盖正文) |
  | `SplitCreate` | `(vw/2, vh/2)` 新分隔线 |
  | `WelcomeToWorkspace` | `(vw/2, vh·0.46)` |
  | `RemoteReconnect` | `(vw/2, vh·0.30)` |
  | `GhostEnter/Exit` | `(vw/2, 44)` 屏顶召唤线 |

**像素小狗投影 `paint_dog(cx,cy,dip,ghost)`。** 复用本品种静态 `rows[9]`(磷光通道里不眨眼/不微动作),格距 `DCELL=4`(岗台本体 `CELL=6`)。雪碧图中心列 7、中心行 4.5 → `ox=cx−7·DCELL`,`oy=cy−4.5·DCELL`,逐格 `pixel_color(ch)` 画 `DCELL×DCELL`。`ghost=Some(a)` 时整只画成 `rgba((PH<<8)|a)` 半透明磷光残影(奔跑拖影 / 收尾消散),否则画**真实毛色**(rule J3:仍是像素狗,不是光团)。

**奔跑段** `cp∈[0, tdur)`(`tdur=switch?0.30:0.28`):`tp=cp/tdur`,`te=ease(tp)`,`ease=1−(1−t)³`。位置 `run=(ex+(ax−ex)te, ey+(base_cy−ey)te − 60·4te(1−te))`(抛物线上拱跨屏)。**奔跑拖影**(非切换):`k=1..2` 在 `tk=ease(tp−0.07k)` 处画 `ghost=max(100−35k,40)` 的残影狗。

**蹲身蓄力 → 猛起拉裂隙**(到位 `tp≥1` 后,`body_shift` 加到 `base_cy`,+下沉/−上提):
  - `cp<0.35`:`0`(刚到位、站定)
  - `cp∈[0.35,0.52)`:`+5·(cp−0.35)/0.17`(蹲下 0→+5,蓄力)
  - `cp∈[0.52,0.70)`:`5−16·(cp−0.52)/0.18`(猛地起身 +5→−11,把裂隙撑开)
  - `cp∈[0.70,0.82)`:`−11`(撑到最开,定格)
  - `cp≥0.82`:`0`

**裂隙 `rift(o,a)`**(画在小狗脚下接缝处):张开度
  - 切换:`o=sin(((cp−0.20)/0.5)·π)`(0→1→0 脉冲)
  - 开门:`cp<0.42→0`;`[0.42,0.70)→ease((cp−0.42)/0.28)`;`≥0.70→1`(与"猛起"同步)
  - 召回:`cp<0.35→1`;`[0.35,0.66)→1−ease((cp−0.35)/0.31)`;`≥0.66→0`(拉合)

  画法:从 `ay` 向下 `segs=(3+10o)` 段,每段 `y=ay+4i`、`x=ax−0.75±(4o)`(`i` 奇偶交替抖动)、`1.5×4px`;首尾段用 `faded(edge=clamp(150a,30,150))`,中间用 `faded(core=clamp(255a,40,255))`;握点再加 `(4+5o)px` 亮核。整体不透明 `a`:开门 `cp<0.85→1` 否则线性淡出(UI 已自显、残影散去);召回/切换恒 1。

**收尾消散。** `exit=clamp((cp−0.82)/0.18,0,1)`,`exit>0` 时小狗以 `ghost=max((1−exit)·200,1)` 化作磷光残影散去(本体从未离开岗台,这里只散投影)。

**绘制顺序:** 裂隙(脚下)→ 奔跑拖影 → 小狗本体(压最上)。

**纪律:** 只用磷光线/裂隙/残影 + 真实毛色像素狗;无烟雾/粒子/全屏遮罩;裂隙峰值 ≤ 召唤线、绝不外发光;小狗不变形、不变光团。

### 10.4 已接线发射点(`workspace.rs`)
`OverlayOpen`(命令面板)· `QuickLookOpen/Close/Switch`(资源管理器点开 + 键盘开关 + ↑↓ 换文件)· `SplitCreate`(分屏)· `WelcomeToWorkspace`(欢迎页启动)· `RemoteReconnect`(SSH 重连)。
**未接线**:`GhostEnter/Exit` —— 幽灵终端是独立顶层窗口(`quick_terminal.rs`),跨窗,留作远期(板 H)。

## 12 · 节律预算 / 可访问性(硬约束,逐条对应实现)

- 主动演出预算:微动作 ≤2/分钟(`micro_times`)、庆祝 ≤2/10 分钟(`celebrate_times`)、气泡 2s 自散不可叠加(新顶旧)。
- 用户主动互动不占预算;叫名字 10s 冷却;久别每天最多 1 次额外心(由日期判定天然限频)。
- Typing/Running/浮层打开 → 暂停一切主动演出(上下文姿态本身除外);通道在浮层打开瞬间触发一次后不继续加戏。
- reduced motion:微动作/庆祝/探头/交替闪/入场/投喂演出/通道飞行全停;保留静态姿态、趴姿、气泡文字、通道 1px 点闪。
- 可关闭:关闭后状态栏保留 `PET · OFF` 席位,功能零损失;关闭后不更新互动记忆。
- 一切上下文来自结构化事件(OSC 133/633、process lifecycle、本地时钟、UI 控制层 cue);**命令行文本可读(结构化字段),输出文本永不扫描**。
- 品种 / 名字 / 落点 / 档案 / 互动记忆入用户配置目录 `pet_state.json`,不进项目配置、不跟随仓库。
