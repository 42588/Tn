# 宠物系统规则(v2 · 陪伴版)

> 2026-06-12 覆写 v1。v1 把宠物做成了「能感知终端的状态指示物」;v2 的目标是把它升级为
> **用户每天期待见面的工作伙伴** —— Tn 最讨人喜欢的部分。
> 已落地基座(状态机、姿态、互动、趴姿、探头)全部保留,新规则叠加其上。
>
> **v2.1(同日修订,四项定夺)**:① 主宠唯一 + 领养命名(新增规则 0,随机刷新废除);
> ② 升级解锁模式**否决**(过度合理化效应:外在奖励排挤内在喜爱,狗会变成进度条);
> ③ 动画立「关键互动三帧律」(预备/主体/收尾);④ 玩具「摆不买」(无货币,远期岗台小物)。
>
> **v2.2(同日修订,互动记忆版)**:用户选择重型互动方案,新增主动互动、互动记忆和记忆透明三章。
> 记住的是宠物相关偏好,不是用户行为画像;只改变演出权重和小细节,不提醒、不评价、不制造负担。

## 北极星

用户清晨打开 Tn 的第一眼,是想看看自己的狗 —— 它探头出来、认出今天又见面了、摇着尾巴等一块小饼干。
衡量标准只有一条:**用户愿意把宠物开着**(可关闭是底线,被关掉是失败)。

## 三条公理

1. **陪伴优先,工作神圣**:宠物是伙伴不是仪表,但它永远不打断工作 —— 不遮挡、不弹窗、不抢焦点、不在你打字和跑命令时加戏。
2. **活着的错觉来自微小与随机**:让宠物显得活的不是大动画,是 30 秒一次的抓痒、偶尔竖起的耳朵、看向屏幕外的一瞥。所有演出都是几个像素的事。
3. **期待靠惊喜,绝不靠愧疚**:没有饥饿值、没有等级条、没有"三天不来狗就难过"。缺席零惩罚,回来永远是开心的重逢 —— 想见它是因为喜欢,不是因为欠它。

## 新规则 0 · 主宠:领养与命名(归属之锚,v2.1)

依恋成立的两个心理条件:**身份连续性**(每次见到的是同一个它)与**不可替代性**。
随机刷新两条全毁(可替换 = 壁纸不是宠物);升级解锁走向反面(狗变进度条)。
因此:**一只、命名、唯一**。

| 项 | 规则 |
|---|---|
| 领养仪式 | 首次启用宠物 → 一次性领养卡(浮层家族样式):七犬品种架直选,或「交给缘分」随机一次 → **起名字**(中文 OK,≤8 显示字,默认 = 品种名)。整个流程 ≤30 秒,可跳过(跳过 = 缘分狗 + 默认名) |
| 唯一性 | 主宠只有一只;名字、羁绊档案、窝的位置、饼干记忆都属于它 |
| 名字的去处 | 状态栏席位(「豆豆 · IDLE」替代品种名)· 右键菜单档案行(「豆豆 · 西高地 · 在一起第 N 天」)· 欢迎页 2× 标签;**不进气泡**(词表封闭不破) |
| 改名 | 右键菜单「改名…」随时可改,档案不动(改名是亲昵,不是重置) |
| 换形象 | 「换个形象…」(品种架直选)保留:名字与档案**延续** —— 像换发型,零失去焦虑、零愧疚弹窗;**「随机刷新」从菜单废除**(持续性老虎机会瓦解身份感) |
| 缘分残留 | 「交给缘分」只在领养卡里存在 —— 随机的魅力留给初遇,不留给日常 |
| 串门偶遇(远期) | 里程碑/节日当天,另一只随机品种"路过"岗台串门 10s(两狗互嗅后离开):让其余六犬保持鲜活,**无收集、无图鉴、无拥有** |

## 新规则 A · 每日见面(仪式核心)

每个自然日**首次现身**是一天中最隆重的演出 —— 这是「每天期待」的锚点。

| 项 | 规则 |
|---|---|
| 触发 | 当日(本地日期)第一次从隐藏/启动进入可见;每天最多一次 |
| 演出 | **入场池四式随机**(v2.1+,回应「出现方式太单调」):式一岗台探头 / 式二小跑入场(右缘跑进窝)/ 式三天降落地(顶落+三帧律回弹)/ 式四「它先到了」(现身即趴睡 → 察觉你 → 哈欠 → 起身摇尾);按性格加权、连续两次不重样 → 落定后开心两连跳(0.8s)→ 问候气泡(时段词表)→ 状态栏席位短暂显示 `第 N 天`(5s 后回常态)。日常显隐只用轻量二式(探头/小跑) |
| 时段感 | 问候词随本地时钟换:清晨(5-9)/白天(9-18)/傍晚(18-23)/深夜(23-5);深夜问候后补一个哈欠帧(眯眼+张嘴 1 格),**不说教**,一个哈欠就是全部提醒 |
| 补货 | 见面时今日小饼干补货 1 块(见规则 B) |
| 静音路径 | reduced motion → 只出问候气泡;关闭宠物 → 一切不发生 |

## 新规则 B · 小饼干(每日投喂仪式)

一天一块、不囤积、不货币化 —— 一个纯粹的"给它点好的"仪式。

| 项 | 规则 |
|---|---|
| 入口 | 右键菜单首组「给小饼干 · 今日还有 1 块」;喂过后变灰「今天吃过啦」 |
| 演出 | 像素饼干(4×4,暖棕)从菜单位置抛向狗 → 狗起跳接住(Success 跳同款)→ 咀嚼 2 帧(腮部 1 格交替)→ 冒一颗爱心 + 尾巴快摆 3s |
| 库存 | 每自然日 1 块,当日有效,不累计、不补偿、不提示"你还没喂" —— 忘了就忘了,明天照常有 |
| 深夜彩蛋 | 深夜(23-5)投喂 → 吃完接一个哈欠,然后主动进入趴姿(它陪你,但它困了) |
| 边界 | 不引入第二种**消耗性**道具;饼干不影响任何数值(没有数值)。岗台玩具(远期)是**摆设**:自由挑选、无货币、不消耗,见「不采用」与分期 |

## 新规则 C · 活物引擎(idle 微动作)

Idle 不再是静止 + 眨眼,而是有生活的小狗。

| 项 | 规则 |
|---|---|
| 节拍 | Idle 持续 20-60s 随机触发一个微动作,时长 ≤1.5s,做完回 Idle;打盹(Sleep)期间停止 |
| 动作池 | ① 抓痒(后腿格上抬 2px 快速抖 3 次)② 舔爪(头低 1 格 + 前爪上抬)③ 追尾转圈(整身水平镜像翻转 2 次 — 翻转即"转身",零新网格)④ 竖耳听声(立耳犬:双耳 +1 格保持 0.6s)⑤ 望屏外(头部整体右移 1 格 0.8s,"看见窗外有鸟")⑥ 伸懒腰(前低后高:头部行 +1px、尾部行 -1px 保持 0.5s) |
| 权重 | 按品种性格加权(见规则 D),同一动作不连续出现两次 |
| 专注保护 | Typing/Running/Error/浮层打开期间一律不触发;微动作进行中如上下文变化,立即弃帧切换 |
| 打扰预算 | 微动作无声、无气泡、不离开栖位;reduced motion 下整章停用 |

## 新规则 D · 性格系统(七犬七脾气)

品种不再只是换皮 —— 参数化性格让"换一只狗"等于"换一种陪伴感",这是品种收集欲的全部来源(没有图鉴、没有成就,好奇心本身就够)。

| 品种 | 性格 | 眨眼间隔 | 尾摆幅度 | 打盹阈值 | 偏好微动作 | 口癖(气泡) |
|---|---|---|---|---|---|---|
| 西高地 | 精神好奇 | 4-6s(勤) | 1px 快 | 120s(坐不住) | 竖耳听声、望屏外 | 「汪!」 |
| 金毛 | 温和可靠 | 5-8s | 2px 大尾 | 90s | 伸懒腰、舔爪 | 「汪~」 |
| 德牧 | 专注警觉 | 6-9s(稳) | 1px 小 | 180s(站岗) | 竖耳听声(高频) | (不出声,点头 1 格) |
| 比熊 | 开心活泼 | 4-7s | 2px 弹 | 90s | 追尾转圈(高频) | 「汪汪!」 |
| 马尔济斯 | 乖巧安静 | 5-8s | 1px 轻 | 60s(爱打盹) | 舔爪、伸懒腰 | 「…汪」 |
| 西施 | 慵懒陪伴 | 6-10s(眯) | 1px 慢 | **45s**(最爱睡) | 伸懒腰(高频)、舔爪 | 「呼…」 |
| 泰迪 | 俏皮黏人 | 4-6s | **3px 最欢** | 120s | 追尾转圈、抓痒 | 「汪!汪!」 |

- 性格只调**参数与权重**,不加品种私有代码路径;表即实现(常量表)。
- 单击「汪」气泡、Play 反应等一切文案走口癖列。
- v2.1:性格差异服务于**领养时刻的选择**与「换形象」的新鲜感,不再暗示轮换收集。

## 新规则 E · 工作共情(它看得懂你的一天)

宠物对工作节奏有恰到好处的反应 —— 全部来自既有结构化事件(块命令行 + exit code),零文本扫描。

| 时刻 | 触发(结构化) | 演出 |
|---|---|---|
| 三连胜 | 连续 3 条命令 exit 0 | Success 跳升级:双爱心(Play 同款心,只此一跳) |
| 大功告成 | 运行 >60s 的命令 exit 0(编译/测试通过的典型形状) | 大庆祝:连跳两次 + 双爱心 + 口癖气泡 |
| 翻盘 | exit ≠0 之后的第一个 exit 0 | 普通 Success 跳 + 气泡「!」(它也松了口气) |
| 连败陪伴 | 连续 ≥3 条 exit ≠0 | **不演出**:就近趴下(趴姿)陪着,气泡「呜…」仅一次;下个 exit 0 自然起身 —— 挫败时安静陪伴,不添乱 |
| 提交时刻 | 命令行以 `git commit`/`git push` 开头且 exit 0(来自 OSC 633 E 结构化命令行) | 小庆祝跳 + 单爱心 —— 一天的成果时刻应该有人摇尾巴 |
| 深夜陪工 | 深夜(23-5)且 Running | 小跑帧减速一半(它困了但还在跑),不打哈欠不打断 |

- 共情演出共享冷却:**每 10 分钟最多 2 次**主动庆祝,超出静默(惊喜稀释自保护)。
- 永不评判:没有"你错太多了"类负反馈;连败时它只是趴下陪你。

|---|---|

| 换形象 | v2.1 语义收紧:主宠是**同一个体**,「换个形象」只改外观 —— 名字、`days_together`、`first_met`、饼干记忆全部延续;陪伴从领养那天起连续计数 |

## 新规则 G · 主动互动(把回应升级为相处,v2.2)

v2.1 已有单击、双击、拖拽和投喂;v2.2 加深的是**用户主动触发的亲密互动**。
它们不占主动演出预算,但必须遵守三帧律、reduced motion 和工作神圣边界。

| 互动 | 触发 | 演出 | 边界 |
|---|---|---|---|
| 摸头 | 长按宠物 350ms,且指针移动未超过拖拽阈值 | 头低 1 格 → 快乐眯眼 → 尾巴快摆 + 单爱心;2s 内重复只延长开心,不叠气泡 | Typing/Running/浮层打开时不触发长按判定;reduced motion 只切到眯眼终帧 |
| 叫名字 | 右键菜单「叫名字」或未来命令面板同名动作 | 竖耳/抬头 → 朝用户方向小跑 1 格 → 回窝坐好;若正在 Sleep,先睁眼再起身 | 名字不进气泡长句;10s 冷却,避免被刷成噪音 |
| 换窝安顿 | 拖拽换窝后松手且移动超过阈值 | 落地回弹 → 嗅嗅地面 → 坐好或趴好;落在常用窝区域时尾巴多摆 1s | 小位移仍按单击;落点继续只入用户配置 |
| 摆玩具 | 右键菜单「摆个玩具…」:球 / 骨头 / 小毯子 / 收起 | 玩具出现在岗台边;Play 收尾时可叼起又放下,或趴在小毯子旁 | 无货币、无稀有度、无收集提示;玩具是布置,不是经济系统 |
| 久别亲近 | 距上次亲密互动 ≥3 个自然日后的第一次摸头/叫名字/逗弄/投喂 | 常规演出后额外一颗爱心 | 不出现"你终于来了"等愧疚文案;久别只加开心,不加压力 |

- 主动互动优先级:Drag > 摸头判定 > Play > 叫名字/投喂/换窝安顿 > 上下文姿态。
- 摸头、叫名字、换窝安顿可先用既有 dy/dx/镜像/眯眼/爱心词汇实现;玩具三件和叼回动作需要 SHEET 05 板 G 审核稿。
- 用户主动触发的互动可以更有手感,但不能扩大宠物占位、不能遮挡输入和选区。

## 新规则 H · 互动记忆(偏好,不是画像,v2.2)

宠物可以记住少量相处习惯,让它显得"懂你"。记忆只来自宠物交互本身,不扫描终端输出、不读取文件、不分析命令内容。

| 记忆项 | 来源 | 用途 | 限制 |
|---|---|---|---|
| `favorite_interaction` | 摸头/投喂/逗弄/叫名字的衰减计数 | 调整主动互动收尾:爱摸头 → 更容易眯眼趴低;爱逗弄 → Play 后偶尔叼玩具回来;爱叫名字 → 叫名字回应更快 | 不展示次数;只展示一句偏好标签 |
| `favorite_perch` | 拖拽落点按屏幕四象限粗分 | 拖回常用区域时触发「认窝」安顿;欢迎页回到工作区时优先回当前窝 | 只记区域,不记精确坐标历史;当前坐标仍由显式拖拽决定 |
| `toy_choice` | 用户显式选择的当前摆设 | 决定岗台小物与 Play 收尾动作 | 不是收集品;同一时间最多一个;可随时收起 |
| `quiet_hours_seen` | 深夜(23-5)首次见面或深夜投喂的低精度计数 | 深夜入场更偏向「它先到了」和趴姿;深夜投喂后更容易哈欠 | 不提醒睡觉、不评价作息;只影响概率 |
| `last_affection_at` | 最近一次摸头/叫名字/逗弄/投喂的本地日期 | 判定久别亲近额外爱心 | 日期级即可,不需要精确时间线 |

- 互动记忆采用小型衰减计数,长期上限固定;实现上不保留完整事件日志。
- 记忆只改变**权重和收尾**,不解锁内容、不生成任务、不影响宠物健康或状态。
- 记忆驱动演出必须搭在已有触发上:用户摸头、叫名字、Play、投喂、拖拽后才显现;宠物不会主动弹出"我记得"。
- 数据仍写入用户配置目录的 `pet_state.json`,不进项目配置、不跟随仓库。

## 新规则 I · 记忆透明与清除(不神秘,不绑架,v2.2)

互动记忆要可见、可解释、可清除。用户应该觉得它贴心,而不是被暗中观察。

| 项 | 规则 |
|---|---|
| 菜单展示 | 右键菜单档案组第二行显示「它记得:爱摸头 · 小球 · 右下角小窝」;最多 3 个短标签,无数据时显示「还在认识你」 |
| 标签词表 | `爱摸头` / `爱逗弄` / `爱叫名` / `爱饼干` / `小球` / `骨头` / `小毯子` / `左下角小窝` / `右下角小窝` / `左上角小窝` / `右上角小窝` |
| 重置入口 | 右键菜单「重置互动记忆…」,二次确认后清空 `favorite_interaction`、`favorite_perch`、`toy_choice`、`quiet_hours_seen`、`last_affection_at` |
| 不清内容 | 重置互动记忆不清名字、品种、首见日期、在一起天数、累计饼干、当前窝坐标和宠物开关 |
| 可访问性 | 屏幕阅读器只读短标签,不朗读内部计数;重置入口键盘可达 |

- 记忆透明行只在菜单里出现,不进状态栏常驻、不进气泡、不做红点。
- 重置后宠物不会"失落"、不会挽留,只回到「还在认识你」。

## 文案词表(气泡全集,≤6 字铁律)

| 场景 | 词 |
|---|---|
| 清晨见面 | 「早!」 |

## 动画手感律(v2.2 · 质量标准)

生命感来自动作的**预备、跟随与次级运动**,不来自动画数量(Heider-Simmel:三角形靠运动时机就能让人共情)。
粘性来自用户主动触发瞬间的"手感"(juice):被动演出保持克制,**主动反馈往死里打磨**。

1. **关键互动三帧律**:一切用户触发的动作(单击/双击逗弄/投喂/领养落地)必须有
   「预备 → 主体 → 收尾」三段 —— 起跳前蹲 1px(挤压)、空中拉伸、落地回弹 1px;
   单帧直切的互动反馈视为未完成。
2. **次级运动**:身体动作时耳朵/尾巴慢半拍跟随(+1 tick 相位差),用既有 dy/相位即可。
3. **缓动**:一切位移用 ease-out(已有 cubic),禁线性;时长 120-500ms 区间。
4. **像素纪律**:全部在 14 列网格 quad 词汇内(dy/dx/镜像/h·w_scale),无新原语;
   reduced motion 直切终帧。
5. **先审后做**:新帧一律先进 SHEET 05 审核稿(板 E 流程),用户定夺后实现。

> 升级清单(对既有动作补三帧):Success 跳、Play 蹦跳、探头落地 —— 排入 P1。
> v2.2 新增审核清单:摸头眯眼、叫名字转头、换窝安顿、球/骨头/小毯子与叼回收尾 —— 排入 SHEET 05 板 G。

## 节律与打扰预算(硬约束)

- 主动演出(非用户触发的一切:微动作、共情庆祝)预算:**微动作 ≤2 次/分钟、庆祝 ≤2 次/10 分钟、气泡 ≤4 条/小时**。
- 用户主动触发互动不占主动演出预算,但同类互动 2s 内合并;叫名字冷却 10s;久别亲近每天最多触发 1 次。
- 记忆驱动变化只能作为既有主动互动的收尾,不能变成独立提醒、弹窗或状态栏常驻。
- Typing / Running / 浮层打开 / alt-screen / 全屏专注 → 暂停一切主动演出(上下文姿态本身除外)。
- 主窗失焦 → 宠物也休息(进入打盹倒计时加速 ×2)。
- 所有气泡 2s 自散、不可叠加(新顶旧)。

## 可访问性与实现边界(沿 v1,逐条有效)

- 可关闭;关闭后状态栏保留 `PET · OFF` 席位,功能零损失。
- reduced motion:微动作/庆祝/探头/交替闪全停,保留静态姿态与气泡文字。
- 屏幕阅读器只读「宠物:品种,第 N 天,已显示/已隐藏」。
- 不进 terminal grid、不写 buffer、不遮 prompt/光标/选区/浮层;固定容器占位。
- 一切上下文来自结构化事件(OSC 133/633、process lifecycle、本地时钟);**命令行文本可读(结构化字段),输出文本永不扫描**。
- 品种、落点、档案与互动记忆入用户配置目录,不进项目配置。
- 互动记忆只记录宠物相关动作的低精度偏好,不记录命令输出、路径、文件名、窗口标题或完整事件日志。
- 记忆透明与重置入口必须键盘可达;关闭宠物后不更新互动记忆。

## 不采用(反规则 — 比规则更重要)

- **不做饥饿/心情/健康值**:宠物永远健康快乐,缺席零惩罚。
- **不做等级、经验条、成就、图鉴、商城**:陪伴不是数值游戏。**升级解锁品种已正式否决**(v2.1:外在奖励排挤内在喜爱,解锁完毕即流失悬崖)。
- **玩具不可"买"只可"挑"**:无货币、无稀有度、无消耗;玩具是给它布置小窝(供养心理),不是经济系统。
- **不做随机刷新常驻入口**:持续性老虎机瓦解身份连续性;随机只活在领养卡的「交给缘分」。
- **不做推送/通知/红点**:期待是用户自己的,不是被提醒出来的。
- **不做失去与告别戏码**:换形象无"送别"弹窗、无挽留文案 —— 愧疚不是粘性。
- 不做长对话、不承载错误详情/用量/权限等正式信息(v1 红线不动)。
- 不做品种间数值差异(性格只是演出参数,没有"更好的狗")。
- **不做行为画像**:互动记忆不是用户分析系统,不推断效率、情绪、作息健康或工作习惯。
- **不做隐藏亲密度分数**:可以有偏好标签,不能有好感度、亲密度、冷淡值或连续互动 streak。
- **不把记忆变成提醒**:它记得你爱摸头,但不会提醒你该摸头。
