# 宠物系统规则

## 定位

本文定义 Tn 宠物系统作为通用趣味功能时的产品规则。当前阶段只提供文档和 SVG 原型,不表示已经接入代码。

## 设计目标

宠物系统用于给终端增加轻松、可爱的陪伴感。它可以感知终端上下文并以像素演出反馈输入、运行、成功或失败,但不应该被设计成错误提示、用量提示或 agent 状态提示的替代品。

核心目标:

- 让终端界面更有温度,降低长时间使用的疲劳感。
- 允许用户选择喜欢的小狗品种,形成个人化桌面感。
- 保持专业终端的可用性:不遮挡、不抢焦点、不污染输出。
- 为后续实现保留稳定资产规范,便于 GPUI 绘制、SVG 资源加载或像素图导出。

## 基础原则

| 原则 | 规则 |
|---|---|
| 通用性 | 宠物服务于终端陪伴和轻量反馈,不绑定某个单一功能或 agent 状态 |
| 上下文感知 | 宠物可以响应 Idle、Typing、Running、Success、Error 等终端上下文 |
| 生命周期稳定 | 宠物种类在终端初始化时决定,持续到该终端进程彻底结束 |
| 可选择 | 用户可以在 7 个品种中选择默认宠物;未固定时按随机策略选择 |
| 可关闭 | 用户可以关闭宠物系统,或仅在欢迎页/空闲时显示 |
| 低干扰 | 宠物不能遮挡 prompt、光标、选区、输出文本和浮层操作 |
| 轻互动 | 允许眨眼、摇尾巴、趴下、睡觉、探头等轻动作 |
| 不拟人化过度 | 不默认弹出长句、吐槽或拟人化对话 |
| 可扩展 | 后续可以增加品种、饰品、姿态和节日皮肤,但必须保持同一规范 |

## 视觉规范

| 项 | 规则 |
|---|---|
| 风格 | 像素风小狗,以可爱、轻量、低分辨率为主 |
| 画布 | 当前 SVG 统一使用 `128x96` viewBox |
| 网格 | 使用 8px 像素格,保证缩放后仍清晰 |
| 背景 | SVG 背景透明,由宿主 UI 决定底色 |
| 眼睛 | 统一使用深色小像素,保证小尺寸可读 |
| 轮廓 | 通过耳朵、毛色、毛型、尾巴和面部区域区分品种 |
| 动效 | 动效只移动少量像素,不改变整体占位尺寸 |
| 状态帧 | 每个品种后续应补齐 idle、typing、running、success、error、hover、drag 等同规格帧 |

## 终端上下文感知

宠物状态由终端宿主事件驱动,不是从 terminal buffer 文本里猜测。状态优先级为 `Drag > Play > Error/Success > Running > Typing > Hover > Sleep > Idle`(Play = 双击逗弄;Sleep = 长空闲打盹,任何活动唤醒);一次性反馈结束后回到当前真实上下文。

| 状态 | 触发条件 | 像素演出 | 时长 / 复原 |
|---|---|---|---|
| Idle 待机陪伴 | 终端没有输入且没有运行中的任务 | 周期性眨眼;轻微像素呼吸沉浮;尾巴做 1px 上下摆动 | 眨眼间隔随机 4-8 秒,持续到其他状态打断 |
| Typing 输入感知 | 捕获到用户键盘按键 `KeyPress` | 进入专注态:眼睛睁大,耳朵立起 1px,身体朝向活动 pane | 按键后短暂保持,连续输入时延长 |
| Running 命令执行中 | 用户回车执行命令后,进程持续时间超过 1 秒 | 切换到小跑帧,脚掌像素前后交替;或尾巴加速摇摆 | 命令结束前持续,作为天然 loading 提示 |
| Success 执行成功 | 命令结束并捕获 Exit Code `0` | 原地蹦跳一次,Y 轴上移 2px 后回落;可在头顶冒像素小爱心或吐舌头 | 短动画结束后回到 Idle 或 Typing |
| Error 执行失败 | 命令因错误退出并捕获 Exit Code 非 `0` | 垂下手或耳朵,眼睛变成类似 `- -` 的委屈状态 | 持续 3 秒后复原 |

## 鼠标交互事件

| 事件 | 触发条件 | 像素演出 | 规则 |
|---|---|---|---|
| Hover 鼠标悬浮 | 鼠标移到宠物命中区域 | **快乐眯眼 ^^**:眼格压成下缘深色横缝,表现享受摸摸 —— 不做整眼消失(无眼诡异,BUG发现 #6) | 不能弹出长说明,不能遮挡终端输出 |
| Click 单击互动 | 鼠标轻点宠物 | 触发歪头动作,可用少许旋转或头部像素错位重绘;冒出 `Woof!` 等短 Bubble | Bubble 2 秒后消失,并支持 reduced motion 下关闭 |
| DoubleClick 逗弄玩耍 | 鼠标双击宠物 | 进入 Play:逐帧蹦跳 + 头顶双爱心交替闪 + 快速摇尾,约 1.4 秒后回真实上下文 | 双击**不再**打开设置(设置走右键菜单/命令面板/状态栏席位);reduced motion 下只静态爱心 |
| Drag 拖拽拾取 | 长按宠物并拖动 | 表现为被拎起的悬空状态像素 | 用户可以把宠物放到屏幕中任意舒适位置,落点应保存到用户配置 |

## 生命周期与随机策略

- 宠物种类在终端初始化时一次性决定,并持续到该终端进程彻底结束。
- 用户已在设置中固定品种时,新终端优先使用固定品种。
- 用户未固定品种时,新终端从可用宠物池中随机选择。
- 手动刷新宠物时不应固定轮换;是否换成新宠物和换成哪一个都走随机策略。
- 刷新只改变后续显示品种,不改变已经结束进程的历史记录,也不写入 terminal buffer。
- 状态动画不能触发重新抽取品种;Idle、Typing、Running、Success、Error 都只改变当前宠物的姿态帧。

## 行为规则

| 行为 | 说明 | 默认 |
|---|---|---|
| 眨眼 | Idle 下低频 2 帧动画,间隔随机 4-8 秒;闭眼帧 = 快乐眯眼横缝(同 Hover) | 开启 |
| 摇尾巴 | Idle 下偶发 1px 摆动;Running 2px 加速;Play 3px 最欢 | 开启 |
| 打盹(趴下+睡觉) | 90 秒无键入/无运行/无互动后趴下贴地、眯眼、头顶 zZ 随呼吸起伏;任何活动即唤醒 | 开启 |
| 探头 | 现身、换品种、欢迎页形态切换时从岗台后短暂冒出(450ms 上浮) | 开启 |
| 玩耍 | 双击逗弄:蹦跳 + 双爱心 + 快速摇尾(约 1.4 秒) | 开启 |
| 跟随 | Hover 时可以看向鼠标,Typing 时可以朝向活动 pane,但不追逐光标 | 关闭 |
| 小跑 | Running 状态下脚掌像素前后交替 | 开启 |
| 蹦跳 | Success 状态下一次性上移 2px 后回落 | 开启 |
| 委屈 | Error 状态下耳朵下垂或眼睛变成 `- -` | 开启 |

## 布局规则

| 场景 | 推荐位置 | 规则 |
|---|---|---|
| 主终端 | 右下角 overlay 或状态栏附近 | 不进入 terminal grid,不影响 scrollback |
| 欢迎页 | 可以更大尺寸展示当前宠物 | 可承担品牌和趣味性 |
| Quick Terminal | 默认不显示或只显示极小版本 | 避免占用空间 |
| 多 pane | 默认全局只显示一只 | 不在每个 pane 重复出现 |
| 全屏专注 | 默认隐藏或缩为小图标 | 避免干扰 |

## 交互规则

- Hover:快乐眯眼 ^^ 享受摸摸(眼缝仍在,不整眼消失),只显示必要短反馈。
- 单击宠物:触发歪头杀和短 Bubble,例如 `Woof!`,2 秒后消失。
- 双击宠物:逗弄玩耍(蹦跳 + 双爱心 + 快速摇尾,约 1.4 秒)—— **不**打开设置。
- 长按拖拽:进入被拎起的悬空状态,松手后把宠物放在用户选择的位置。
- 右键宠物:显示菜单,包含随机刷新、更换品种、隐藏、关闭、仅欢迎页显示(= 设置入口)。
- 键盘:命令面板必须能打开宠物设置,不能只依赖鼠标。

## 文案规则

- 默认不让宠物说长句。
- 气泡只允许短句或短颜文字,例如“汪”“休息一下?”、`Woof!`。
- 气泡必须可关闭,且不能覆盖终端输出。
- 不把错误详情、限额或权限确认交给宠物表达;这些仍由正式 UI 承担。

## 可访问性

- 宠物必须可关闭。
- 支持 reduced motion:禁用眨眼、摇尾巴和探头动画。
- 支持 reduced motion:禁用 Running 小跑、Success 蹦跳、Error 委屈停留等非必要动画,保留静态状态即可。
- 屏幕阅读器只读“宠物:品种名,已显示/已隐藏”,不朗读 SVG 像素细节。
- 用户不能因为关闭宠物而丢失任何功能信息。

## 实现边界

后续接入代码时应遵守:

- 宠物组件不写入 terminal buffer。
- 宠物组件使用固定尺寸容器,避免布局跳动。
- 品种选择进入用户配置,不写入项目配置。
- 宠物行为状态可以订阅终端宿主事件,但不能反向影响 PTY、shell、agent 或命令执行。
- 命令运行与退出结果应来自 shell integration / process lifecycle 等结构化事件,不能靠扫描输出文本判断。
- KeyPress、Hover、Click、Drag 事件必须先保证终端输入、选择、复制和浮层交互不被抢占。
