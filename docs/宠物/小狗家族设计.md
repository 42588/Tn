# 小狗家族设计

## 定位

本文记录 Tn 宠物系统第一批小狗家族原型。目标是建立一套可选、可爱、轻量的终端小狗形象,用于增强终端趣味性和个人化体验。

## 设计取向

本轮不再把 7 个品种硬映射到 7 个功能状态,而是按品种设计 7 个可选角色。每个角色都应像“用户选择的终端小伙伴”,并能用同一套姿态帧响应终端上下文,而不是把某个品种固定当作系统状态图标。

共同风格:

- 像素风,透明背景,适合深色终端和浅色欢迎页。
- 身体比例偏可爱:头大、腿短、眼睛小、动作轻。
- 通过少量像素块表达品种特征,避免复杂插画。
- 默认静态可用,后续按终端上下文补姿态和动画帧。

## 品种识别规则

### 西高地

- 主色接近白色,但用浅灰区分阴影。
- 耳朵必须是尖耳,轮廓比其他白色品种更精神。
- 体型短小紧凑,不要做成长毛垂坠。

### 金毛

- 主色为暖金色,可用浅金色做脸部高光。
- 耳朵下垂,表情温和。
- 尾巴可以略长,用于后续摇尾动画。

### 德牧

- 必须有黑背或深色背部区域。
- 耳朵立起,鼻梁略长。
- 整体姿态比其他品种更警觉。

### 比熊

- 轮廓应圆,像蓬松的白色球。
- 耳朵和身体边缘尽量软,不要做成尖耳。
- 表情可以更开心。

### 马尔济斯

- 白色长毛是核心,脸部可做中分竖线。
- 可以保留一个小蝴蝶结作为识别点。
- 整体更精致、安静。

### 西施

- 双色脸和短鼻是核心。
- 头顶毛束或小发饰可作为识别点。
- 整体姿态偏慢、贴地、陪伴感。

### 泰迪

- 棕色卷毛和圆耳是核心。
- 头部可以做得更圆。
- 后续动画适合俏皮和黏人的动作。

## 上下文姿态扩展

下一阶段如果要继续深化,建议每个品种至少补齐这些姿态,并保持同一画布、同一占位和同一识别特征:

- `idle`: 默认站立或坐姿。
- `blink`: Idle 下 4-8 秒随机触发的眨眼帧。
- `typing`: KeyPress 后的专注帧,眼睛睁大,耳朵立起 1px。
- `running`: 命令超过 1 秒仍在运行时的小跑或快速摇尾帧。
- `success`: Exit Code `0` 后的 2px 蹦跳或像素小爱心帧。
- `error`: Exit Code 非 `0` 后的垂耳或 `- -` 委屈眼帧。
- `hover`: 看向鼠标或闭眼享受摸摸的帧。
- `click`: 歪头杀和短 Bubble 配套帧。
- `drag`: 被拎起的悬空状态帧。
- `sleep`: 睡觉或趴下。
- `play`: 摇尾巴或探头。

每个品种仍保持同一个文件名前缀,例如 `01-westie-idle.svg`、`01-westie-sleep.svg`。

> 2026-06-12 落地状态(`crates/tn-ui/src/pet.rs`,参数化姿态而非每品种独立帧):
> 全部 11 个姿态已实现 —— `sleep` = 90s 空闲趴下 + 眯眼 + zZ;`play` = 双击逗弄
> (蹦跳 + 双爱心 + 3px 快摇尾);`hover`/`blink` 闭眼统一为快乐眯眼横缝(不整眼
> 消失);「探头」挂在现身/换品种/欢迎形态切换的入场动画上。

## 不采用规则

- 不把 7 个品种硬映射到 7 个系统状态。
- 不用高饱和单色块替代品种毛色。
- 不做写实插画,避免和终端像素风脱节。
- 不在宠物上承载错误详情、权限、用量等正式信息。
- 不让状态动画改变当前终端已经决定好的宠物品种。
