## 2. 测试分层策略

1. **纯函数单元测试**(主力):对不依赖 GPUI/Win32/网络的逻辑直接断言。**项目已有正确范式**——把可测逻辑从 render 抽成自由函数(如 `fit_grid_size_from_bounds`、`row_runs`、`launch_rows`),本计划延续此法。
2. **模型/状态机测试**:对 `Terminal`、`BlockModel`、`Document`、`AgentRegistry`、宠物状态机用「构造→喂事件→断言快照」。
3. **快照(golden)测试**:`tn-core` 已有 `golden_snapshot_of_a_representative_ansi_program`;建议引入 [`insta`](https://docs.rs/insta) 管理快照,扩展到渲染 run、宠物姿态指纹、Lottie 段选择。
4. **属性测试(property)**:对解析器/几何/折行类引入 [`proptest`],用不变量替代枚举用例(见各 crate 清单)。
5. **基准(bench)**:`criterion`,与性能报告 §7 对齐,作为「性能不回归」门禁。
6. **不可测边界**:GPUI paint、DirectX、真实 ConPTY/SSH 网络——不强行单测,改为①抽纯逻辑下沉②集成测试(headless)③真机验证清单。

## 4. GPUI / Win32 不可测部分的处理原则

- **下沉**:凡是「输入 → 决策 → 输出」的逻辑(几何、文案、状态迁移、编码)一律抽自由函数,在 `#[cfg(test)]` 内断言。项目已有大量先例(`fit_grid_size_from_bounds`、`activity_rail_layout`、`launch_rows`、`resize_anchoring_for_pane`)。
- **headless 集成**:复用现有 `TN_AUTOQUIT` 自测通道做端到端冒烟(spawn → 输出到达网格 → 退出),CI 可跑。
- **真机清单**:GPU paint、IME 上屏、ConPTY 转发、SSH 握手保留为人工验证项,沿用任务文档「真机验证通过」惯例。

## 5. 工具与门禁

| 工具 | 用途 | 引入成本 |
|---|---|---|
| `cargo test --workspace` | 现有主门禁,保持全绿 | 已有 |
| `cargo bench -p tn-core`(扩展) | 性能不回归基线 | 低 |
| [`insta`] | 快照管理(渲染 run / 宠物姿态 / Lottie 段) | 低 |
| [`proptest`] | 解析器/几何/折行不变量 | 低 |
| [`cargo-llvm-cov`] | 覆盖率量化,定位真实盲区 | 低(本地) |
| CI(GitHub Actions) | PR 上自动跑 test + clippy + fmt | 中(仓库已有 repo 字段,无 CI 配置) |

**建议门禁线**:`cargo test --workspace` 全绿 + `cargo clippy --workspace -- -D warnings` 无新增 + 覆盖率不下降。
