## 原则

- Tn 是 **Windows 优先、为 vibe coding 打造的终端**:把通用 AI 编码 agent 当一等公民托管,同时保持终端基础能力扎实。
- 产品取向是:像现代编辑器一样顺手,像优秀终端一样稳定,视觉靠 Calm Glass 的光影层次而非堆砌效果。
- 用户可见工作流、文案、交互状态和体验取舍写入产品体验子文档;crate 边界、数据流、实现细节写入 [系统架构索引](系统架构索引.md) 或 `docs/架构/`。
- 视觉令牌、圆角、间距、CSS 到 GPUI 翻译和守卫命令写入 [界面样式实现规则](界面样式实现规则.md) 或 `docs/界面样式/`。
- 已修问题、根因、验证结果写入 [修复与优化记录索引](修复与优化记录索引.md) 或 `docs/修复与优化/`;仍未完成的限制写入 [已知问题索引](已知问题索引.md)。
- 设计真源是 [design/mockup.html](../design/mockup.html) 与 [design/panels/](../design/panels/),共享样式为 [design/calm-glass.css](../design/calm-glass.css)。改样子先看或改原型,再按样式规则端口到 GPUI。
- 已属于其它责任域的专题只链接,不复制长段。例如 SSH 实现明细链接到 [新增模块索引](新增模块索引.md),远端文件与 SFTP 链路链接到对应新增模块或已知问题专题。

## 状态

- Quick Look、平铺分屏、命令面板、app 菜单、Agent 活动栏、用量读数、终端滚动和 Calm Glass 主体体验已落地。
- SSH 连接器、密码卡、TOFU 指纹卡、断线横幅、Quick Terminal SSH 入口已有实现和 headless 覆盖,仍需真实 SSH/SFTP 场景持续复验。
- 最近目录数据源、搜索 UI、URL 点击、拖拽停靠、完整会话管理器和部分远端路径交互仍按已知问题或架构路线图跟踪。
