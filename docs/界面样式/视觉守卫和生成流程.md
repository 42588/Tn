# 视觉守卫和生成流程

## 定位

本文负责记录界面样式责任域的自动守卫、生成流程和失败时的处理入口。具体译法和组件规格分别落在 `界面翻译规则.md` 与 `组件规格与自动生成数值.md`。

## 背景

原母文档要求图形界面改动必须同时经过 headless 守卫和真实 app 视觉比对。守卫目标是防止设计真源、主题 token、生成规格和 GPUI 实现之间漂移。

## 具体内容

### 四道 headless 守卫

运行入口：`cargo test -p tn-ui`。

| 守卫 | 作用 |
|---|---|
| `token_drift` | 校验 mockup `:root` 与主题 / 常量一致。 |
| `mockup_and_calm_glass_roots_mirror` | 校验 `design/mockup.html` 与 `design/calm-glass.css` 两份 `:root` 镜像一致，防止手动同步副本漂移。 |
| `no_hardcoded_theme_colors` | 命中主题色的裸 hex 即红，并点名违规位置。 |
| `spec_gen` | 从 `design/calm-glass.css` 生成 / 校验组件规格表。 |

### 规格生成

`TN_GEN_SPEC=1 cargo test -p tn-ui spec_gen` 用于刷新自动规格。组件规格取自 `design/calm-glass.css`，令牌取自 `tn-dark.toml` 和 `style.rs`；一致性由 `token_drift` 与 `mockup_and_calm_glass_roots_mirror` 守卫。

### 真机比对

图形界面改动还需要运行 `cargo run -p tn-app`，把真实 app 与浏览器打开的 `design/mockup.html` / `design/panels/` 原型并排看，重点核对圆角、间距、颜色、阴影深度和面板材质。

### 失败处理

- 主题色或白叠加漂移：先看 [组件规格与自动生成数值](组件规格与自动生成数值.md) 中的令牌表，再回到主题 / `style.rs`。
- CSS 到 GPUI 写法不确定：先查 [界面翻译规则](界面翻译规则.md)，尤其是 `cola()`、`rgba()`、`FontWeight(N.)`、`min_w(px(0.))` / `min_h(px(0.))` 和 gpui 0.2.2 无法复制特性表。
- 视觉与原型不一致：先确认设计真源路径，再按 [设计真源与还原工作流](设计真源与还原工作流.md) 做真机并排比对。

## 验证或状态

- 状态：已从母文档工作流和自动规格说明迁移。
- 文档重构本身不运行编译；后续 UI / GPUI 样式改动应按本文执行验证。
