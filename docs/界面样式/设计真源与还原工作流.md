# 设计真源与还原工作流

## 定位

本文负责说明界面样式实现的设计真源、还原顺序和真机比对入口。具体 CSS → GPUI 写法见 `界面翻译规则.md`，组件精确数值见 `组件规格与自动生成数值.md`。

## 背景

图形界面样式以设计原型为准，Rust/GPUI 实现需要从设计真源出发，再用守卫和真机比对确认还原程度。

## 具体内容

### 0. 工作流（图形界面**必走此路**）

1. **设计真源** = [`design/panels/`](../../design/panels/)(逐面板原型,**共享** [`design/calm-glass.css`](../../design/calm-glass.css))+ [`design/mockup.html`](../../design/mockup.html)(全窗合成,浏览器打开即标准);**每个组件的确切数值**见文末 §16(自动生成、最精准——组件值取自 `calm-glass.css`,令牌取自主题)。
2. **写 gpui**:颜色一律 `col()` / `cola()`(§3,**禁止硬编码主题色**)· 尺寸/圆角照 §16 或 `style.rs` 令牌 · 字重 `FontWeight(N.)`(精确)。
3. **四道 headless 守卫**(`cargo test -p tn-ui`):`token_drift`(mockup `:root` ↔ 主题/常量一致)· `mockup_and_calm_glass_roots_mirror`(mockup ↔ calm-glass 两份 `:root` 镜像,手动同步副本防漂)· `no_hardcoded_theme_colors`(命中主题色的裸 hex 即红、点名)· `spec_gen`(从 `calm-glass.css` 生成/校验 §16)。
4. **真机比对** = `cargo run -p tn-app` 跑真实 app,与浏览器开的 [`design/mockup.html`](../../design/mockup.html) / [`design/panels/`](../../design/panels/) 原型并排看。

> 操作总纲见 [CLAUDE.md「图形界面 / 样式开发(必走此路)」](../../CLAUDE.md)。
