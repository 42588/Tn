## 输入:按键到 VT 字节

Tn 的 legacy 输入编码应对齐 Windows Terminal `_encodeRegular` 的规则,并用 Ghostty 式单入口结构组织:

- 模式影响编码:`DECCKM`、`DECKPAM`、`LNM`、`BackarrowKey`、`SendC1`。
- Tn 不应自建这些模式位,而应从 alacritty `Term::mode()` 派生。
- 修饰键位是 `SHIFT=1, ALT=2, CTRL=4`;写到 VT 参数时使用 `bits + 1`。
- 输出形态包括 CSI、SS3、字面串和 Alt 前缀。

## 逐键规则摘要

| 键 | 规则 |
|---|---|
| 方向键 | 无修饰且应用光标模式开时用 SS3;否则 CSI;带修饰使用 `ESC[1;<mod><final>` |
| Home / End | 同方向键的 CSI / SS3 逻辑 |
| F1-F4 | 无修饰 SS3,带修饰 CSI |
| F5-F20 | DECFNK `ESC[<n>~`,n 来自 WT 的跳号 LUT |
| Insert / Delete / PgUp / PgDn | `ESC[2~ / 3~ / 5~ / 6~`,带修饰时加 `;<mod>` |
| Backspace | 默认 `0x7f`;BackarrowKey 模式发 `0x08`;Ctrl 反转两者;Alt 前缀 |
| Tab | `\t`;Shift+Tab 发 `ESC[Z` |
| Enter | 默认 CR;LNM 模式 CRLF;Ctrl+Enter 发 LF;Alt 前缀 |
| Ctrl+字母/符号 | 使用 `_makeCtrlChar` 语义,例如 `@..~` 取 `ch & 0x1f` |
| 可打印字符 | 直接发送 GPUI `key_char`;Alt 时前置 ESC |

## Tn 取舍

- 采纳 WT legacy 编码算法。
- 采纳 Ghostty 的 `encode(key_event, options)` 结构思想,让 options 从终端模式派生。
- 跳过 win32-input-mode。
- kitty 键盘协议和 DECKPAM 小键盘编码后置。

## ConPTY 操作纪律

`portable-pty` 已封装 ConPTY,但 Tn 仍必须遵守 Windows Terminal 证明过的运行规则:

1. EOF 只有两种:读到 0 字节,或 `ERROR_BROKEN_PIPE`。不要依赖额外哨兵。
2. 开 `INHERIT_CURSOR` 时,ConPTY 会发 `ESC[6n` 并等待 `ESC[<row>;<col>R` 回复。必须把 alacritty 产生的 `PtyWrite` 回写到 PTY writer。
3. 读循环应流水线化,不要逐字节解析拖慢输入回显。
4. UTF-8 解码要跨读取保持状态,因为多字节序列可能被读边界切断。
5. resize 传参顺序是列在前、行在后,并且都要 clamp 到至少 1。
6. 关闭时先 `CancelIoEx`,join reader 线程后再拆回调,避免 use-after-free。
7. 清屏可用 ConPTY 的 clear pseudo console 信号清 PTY 侧滚动区。

## 路线图影响

- 输入层:legacy 编码规则进入 `encode_key` / `to_esc_str`。
- PTY 后端:保留 PtyWrite 回写、EOF 轮询、resize clamp、关闭先 cancel+join。
