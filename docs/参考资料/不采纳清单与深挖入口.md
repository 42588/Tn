# 不采纳清单与深挖入口

## 定位

本文记录外部终端源码研读中明确不采纳的设计、原因、可继续深挖的文件和对 Tn 路线图的索引影响。

## 明确不采纳

| 不采纳 | 原因 |
|---|---|
| Ghostty PageList 的分页内存、偏移指针和自管 mmap | alacritty 已有网格、滚动区和 reflow;在 Rust 中重做成本高且 unsafe 重 |
| Ghostty / AtlasEngine 整套字体引擎 | GPUI + DirectWrite 已承担发现、回退、整形和光栅化;Tn 只吸收 run 整形和 box sprite 原则 |
| 自造 VT / CSI / DCS 解析器 | alacritty 的 vte 已做;Tn 只需要在其旁路或上层拦截 OSC |
| libxev 事件循环 | Tn 用 Rust 线程、channel、GPUI executor 和帧时钟 |
| Windows Terminal 的 WinRT + X-macro 设置机制 | Tn 用 serde 结构体、TOML 和迁移表 |
| Ghostty 自有配置行格式 | Tn 使用 TOML / serde,只吸收结构模式 |
| 初期实现 kitty 键盘协议、彩色 emoji 直采、双高字符 | 后置;先保证 legacy 编码和基础 cell 渲染扎实 |

## 对路线图的索引影响

- 输入层:按 Windows Terminal `_encodeRegular` 重写或维持 legacy 编码完整性,模式位取自 alacritty。
- 渲染层:后续自定义 GPUI `TerminalElement`,使用 typed-quad 思想、run 整形、fg/bg 分离和 box sprite 自绘。
- 重绘层:push notify + vsync 合并,同步输出由底层解析器或状态机保证整帧一致。
- 配置层:profile / scheme / actions / keybindings 解耦,加迁移表和 diff-on-reload。
- `tn-core`:滚动区字节上限、行级可能位和 block 拥有式快照是后续优化方向。
- `tn-shell` / `tn-blocks`:按 OSC 133/7/8 语义实现 block、cwd 和 hyperlink。
- `tn-pty`:坚持 ConPTY EOF、DSR、resize、clear、关闭纪律。
