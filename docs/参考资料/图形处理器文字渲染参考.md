# GPU 文字渲染参考

## 定位

本文记录 Windows Terminal AtlasEngine 和 Ghostty 字形处理对 Tn 终端渲染的参考价值。本文只记录可迁移思想和 Tn 取舍,不承载 GPUI 组件规格。

## Windows Terminal AtlasEngine

参考文件:

- `src/renderer/atlas/BackendD3D.cpp`
- `src/renderer/atlas/common.h`

核心思想:不要逐个“画文字”,而是把整屏拍平成一串带类型的 quad,用批处理和 shader 分支画出来。

可迁移结构:

- 统一紧凑 `QuadInstance`,包含 shading type、位置、大小、纹理坐标和颜色。
- 字形、背景填充、光标、下划线、选区都进入同一实例缓冲。
- `shadingType` 区分灰度文字、ClearType、彩色 emoji、实线、虚线、光标和填充。
- 字形图集按 2 的幂增长,用矩形装箱打包。
- 缓存键按字体、line rendition 和 glyph id 分层。
- 每行 shape 一次,生成类似 `ShapedRow` 的 SoA 数据并缓存复用。
- 前景色和背景色分平面处理。
- 连字或 overhang 字形按 cell 边界切分,以支持逐格上色。

## Ghostty 可迁移原则

- 按 run 整形,即同字体同样式连续段一起 shape,不要按 cell 独立 shape。
- box-drawing 和 Powerline 等图形字符用内部 sprite 或 quad 自绘,不要完全依赖字体字形,以保证像素级对齐和无缝隙。

## Tn 纠偏

Tn 使用 GPUI,而 GPUI 已经提供文字系统、DirectWrite 和字形图集能力。因此:

- 不重写 D3D 渲染器。
- 不复制 AtlasEngine 的字体引擎或图集管理。
- 目标是写自定义 GPUI `TerminalElement`,把可见网格组织成批量 `paint_quad` 和文本 shape / paint 调用。
- 背景、光标和选区优先走 quad。
- 字形按 run 整形。
- fg/bg 分离上色。
- box-drawing / Powerline 用 quad 或 sprite 自绘。

## 判定

- AtlasEngine 的数据组织思想:改造采纳。
- Ghostty 的 run 整形和 box sprite 原则:采纳。
- 裸 D3D、字体 collection、HarfBuzz/Atlas/SharedGrid 整套实现:跳过。
- 彩色 emoji 直采和双高字符可后置。

## 路线图影响

- 自定义 `TerminalElement` 是后续精修项。
- 当前 `div` + run 批处理可继续服务已落地体验。
- 终端渲染的用户体验原则写入 [终端交互体验](../产品体验/终端交互体验.md),具体实现路线写入系统架构。
