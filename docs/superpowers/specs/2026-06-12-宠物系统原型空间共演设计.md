# 宠物系统原型空间共演设计

## 定位

本规格定义 `design/panels/05-pet-system.html` 的下一轮原型更新:新增 **SHEET 05 板 H · 磷光通道空间共演**。

目标不是给宠物追加普通动作,而是把宠物确立为 Tn 空间转场的可见钥匙:幽灵终端、Quick Look、浮层、分屏等关键 UI 不再凭空出现,而是从真实 UI 接缝被宠物用磷光裂缝、光点和残影打开。

## 已确认方向

- 本轮优先做 **板 H 旗舰方案**。
- 板 H 是宠物系统原型的空间共演审核稿,先不拆成新的跨页图纸。
- 板 G 主动互动/玩具/互动记忆仍保留为后续入口,但不稀释本轮主轴。
- 设计以 [宠物系统规则](../../宠物/宠物系统规则.md) 的规则 J 和 `PetSpatialCue` 为权威来源。

## 核心设计

板 H 分为五段:

1. **H1 空间语法总览**
   展示开门、穿梭、召回三个动词,并说明它们如何映射为 `PetSpatialCue`。

2. **H2 幽灵终端共演分镜**
   主岗台裂缝亮起,宠物跃入;屏顶召唤线点亮;幽灵终端垂下;宠物从幽灵窗底边探头。隐藏时反向召回,宠物回到主岗台。

3. **H3 Quick Look 共演分镜**
   宠物叼出磷光光点;光点贴住 Quick Look 入场边缘并展开面板;切换文件只做嗅光点/抬头换向;关闭时宠物拉回光点并回窝。

4. **H4 浮层与分屏短句法**
   命令面板/设置浮层使用岗台探头和光点升起;分屏使用宠物沿分隔线小跑、分隔线短暂发光。强调普通 hover、普通菜单项和状态更新不使用通道语法。

5. **H5 SPEC 表**
   记录事件名、来源 UI、源锚点、目标锚点、完整共演时长、短版时长、reduced motion 降级、实现边界。

## 视觉纪律

- 通道只能由 1px 磷光裂缝、光点、真实 UI 接缝和幽灵残影组成。
- 禁止烟雾、粒子风暴、大面积渐变、全屏遮罩、柔光雾和装饰性发光。
- 宠物仍是像素小狗,不变成光团、不分身、不长出额外形态。
- 通道必须依附真实 UI 接缝:主岗台线、屏顶召唤线、幽灵窗底边、Quick Look 入场边缘、pane 分隔线、浮层出现点。
- UI 本体不得等待宠物动画完成;动画只补足空间叙事。

## 实现边界

本轮更新的是 HTML 原型审核稿和项目文档,不是 Rust 功能实现。

原型应继续遵守磷光设计语言:

- 大面积使用 L0-L4 不透明海拔。
- 平铺板面不加投影,真浮层才有外投影。
- 小渐变只允许用于现有身份棒、幽灵顶缘、召唤线等 4px 内元素。
- 宠物和通道示意以固定尺寸容器呈现,不能因分镜内容变化导致布局抖动。

## 验收标准

- `design/panels/05-pet-system.html` 新增板 H,并能直接用浏览器打开。
- 板 H 至少覆盖幽灵终端、Quick Look、浮层、分屏四个关键转场。
- 每个关键转场都能看出源锚点、目标锚点、宠物动作和磷光材料。
- SPEC 表明确 `GhostEnter`、`GhostExit`、`QuickLookOpen`、`QuickLookClose`、`QuickLookSwitch`、`OverlayOpen`、`SplitCreate` 的降级策略。
- `design/README.md` 和 `docs/设计索引.md` 摘要同步到“空间共演/磷光通道”。
- `TODO.md` 与 `docs/任务/` 记录本轮原型更新、验证结果和后续实现入口。

## 非目标

- 不实现 `PetSpatialCue` Rust 事件模型。
- 不重写板 A-F。
- 不把所有 UI 打开都魔法化。
- 不让宠物成为错误提示、权限提示或焦点提示的唯一载体。
- 不引入新宠物品种、新数值系统或宠物健康/饥饿机制。
