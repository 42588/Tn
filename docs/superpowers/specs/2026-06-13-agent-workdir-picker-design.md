# Agent 工作目录选择设计

## 背景

BUG #4 要求 Agent 启动前可以选择工作目录。该行为需要同时覆盖幽灵终端启动器和主窗口欢迎页,并保持两处键位规则一致。

## 交互设计

- 幽灵终端:点击 Agent 磁贴后,原 Ghost 启动器卡片就地切换为单层工作目录选择器,不再继续嵌套磁贴或浮层。
- 欢迎页:不改变欢迎页布局。点击 Agent 磁贴后,Workspace 在欢迎页上方显示临时 scrim/modal 目录选择器。
- 两处都显示三块内容:最近工作目录、当前目录子目录、浏览本地文件夹。
- 欢迎页打开时以 Explorer 当前 Host root 作为最近目录第一项 seed;幽灵终端不依赖 Explorer,从最近目录、进程当前目录或 `C:\` 回退。

## 键位规则

- `Tab` / `Shift+Tab`:在最近工作目录、当前目录子目录、浏览之间切换焦点。
- `↑↓`:只在当前焦点区域内移动选择。
- `←`:返回上一级目录;没有上级目录时不返回。
- `→`:进入当前选择目录;焦点在浏览时打开系统目录选择器。
- `Enter`:用当前高亮目录启动 Agent。

## 数据与边界

- 最近目录保存到 `workdir_recents.json`,最多保留 12 条。
- 目录读取只列出本地主机目录,并标记含 `.git` 的子目录。
- Windows 目录列表固定显示可用盘符入口,避免从 `C:\` 起步时无法切换到其他盘。
- Agent 启动通过 `LaunchSpec::with_cwd` 写入 cwd;普通 shell、WSL、SSH 启动路径不改变。

## 验收

- 幽灵终端 Agent 磁贴点击后不直接启动,先显示工作目录选择器。
- 欢迎页 Agent 磁贴点击后出现 overlay,欢迎页原布局不移动。
- 两处键位规则一致,IME 在导航面板中关闭。
- `cargo test -p tn-ui --lib` 与 `cargo build -p tn-ui` 通过。
