## 0. 一句话定位

Tn 是一个 **Windows 优先、Rust 编写、GPU 加速**的终端模拟器,主打**性能与颜值**,围绕「**vibe coding**」体验设计:把 Claude Code / Codex 等 AI CLI 当一等公民来托管,提供 Warp 风格的 block 历史,并原生支持 WSL 与远程 Linux(SSH)。

## 1. 核心原理:终端 ≠ shell

这是理解整个项目的基础。**Shell 和终端模拟器是两个完全不同的层**,经常被混淆:

|                                               | 是什么                                                                                                                   | 在 Tn 里谁负责        |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ | --------------------- |
| **Shell**(PowerShell / cmd / bash)            | 命令解释器。解析命令、跑程序、管道。**它本身没有窗口、没有 GUI**,只从一个管道读字节(stdin)、往另一个管道写字节(stdout)。 | **复用现成的**,不重写 |
| **终端模拟器**(Tn / Windows Terminal / iTerm) | 屏幕 + 键盘。把 shell 输出的字节流(文字 + ANSI/VT 转义码)解析成二维字符网格并渲染;把键盘事件编码成字节喂回 shell。       | **我们写**            |

历史上 Windows 默认终端是 `conhost.exe`。**Tn 做的事 = 用我们自己的实现取代 conhost**,PowerShell 本身不变,而且**可替换**——同一套机制能跑 `cmd`、`wsl`、`bash`、`vim`,乃至 `claude` / `codex`。

如果不做解析直接显示,你看到的是这种东西:

```
PS C:\> ls
\x1b[32m\x1b[1mDirectory:\x1b[0m C:\... \x1b[?25l\x1b[2K\x1b[38;5;39m...
```

终端的实活就是把这些 **VT/ANSI 转义序列**翻译成「光标移到第 N 行」「这段变绿」「清屏」「进入全屏(alt-screen)」并维护网格 + 滚动历史 + 光标状态。这部分极繁琐(VT100 几百条序列),所以我们**复用 alacritty 的解析引擎**(WezTerm、Zed 同理),不自己重写。

类比:**Tn 是「屏幕和键盘」,PowerShell 是「插上去的大脑」,ConPTY 是中间的「电线」。**

## 2. 系统架构

### 2.1 Cargo 工作区 ✅

多 crate 工作区。这样划分的核心理由:① 把 **GPUI(pre-1.0,API 易变)隔离**在 `tn-ui` 一个 crate;② 让终端内核 **headless、可单测**,不依赖 GPU;③ 职责清晰。

```
Tn/
├── Cargo.toml            # [workspace] + [workspace.dependencies](统一版本)
├── rust-toolchain.toml   # stable, x86_64-pc-windows-msvc
├── .cargo/config.toml    # cargo 别名(smoke/lint/...)
├── deny.toml             # cargo-deny 许可证/安全门
├── crates/
│   ├── tn-core/   # 终端引擎:alacritty 包装(Term+解析+快照)。无 GPUI/无 IO  ✅
│   ├── tn-pty/    # PTY 后端:PtyBackend trait + LocalPty(ConPTY)            ✅
│   ├── tn-config/ # 配置 + 主题(TOML、路径、热重载)                          ✅ M1.3
│   ├── tn-shell/  # OSC 133/633/7 旁路解析 → BlockEvent;shell 集成脚本        ✅ M3
│   ├── tn-blocks/ # Warp 式 block 模型(命令/输出/退出码/时长)               ✅ M3
│   ├── tn-agent/  # Agent Host 平台:AgentId/descriptor/event/registry/usage  ✅
│   ├── tn-ai/     # 内置 Claude/Codex adapter(JSONL 用量),可移除             ✅ M4
│   ├── tn-ui/     # GPUI 前端(唯一链接 gpui 的库):TerminalView 等          ✅
│   └── tn-app/    # 二进制 `tn`:开窗 + 接线 + 日志                          ✅
└── crates/tn-cli/ # headless 调试/烟雾测试工具                               ✅
```

### 2.2 依赖关系图(无环,评审 + deny.toml 把守)

```
tn-app ──► tn-ui ──► gpui            ◄── 许可证 & API 抖动的防火墙(gpui 只出现在这里 + tn-app)
```

## 3. 数据流

### 3.1 全景图 ✅

```
   你按键盘                                          子进程输出
      │                                                   ▲
      ▼  键 → 字节(Enter→"\r", Ctrl+C→0x03, ↑→"\x1b[A")    │ 字节流(文字 + ANSI/VT)
┌──────────────┐            ConPTY(内核伪终端)            ┌──────────────┐
│ TerminalView │◄──────────── 双向字节管道 ──────────────►│ powershell.exe│
│ (tn-ui/GPUI) │                                          │ (或 cmd/wsl…)│
└──────┬───────┘                                          └──────────────┘
       │ reader 线程读到字节
       ▼
┌──────────────────────────────────┐
│ Terminal (tn-core)               │
│  Processor.advance(&mut Term, …) │  把字节喂给 alacritty:更新网格、光标、滚动
│  → 同时产生 PtyWrite 事件(见3.3) │
└──────┬───────────────────────────┘
       │ snapshot() → TerminalSnapshot(可见网格)
       ▼
   GPUI div/text → DirectWrite 画成像素(tn-ui)
```

### 3.2 输出路径 ✅

1. `LocalPty` 通过 ConPTY 开伪终端,把 shell 挂为子进程([crates/tn-pty/src/local.rs](../crates/tn-pty/src/local.rs))。
2. **reader 线程**循环 `reader.read(buf)`,把字节交给 `Terminal::advance`([crates/tn-core/src/lib.rs](../crates/tn-core/src/lib.rs))。
3. `advance` = `vte::ansi::Processor::advance(&mut Term, bytes)`,alacritty 维护 `Term` 的网格/光标/滚动。
4. `Terminal::snapshot()` 遍历 `term.renderable_content()` 的可见行,产出不可变的 `TerminalSnapshot`(行×列的 `SnapshotCell`)。
5. `TerminalView::render` 把每行渲染成一个等宽 `div`([crates/tn-ui/src/terminal_view.rs](../crates/tn-ui/src/terminal_view.rs))。

### 3.3 输入路径 ✅

键盘事件经 `on_key_down` → `crate::input::encode_key(&Keystroke, InputMode)`([crates/tn-ui/src/input.rs](../crates/tn-ui/src/input.rs))把 GPUI `Keystroke` 映射成终端字节,写回 PTY 的 writer(shell 的 stdin)。**已照搬 Windows Terminal `_encodeRegular` 算法**:方向键/Home/End 按 DECCKM 选 CSI(`ESC[A`)或 SS3(`ESC OA`),带修饰 → `ESC[1;<mod><final>`(`<mod> = bits(SHIFT1/ALT2/CTRL4)+1`);F1–F4 SS3/CSI;F5–F20 DECFNK `ESC[<n>~`(跳号 LUT);Insert/Del/PgUp/PgDn `ESC[n~`;Backspace `0x7f`(Ctrl→`0x08`);Tab + Shift-Tab `ESC[Z`;Enter CR / LNM-CRLF / Ctrl-LF;`_makeCtrlChar`;Alt = ESC 前缀(CSI 键则折进 `<mod>`)。模式位经 `tn_core::Terminal::input_mode()`(读 alacritty `Term::mode()`)。`Ctrl+Shift+*`、`Ctrl+Tab` 保留给 UI(返回 None)。🧭 后置:kitty 键盘协议、DECKPAM 小键盘编码、win32-input-mode、bracketed-paste 包裹(标志已暴露)。详见 [参考资料.md](参考资料.md) §一。

### 3.4 PtyWrite 回写(关键坑)✅

**alacritty 在解析到设备查询时,会通过 EventListener 产生 `Event::PtyWrite(reply)` 事件,这些回复必须写回 PTY。** 否则:ConPTY 启动时发 `ESC[6n`(查询光标位置)并**阻塞等待回复**,不回复 → 子进程永远卡住(实测:只读到 4 字节、永不退出)。所以 reader 线程在 `advance` 后会 `drain_events()`,把所有 `PtyWrite` 写回 writer。

详见记忆 `conpty-dsr-ptywrite`。另一个 ConPTY 坑:**它不可靠地发 EOF**,所以判断子进程退出要用 `try_wait` 轮询,而不是等 `read()==0`。

### 3.5 重绘循环 ✅

✅ 现状(M1.5,[crates/tn-ui/src/terminal_view.rs](../crates/tn-ui/src/terminal_view.rs)):**push + vsync 合并**——reader 线程在有新输出时往 `futures::channel::mpsc::unbounded` 发一个 wake(由 `dirty` 原子标志去重,通道里至多 1 个待处理 wake);前台 `cx.spawn` 任务 `await` 该 wake 后 `cx.notify()`,GPUI 把多次 notify 合并到 vsync 帧时钟。空闲时**零唤醒**(无轮询)。**DEC 2026 同步输出由 alacritty `vte` `Processor`(`StdSyncHandler`)内部处理**——网格仅在 BSU→ESU 完成或其超时时才变更,故每次 `snapshot()` 都是整帧、无半更新撕裂,tn-ui 侧无需额外代码。遵 Ghostty 经验:不过度工程化批渲染,让 vsync 节流。_边角:同步超时仅在下次 `advance` 时复查,中途卡住的同步流会延迟显示——可接受,真实程序会及时发 ESU。_(见 [参考资料.md](参考资料.md) §六)

### 3.6 resize 联动 ✅

`render` 里用 `window.viewport_size()` 除以**实测的等宽字符宽度**(`text_system().advance(font_id, size, 'm')`)算出列数、除以行高算出行数;若与当前网格不同,则 `Terminal::resize_conpty_with_anchoring` + `PtyBackend::resize`(ConPTY `window_change`)。尺寸计算走 `TerminalView::fit_grid_size_from_bounds`:隐藏 / 最小化 / 标签切换期间若 canvas bounds 小到扣除 `BODY_PAD` 后不足一个完整 cell / row,本帧跳过 resize,避免把瞬态布局写成 `1x1` PTY 尺寸。grow 锚定策略按 pane 分流:普通 shell 用 `ResizeAnchoring::Top` 保历史;agent / alt-screen 用 `ResizeAnchoring::Bottom`,避免把当前 TUI 屏推进 scrollback。

**为何是 `resize_conpty` 而非 `resize`**:ConPTY 行/列仍**精确跟随** alacritty(尺寸不匹配会复活高频空白,见 [CLAUDE.md 踩过的坑](../CLAUDE.md));但 alacritty 标准 resize 在**长高时底锚定**——把 `delta` 行从滚动历史**提升进视口顶部**,而 ConPTY 紧接着用**顶锚定**的 repaint 覆盖这些格、且 alacritty 已把它们移出历史环 → 永久丢失(探针 `TN_RESIZE_EXP`:12→24 丢 12 行)。`resize_conpty` 在 ConPTY 重绘**之前**把被提升的行用 `scroll_up` 推回滚动区,使长高变**顶锚定**(新行是底部空白、历史留在历史环不被 repaint 触及)——与 ConPTY 的 repaint 对齐 → 零丢失(`TN_RESIZE_EXP=topgrow` 验证 40/40 存活),且恰好与原生 Windows 控制台"长高=内容留原位、下方开空白"一致。

### 4.2 `tn-pty` — PTY 后端 ✅ / 🧭

- `PtyBackend` trait:`resize / take_reader / writer / killer / wait / try_wait`——**统一同步的 Read/Write 接口**,让驱动层与后端无关。
- `PtySize`、`SpawnSpec`(program/args/cwd/env,builder 风格)、`Killer`。
- ✅ `LocalPty`(ConPTY,经 portable-pty):`openpty` → `spawn_command` → `try_clone_reader`/`take_writer`;**保留 slave 句柄**存活(ConPTY 上过早 drop 会扰动伪控制台)。
- ✅ **WSL**(`wsl.rs`)——**无需专属 backend**:ConPTY 托管 `wsl.exe -d <distro> --cd ~` 如普通程序,复用 `LocalPty`;`parse_distros` 解码 `wsl -l -q`(UTF-16LE)枚举发行版(纯函数,单测)。WSL 文件命名空间独立于 Windows cwd,Explorer 通过 `ExplorerRoot::Wsl` 把 Linux cwd 显式映射到 `\\wsl$\<distro>\...` 驱动 Linux 目录树并显示 dotfiles,不把 `/home/...` 映射成宿主盘路径。端到端 SMOKE PASS。
- ✅ `SshBackend`(russh,**首次真机验证 2026-06-05**):russh 给 async channel 而非 fd → 专属线程跑 current-thread tokio,`select!` 把 `ChannelMsg::Data` ↔ 同步 `std::mpsc` 桥成同步 Read/Write;`window_change`/keepalive/drop 断开;认证链 = **`none` 探测服务器开放方式 → key → password(同密码并试 `password` 与 `keyboard-interactive`)**。**已真机端到端**:密钥(id_ed25519)+ 交互 shell 登录 AlmaLinux 9.7(WSL)。仍未真机走过:密码框 UI / kbi / 重连 / TOFU 首连(见 §8)。

### 4.5 `tn-shell` / `tn-blocks` / `tn-ai` ✅

- **`tn-shell`**:alacritty **不解析** OSC 133/633 →在 PTY 字节上挂一个**旁路 `vte::Parser`**(只处理 `osc_dispatch`),产出 `BlockEvent`,同时原始流照常喂 `Term`。pwsh 集成脚本(per-session nonce 防伪)经 `-EncodedCommand` 注入(`TN_NO_SHELL_INTEGRATION` 可关)。
- **`tn-blocks`**:`BlockModel` 状态机 `Prompt→Input→Running→Finished` → `Block`(命令/cwd/输出行区间/退出码/时长),行锚点用 `cursor_abs_line()`。**alt-screen(vim/agent TUI)进入时整体隐藏 block chrome、全幅渲染**,退出恢复——「让位给全屏程序」的关键(正确性门槛)。
- **`tn-ai`**(现为**内置 adapter** 层,见 §4.6):`claude.rs`/`codex.rs` 解析本地会话 JSONL(Claude `~/.claude/projects`、Codex `$CODEX_HOME/sessions` + 日志里真实 `model_context_window`)+ pricing 表,**AI 用量**(上下文占用 + token + 估算花费)展示为每 pane 头环形读数 + 状态栏读焦点 pane——详见 [产品设计.md](产品设计.md) §5。`detect.rs::resolve_pane_session` 按 **launch 后 stale→fresh** 绑定 pane 的会话(吃 `&dyn AgentAdapter`,agent-agnostic)。

### 4.6 `tn-agent` — Agent Host 平台 ✅(2026-06)

**做了什么**:把「Claude/Codex 特判」抽象成对具体 agent **零知识**的宿主平台。`tn-agent`(headless,平台核心)定义 `AgentId`(开放字符串身份,**替代闭合 `AgentKind` 枚举**)· `AgentDescriptor`/`AgentCapabilities`/`AgentRuntimeKind`(身份 / 能力插槽 / 运行位置)· `AgentEvent`/`AgentStatus`(UI 唯一输入契约)· `AgentAdapter` trait + `GenericAdapter` · `ExternalEventAdapter`/`ExternalProcessAdapter`(JSONL / stdout 实时事件入口)· `AgentRegistry`(按 id/命令解析,**空 = 纯 shell 宿主**)· `AiUsage`+pricing(通用用量契约)。Claude/Codex 降为 `tn-ai` 里的**可移除内置 adapter**(`builtin_registry()` 保留但默认不注册 → 出厂无内置 agent);`tn-ui` 经注册表(gpui Global)解析 descriptor,**render 路径零 agent 特判**,capability 驱动 header/活动栏插槽,`AgentEvent` 归约器复用既有用量轮询,并消费外部 adapter 明确发出的状态/模型/权限/错误/摘要事件。用户写 config `[[agents]]` manifest 即可加 agent(generic 无遥测),也可声明 `runtime_support` / `allow_network` 进入非 PTY runtime 安全契约。

**为什么**:在加第三个 agent(Gemini/Aider/…)前先平台化,否则每加一个就要再串一遍全 UI 的 `match` 臂,技术债指数增长。

**层次与边界(铁律)**:`tn-config ← tn-agent ← tn-ai`(平台 ← 内置 adapter)无环;平台对具体 agent 零知识,认 Claude/Codex 的代码只准在 `tn-ai`。**`AgentRuntimeKind`(进程/协议运行位置,`LaunchSpec::runtime()` 派生)≠ `FileNamespace`(文件 I/O 命名空间)**——SSH-runtime 的文件 I/O 只能经 SFTP-backed `ExplorerRoot::Ssh` / `RemoteFileService`,不得落回本机 `PathBuf` / git watcher;`AgentEvent::CwdChanged` 只喂目标 pane 上下文,不碰全局 Explorer。非 PTY runtime 已开放 `RemoteDaemon`/`Http`/`WebSocket`/`Structured`;网络 runtime 默认拒绝,`allow_network=true` 也只是 `AgentNetworkPolicy::Ask`,需要宿主层用户确认。守卫测试 `ui_has_no_closed_agent_enum` 锁死 UI 零闭合枚举。分阶段实现明细见 [CHANGELOG.md](../CHANGELOG.md) 的 Agent Host 段。

**应用内 Agent 编辑器(2026-06,建在平台之上)**:把「加 agent」从手编 `config.toml` 升级为应用内交互(用户反馈:编辑配置太极客)。欢迎页 launchpad「+ 添加 Agent」磁贴 → 居中玻璃浮层(`workspace::render_agent_form`)收集 名称/命令/颜色/Ink 自绘光标 → 写 `[[agents]]`+`[[profiles]]`(`tn_config::append_agent`/`remove_agent`,块级保注释)→ `reload_agents` 重读盘 + 重装 `AgentRegistry` global + 重建 launchpad **即时生效**(无需重启)。自定义磁贴 hover ✎/✕ 编/删。**架构意义:验证了平台的"配置即接入"层 —— 编辑器只产出 config 数据,`tn-agent`/`tn-ai` 零改动**;诚实保持(config-only = generic,无用量遥测,编辑器不给 usage 勾选)。UX 见 [产品设计 §6.4](产品设计.md),明细见 [CHANGELOG.md](../CHANGELOG.md)。

**实时(观测)层 = 配置可达的 sidecar(2026-06,接入深度 ③)**:`[[agents]] sidecar = "cmd"` → `AgentDescriptor.realtime_command`;launched agent pane 据 `AgentDescriptor::sidecar_launch()`(默认拒绝网络的**单点决策**)**每 pane** 拉起自己的 `ExternalProcessAdapter`(非 ConPTY 进程,= 第一个真正执行的非 PTY 运行时)读 stdout JSONL → `AgentEvent` → 同一 `reduce_agent_event` 漏斗 → agent 头状态/权限/transcript/用量。**这让配置声明的 agent 也能有真实遥测,无需内置 adapter** —— 兑现「配置可达 + 网络确认」。网络 sidecar(http/ws/remote-daemon)默认拒绝,仅 `allow_network` 时弹确认卡、确认才连(`AgentNetworkPolicy::Ask`,从不静默)。**仍按原则不做**:纯终端-less pane(无本地进程的 Http-only agent)—— 无真实 agent 可测、要重写 TerminalView 的 PTY 内核,属投机(§5「AgentRuntime 只落实测得到的运行时」)。坑见 [CLAUDE.md](../CLAUDE.md)「sidecar 遥测必须 per-pane」。

## 5. 关键技术决策(锁定)

| 领域        | 决策                                                                                                    | 理由                                                                                                                  |
| ----------- | ------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| UI/渲染     | **GPUI**(Windows 走 DX11+DirectWrite)                                                                   | 性能+颜值兼顾、纯 Rust、为文字密集型应用而生;参考 Zed 的 terminal crate                                               |
| 引擎        | **alacritty_terminal** 库                                                                               | 久经考验的 VT 解析+网格,绝不自己重写                                                                                  |
| PTY/远程    | **portable-pty**(ConPTY+WSL)+ **russh**(SSH)                                                            | 纯 Rust、可内置、连接管理可自控                                                                                       |
| AI 模式     | **托管 CLI** + Warp 式 **block UI**                                                                     | 不做原生侧栏/内联补全(暂);深度 = 优秀托管 + chrome 装饰 + 优雅降级                                                    |
| 许可证      | **开源 GPL-3.0-or-later**                                                                               | 规避 GPUI 依赖树里 GPL-3.0 传递依赖的冲突([zed#55470](https://github.com/zed-industries/zed/issues/55470))            |
| MVP 顺序    | 本地内核 → WSL/SSH → blocks → AI                                                                        | 先有能日用的扎实终端                                                                                                  |
| 渲染        | **按 `generation` 缓存渲染数据**                                                                        | 引擎每次 grid 变更 +1;空闲/光标闪烁等无变化帧复用 snapshot+row_runs,不每帧重走全网格                                  |
| ConPTY 行数 | **始终精确跟随 alacritty**(曾试"shell 行锁定"防 resize 吃历史,因 ConPTY≠alacritty 致高频空白已**撤销**) | 尺寸不匹配会复活空白;故不锁行                                                                                         |
| resize 长高 | **普通 shell 顶锚定长高;agent / alt-screen 底锚定长高**(`ResizeAnchoring`)                              | shell 顶锚定避免 ConPTY repaint 覆盖历史;agent / TUI 底锚定避免把当前主屏帧推进 scrollback、滚轮翻出重复旧帧           |
| resize 临时小尺寸 | **跳过不足一个 cell / row 的隐藏帧 bounds**(`fit_grid_size_from_bounds`)                               | 标签切换 / 最小化恢复会产生很小但非零的 canvas bounds;若强行兜成 `1x1` 写给 PTY,光标会被推到左上角并可能覆盖历史       |
| scrollback 光标 | **`display_offset > 0` 时 snapshot 隐藏 live cursor;输入前先回 bottom**                              | 阅读历史时不应把 live cursor 投影到历史视图;`display_offset == 0` 才是 live bottom                                  |
| agent 退出  | **`-NoExit` 托管 agent 用哨兵标题自报退出**                                                             | pwsh 存活故无 `ProcessExited`;退出后清身份回落 shell,而非进程树轮询(脆弱)                                             |

## 7. 已知坑与经验(踩过的)

1. **ConPTY 启动 DSR 阻塞**:必须把 alacritty 的 `PtyWrite` 回复写回 PTY(见 §3.4)。
2. **ConPTY 无 EOF**:用 `try_wait` 轮询判断退出;烟雾测试加硬超时 + watchdog kill。
3. **保留 slave 句柄**:ConPTY 上过早 drop `SlavePty` 会出问题。
4. **CommandBuilder 默认继承父环境**(`std::env::vars_os`),不是空环境——所以 `powershell.exe` 能正常启动(早期误判过这点)。
5. **`gpui::Pixels.0` 私有**:取 f32 用 `f32::from(pixels)`,不能 `.0`。
6. **gpui async 接口**:`cx.spawn(async move |this: WeakEntity<Self>, cx: &mut AsyncApp| …)`;`WeakEntity::update(cx, |v, cx| cx.notify())`;定时器 `background_executor().timer(dur).await`;退出 `cx.quit()`。
7. **cargo 不在 bash PATH**:在 `%USERPROFILE%\.cargo\bin\cargo.exe`,PowerShell 里用全路径或重开 shell。

> 更多近期硬坑(ConPTY 行**增高**吃历史/顶飞提示符、reader panic 经 Mutex 中毒拖垮 app、`-NoExit` 托管 agent 退出检测、**`overflow_hidden` 既裁投影又不跟圆角 / 分屏间距用 wrap 内侧 padding / 大面积半透渐变色带 / acrylic 透背 dogfood 后被否**)详见 [CLAUDE.md](../CLAUDE.md)「踩过的坑」,不在此复述。

**M2 SSH(✅ 首次真机验证 2026-06-05)**:`SshBackend` 实现 `PtyBackend`——专属线程跑 current-thread tokio,connect→**auth(`none` 探测 → key → password,password 方法与 keyboard-interactive 同密码并试)**→pty→shell→`select!` 把 async channel 桥成同步 Read/Write;`window_change`/keepalive 30s/drop 断开;`SshConfig`(host[:port]/user/自动找 `~/.ssh/id_*`);`TerminalView` 已抽象到 `Box<dyn PtyBackend>`,`LaunchSpec.ssh` + `from_profile(kind="ssh")` 接线。**真机端到端**:Tn 以密钥(id_ed25519)+ 交互 shell 登录 AlmaLinux 9.7(WSL)。诊断价值印证:认证前 `authenticate_none` 打印的"服务器支持的认证方式"与 `ssh -v` 的 "Authentications that can continue" 一字不差,直接帮 owner 定位到是服务器侧 `PermitRootLogin no`(drop-in 覆盖)而非 Tn 问题。

## 9. 开发指南

### 9.1 环境

- Rust **stable**,目标 `x86_64-pc-windows-msvc`(`rust-toolchain.toml` 已固定)。需 **VS C++ 生成工具 + Windows SDK**(GPUI/DX 必需)。
- cargo 路径:`%USERPROFILE%\.cargo\bin\cargo.exe`。

### 9.2 常用命令

```powershell
cargo build --workspace
cargo test  -p tn-core               # 引擎单测
cargo run   -p tn-cli                # ConPTY 烟雾测试(headless,打印网格)
cargo run   -p tn-app                # 开窗(主程序)
$env:TN_AUTOQUIT="1"; cargo run -p tn-app   # GUI 自测:dump 网格后自动退出
# 别名(.cargo/config.toml):cargo smoke / cargo lint / cargo t
```

### 9.3 约定

- **`gpui` 只准出现在 `tn-ui` / `tn-app`**;`tn-core`/`tn-pty`/`tn-config` 必须 headless 可编译可测。
- **`tn-pty::LocalPty` 是本地进程环境边界**:它负责 ConPTY spawn、清理 VS Code/Cursor 注入的 IDE 环境标记,并在 Windows 上把当前 Tn 进程 PATH 的额外目录合并回 `portable-pty` 的注册表 base PATH,保证 npm/cargo shim(如 `codex.ps1`)在 Tn 子 shell 内仍能解析。
- 不要把 alacritty 的类型泄漏到 crate 边界外——在 `tn-core` 里包装(目前 `Event` 经 `TermEvent` 重导出是唯一例外,慎用)。
- 改依赖版本走 `[workspace.dependencies]`;git 依赖必须钉死 `rev=`。
- 提交信息结尾带 `Co-Authored-By` 行(若用 AI 协作);行尾 LF(`.gitattributes` 已设)。

### 9.4 新增一个 crate

1. `crates/<name>/{Cargo.toml, src/lib.rs}`,字段用 `*.workspace = true`。
2. 在根 `Cargo.toml` 的 `members` 加入;若被复用,在 `[workspace.dependencies]` 加 `tn-<name> = { path = ... }`。

## 10. 术语表

- **PTY / ConPTY**:伪终端。一对内核管道,让程序以为自己连着真终端。ConPTY 是 Windows 10+ 的实现。
- **VT / ANSI 转义序列**:`ESC[...` 这类控制码,表达颜色、光标移动、清屏等。
- **OSC**:Operating System Command(`ESC ] … ST`),用于标题、cwd(OSC 7)、shell 集成(OSC 133/633)、超链接(OSC 8)。
- **DSR**:Device Status Report,`ESC[6n` 查询光标位置,终端须回 `ESC[<row>;<col>R`(见 §3.4)。
- **alt-screen**:全屏程序(vim、agent TUI)切换到的备用屏缓冲(DECSET 1049),退出后恢复主屏。
- **block**:Warp 式概念——一条命令及其输出作为一个可折叠/复制/重跑的单元。
- **damage / 脏行**:本帧相对上帧变化的行,用于局部重绘。
- **snapshot**:`tn-core` 产出的不可变可见网格视图,供渲染层消费。
