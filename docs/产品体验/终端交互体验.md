# 终端交互体验

## 定位

本文记录终端正文的用户可见体验:颜色、选择、scrollback、Warp 式 block、滚动条、滚轮归属、resize 和光标语义。底层 VT、ConPTY、渲染和数据流见系统架构。

## 基础能力

- 支持每格颜色、选择、复制粘贴、滚动历史、URL 检测和子串搜索内核。
- URL 点击 UI 与完整搜索 UI 后置,但内核能力已按架构路线图准备。

## Block 与全屏程序

- Warp 式 block 显示命令、输出、退出码和时长卡片。
- 当 alt-screen 或 agent surface 拥有 viewport 时,整体隐藏 block chrome 并全幅渲染,让位给全屏程序。
- shell 内敲 `claude` / `codex` 这类不一定进入 alt-screen 的主屏 TUI,也按 agent surface 处理。

## 滚动与选择

- 滚动条在右缘显示为细指示条,平时隐形或 hover 微亮。
- 可见 bar 保持纤细,命中区放大到透明带,保证好抓但不抢视线。
- 拖出元素外松开不应粘连。
- 普通 shell pane 的鼠标滚轮滚 Tn 的 scrollback。
- agent 或 alt-screen pane 的滚轮交给程序自身,避免把 Codex 或类 TUI 的旧主屏帧从终端历史里翻出来。

## 历史与 resize

- 阅读历史时不显示 live cursor。scrollback 视图只展示历史内容,不把当前输入光标投影到历史顶部。
- 从任何历史位置开始输入,都会先回到底部。
- 普通 shell 拖大窗格时内容留原位,下方开空行,符合 Windows 原生控制台语义,不吞滚动历史。
- agent / alt-screen 保持程序主屏语义,不把当前 TUI 屏推进 scrollback。

## 光标语义

- 块光标盖宽字符时变成 2 列宽。这是块光标固有行为,owner 确认保留,不是 bug。
- 编辑器光标和终端光标都优先保证瞬时、准确、对齐,不追求装饰性滑动。

## 状态

- 每格颜色、选择复制、滚动历史、右缘滚动条、滚轮归属、resize 锚定分流和 agent surface 处理已落地。
- 搜索 UI、URL 下划线和 Ctrl+Click 打开仍后置。
