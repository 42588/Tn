# 玻璃视觉原则

## 定位

本文记录 Calm Glass 的用户可见视觉取向和产品级取舍。确切颜色、圆角、字重、间距、token 和 CSS 到 GPUI 译法写入界面样式责任域。

## 核心取向

Calm Glass 是磨砂玻璃感,但深度来自光影分层,不是光污染。界面应显得安静、坚实、可读,避免自发光、霓虹描边、大面积炫光和装饰性堆砌。

## 产品决策

- 窗口保持 `Opaque`。曾试过默认开启 acrylic 透背,面板会透出桌面模糊;owner dogfood 后否决,原因是透明观感不喜欢、磨砂浓度不一致,并且容易显成“透明矩形框”。
- 玻璃感只靠应用内部面板层叠、1px 渐变边框环和投影表达。
- 大面积渐变改为纯色中点。8-bit 无噪点 dither 且无逐元素 `backdrop-filter` 时,大渐变容易产生断层色带。
- 深度感只在必要处用边框环、微弱 specular、高度层级和阴影表达,不要用大面积发光。
- 逐面板原型已经对齐 mockup。改视觉先看或改 [design/mockup.html](../../design/mockup.html) 与 [design/panels/](../../design/panels/),再端口到 GPUI。
- 主题色、圆角、字重和间距由样式 token 单一真源控制,避免在实现里散落硬编码颜色。

## 状态

- 标题栏、侧栏、Quick Look、命令面板、Agent 活动栏和主工作区的 Calm Glass 主体视觉已落地。
- 样式守卫、token 生成和组件数值不在本文维护,以界面样式责任域为准。
