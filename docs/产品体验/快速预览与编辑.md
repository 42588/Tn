# 快速预览与编辑

## 定位

本文记录 Quick Look 的用户可见体验:浮层定位、预览/编辑两态、Diff 审阅、保存安全、光标与 CJK 选择语义。实现细节和修复复盘不在本文展开。

## 背景

Quick Look 是点文件树或在树焦点按 `Space` 弹出的玻璃浮层。它贴文件树右缘、浮于终端、不占分屏,目标是快速看一眼或顺手小改,不是替代重度编辑器。真要大改仍应回终端使用 vim 或后续 Editor Pane。

## 两态交互

| 态 | 键 | 行为 |
|---|---|---|
| 预览 | `Up` / `Down` | 换文件,实时跟随文件树选中 |
| 预览 | `Tab` | 切换 File / Diff |
| 预览 | `Enter` | File tab 进入编辑态;Diff tab 跳到对应 File 行 |
| 预览 | `Ctrl+Enter` / 点击“打开为编辑器” | 把当前文档升级到 Editor Pane,继续使用同一份 buffer、cursor、undo 和 dirty 状态 |
| 预览 | `PgUp` / `PgDn` | Diff tab 跳上/下一个 hunk |
| 预览 | `Esc` / `Space` | 收起浮层,焦点还给终端 |
| 预览 | 鼠标拖选 + `Ctrl+C` / `Ctrl+A` | 只读复制,不可修改 |
| 编辑 | 打字、`Backspace`、`Delete`、`Enter`、`Tab` | 编辑文本,`Ctrl+S` 写盘 |
| 编辑 | 方向键、`Home`、`End`、`PgUp` / `PgDn` | 移动编辑器光标,不再换文件 |
| 编辑 | `Shift` + 方向、鼠标拖选 | 选区,语义是“实心块拖到哪就选到哪”;软换行文本中 `Shift+Up/Down` 按视觉行扩展 |
| 编辑 | `Ctrl+C/X/V`、`Ctrl+Z/Y`、`Ctrl+A` | 复制剪切粘贴、撤销重做、全选 |
| 编辑 | `Ctrl+F` / `Ctrl+H` | 查找 / 替换 |
| 编辑 | `Esc` | 回预览 |
| 编辑 | `Ctrl+Enter` / 点击“打开为编辑器” | 保留当前未保存内容、光标、撤销栈和 dirty 状态,升级到 Editor Pane |

## 体验决策

- 光标使用终端式就地反相块,真实位置始终在输入事件提交时立即更新,随字符列宽变化:中文 2 列宽,英文 1 列宽。自绘 File/Edit 光标使用与终端一致的 1px 小圆角,视觉高度贴近代码字形并居中在行盒内;行高、IME 锚点、命中测试、selection 和 copy 语义不随视觉块变化。开启 `[editor] animations = "subtle"` 时只在绘制层给同一视觉行小跨度打字/删除增加与终端一致的 cursor glide 和块光标 squash/pop;运动使用终端同款 chase 模型,连续输入从当前绘制位置继续追逐最新目标,不做字符残影。关闭动画、Windows reduced-motion、高负载/4000 行、大文件、IME 合成、拖选、选区、普通方向移动、查找跳转、滚动和结构性编辑均立即 snap。
- CJK 精确依赖固定单元格渲染,列到像素严格对齐,避免中文行的光标、选区和横向滚动漂移。
- 正文中文 IME 候选框应贴当前 caret 底部,并随软换行、CJK 双宽、横滚和纵滚位置移动;查找/替换框中文输入时候选框仍贴输入框。
- 自绘编辑器连续输入不应每个字符强制把光标行居中;只在光标离开视口时跟随,避免长行或 IME 连续提交时卡顿和跳动。
- Diff 审阅保持在浮层内完成。Diff tab 支持鼠标拖选、`Ctrl+C`、`Ctrl+A`;`PgUp` / `PgDn` 在 hunk 间跳转;`Enter` 或双击当前 diff 行切回 File tab 并定位到对应的新文件行,目标行用短暂 accent 高亮。
- 删除行定位到同一 hunk 内最近可对应的新文件行,等距时偏上一行。远端 diff 只在当前 Quick Look 内切 File,不把远端路径当本机路径打开;该路径仍需 SSH 实机复验。
- 本地文本保存记录打开时的编码、换行风格、mtime、size 和 hash。`Ctrl+S` 前若磁盘文件被外部修改或删除,先显示冲突条“重新载入 / 取消 / 覆盖保存”,默认不静默覆盖。
- 外部编辑器保存当前本地文件时,Quick Look 在目录 watcher debounce 后检查当前文件。预览态没有本地 dirty 编辑时自动刷新内容;编辑态不自动同步、不退出编辑、不立即弹冲突,后续 `Ctrl+S` 仍通过保存 guard 提示冲突。
- 正常保存保留原 UTF-8 BOM、UTF-16、GBK 与 LF/CRLF 风格。
- 代码长行可横向滚动,底部使用自绘细滚动条;编辑态光标会跟随横滚。预览每次从行首开始,不残留横滚位置。
- Markdown、txt、log 和说明文档长段落默认软换行;滚动、选区、复制和查找仍按逻辑文本语义工作,键盘上下移动/选择在编辑态按视觉行穿过同一长逻辑行。
- 浮层下铺 click-away scrim,故意不盖文件树、标题栏、状态栏;点空白收起。面板根吞点击,避免穿透到终端。
- Quick Look 打开期间必须持续持有键盘焦点。即使当前目录被其它软件修改并触发文件树或 agent 改动列表刷新,`Esc`、`Up` / `Down`、编辑态输入和保存仍归 Quick Look 处理,不落到底层终端或 workspace。
- Editor Pane 当前是实验性轻量编辑宿主,从 Quick Look 显式升级进入。它显示文件标题、dirty badge 和状态栏行列 / UTF-8 / LF 信息;未保存时关闭 pane、关闭 tab 或退出应用会显示“保存 / 放弃 / 取消”提示,避免静默丢失。当前用户可依赖的范围是基础本地文本编辑和 dirty close 保护;完整复用 Quick Look 的数据语义、真实光标/selection/IME 手感、格式保持、外部变更 guard、保存冲突 guard、Markdown 预览和终端往返仍属于后续改造。

## 状态

- 预览、编辑、只读选择、Diff 选择、Diff 跳转、本地保存守卫已落地。
- 外部目录变更后的 Quick Look 焦点保持已加 headless 回归守卫,仍建议真机复验。
- 外部编辑器保存后的 Quick Look 自动刷新已加 headless 回归守卫,仍建议真机复验预览态 debounce 后刷新、编辑态不被打断以及保存时冲突提示。
- 散文或 Markdown 长段落软换行已接入 File/Edit,软换行编辑态 `Shift+Up/Down` 已加 headless 守卫;仍建议真机复验跨视觉行选择、复制、查找和代码横滚。
- 轻量 typing effects 已接入 Quick Look 编辑态并有 headless motion gate 守卫,当前为终端同款 cursor-only chase 动画,光标圆角与视觉高度也已按终端观感收敛;仍建议真机用 `TN_PERF` 复验连续输入/删除、IME、4000 行和大文件。
- TnE-19/20 已接入 Quick Look “打开为编辑器”升级入口和 Editor Pane 基础宿主;`cargo test -p tn-ui --lib` 177 passed。Phase 0 已明确它当前只是实验性轻量编辑宿主,不是完整轻量 IDE。仍建议真机复验 Quick Look 编辑几步后升级到 Editor Pane 的输入、undo、保存和 dirty 关闭路径。
- 远端 Diff 路径和远端 guarded write-back 需继续按真实 SSH/SFTP 场景复验。
