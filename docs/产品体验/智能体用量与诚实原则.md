# 智能体用量与诚实原则

## 定位

本文记录 Tn 托管 AI agent 时的用户可见用量展示和诚实状态原则。adapter、事件模型、pricing、会话解析和 crate 边界见系统架构。

## 用量展示

托管带遥测 adapter 的 agent 时,Tn 解析其本地会话记录,计算上下文占用、token 和估算花费。展示位置:

- 每个 agent pane 头部显示一枚环形读数,用量环按百分比动态合成。
- 状态栏读取焦点 pane 的用量。
- 匹配策略优先使用启动意图。
- `codex` 默认可能在 `~` 跑,会话 cwd 不等于 app cwd 时,回退到该 agent 最新会话。
- 普通 shell 不靠会话新鲜度反推 agent,避免同目录 dev agent 误标;只认 launch intent 和 shell 集成当前运行命令首 token。

## 诚实状态原则

核心原则:看不到的状态绝不编造。

- 不伪造 agent “运行中”或 “Thinking”。普通 PTY 只能观察字节流,没有“正在想”的信号。
- Tn 不画转圈、不脉冲来假装 agent 在忙。
- 只有外部或 sidecar adapter 明确发出 `AgentEvent::StatusChanged`、`PermissionRequested`、`ErrorReported`、`TranscriptAppended`、`ModelChanged` 时,agent 头才显示对应状态、权限、错误或摘要 chip。
- 没有结构化事件时,界面保持安静。

## 活动栏与文件变化

- 活动栏“本次改动”来自真实 `git diff HEAD --numstat`,不解析终端正文猜测改了什么。
- 文件变化即刷新,但需要过滤噪声目录并防抖。
- 状态行只显示 git 摘要,例如 N 文件、增加删除行数或工作区干净,不显示伪造的运行态。

## shell 内 agent 检测

- 用户在 shell 内敲 `claude` 或 `codex` 时,Tn 可以自动切 agent 态,依据是 shell 集成报告的当前运行命令。
- 命令结束即还原。
- 不通过进程树轮询,也不靠会话新鲜度判断,因为这些信号脆弱且容易误标。
- shell-detected agent 的活动栏走 overlay,不改变终端正文宽度。

## 文件系统与 pane 上下文

- Host、WSL、SSH/macOS 远端 cwd 分属不同命名空间,不能串台。
- Host 使用 Windows 路径。
- WSL 通过 `\\wsl$\<distro>` 驱动 Linux 目录树并显示 Linux dotfiles。
- SSH/macOS 远端通过 SFTP 后端驱动远端 Explorer root、Quick Look 有界预览和文本 guarded write-back,不能把 `/home/...` 或 `/Users/...` 当 Windows 目录。
- 每个分屏 pane 记自己的文件树 root、展开态、选中文件、用量环和 git 改动。切换焦点 pane 时各自恢复,不被别的 pane 覆盖。
- “打开文件夹”只影响当前焦点 pane,绝不广播 `cd`,尤其不打扰正在交互的 agent pane。

## 状态

- 内置 Claude/Codex 用量、状态栏焦点读数、真实 git diff 活动栏、shell 内 agent 检测和 per-pane 上下文已落地。
- 外部 sidecar adapter 与联网确认按 Agent Host 平台继续扩展。
