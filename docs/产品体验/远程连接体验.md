# 远程连接体验

## 定位

本文记录 SSH 连接的用户可见体验:连接器、进度、密码、主机指纹、错误、最近连接、收藏和 Quick Terminal 入口。底层 SSH、SFTP、远端文件服务和实现记录只链接不复制。

## 核心原则

SSH 连接遵循“先解释、再进入 shell”。进度、密码、主机指纹和错误都应在窗格内用 Calm Glass 覆盖卡表达,不要把关键决策埋进终端滚动文本。

## 连接与安全

- 首次连接未知主机时,必须由用户确认主机指纹。
- 只有用户选择“记住”时才写入 `known_hosts`。
- 如果写入 `known_hosts` 失败,Tn 必须明确提示并中止本次连接,不能让用户误以为已保存信任。
- 连接覆盖卡存在时,普通按键、粘贴、重跑命令、alt-screen 滚轮输入都不得进入远端 PTY。
- shell 真正打开前会清掉排队字节,避免用户在进度、密码或指纹卡上敲的键在连接后 replay 到远端 shell。

## 状态表达

- 连接中只显示真实阶段:建立连接、认证、打开 shell。
- 断线重连使用非模态横幅和连接态信息胶囊,不伪造不可观测的转圈状态。
- SSH pane 头部统一显示类似“已连接 user@host:port 登录方案 密钥/密码/交互”的连接信息。
- 覆盖卡中的“取消 / 关闭”表示退出本次 SSH 尝试。若这是最后一个窗格,界面回到欢迎启动页,不滞留失败卡或进度卡。

## 连接管理

- 成功连接会自动进入“最近”。
- 点星标即收藏并长期置顶;再点一次取消收藏。
- 收藏行提供重命名入口,昵称支持中文,只改变显示名,不改变真实 `user@host:port` 连接目标。
- 连接器面板有视口内最大宽高,列表不会撑爆窗口。
- `~/.ssh/config` 行作为只读来源显示;要移除源项需要编辑 OpenSSH 配置文件。
- 连接器不再提供“保存为连接”或删除已保存连接的子流程。长期保留通过收藏承担。

## Quick Terminal 入口

- Quick Terminal 启动器中的 SSH 卡片打开紧凑连接器。
- 连接器可从 SSH profiles、最近连接、`~/.ssh/config` alias 或手输 `user@host[:port]` 启动远程 shell。
- 最近连接行同样支持星标收藏、取消收藏和中文昵称重命名。
- SSH 目标输入阶段禁用中文输入法,避免 host 字段混入非 ASCII。

## 状态

- 连接器、最近连接、收藏、中文昵称、进度卡、错误卡、密码卡、TOFU 指纹卡、连接态胶囊和 Quick Terminal SSH 入口已实现。
- 密码框、TOFU 首连信任卡、断线重连横幅、ssh-config 行和拨号校验仍需要真实 SSH 目标持续复验。
- `keyboard-interactive`、ssh-agent、`ProxyJump` 和低优打磨仍按新增模块或已知问题跟踪。
