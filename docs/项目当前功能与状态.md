# 项目当前功能与状态

## 定位

本文是当前仓库的功能地图和源码状态快照,用于新一轮产品、架构和体验设计前快速了解“项目里现在有什么”。本文不是旧方案恢复、不是路线图,也不是运行验收报告。

信息来源限定为当前源码、`Cargo.toml` 和现存最小文档体系。凡是本轮没有实际运行验证的内容,只写为“源码入口已存在”或“待运行验收”。

## 总体判断

Tn 当前不是空仓库,而是一个已有较多实现的 Rust / GPUI 终端应用工程。现有代码已经覆盖桌面入口、主工作区、终端 pane、PTY 抽象、本地/WSL/SSH 会话、文件浏览、Quick Look、远端文件、远端 Git diff、Agent Host、用量展示、Quick Terminal、布局槽位、配置与主题等功能。

但项目文档已按用户要求归零重启,旧产品方案、旧路线图、旧任务历史和旧架构文档不再作为权威来源。因此后续重新设计时,应把本文当作“现有资产盘点”,而不是必须延续的产品方案。

## 用户可见功能

### 1. 启动与窗口

- `tn-app` 是二进制入口,负责初始化 stderr + 文件日志、panic hook、Windows AppUserModelID 和 mimalloc 空闲回收。
- release 构建使用 `windows_subsystem = "windows"` 隐藏控制台,日志写入配置目录下的 `logs/tn.log`。
- `tn-ui::run()` 加载配置和主题,注册 workspace keybindings,嵌入 `CaskaydiaCove Nerd Font` 字体。
- 主窗口默认居中打开,初始尺寸约 1100x720,使用透明 titlebar,窗口背景跟随主题 backdrop。
- 主实例会尝试注册 Quick Terminal 全局热键;能注册时才启用 tray listener 和隐藏弹出窗口。

### 2. 欢迎页与启动器

- 新标签默认打开 Welcome launchpad,不是立即创建 shell pane。
- Welcome 中的启动项来自配置 `[[profiles]]`、已发现 WSL distros 和 SSH 入口。
- 启动项类型包括普通 shell、WSL、SSH、Agent。
- Agent tile 支持添加、编辑和删除自定义 agent。
- 点击 Welcome tile 会把当前欢迎 tab 转换为实际 pane tree。
- 命令面板和 Quick Terminal 复用类似的 profile / WSL / SSH 启动逻辑。

### 3. 工作区、标签和分屏

- 支持多标签页,`Ctrl+Shift+T` 新建欢迎页 tab,`Ctrl+Tab` 切换 tab。
- 每个 tab 的内容是 n-ary pane tree,支持横向和纵向分屏。
- `Ctrl+Shift+D` 向右分屏,`Ctrl+Shift+E` 向下分屏。
- 支持关闭 pane、切换 pane、关闭 tab,并在关闭前处理 Quick Look 的未保存改动。
- 支持键盘调整 pane 权重:宽度和高度增减。
- 鼠标拖动分隔线时先显示预览线,松手后一次性提交 resize,减少 ConPTY 重绘导致的内容跳动。
- 支持后台 tab 终端渲染缓存清理,避免打开很多 tab 后 GPUI 元素缓存线性膨胀。

### 4. 命令面板和新会话分屏

- `Ctrl+Shift+P` 打开命令面板,可搜索并启动 profile。
- 命令面板启动 profile 时会新建 tab。
- 支持 WSL 聚合卡片:多个 distro 时先 drill down,单个 distro 时直接启动。
- SSH 启动进入内置 SSH prompt,而不是普通 profile 列表直接启动。
- `Ctrl+Shift+N` 或 App Menu 的“新会话”打开分屏启动器。
- 分屏启动器先选择方向,再选择 profile;启动后在当前 pane 的指定方向创建新 pane。
- 新会话分屏与命令面板不同:前者进入当前 tab 的分屏树,后者进入新 tab。

### 5. App Menu 与布局管理

- 点击 Tn brand 打开 App Menu。
- 菜单入口包括新会话、新标签、打开文件夹、布局、文件浏览器、设置、主题、重载配置、关于和退出。
- “设置”会用 Quick Look 的编辑模式打开 `config.toml`,支持在应用内编辑并保存。
- “打开文件夹”对 Host 使用系统目录选择器,对 WSL / SSH 进入内置远端目录选择器。
- 布局管理提供 7 个槽位,可保存当前 tab 的 pane tree、加载到当前 tab、删除槽位。
- 布局保存的是 pane 结构和 launch spec,不会序列化正在运行的进程状态。

### 6. Quick Terminal

- Quick Terminal 是独立 `WindowKind::PopUp` 浮窗,由全局热键唤出。
- 配置默认启用,默认热键来自 `[quick_terminal]`。
- 第一次唤出且没有运行会话时展示启动器。
- 启动器支持 Agent、shell、WSL 和 SSH prompt。
- 选择会话后嵌入普通 `TerminalView`,因此 agent header、usage、SSH 卡片等能力可复用。
- 窗口可按配置从屏幕边缘滑入,支持 autohide。
- 会话在隐藏后保留,再次唤出继续使用;也可切回启动器替换会话。
- Quick Terminal 对 Win32 窗口操作做了异步延迟处理,避免 GPUI 回调中重入窗口过程。

## 终端能力

### 7. Headless 终端引擎

- `tn-core::Terminal` 封装 `alacritty_terminal` 的 `Term` 和 VTE `Processor`。
- PTY 输出通过 `advance()` 进入终端状态机,UI 通过 `snapshot()` 拿不可变快照。
- `TerminalSnapshot` 包含行列数、光标、scrollback、默认色、单元格和可见文本。
- 支持 ANSI 0..255、24-bit RGB、OSC live color overrides、默认主题 palette。
- 支持按 row 合并同样式 cell run,减少 UI 渲染时每格一个元素的成本。
- 支持 CJK / double-width 字符列宽,避免 glyph advance 与网格列宽不一致导致错位。
- 支持选择、单词选择、整行选择、复制 selection text。
- 支持搜索可见区和 scrollback,结果使用绝对行号。
- 支持检测可见区 `http://` / `https://` URL,并记录 viewport 坐标。
- 支持 bracketed paste、alt screen 等输入模式查询。
- 支持 ConPTY resize 专用 top-anchor grow 逻辑,避免长高时把历史内容错误拉进 live screen。
- 支持 grid swap out / restore,用于长期停驻终端的内存压力控制。

### 8. Terminal Pane 交互

- `TerminalView` 负责把 `tn-core`、PTY backend 和 GPUI 渲染连接起来。
- Reader thread 从 PTY 读取字节,喂给 `tn-core`,并把 `PtyWrite` 回复写回 PTY。
- Foreground repaint loop 合并输出 wake,处理 SSH/PTY 事件、agent 状态、cwd 变化和 bell。
- 支持键盘输入编码、IME composition、中文输入、复制、粘贴和 bracketed paste。
- 支持鼠标选择、拖拽选择、scrollbar thumb 拖动和滚动到历史。
- 支持光标 blink、短距离 cursor glide、visual bell 和可选 audio bell。
- 支持点击/快捷操作命令块:复制命令、重跑命令。
- 在 alt screen 或 agent 占用视图时隐藏命令块 bar,避免干扰 TUI/agent 正文。
- 支持以 cwd 更新 Explorer root、tab path badge 和 agent activity rail root。

### 9. Shell Integration 与命令块

- `tn-shell` 旁路解析 OSC 133、OSC 633 和 OSC 7,不干扰终端网格。
- 可识别 prompt start、command start、output start、command finished、command line 和 cwd。
- `tn-blocks` 把事件组织成命令块,记录命令文本、cwd、输出起止行、exit code 和耗时。
- Terminal Pane 的 block bar 从 `BlockModel` 读取当前或最近命令状态。
- shell-detected agent 依赖 command line 事件识别用户在普通 shell 中输入的 agent 命令。

## 会话与远端能力

### 10. LaunchSpec 与会话类型

- `LaunchSpec` 把配置 profile 转为可启动 pane。
- 默认本地 shell 是 `powershell.exe -NoLogo`,并注入 PowerShell shell integration。
- WSL profile 通过 `wsl.exe -d <distro> --cd ~` 启动,文件命名空间标记为 WSL。
- SSH profile 生成 `SshConfig`,由 `SshBackend` 运行远端 PTY。
- 非 PowerShell 的 command 会被托管在 PowerShell 中执行,用于兼容 npm shim / agent 命令。
- Agent profile 会解析到开放 `AgentId`,再从 registry 获取展示、能力和 usage adapter。
- `FileNamespace` 明确区分 Host、WSL、SSH 的文件命名空间,避免把远端 cwd 当成本机路径用。

### 11. SSH 交互

- SSH 后端基于 `russh`,并通过统一 `PtyBackend` 暴露给 TerminalView。
- 连接过程会发出 progress event:连接、认证、打开 shell。
- 认证支持 public key、password 和 keyboard-interactive。
- 首次主机指纹进入 TOFU host key 确认卡片,可选择仅本次信任或写入 `known_hosts`。
- 密码 prompt 有遮罩输入、显示/隐藏、记住本会话和错误重试。
- 失败卡片区分认证失败和 host key mismatch,并给出可操作提示。
- 连接成功后记录 auth method,用于最近连接列表显示密钥/密码/交互 badge。
- 断开后有 disconnected / reconnect 相关 UI 状态。
- SSH 最近连接支持收藏、重命名、过滤、按收藏和最近使用排序。

### 12. 远端文件和远端目录

- `tn-pty::remote_fs` 提供 SFTP v3 风格 API:列目录、读文件、stat、写文件。
- `RemotePath` 负责远端路径 normalize、join、parent 和 name。
- `RemoteId` 用 user、host、port 和 path 唯一标识远端文件。
- Explorer 可以以 SSH root 展示远端目录树。
- Quick Look 可以打开远端文件,并以虚拟路径展示在 UI 中。
- 远端文件读取有限制,当前 `REMOTE_READ_LIMIT` 为 2 MiB。
- Quick Look 保存远端文本前会 stat + read 做冲突检测,再通过 SFTP 写回。
- WSL root 通过 `\\wsl$` / `\\wsl.localhost` UNC 路径映射,Explorer 可按 Linux 路径展示。
- 远端目录选择器用于 SSH / WSL 的“打开文件夹”,支持目录列表、父级、进入子目录和错误状态。

## 文件、预览与编辑

### 13. Explorer 文件树

- Explorer 默认 root 是 `USERPROFILE` / `HOME` / 当前目录。
- root 类型包括 Host、WSL、SSH。
- Host root 隐藏 dotfiles,WSL / SSH root 显示 dotfiles。
- 目录和文件按“目录优先、名称不区分大小写”排序。
- 每个 pane 保存独立 ExplorerSnapshot,包括展开状态和选中文件。
- focus 在 split pane 间切换时,Explorer 恢复对应 pane 的 root 和树状态。
- 同一 pane 内 cwd 变化时,Explorer follow root 并尽量保留祖先展开状态。
- Host root 支持 `notify` 文件监听,经过 debounce 后重建树。
- Host root 支持异步 git status,显示 M / A / D / U / R 等状态并向父目录聚合。
- 点击文件会发出 `OpenFile`,由 Workspace 打开 Quick Look。
- 键盘可在 Explorer 中上下移动、Enter/Space 打开文件或展开目录。

### 14. Quick Look 预览

- Quick Look 是工作区浮层,可从 Explorer 文件、agent activity rail 或菜单设置入口打开。
- 支持 File / Diff 两个 tab。
- 文本文件在后台线程读取,避免 UI 阻塞。
- 支持 UTF-8、UTF-8 BOM、UTF-16 LE、UTF-16 BE、GBK 解码与保存时编码保持。
- 支持保留换行风格 LF / CRLF 和 final newline。
- 大文件或含 null byte 文件按二进制显示。
- 支持图片预览:png、jpg、jpeg、webp、gif、bmp 等。
- 支持 PDF 预览,通过 Pdfium 渲染页面图片。
- Office / 二进制类文件只读,不显示编辑入口。
- 关闭 Quick Look 时会显式移除 GPUI image / PDF asset 并释放缓存容量。
- 支持外部文件变更检测:未编辑时自动 reload,有未保存改动时提示冲突。

### 15. Quick Look 编辑模式

- 可编辑文本文件可进入编辑模式。
- 编辑状态使用 `DocumentSession` / `tn-editor` 文档模型。
- 支持输入、换行、退格、delete、方向键、Home/End、Page、选择、复制、粘贴、全选。
- 支持查找、替换当前、替换全部。
- 支持 Ctrl+S 保存本地或远端文件。
- 保存前用 `FileGuard` 检查 mtime、size 和 sample hash,防止覆盖外部改动。
- 遇到文件已被外部修改、缺失或无法确认状态时显示冲突。
- 支持强制保存当前来源。
- 有 pending leave 机制:切换文件、切换 tab、关闭、退出前都先处理未保存改动。
- 编辑后返回预览时会把未保存缓冲镜像到 File tab,让预览显示当前编辑内容。
- Quick Look 自绘 File / Diff / 编辑路径默认开启,`TN_QL_LEGACY=1` 可回退旧渲染路径。

### 16. Diff 与远端 hunk 操作

- Diff tab 懒加载,只有进入 Diff 时才运行 `git diff`。
- 本地文件 diff 使用 bounded git helper,避免慢 git 卡住 UI。
- 远端 diff 通过 SSH remote command 运行 `git diff --no-color -- <file>`。
- Diff 解析会区分 hunk header、context、addition、deletion。
- 支持 hunk 跳转、从 Diff 行跳回 File 行并做高亮。
- agent activity rail 点击文件卡片时直接打开 Quick Look 的 Diff tab。
- 远端 Diff tab 的每个 hunk header 可显示“接受/拒绝”按钮。
- 接受远端 hunk 使用 `git apply --cached -`,拒绝使用 `git apply --reverse -`。
- 远端 hunk 操作会重新抓取 diff,并通知 workspace 刷新 agent activity rail。

## Agent 与智能体功能

### 17. Agent Host 平台

- `tn-agent` 是无 UI 的 Agent Host 平台抽象。
- Agent 身份使用开放 `AgentId`,不再用封闭 enum 限制 Claude/Codex。
- `AgentDescriptor` 描述 label、short、aliases、accent、glyph、能力、运行时和网络策略。
- 基线能力包括 terminal、cwd sync、git diff;usage、transcript、permission prompts 等是可选能力。
- `AgentRuntimeKind` 描述运行位置/协议,不等同于文件命名空间。
- `AgentRegistry` 支持按 command alias 匹配 agent,也支持 fallback generic descriptor。
- UI 通过 GPUI global `AgentHost` 读取 active registry。

### 18. Claude / Codex usage adapter

- `tn-ai` 提供内置 Claude Code 和 Codex adapter。
- Claude adapter 读取 `~/.claude/projects/<proj>/<session>.jsonl`。
- Codex adapter 读取 `$CODEX_HOME/sessions/**/rollout-*.jsonl`。
- usage poller 会绑定到 pane 启动后最新的 session log,避免读取旧会话。
- usage 展示支持 cost、context percentage 和 tokens 三种模式。
- `[general].billing_mode` 是全局起始模式,`[general.billing]` 可按 agent id 覆盖。
- usage pill 点击后在当前 pane 内循环显示模式,不影响其他 pane。

### 19. Agent UI Surface

- Agent pane header 显示头像/名称/模型/usage ring/pill。
- sidecar / realtime adapter 可更新 status、model、transcript tail、permission prompt 和 error。
- 网络 sidecar 默认需要用户确认,不会静默连接。
- Agent activity rail 显示本次改动文件列表。
- 本地 activity rail 通过 git diff / numstat 计算;远端通过 remote git command 计算。
- Activity rail 有 Loading / Ready / Idle 状态,UI render 不直接做 git I/O。
- 文件变更 watcher 会 debounce 后刷新 activity rail。
- 点击 activity rail 文件卡片打开 Quick Look Diff。
- Quick Look 在 rail 模式下用上下键在当前 pane 的改动文件中导航,不跳回 Explorer 全树。

### 20. Agent 配置与管理

- `[[agents]]` manifest 可声明 id、label、short、aliases、accent、glyph、cursor、capabilities、runtime support、network policy 和 sidecar。
- `[[profiles]]` 中 `kind = "agent"` 的 profile 负责把 agent 放入启动器。
- Welcome 中可添加、编辑和删除自定义 agent。
- Agent 表单支持名称、命令、sidecar、是否网络、颜色和是否由 agent 自己绘制 cursor。
- 保存表单会向 `config.toml` 追加 `[[agents]]` 和 `[[profiles]]`,尽量保留原文件注释。
- 编辑 agent 时先移除旧 block 再追加新 block,避免重复。
- 删除 agent 会移除对应 manifest 和 profile。
- 保存后 workspace 会 reload config、重建 AgentRegistry、重建 Welcome,无需重启。

## 配置、主题与工程支撑

### 21. 配置系统

- `tn-config` 使用 TOML schema,所有字段默认继承内置配置。
- 首次运行会写入默认 `config.toml` 和 `themes/tn-dark.toml`。
- 配置无法读取或解析失败时回退内置默认。
- 支持 `[general]`、`[font]`、`[appearance]`、`[editor]`、`[quick_terminal]`、`[[profiles]]`、`[[agents]]`、`[[actions]]`、`[[keybindings]]`。
- keybinding 通过 action id 映射到 workspace command。
- `reload_config` 可重新读取配置并把 palette 应用到现有 pane;字体和 scrollback 类变更需要新 pane。
- App Menu 的“重载配置”实际是 reset default config 的破坏性路径,和 `Ctrl+Shift+R` 的 hot reload 不同。

### 22. 主题与样式

- 内置主题是 Tn Dark。
- Theme 包含 terminal colors、UI colors、agent colors、window chrome 等。
- UI 样式集中在 `tn-ui/src/style.rs`,包含玻璃面板、radius、shadow、颜色 helper 和 icon helper。
- 终端 palette 可以从 config theme 映射到 `tn_core::Palette`。
- 桌面窗口 backdrop 可根据主题使用 Opaque / Blurred。
- Editor motion 通过 `[editor].animations` 配置,并根据 reduced motion / high load 降级到 instant。

### 23. 运行与验证工具

- `tn-cli` 是 headless debug harness,可跑 ConPTY -> `tn-core` -> snapshot 链路。
- 默认 smoke 会启动 `cmd.exe /c echo HELLO_TN_MARKER`,检查快照中是否出现 marker。
- `TN_RESIZE_EXP` 提供 resize probe,用于验证 ConPTY grow/shrink 是否丢 scrollback。
- `TN_AUTOQUIT=1` 用于 headless self-test。
- `TN_DEMO=1` 会启动脚本化 GUI demo,按步骤展示 prompt、彩色输出、scrollback、selection、split 和 resize。
- `TN_QL_BENCH=<path>` 可打开 Quick Look 后定时退出,用于复现预览 paint freeze。
- `TN_PERF` / `TN_AGENT_DUP` 等 env 开关用于渲染和 agent 重复诊断。

## 重新设计前建议

1. 先读取 [踩坑记录](共享记忆/踩坑记录.md),尤其是终端生命周期、焦点输入法、GPUI 渲染、远端命名空间和视窗工具链坑位。
2. 把现有代码当作“可复用资产池”而不是产品约束:终端核心、PTY、Explorer、Quick Look、Agent Host 和配置系统都可复用,但旧 UI 信息架构和旧路线图不应自动继承。
3. 重新定义产品方向前,先跑最小验证:文档链接检查、`cargo check`,再按需要跑 `tn-core` / `tn-editor` / `tn-config` / `tn-agent` / `tn-ui` pure helper 测试和 `tn-cli` smoke。
4. 如果准备继续做产品设计,建议下一步新建独立的“新设计功能边界”或“产品体验索引”,把“要保留、要砍掉、要重做”的决策从本文中分离出去。
