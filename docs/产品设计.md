## 1. 产品定位

Tn 是 **Windows 优先、为 vibe coding 打造的终端**:把 AI 编码 CLI(Claude Code / Codex)当**一等公民**托管,给它出色的 UX、灵活平铺、原生 WSL/SSH。设计取向一句话:**像现代编辑器一样顺手,像优秀终端一样扎实,颜值靠光影分层而非堆砌**。

## 2. 交互

### 2.1 Quick Look 速览 / 编辑(两态)

**速览浮层** = 点文件树(或树焦点按 `Space`)弹出的、**贴树右缘、浮于终端、不占分屏**的玻璃卡。它的设计意图是「**快速看一眼 / 顺手小改**」,不是替代重度编辑——真要大改回终端用 vim。

**两态键位**(焦点在浮层时):

| 态 | 键 | 行为 |
| --- | --- | --- |
| **预览(只读)** | `↑`/`↓` | 换文件(实时跟随,树选中同步移动) |
| | `⇥` | 切 File / Diff |
| | `Enter` | 进编辑态 |
| | `Esc` / `Space` | 收起(焦点还给终端) |
| | 鼠标拖选 + `Ctrl+C`/`Ctrl+A` | 只读复制(**不可改**) |
| **编辑** | 打字 / `Backspace` / `Delete` / `Enter` / `Tab` | 编辑(`Ctrl+S` 写盘) |
| | 方向键 / `Home` / `End` / `PgUp·Dn` | 移光标(归编辑器,不再换文件) |
| | `Shift`+方向 / 鼠标拖选 | 选区(「实心块拖到哪、就选到哪」) |
| | `Ctrl+C/X/V` · `Ctrl+Z/Y` · `Ctrl+A` | 复制剪切粘贴 / 撤销重做 / 全选 |
| | `Ctrl+F` / `Ctrl+H` | 查找 / 替换 |
| | `Esc` | 回预览 |

**关键体验决策**:

- **光标 = 终端式就地反相块**(瞬时、精确、随字符列宽:中文 2 列粗、英文 1 列细)。**曾试 Q 弹平滑滑动光标,owner 实测卡顿 + 字飘后撤回**——结论:文本编辑器的光标要的是「**瞬时准**」不是「平滑漂」(见 [优化日志 §56](优化日志.md))。
- **CJK 精确**靠固定单元格渲染(列↔像素严格对齐),中文行光标 / 选区 / 横滚不再漂(见 [优化日志 §55](优化日志.md))。
- **横向**:长行可横滚(自绘底部细滚动条 + 编辑态光标跟随);预览每次从行首开始,不残留横滚位置。**散文/Markdown 长段落暂只能横滚不能软换行**(技术约束,见架构蓝图 §8 ⏳后续)。
- **不占分屏 / 不抢焦点**:浮层下铺 click-away scrim(故意不盖文件树/标题栏/状态栏),点空白干净收起;面板根吞点击防穿透到终端(见 CLAUDE.md 坑)。

### 2.2 平铺 / 分屏 / 标签 / 欢迎页

- **n-ary 平铺分屏**:任意方向切分,键盘改尺寸 + 分隔线拖拽(**commit-on-release**:拖动只移预览线、释放才 resize 一次,避免 ConPTY 每帧抖动)。
- **欢迎 launchpad**:新标签 / 首个标签 = 启动磁贴(发现的 profile)+ 快捷键提示,点磁贴起会话。
- **默认布局**:`Explorer(窄) | Claude(大) + 小 shell`。

### 2.3 命令面板 / 快捷键

`Ctrl+Shift+P` 命令面板;侧栏 `Ctrl+Shift+B`、Quick Look 收起 `Ctrl+Shift+J`。**注**:`Ctrl+Shift+*` 在中文/多布局 Windows 可能被系统/IME 吞,真机不达时可改键(见 CLAUDE.md 坑)。命令面板中文搜索 parked(IME 合成只经 `EntityInputHandler`,面板未接,目前只搜英文)。

### 2.4 SSH 连接 UX

SSH 连接遵循「先解释、再进入 shell」:进度、密码、主机指纹、错误都用窗格内 Calm Glass 覆盖卡表达,而不是把关键决策埋进终端滚动文本。

- **安全默认**:首次连接未知主机必须由用户确认指纹;选择「记住」时才写入 `known_hosts`。如果写入失败,Tn 会明确提示并中止,不会让用户误以为已保存信任。
- **输入隔离**:连接覆盖卡存在时,普通按键、粘贴、重跑命令、alt-screen 滚轮输入都不进入远端 PTY。连接真正打开 shell 前还会清掉排队字节,避免用户在进度/密码/指纹卡上敲的键连接后 replay 到远端 shell。
- **诚实状态**:连接中只显示真实阶段(建立连接 / 认证 / 打开 shell);重连用非模态横幅和连接态信息胶囊,不伪造不可观测的转圈状态。SSH 窗格右上角统一为「已连接 user@host:port 登录方案 密钥/密码/交互」格式,不再拆成孤立的状态文字和 host chip。
- **明确退出**:覆盖卡里的「取消 / 关闭」表示退出本次 SSH 尝试;如果这是最后一个窗格,界面回到欢迎启动页,不把失败卡或进度卡滞留在屏幕上。
- **连接管理**:成功连接会自动进入「最近」;点星标即收藏并长期置顶,再点一次取消收藏。收藏行提供重命名入口,昵称支持中文,只改变显示名,不改变真实 `user@host:port` 连接目标。连接器面板有视口内最大宽高,列表不会撑爆窗口。连接器不再提供「保存为连接」或删除已保存连接的子流程;`ssh-config` 行来自 `~/.ssh/config`,作为只读来源显示,要移除源项需编辑 OpenSSH 配置文件。
- **Quick Terminal 入口**:幽灵终端启动器里的 SSH 卡片打开紧凑连接器,可从 SSH profiles、最近连接、`~/.ssh/config` alias 或手输 `user@host[:port]` 直接启动远程 shell;最近连接行同样支持星标收藏/取消收藏和中文昵称重命名,SSH 目标输入阶段禁用中文输入法以避免 host 字段混入非 ASCII。

## 3. 视觉系统(Calm Glass)

**取向:磨砂玻璃靠光影分层,拒绝「光污染」(自发光、霓虹描边)。** 深度感靠**层叠 + 1px 渐变边框环 + 投影**,不靠大面积发光。

- **窗口保持 `Opaque`**:试过默认开 acrylic 透背(面板透出桌面模糊),owner dogfood 后**否决**(透明观感不喜欢、磨砂浓度不一致显「透明矩形框」、大面积半透色带)。玻璃感只靠内部面板层叠。
- **大面积渐变改纯色中点**:8-bit 无噪点 dither + 无逐元素 `backdrop-filter`,大渐变会断层色带 → 所有大面积渐变改纯色,深度仅靠 1px 渐变边框环(见 CLAUDE.md 坑)。
- 逐面板已 **1:1 复刻 mockup**;颜值由 `style.rs` token 单一真源 + 四道机器守卫(`token_drift`/`roots_mirror`/`no_hardcoded`/`spec_gen`)把关,主题色硬编码清零。

> 确切颜色/圆角/字重/间距数值 → [样式还原手册 §16](样式还原手册.md)。

## 4. 终端体验

- **每格颜色** · 选择/复制粘贴 · 滚动历史 · URL 检测 · 子串搜索(内核就绪)。
- **Warp 式 block**:命令/输出/退出码/时长卡片;**alt-screen(vim/agent TUI)进入即整体隐藏 block chrome、全幅渲染**——「让位给全屏程序」。
- **滚动条**:右缘细指示条,平时隐形/hover 微亮;**可见 bar 纤细、命中区放大到 14px 透明带**(好抓又不抢视线);拖出元素外松开不粘连(见 [优化日志 §58](优化日志.md))。
- **resize 长高顶锚定**:拖大窗格内容留原位、下方开空行(合 Windows 原生控制台语义),不吃滚动历史。
- **光标随字符宽**:块光标盖宽字符(CJK)即变 2 列宽——这是块光标固有、owner 确认保留(非 bug)。

## 5. AI 用量展示

托管 Claude/Codex 时,Tn 解析其**本地会话 JSONL**(Claude `~/.claude/projects`、Codex `$CODEX_HOME/sessions`)算出**上下文占用 + token + 估算花费**,展示为:

- **每 pane 头一枚环形读数**(用量环按 % 算 `stroke-dashoffset` 动态合成 SVG)。
- **状态栏读「焦点 pane」**的用量。
- 匹配策略:**启动意图优先**;codex 默认在 `~` 跑(会话 cwd ≠ app cwd)→ 回退「该 agent 最新会话」。普通 shell **不靠会话新鲜度反推 agent**(同目录的 dev agent 会误标),只认 launch intent + shell 集成当前运行命令首 token。

> 实现/数据流见 [架构蓝图 §4.5](架构蓝图.md)(`tn-ai`)。

## 6. Chrome 与「诚实」原则

### 6.1 标题栏 / 状态栏

自绘集成标题栏(`appears_transparent`,OS 直接执行拖窗/最小化/最大化/关闭的 NC 命中,不双触发);多段状态栏(分支 / 会话数 / 文件类型 / 编码 / 主题 / 焦点 pane 用量)。

### 6.2 app 菜单(logo 可点)

点左上 **`Tn ▾` 品牌**弹下拉(会话 / 文件·工作区 / 设置)。**品牌从「拖窗把手」改成「可点按钮」**(`caret` 提示可点),**拖窗改靠标签条空白**(spacer)——这是有意取舍:logo 当菜单入口比当拖把更有用。点外部 scrim 关闭。各项接真实动作:新会话(分屏启动器)/ 新标签(欢迎页)/ 打开文件夹(重定文件树根 + `cd` 纯 shell 窗格)/ 布局(7 槽存取删)/ 文件浏览器开关 / 设置(在 Quick Look 编辑器开 `config.toml`)/ 主题 / 重载配置(还原默认,panic button)/ 关于 / 退出。

### 6.3 诚实状态(不伪造)

**核心原则:看不到的状态绝不编造。**

- **不伪造 agent「运行中 / Thinking」**:agent 思考/运行态对终端是**不可观测**的(PTY 里只有字节流,没有「正在想」信号),所以 Tn **不画转圈、不脉冲**假装它在忙。`@keyframes pulse` 只用于真实可观测的指示,不用来伪造 Thinking。
- **活动栏「本次改动」= 真实 `git diff HEAD`**(`--numstat`),**不解析终端正文**猜测改了什么;文件变化即刷新(`notify` 监听 cwd,过滤噪声目录 + 防抖)。状态行只显 git 摘要(N 文件 +X −Y / 工作区干净),不显伪造的运行态。
- **shell 内敲 `claude`/`codex` 自动切 agent 态**:认 shell 集成的**当前运行命令**(诚实——用户真敲了),命令结束即还原;不进程树轮询(脆弱)、不靠会话新鲜度(会误标同目录 dev agent)。
- **文件系统不串台**:本地 Host、WSL、SSH/macOS 远端 cwd 分属不同命名空间。Explorer 只接管可由本机枚举的 root:Host 走 Windows 路径,WSL 走 `\\wsl$\<distro>` 驱动 Linux 目录树并显示 Linux dotfiles;SSH/macOS 远端在没有 SFTP/远端 FS 后端前只保留终端自身 cwd 状态,不重定 Explorer root,也不把 `/home/...` 或 `/Users/...` 误当 Windows 目录。
