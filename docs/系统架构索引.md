## 架构总览

Tn 是 Windows 优先、Rust 编写、GPU 加速的终端模拟器。它不重写 shell,而是替代 `conhost.exe` 的窗口、输入、VT/ANSI 解析和渲染职责:PowerShell、cmd、WSL、SSH shell、`vim`、`claude`、`codex` 等子进程仍通过 PTY/ConPTY 读写字节。

系统边界:

- 终端模拟器负责屏幕、键盘、VT/ANSI 解析、网格、滚动历史、光标、选择、渲染和 UI chrome。
- shell 或 TUI 程序负责命令解释、程序执行、管道和自己的 stdout/stderr 输出。
- `tn-core` 是 headless 终端内核;`tn-ui` 是唯一 GPUI 前端;`tn-pty` 是本地/远端 PTY 边界;Agent Host 是配置可扩展的智能体宿主平台。
- `gpui` 只能出现在 `tn-ui` / `tn-app`;其它 crate 必须 headless 可编译、可测试。

子系统入口:

- crate 和依赖方向见 [工作区与代码包边界](架构/工作区与代码包边界.md)。
- 技术决策、依赖许可证、术语和已知坑入口见 [架构总览与关键决策](架构/架构总览与关键决策.md)。

## 数据流入口

Tn 的主数据流是双向字节管道:

```text
键盘/鼠标 -> tn-ui 输入编码 -> PTY writer -> shell / TUI
shell / TUI -> PTY reader -> tn-core VT 解析 -> TerminalSnapshot -> tn-ui 渲染
```

关键入口:

- 输出、输入、PtyWrite 回写、push+vsync 重绘和 resize 联动见 [终端数据流](架构/终端数据流.md)。
- 终端滚动、隐藏帧 resize、agent/alt-screen 滚轮和 scrollback 修复记录见 [终端滚动与窗口尺寸](修复与优化/终端滚动与窗口尺寸.md)。
- 远端文件、SFTP、远端 Quick Look 和远端 git 数据流见 [远端文件服务与改动流](新增模块/远端文件服务与改动流.md)。
- Agent 事件、用量和 sidecar 遥测流见 [智能体宿主平台](架构/智能体宿主平台.md) 与 [智能体宿主平台化](新增模块/智能体宿主平台化.md)。
