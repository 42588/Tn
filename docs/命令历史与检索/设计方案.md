- 日期:2026-06-19
- 范围(预估):`tn-blocks`、`tn-shell`、新增 `tn-history`(或并入 `tn-blocks`)、`tn-ui`(历史面板 / 命令面板)、`tn-config`
- 状态:设计稿 · 未实现
- 来源:[终端项目互联网调研报告](../参考资料/终端项目互联网调研报告.md);用户保留点 1 / 点 2,点 3 搁置。

## 1. 源项目机制(一手来源,实现据此对齐)

### 1.1 zoxide — frecency

- 每个目录初始分 `1`,每次访问 `+1`。
- 排序权重按「距上次访问」分桶:**1 小时内 ×4 / 1 天内 ×2 / 1 周内 ÷2 / 更早 ÷4**。
- 老化:当总分超过 `_ZO_MAXAGE`(默认 **10000**)时,全体乘系数 k 使总分回落到 ≈ 90% MAXAGE;缩放后 <1 的条目删除。理论库上限 `4 × MAXAGE`。
- 匹配:大小写不敏感,查询词须**按顺序**出现;且「最后一个关键词的末段必须匹配路径末段」。
- 文件系统中已不存在且超 90 天的目录惰性清除。

### 1.2 Atuin — 持久化历史

- **SQLite** 存储;每条记录:`command / cwd / exit code / duration / hostname / session / timestamp`。
- 检索作用域三档:**session / directory / global**(Ctrl-R 循环)。
- CLI 过滤:`--exit`、`--after/--before`(时间)、关键词。
- 可配 `search_mode`:prefix / full-text / fuzzy(skim 算法)。
- 导入:`atuin import auto`,**不替换**原 shell 历史文件。
- 同步可选、端到端加密、可自托管或关闭。

### 1.3 fzf — 检索语法 + 预览

- 扩展语法:fuzzy(`sbtrkt`)默认;`'wild` 精确、`'wild'` 词边界;`^pre` 前缀、`suf$` 后缀;`!term` 取反;`|` 作 OR。
- 预览:`--preview` 跑命令,`{}` = 当前焦点行;`--preview-window` 控制尺寸/位置;支持 ANSI。
- 键位:Ctrl-J/K 移动、Enter 选中、Esc 退出;`-m` 多选 Tab/Shift-Tab。

> 来源:zoxide Algorithm wiki / README、Atuin README+docs、fzf README(2026-06-19 取材)。

## 2. 点 1 — 持久化结构化命令历史

### 2.1 目标

命令历史从「内存、按会话、重启即丢」升级为「本地 SQLite 持久、跨会话、带上下文、可跳回原始输出」。

### 2.2 数据采集:免 shell hook(Tn 的结构优势)

Atuin 需要往用户 shell 注入 preexec/precmd hook。**Tn 不需要**——[tn-shell](../../crates/tn-shell/src/lib.rs) 已旁路解析 OSC 133/633/7,得到 prompt/command start、output start、command finished(exit code)、command line、cwd。[tn-blocks](../../crates/tn-blocks/src/lib.rs) 已把这些组织成块,字段与 Atuin 几乎一一对应,**外加 output 起止行**。

**降级策略(必须明确)**:命令捕获依赖 shell integration 存在。

| 会话类型 | integration | 历史采集 |
|---|---|---|
| 内置 PowerShell | 自动注入 ✅ | 全字段入库 |
| 裸 WSL / SSH / 其他 shell | 取决于远端是否有 OSC 注入 | 有则入库;无则仅记「命令行(若能从 633 CommandLine 取到)」或不记,**不伪造 exit/duration** |
| Agent pane(alt-screen TUI) | 不适用 | 不入命令历史(它不是 shell 命令流) |

### 2.3 SQLite schema(建议)

选 **`rusqlite` + `bundled` feature**:把 SQLite 编进二进制,零系统依赖,符合 Tn「资源全捆绑」既有约定(参 `vendor/`)。库文件落配置目录(`%APPDATA%/Tn/history.db`)。

> 备选 append-only JSONL 已评估:写简单,但作用域/时间/exit 过滤与 fulltext 都要全量扫,查询成本不划算,**否决**。

```sql
CREATE TABLE command_history (
  id           INTEGER PRIMARY KEY,
  command      TEXT    NOT NULL,
  cwd          TEXT,
  exit_code    INTEGER,            -- NULL = 未知/仍在运行
  exit_known   INTEGER NOT NULL,   -- 是否拿到真实 OSC 133;D
  duration_ms  INTEGER,
  started_at   INTEGER NOT NULL,   -- epoch ms
  finished_at  INTEGER,
  session_id   TEXT    NOT NULL,   -- Tn pane/会话 uuid
  namespace    TEXT    NOT NULL,   -- Host / WSL / SSH
  host         TEXT,               -- NULL=本机;否则 user@host 或 wsl:<distro>
  shell        TEXT,               -- powershell / bash / ...
  out_start    INTEGER,            -- scrollback 绝对行(仅 live 会话有意义)
  out_end      INTEGER
);
CREATE INDEX idx_hist_cwd     ON command_history(cwd);
CREATE INDEX idx_hist_started ON command_history(started_at);
CREATE INDEX idx_hist_session ON command_history(session_id);

-- 可选:命令文本全文检索(对齐 Atuin full-text 模式)
CREATE VIRTUAL TABLE command_fts USING fts5(command, content='command_history', content_rowid='id');

-- 目录 frecency(与「最近工作目录」合并为唯一来源)
CREATE TABLE dir_visits (
  path        TEXT    PRIMARY KEY,
  score       REAL    NOT NULL,
  last_access INTEGER NOT NULL
);
```

### 2.4 写入接线点

- **命令完成**:block 收到 OSC 133;D(command finished)→ 落 `command_history` 一行(含 exit/duration/out 区间)。接线在 [tn-blocks](../../crates/tn-blocks/src/lib.rs) 块封口处或 [terminal_view/io.rs](../../crates/tn-ui/src/terminal_view/io.rs) reader 侧,异步写,不阻塞渲染。
- **cwd 变化**:OSC 7 → `dir_visits` upsert(score += 1, last_access = now),并跑 zoxide 老化判定。

### 2.5 检索作用域与过滤(对齐 Atuin)

- 三档作用域:`session`(当前 pane)/ `directory`(当前 cwd 及子树)/ `global`,UI 上循环切换。
- 过滤:exit≠0、时间范围、namespace/host。
- 去重:存全量,查询时按 `command` 去重保留最近一条(Atuin 同款,默认开,可关)。

### 2.6 差异化:跳回原始输出(Atuin 做不到)

历史项带 `out_start/out_end` + `session_id`。若该会话仍存活且行未被 scrollback 淘汰 → 选中可**滚动定位到当时输出**(复用现历史面板的 BlockBar 定位联动 + 已有「复制输出」)。会话已结束或行已淘汰 → 标注「仅元数据」,不假装有输出。

> **体积权衡**:输出正文**不全量入库**(可能极大),只存行区间引用。这是与 Atuin 的有意取舍——Atuin 干脆不存输出。

### 2.7 隐私 / 同步

- **本地优先**,默认不联网;UI 诚实标注「历史仅存本机」。
- 同步(Atuin e2e 模型)**只留扩展口子、本轮不实现**:schema 预留即可,避免后续迁移。

### 2.8 复用 / 新增

- 复用:tn-blocks 字段、tn-shell OSC、历史面板 UI、BlockBar 定位。
- 新增依赖:`rusqlite`(bundled)。新增 crate `tn-history` 或并入 `tn-blocks`(倾向独立 crate,隔离 IO)。

## 3. 点 2 — 统一模糊检索入口(picker)

### 3.1 目标

把命令面板(现仅启动 profile)升级为单入口模糊 picker,聚合多数据源,带预览。

### 3.2 匹配引擎

- 采用 **fzf 扩展语法子集**:fuzzy 默认 + `'`精确 + `^`/`$`锚定 + `!`取反(`|` OR 视情况)。
- **不手写匹配**:Tn 是 Rust,选 **`nucleo`**(Helix 所用的 fzf 式高速匹配器)或等价 `fuzzy-matcher`;实现前先查 GPUI/zed 生态是否已捆绑可复用 matcher(Tn 当前无任何 fuzzy 依赖,已确认)。

### 3.3 数据源(分类聚合)

| 源 | 来自 | 选中动作 |
|---|---|---|
| 命令历史 | 点 1 SQLite | 回填命令到输入行 / 跳回输出 |
| 文件 | Explorer / cwd 扫描 | Quick Look 打开 |
| 目录(frecency) | `dir_visits` + 最近工作目录 | set cwd / 打开文件夹 |
| git 分支 | gitutil | 切分支(只读列举,切换可后置) |
| 布局 / 会话 | layout 槽位 | 加载布局 |

### 3.4 目录排序:zoxide frecency

`dir_visits` 直接套 §1.1 公式(×4/×2/÷2/÷4 + MAXAGE=10000 老化)。**与「最近工作目录」合并为唯一目录库**,不建两套。

### 3.5 预览契约

对应 fzf `--preview {}`:右侧预览窗——文件复用 [quick_look.rs](../../crates/tn-ui/src/quick_look.rs);历史项预览其输出 / diff;目录预览 ls。GPUI 浮层(非终端内 TUI),基于现命令面板浮层扩展。

### 3.6 键位 / 作用域

- 沿用现命令面板键位 + fzf 习惯:Ctrl-J/K 移动、Enter 选中、Esc 退出;可选 Tab 多选。
- 作用域切换借 Atuin:在历史源内循环 session→dir→global。

### 3.7 复用 / 新增

- 复用:命令面板浮层、Quick Look 预览、点 1 历史库、layout/gitutil。
- 新增依赖:`nucleo`(或等价)。

## 4. 实现分期

| 阶段 | 内容 | 产出验收 |
|---|---|---|
| **Phase 1** | 点 1:`rusqlite` 历史库 + 写入接线 + 历史面板升级(三档作用域 + 过滤 + 跳回输出) | 命令完成即落库;重启后跨会话可搜;exit/time/cwd 过滤;能跳回存活会话的输出 |
| **Phase 2** | 点 2:命令面板 → 统一 picker(fzf 子集语法 + nucleo + 预览 + 多数据源 + frecency 目录) | 单入口搜历史/文件/目录/分支/布局;右侧预览;目录按 frecency 排序 |

Phase 1 必须先行——它同时是 Phase 2 的历史数据源。

## 5. 验收口径

- 单元测试:frecency 公式(分桶/老化/删除边界)、fzf 语法解析(`'`/`^`/`$`/`!`)、schema 读写与去重、降级路径(无 integration 不伪造字段)。
- 真机:长会话落库不卡输入;跨重启检索;picker 预览不阻塞;Windows 下 `history.db` 路径与并发写正常。

## 6. 风险与边界

- **shell integration 覆盖面**是点 1 的真正边界,降级策略(§2.2)必须先定,不能对无 integration 的会话伪造 exit/duration。
- **输出持久化体积**:只存行区间,不存正文。
- **并发写**:多 pane 同时落库,`rusqlite` 单连接 + 队列 或 WAL 模式;避免锁竞争阻塞渲染线程(写走后台)。
- **GPUI 无全屏 TUI overlay**:picker 是浮层,已有先例,非新风险。
- 不做云同步实现、不做 lazygit 写操作、不动终端核心。
