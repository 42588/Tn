## Quick Look 速览 / 编辑器

- **散文 / Markdown 长段落不能软换行**:固定行高 `uniform_list`(只 measure row 0)→ wrap 会变高破坏虚拟化,长段落只能横向滚。prose 体验欠佳。要 wrap 得放弃该场景虚拟化或改测每行高。
- **Diff tab 不支持拖选**:鼠标拖选只做了 File / 编辑态;Diff 行未接。
- **编辑态每键 `Rc::make_mut` 深拷整 buffer**:412 行亚毫秒、非冻结主因,但接近 `MAX_LINES`(4000)会有延迟感;真要治改 `Rc<RefCell<Vec<String>>>`。

## 输入 / 键位

- **命令面板中文搜索 parked**:面板靠 `on_key` 读 ASCII,IME 合成的中文只经 `EntityInputHandler` 投递、面板未接 → 目前只能搜英文。补法见 CLAUDE.md「未做/后续」。
- **`Ctrl+Shift+*` 可能被中文 / 多布局 Windows 的 IME / 系统热键吞**:真机不达时可改键,或把系统「输入语言热键」设未分配。
- **IME 合成键子类化修复(`VK_PROCESSKEY`)真机待验**:理论成立(IMM32 标准),须真机肉眼 + 看日志(`RUST_LOG=tn::ime=debug`)确认微软拼音确以 `VK_PROCESSKEY` 投递合成键。

## 〔愿景稿〕

当前 Quick Look 编辑器已经能完成“点开文件、快速修改、保存”的闭环,但它仍然是一个塞在 `quick_look.rs` 里的轻量快编器。继续直接堆软换行、多光标、Diff 拖选、LSP、打字动画,会让用户体验变得不可预测:有些场景很漂亮,但保存不安心、长文件不流畅、远端文件不可用、动画还可能拖慢输入。

这项优化的目标不是把 Tn 变成 VS Code,而是把编辑能力升级成 **Tn 工作流里的可信编辑器**:打开快、编辑稳、保存安全、审查 agent 改动顺手,动画作为细节增强而不是主目标。

**用户视角目标**

- **不离开上下文**:用户在终端、文件树、agent 改动卡片之间流转时,能直接预览、编辑、保存,不需要跳到外部编辑器。
- **不怕误改和丢数据**:编辑前后有 dirty 状态、撤销/重做可靠、保存前能发现磁盘外部变化,关闭浮层/切文件时不会静默丢修改。
- **读得舒服**:代码长行继续横滚,Markdown/日志/说明文档应支持软换行;大文件要能快速只读打开,不能因为一个文件拖垮 UI。
- **改得顺手**:鼠标选区、键盘选择、查找替换、复制粘贴、Diff 查看和快速保存都要统一;未来多光标/列选/LSP 要有清晰落点。
- **反馈高级但不打扰**:打字动画、caret 过渡和插入字符反馈应该让输入更有质感,但任何时候都不能牺牲“光标真实位置立即到位”和“连续输入不卡”。

**产品形态**

- **Quick Look 快编**:仍是默认入口。文件树 `Space`/`Enter`、agent 改动卡片、设置文件都先打开浮层。适合看文件、小改动、配置修改、快速 Diff。
- **Editor Pane**:当用户需要长时间编辑或多文件处理时,Quick Look 可升级为正式编辑 pane/tab。它共享同一编辑核心,但拥有更稳定的空间、状态栏和后续 LSP 能力。
- **Diff Review**:agent 改动卡片进入审查视图。短期支持 Diff 拖选/复制/跳转;中期支持 hunk 级 accept/reject、打开对应文件定位编辑。

**架构方向**

- 新增 headless `tn-editor` 层,把编辑核心从 `quick_look.rs` 中拆出。它不依赖 GPUI,负责 `Document`、`CursorSet`、`Selection`、`EditTransaction`、`UndoStack`、`SearchState`、`LineLayout`、编码/换行符、dirty 状态和保存冲突检测。
- UI 层新增 `editor::element` / `QuickLookCodeElement`,用 GPUI `Element` 的 `layout/prepaint/paint` 统一绘制文本、行号、选区、搜索命中、diff decoration、IME preedit、caret 和 typing effects。
- Quick Look、Editor Pane、Diff Review 都消费同一 `tn-editor` 文档模型和同一 renderer。Quick Look 只是宿主形态之一,不再拥有完整编辑业务。
- 文件访问继续受命名空间约束:Host 可直接读写;WSL 走 `\\wsl$` 映射;SSH/macOS 远端必须等 SFTP/远端 FS 后端,不能把远端路径伪装成本机 `PathBuf`。

**编辑体验要求**

- **保存安全**:打开文件时记录 mtime/size/hash;保存前若磁盘已变化,提示用户选择覆盖、重新加载、另存为或查看差异。默认不做静默覆盖。
- **关闭安全**:Quick Look 关闭、切文件、切 Diff、退出应用时,若当前文档 dirty,必须给出保存/放弃/取消。小改动也不能静默丢。
- **换行与编码**:保留原文件换行符风格和编码信息。无法可靠识别时明确降级为 UTF-8 文本编辑或只读预览。
- **大文件策略**:小文件可编辑;接近阈值时提示性能风险;超大文件默认只读流式/分页预览。用户需要强制编辑时必须明确确认。
- **软换行策略**:代码默认横滚;Markdown、txt、log、说明文档默认软换行。实现上采用 logical line -> visual line 映射,视觉行仍固定行高,避免破坏虚拟化。

**打字动画纳入范围**

打字动画保留,但它是 renderer 的可选 effect,不是编辑核心能力。

- 逻辑 cursor、selection、undo、文本内容必须在输入事件提交时立即更新;动画只能表现视觉过渡。
- 允许的效果:同一视觉行内 60-90ms caret 轻微 glide,块光标 squash/pop,新插入字符 70-110ms 的轻微亮度/alpha settle。
- 禁止的效果:不允许在 `uniform_list + div/span` 外层用绝对叠层和 16ms 帧泵推动光标;不允许积累 per-char 动画队列;不允许让动画改变文本宽度、行高或真实命中测试。
- 自动 snap 场景:IME composition、拖选、选区存在、查找跳转、PageUp/Down、跨行移动、滚动、文件切换、窗口失焦、大文件、高帧耗时。
- 配置项:提供 `editor.animations = "off" | "subtle" | "full"` 或 reduced-motion 开关。默认建议 `subtle`;性能不达标时自动降级到当前瞬时反相块。

**渲染方案**

- `EditorElement::prepaint` 计算可见 visual lines、行号槽、文本区、caret rect、选区 rect、IME candidate bounds 和 scrollbar geometry。
- `EditorElement::paint` 只绘制可见区域。文本 shaping 缓存按 `(line_id, line_revision, theme_revision, font_key, wrap_mode)` 失效。
- 固定单元格模型保留:ASCII 1 列,CJK/宽字符 2 列。软换行在 visual line 层做,不改变字符命中模型。
- Diff 视图作为 decoration 渲染,不单独维护另一套文本控件。File / Edit / Diff 三态共用选择、复制、滚动和命中测试。

### P0 — 保存 / 关闭安全(先做,纯逻辑、零 UI 重构风险)

**为何先做**:这是「可信编辑器」的底线,且不依赖任何渲染重构,可立即在现有 `quick_look.rs` 落地,提前消除「保存不安心 / 静默丢改」的最大风险。

- ☐ 打开文件时记录 `FileGuard { mtime, size, hash }`(hash 用现成轻量算法,大文件可只 hash 头尾 + size);存到 `QuickLook` 新字段。
- ☐ `save()` 前比对磁盘:若 mtime/size/hash 变化 → 弹冲突浮层,选项 **覆盖 / 重新加载(放弃本地改)/ 另存为 / 查看差异**,默认**不覆盖**。
- ☐ dirty-close 守卫:`dirty == true` 时,关闭浮层 / 切文件(预览 `↑↓`)/ 切 Diff tab / 退出 app,必须弹 **保存 / 放弃 / 取消**;取消 = 留在当前文档。
- ☐ 换行符 + 编码探测:打开时记录原换行风格(LF/CRLF)与编码;保存时**原样回写**;无法可靠识别 → 明确降级为 UTF-8 文本编辑或只读预览(UI 标注,不静默猜)。
- **产出物**:`quick_look.rs` 新增 `FileGuard` + 冲突/脏关确认浮层 + 换行/编码字段;冲突检测为纯函数(`fn detect_conflict(guard, disk) -> Conflict`)。
- **验收**:外部改文件后保存 → 弹冲突卡,选项各自行为正确不静默覆盖;dirty 时所有离开路径都拦截;CRLF 文件保存后仍 CRLF。
- **验证**:`detect_conflict` / 换行探测 headless 单测;`cargo test -p tn-ui --lib`;真机:edit 中用别的程序改同文件 → Ctrl+S 弹冲突。
- **依赖**:无。**风险**:hash 大文件成本 → 用 size+mtime 先判,仅可疑时才 hash 或只 hash 采样。

### P1 — 核心抽离:新建 headless `tn-editor`

**目标**:把编辑核心从 GPUI 剥离成可 headless 单测的库;**纯重构,Quick Look 行为逐字不变**。

- ☐ 新建 crate `tn-editor`(headless,**禁止依赖 gpui**,加进 workspace + 依赖方向文档)。模型:`Document`(行缓冲 + 换行/编码 + P0 的 `FileGuard`)、`Selection`、`EditTransaction`、`UndoStack`、`SearchState`、`CursorSet`(单光标起步,为多光标留位)。
- ☐ 迁移现有纯函数为 `Document` 方法 / transaction:`op_*` → `EditTransaction`;`all_matches` → `SearchState`;`char_to_byte` 等工具下移。`parse_diff`/`highlight` 视情况一并下移(headless 可测)。
- ☐ **治 `Rc::make_mut` 深拷**(本文件「编辑态每键深拷整 buffer」):`Document` 内部改 rope 或 `Rc<RefCell<Vec<String>>>` + 增量 undo(记 patch 而非整 buffer 快照);保留快照语义但去掉每键全量拷。
- ☐ `quick_look.rs` 改为薄壳:`buf/cursor/undo/redo/search` 委托给 `tn-editor::Document`,UI 只读模型 + 发输入。
- **产出物**:`crates/tn-editor/`(lib + 单测);`quick_look.rs` 编辑逻辑减重。
- **验收**:Quick Look 全部既有交互(打字/选区/复制粘贴/撤销重做/全选/鼠标点位/查找替换/保存)**逐项无回归**;4000 行(`MAX_LINES`)文件连续输入无新增延迟。
- **验证**:`tn-editor` 单测全覆盖迁移的纯函数(对照原 `op_*` 单测);`cargo test --workspace --lib`;`cargo build --workspace`;四道 UI 守卫绿。
- **依赖**:P0(`FileGuard` 进 `Document`)。**风险**:迁移漏改语义 → 用「先 1:1 迁移 + 跑原单测全绿,再优化深拷」两步走,别一次重写。

### P2 — 只读 Renderer Spike:`EditorElement`

**目标**:在 **File 预览态(只读)** 接入自绘 `EditorElement`,先验证渲染管线,不碰编辑。

- ☐ UI 层新建 `editor::element`(`EditorElement` 实现 GPUI `Element` 的 `layout/prepaint/paint`)。`prepaint` 算可见 visual lines / 行号槽 / 文本区 / 纵横滚 geometry;`paint` 只绘可见区。
- ☐ 文本 shaping 缓存按 `(line_id, line_revision, theme_revision, font_key, wrap_mode)` 失效。
- ☐ 固定单元格模型保留(ASCII 1 列 / CJK 2 列);命中模型不变。
- ☐ File 预览态切到 `EditorElement` 只读路径,验证:文本绘制、行号、纵横滚、CJK 命中、预览拖选 + 复制(对照现 `uniform_list` 路径)。
- **产出物**:`crates/tn-ui/src/editor/element.rs`;File 预览态双轨(env 门控可切回旧路径回归对照,如 `TN_QL_ELEMENT=1`,同 `TN_QL_BENCH` 取向)。
- **验收**:只读预览观感、纵横滚、CJK 行点击/拖选/复制 = 当前或更好;长文件(`TN_QL_BENCH`)paint 不冻、不 OOM。
- **验证**:`TN_QL_BENCH=<大文件/CJK 文件>` 探针;真机肉眼对照;`cargo test -p tn-ui --lib`。
- **依赖**:P1(消费 `Document`)。**风险**:`Element` 三段式比 `uniform_list` 底层 → 先只读、小步,paint 成本用 `coalesce_spans`/封顶 span 思路守住。

### P3 — 编辑态接入(瞬时反相块,无动画)

- ☐ 编辑态改走 `EditorElement` + `tn-editor::EditTransaction`:单光标、选区、IME preedit、查找替换、保存、dirty 全部复用 P1 模型。
- ☐ IME 完整复用既有链路(`EntityInputHandler` + 可打印键放行 + `VK_PROCESSKEY` 子类化,见 CLAUDE.md);caret = **瞬时反相块**,**不启用任何动画**。

## 对面板进行解耦:per-pane 工作区上下文 ✅(2026-06,SSH 远端 FS 留待 SFTP)

**目标(需求原文)**:让每个终端窗格拥有自己的「工作区上下文」,而不是让左侧文件树、Agent 改动列表、用量统计都依赖一个全局状态。

### 已完成(本轮做了什么)

- **每个 pane 绑定自己的文件树状态** ✅:`ExplorerView` 仍是单一渲染实例,但 Workspace 按 `PaneId` 存 `explorer_states`(`ExplorerSnapshot` = 展开态 + 选中文件)+ `explorer_pane`(记当前展示的 pane)。render 焦点跟随**区分两种事件**:焦点**换 pane** → `ExplorerView::switch_pane`(保存旧 pane 快照 / 恢复新 pane 快照,各 pane 文件树互不串台);**同一 pane 内 `cd`**(OSC 7)→ 沿用 `follow_root`(保留子目录展开态)。切回某 pane 即恢复它自己的 root / 展开态 / 选中文件。快照保存时惰性 `retain` 裁掉已关闭 pane(免逐 `panes.remove` 钩子);恢复经纯函数 `snapshot_under_root` 过滤到新 root 内(越界展开/选中丢弃,headless 单测覆盖)。
- **Agent 信息跟 pane 绑定** ✅(本就如此,本轮只验不回归):agent 身份 / 用量环 / 「本次改动」/ git diff watcher / 会话态全在 `TerminalView`(per-view),状态栏只读**焦点 pane** 的用量。不会被其它 pane 的 cwd 或新启动会话串台。
- **「打开文件夹」收敛到当前焦点 pane** ✅:`cd_panes_to_root`(无差别广播)→ `cd_pane_to_root(id, …)`(单 pane)。`menu_open_folder` 进入前捕获焦点 pane,只对它 `cd` + `set_rail_root`,其它 pane 保持各自目录;agent pane **永不被 `cd`**。Host pane 选 Windows 路径、WSL pane 经原生 picker 浏览 `\\wsl$\<distro>\`(`from_accessible_path` 识别 UNC)后映射回 Linux 路径 `cd`。
- **命名空间边界不混淆** ✅:Windows `PathBuf` / WSL Linux 路径 / SSH 远端路径经 `ExplorerRoot::path_for_namespace` 显式转换;`is_host_process_path` 拦截不兼容映射(WSL UNC 不会塞给 cmd/pwsh)。

### 还没做(本轮范围外,需远端 FS 后端)

- **SSH pane 的「打开文件夹」** 🧭:远端目录浏览需先实现 SFTP / 远端文件服务。本轮**禁用 + echo 提示**(「远端目录浏览需要 SFTP / 远端文件后端(后续支持),本次已跳过」),不把本机路径误塞进远端 shell。SFTP 落地后应改为打开远端目录选择器、选目录后让 SSH shell `cd` 到远端路径——与上面「远端 / WSL 文件系统」条同一后端。

### 如何验证

- **自动**:`cargo test -p tn-ui --lib`(含 `snapshot_restore_filters_to_new_root` / `snapshot_restore_drops_out_of_root_selection` 两个纯函数单测 + 四道守卫全绿);`cargo build --workspace`;`$env:TN_AUTOQUIT="1"; cargo run -p tn-app`(网格收敛、exit 0)。
- **真机肉眼**(`cargo run -p tn-app`,改完先关旧 `tn.exe` 防重链锁):分屏两 pane 各 `cd` 到不同目录、各展开不同子树、各选中不同文件 → 来回切焦点,文件树**各自恢复**;同 pane 内 `cd` 进子目录展开态保留;「打开文件夹」只当前焦点 pane `cd` 过去、其它 pane 与 agent pane 不动;焦点 SSH pane 点「打开文件夹」→ 提示 / no-op;agent pane 用量环 / 「本次改动」切到别的 pane 不串台。
