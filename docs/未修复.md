## Quick Look 速览 / 编辑器

- **散文 / Markdown 长段落不能软换行**:固定行高 `uniform_list`(只 measure row 0)→ wrap 会变高破坏虚拟化,长段落只能横向滚。prose 体验欠佳。要 wrap 得放弃该场景虚拟化或改测每行高。
- **Diff tab 不支持拖选**:鼠标拖选只做了 File / 编辑态;Diff 行未接。
- **编辑态每键 `Rc::make_mut` 深拷整 buffer**:412 行亚毫秒、非冻结主因,但接近 `MAX_LINES`(4000)会有延迟感;真要治改 `Rc<RefCell<Vec<String>>>`。

## 输入 / 键位

- **命令面板中文搜索 parked**:面板靠 `on_key` 读 ASCII,IME 合成的中文只经 `EntityInputHandler` 投递、面板未接 → 目前只能搜英文。补法见 CLAUDE.md「未做/后续」。
- **`Ctrl+Shift+*` 可能被中文 / 多布局 Windows 的 IME / 系统热键吞**:真机不达时可改键,或把系统「输入语言热键」设未分配。
- **IME 合成键子类化修复(`VK_PROCESSKEY`)真机待验**:理论成立(IMM32 标准),须真机肉眼 + 看日志(`RUST_LOG=tn::ime=debug`)确认微软拼音确以 `VK_PROCESSKEY` 投递合成键。

## Tn Editor 用户体验升级:从 Quick Look 快编到可扩展编辑器

当前 Quick Look 编辑器已经能完成“点开文件、快速修改、保存”的闭环,但它仍然是一个塞在 `quick_look.rs` 里的轻量快编器。继续直接堆软换行、多光标、Diff 拖选、LSP、打字动画,会让用户体验变得不可预测:有些场景很漂亮,但保存不安心、长文件不流畅、远端文件不可用、动画还可能拖慢输入。

这项优化的目标不是把 Tn 变成 VS Code,而是把编辑能力升级成 **Tn 工作流里的可信编辑器**:打开快、编辑稳、保存安全、审查 agent 改动顺手,动画作为细节增强而不是主目标。

**用户视角目标**

- **不离开上下文**:用户在终端、文件树、agent 改动卡片之间流转时,能直接预览、编辑、保存,不需要跳到外部编辑器。
- **不怕误改和丢数据**:编辑前后有 dirty 状态、撤销/重做可靠、保存前能发现磁盘外部变化,关闭浮层/切文件时不会静默丢修改。
- **读得舒服**:代码长行继续横滚,Markdown/日志/说明文档应支持软换行;大文件要能快速只读打开,不能因为一个文件拖垮 UI。
- **改得顺手**:鼠标选区、键盘选择、查找替换、复制粘贴、Diff 查看和快速保存都要统一;未来多光标/列选/LSP 要有清晰落点。
- **反馈高级但不打扰**:打字动画、caret 过渡和插入字符反馈应该让输入更有质感,但任何时候都不能牺牲“光标真实位置立即到位”和“连续输入不卡”。

**产品形态**

- **Quick Look 快编**:仍是默认入口。文件树 `Space`/`Enter`、agent 改动卡片、设置文件都先打开浮层。适合看文件、小改动、配置修改、快速 Diff。
- **Editor Pane**:当用户需要长时间编辑或多文件处理时,Quick Look 可升级为正式编辑 pane/tab。它共享同一编辑核心,但拥有更稳定的空间、状态栏和后续 LSP 能力。
- **Diff Review**:agent 改动卡片进入审查视图。短期支持 Diff 拖选/复制/跳转;中期支持 hunk 级 accept/reject、打开对应文件定位编辑。

**架构方向**

- 新增 headless `tn-editor` 层,把编辑核心从 `quick_look.rs` 中拆出。它不依赖 GPUI,负责 `Document`、`CursorSet`、`Selection`、`EditTransaction`、`UndoStack`、`SearchState`、`LineLayout`、编码/换行符、dirty 状态和保存冲突检测。
- UI 层新增 `editor::element` / `QuickLookCodeElement`,用 GPUI `Element` 的 `layout/prepaint/paint` 统一绘制文本、行号、选区、搜索命中、diff decoration、IME preedit、caret 和 typing effects。
- Quick Look、Editor Pane、Diff Review 都消费同一 `tn-editor` 文档模型和同一 renderer。Quick Look 只是宿主形态之一,不再拥有完整编辑业务。
- 文件访问继续受命名空间约束:Host 可直接读写;WSL 走 `\\wsl$` 映射;SSH/macOS 远端必须等 SFTP/远端 FS 后端,不能把远端路径伪装成本机 `PathBuf`。

**编辑体验要求**

- **保存安全**:打开文件时记录 mtime/size/hash;保存前若磁盘已变化,提示用户选择覆盖、重新加载、另存为或查看差异。默认不做静默覆盖。
- **关闭安全**:Quick Look 关闭、切文件、切 Diff、退出应用时,若当前文档 dirty,必须给出保存/放弃/取消。小改动也不能静默丢。
- **换行与编码**:保留原文件换行符风格和编码信息。无法可靠识别时明确降级为 UTF-8 文本编辑或只读预览。
- **大文件策略**:小文件可编辑;接近阈值时提示性能风险;超大文件默认只读流式/分页预览。用户需要强制编辑时必须明确确认。
- **软换行策略**:代码默认横滚;Markdown、txt、log、说明文档默认软换行。实现上采用 logical line -> visual line 映射,视觉行仍固定行高,避免破坏虚拟化。

**打字动画纳入范围**

打字动画保留,但它是 renderer 的可选 effect,不是编辑核心能力。

- 逻辑 cursor、selection、undo、文本内容必须在输入事件提交时立即更新;动画只能表现视觉过渡。
- 允许的效果:同一视觉行内 60-90ms caret 轻微 glide,块光标 squash/pop,新插入字符 70-110ms 的轻微亮度/alpha settle。
- 禁止的效果:不允许在 `uniform_list + div/span` 外层用绝对叠层和 16ms 帧泵推动光标;不允许积累 per-char 动画队列;不允许让动画改变文本宽度、行高或真实命中测试。
- 自动 snap 场景:IME composition、拖选、选区存在、查找跳转、PageUp/Down、跨行移动、滚动、文件切换、窗口失焦、大文件、高帧耗时。
- 配置项:提供 `editor.animations = "off" | "subtle" | "full"` 或 reduced-motion 开关。默认建议 `subtle`;性能不达标时自动降级到当前瞬时反相块。

**渲染方案**

- `EditorElement::prepaint` 计算可见 visual lines、行号槽、文本区、caret rect、选区 rect、IME candidate bounds 和 scrollbar geometry。
- `EditorElement::paint` 只绘制可见区域。文本 shaping 缓存按 `(line_id, line_revision, theme_revision, font_key, wrap_mode)` 失效。
- 固定单元格模型保留:ASCII 1 列,CJK/宽字符 2 列。软换行在 visual line 层做,不改变字符命中模型。
- Diff 视图作为 decoration 渲染,不单独维护另一套文本控件。File / Edit / Diff 三态共用选择、复制、滚动和命中测试。

**分阶段路线**

1. **核心抽离**:建立 `tn-editor`,迁移当前 `op_*`、选区、查找替换、undo/redo、保存逻辑。Quick Look 行为保持不变,先解决 `Rc::make_mut` 深拷和保存冲突基础设施。
2. **只读 Renderer Spike**:在 File 预览态接入 `EditorElement` 只读路径,验证文本绘制、行号、纵横滚、CJK 命中、预览拖选和复制。
3. **编辑态接入**:复用 `tn-editor` transaction,接入单光标、选区、IME、查找替换、保存、dirty 状态。先使用瞬时反相块,不启用动画。
4. **Diff Review 接入**:Diff tab 复用同一 renderer,补拖选/复制/hunk 跳转;后续再做 accept/reject hunk。
5. **软换行**:实现 logical line -> visual line 表。先对 Markdown/txt/log 开启,代码保持横滚;确认虚拟化和鼠标命中稳定后再开放配置。
6. **打字动画**:在 renderer effect 层加入 subtle 动画,默认可关、可降级。用 `TN_PERF` 记录连续输入、中文长行、4000 行文件和 IME 场景。
7. **Editor Pane**:Quick Look 增加“打开为编辑器”入口,共享 `tn-editor` 文档会话。再考虑多文件标签、LSP、diagnostics、补全。

**通过标准**

- 打开普通文本文件即时可预览;进入编辑不改变原有快捷键心智。
- 连续输入 200 字符无明显卡顿,真实 cursor 不落后文本状态。
- 关闭动画 / reduced-motion / 高负载自动降级时,体验回到当前瞬时反相块且无行为差异。
- 中文长行点击、拖选、横滚、IME candidate 位置与当前版本同等精确或更好。
- Markdown/txt/log 软换行不会破坏滚动、选区、复制和查找。
- 4000 行文本滚动和编辑可用,无整面板每帧重建导致的 CPU 尖峰。
- 保存冲突、dirty close、外部修改检测有明确 UI,不静默覆盖或丢弃用户修改。
- File / Edit / Diff 三态交互无回归;`tn-editor` 核心逻辑必须有 headless 单测覆盖。

## Agent Host 平台化:从 Claude/Codex 特判迁移到可扩展 Agent Runtime

> **✅ 已实现(2026-06,P0–P6)**。新增 headless `tn-agent` 平台 crate(`AgentId`/`AgentDescriptor`/`AgentCapabilities`/`AgentRuntimeKind`/`AgentEvent`/`AgentAdapter`/`AgentRegistry` + `AiUsage`/pricing);Claude/Codex 包成 `tn-ai` 内置 adapter(`builtin_registry()`,**默认不注册** → 出厂纯 shell 宿主);`tn-ui` 全面去 `AgentKind` 特判,身份走 `AgentId` + 注册表 descriptor(gpui Global),capability 驱动 header/活动栏插槽,AgentEvent 归约器复用既有轮询;config `[[agents]]` manifest → `register_manifest` 让用户写 TOML 加 agent(generic 无遥测);`AgentRuntimeKind`(`LaunchSpec::runtime()`)与 `FileNamespace` 严格分离;守卫测试 `ui_has_no_closed_agent_enum` 锁死 UI 零闭合枚举。**闭合 `AgentKind` 枚举已全删。** 详见 [CLAUDE.md](../CLAUDE.md) tn-agent/tn-ai 条 + Agent Host 平台化坑。
>
> **后续(未做,低优先)**:外部进程 adapter **实时** stdio/JSON-RPC 通道(目前只定契约 = `AgentAdapter` trait + `GenericAdapter` + manifest;实时客户端 + manifest 网络权限默认拒绝/用户确认待有真实外部 agent 时实装)· 非 PTY 运行时(`AgentRuntimeKind` 已开放,未实装)· AgentEvent 的 transcript/permission/tool-call/status 渲染槽(变体已在契约,无 UI 槽)· tn-ai 仍保留的 `claude::`/`codex::` 原始解析 pub 函数(adapter 内部用)。

当前 Agent 面板仍然把 Claude/Codex 当成固定类型处理:启动器、Quick Terminal、`TerminalView`、状态栏、用量解析、颜色、session 绑定和 shell 命令识别都存在具体 agent 特判。这个模型只能支撑少量内置 agent,无法长期承载 Gemini CLI、Aider、OpenCode、Qwen Code、自研 agent、远端 agent 或非 PTY 型 agent。

目标不是简单做一个“通用面板”,而是把 Tn 抽象成 **Agent Host**:Claude/Codex 只是内置 provider,未来任意 agent 都按同一套宿主模型进入 Tn。

- **AgentDescriptor 描述身份**:每个 agent 用 `id`、名称、命令别名、图标/字标、主题色、默认参数、平台支持、默认 cwd 策略声明自己。UI 只从 descriptor 取展示信息,不再写死 Claude/Codex。
- **AgentRuntime 描述运行位置和协议**:同一个 agent 可以运行在 Local PTY、WSL PTY、SSH PTY、远端 daemon、HTTP/WebSocket 或未来的非终端结构化流上。Terminal pane 不应默认假设 agent 一定是本地进程。
- **AgentAdapter 负责私有格式转换**:Claude/Codex 的 JSONL/rollout 解析属于内置 adapter,不能继续散落在 UI。adapter 把私有日志、usage、transcript、状态转换为统一 `AgentEvent`。
- **AgentEvent 作为 UI 唯一输入**:UI 订阅 `session.started`、`session.ended`、`cwd.changed`、`model.changed`、`usage.updated`、`status.changed`、`transcript.appended`、`diff.changed`、`permission.requested`、`error.reported` 等事件,而不是直接读某个 agent 的日志文件。
- **Universal Agent Surface 按能力渲染**:Agent 面板拆成 Header、Body、Rail、Footer、Inspector 等通用插槽。每个插槽按 capability 开关显示,例如 terminal、usage、transcript、cwd_sync、git_diff、permission_prompts、tool_calls、checkpoints。没有 usage 的 agent 仍能作为普通 agent session 托管,只是隐藏用量区。
- **分层接入深度**:新 agent 至少支持配置级接入(声明命令即可启动);进一步可以通过观测级 adapter 获得日志/usage;再进一步通过 wrapper/sidecar 获得结构化事件;最高级是原生输出 Tn Agent Protocol。
- **扩展机制优先外部进程 adapter**:不优先做动态 Rust 插件,避免 ABI、崩溃隔离和安全边界问题。外部 adapter 用 stdio/JSON-RPC 通信,manifest 声明可读日志路径、可执行命令和网络权限。
- **Session 绑定必须去全局猜测**:agent session 归属应以 pane id + runtime + cwd namespace + adapter session id 为准。不能继续依赖“最新日志文件”这类全局新鲜度猜测,避免其它正在运行的开发 agent 串台。
- **与“对面板进行解耦”的边界**:Agent Host 只负责识别、启动、观测和翻译 agent session;每个 pane 的文件命名空间、cwd、Explorer root、git diff rail root、Quick Look 文件访问仍归属 pane/workspace context。`AgentEvent::cwd.changed` 只能作为目标 pane 的上下文输入,不能直接改全局 Explorer。`AgentRuntime` 描述进程/协议运行位置,`FileNamespace` 描述文件 I/O 命名空间,两者不能混用。SSH agent 即使能运行,在 SFTP/远端文件服务完成前也不能驱动 Explorer/Quick Look。

建议迁移顺序:

1. 新增 headless `tn-agent` 层,定义 `AgentId`、`AgentDescriptor`、`AgentRuntime`、`AgentCapabilities`、`AgentSessionRef`、`AgentEvent`、provider/adapter trait。先把 Claude/Codex 包成内置 provider,不改变现有 UX。
2. 将 `tn-ui` 中 `AgentKind` 的展示用途迁移为 `AgentId + AgentSessionViewModel`:欢迎页、Quick Terminal、tab 标题、agent 颜色、状态栏聚合都从 registry 读取。
3. 将 Claude/Codex usage poller 和 session 解析迁入 adapter,UI 不再知道日志目录或解析细节。
4. 支持配置级 agent:用户只写 TOML 就能让新 agent 出现在启动器/Quick Terminal,并获得通用 Agent 面板。
5. 支持外部 adapter 协议:让社区或用户为新 agent 提供 usage/transcript/tool-call/permission 等高级能力,而无需改 Tn UI。

## 对面板进行解耦

目标是让每个终端窗格拥有自己的“工作区上下文”,而不是让左侧文件树、Agent 改动列表、用量统计都依赖一个全局状态。

- **每个 pane 绑定自己的文件上下文**:每个 Host / WSL / SSH pane 都应记录自己的文件命名空间、当前 cwd、对应的 Explorer root。切换焦点 pane 时,左侧文件树显示该 pane 对应的文件系统视图;切回其它 pane 时恢复它自己的目录状态、展开状态和选中文件。
- **Agent 信息跟 pane 绑定**:Claude / Codex 这类 agent pane 的“本次改动”、git diff、用量环、会话状态都必须归属到具体 pane,不能被其它 pane 的 cwd 或新启动的 agent 会话串台。
- **左上角「打开文件夹」按当前 pane 的命名空间执行**:
  - Host / Windows pane:继续使用系统原生文件夹选择器,选择 Windows 路径后更新当前 pane 的 Explorer root,并让对应本地 shell `cd` 到该目录。
  - WSL pane:用户应能像打开 Windows 文件夹一样浏览 Linux 目录,但实际显示的是 `\\wsl$\<distro>\...` 映射出的 Linux 文件树;选择目录后要转换回 Linux 路径,让对应 WSL bash pane `cd` 到该目录。
  - SSH pane:需要先实现 SFTP / 远端文件服务。完成后,「打开文件夹」应打开远端目录选择器或远端 Explorer,显示远端 Linux/macOS 目录;选择目录后让对应 SSH shell `cd` 到远端路径。
- **同步范围要收敛到目标 pane**:用户对某个 pane 打开的文件夹或执行的目录切换,默认只影响该 pane 及其绑定的文件树/Agent rail;不要无差别把 `cd` 发送给所有 pane,尤其不能发送给 Claude/Codex 交互中的 agent pane。
- **命名空间边界不能混淆**:Windows `PathBuf`、WSL Linux 路径、SSH 远端路径必须显式转换。没有远端 FS 后端时,SSH cwd 只能作为终端状态展示,不能驱动本机 Explorer / Quick Look / git watcher。

## 对每一页面的样式重新设计，增加更多细节，尝试使用更高级的界面库或者自研界面库
