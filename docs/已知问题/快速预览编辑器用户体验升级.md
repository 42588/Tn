# 快速预览编辑器用户体验升级

## 定位

本文承载 Quick Look / Tn Editor 从轻量快编升级为可信编辑器的愿景和执行计划。母索引只保留问题状态;已完成实现细节见修复专题。

## 背景

Quick Look 已能完成打开、预览、快速修改和保存,但早期实现把预览、编辑、Diff、保存和渲染业务集中在 `quick_look.rs`。继续在原结构上叠加软换行、多光标、Diff 交互、LSP 和打字动画,会放大保存安全、长文件性能、远端文件语义和 IME 兼容风险。

目标不是把 Tn 做成完整 IDE,而是让 Tn 工作流里的文本编辑可靠:打开快、编辑稳、保存安全、审查 agent 改动顺手,动画只作为可降级的细节增强。

## 具体问题或计划

### 用户体验目标

- 用户在终端、文件树和 agent 改动卡片之间流转时,可直接预览、编辑、保存,不必跳到外部编辑器。
- 编辑前后有 dirty 状态,撤销/重做可靠,保存前能发现磁盘或远端外部变化。
- 代码长行继续横滚;Markdown、txt、log、说明文档默认软换行。
- 大文件能快速只读打开,接近阈值时提示性能风险,超大文件不默认进入可编辑模式。
- 鼠标选区、键盘选择、查找替换、复制粘贴、Diff 查看和保存语义在 File / Edit / Diff 三态中统一。
- 打字动画不能影响真实 cursor、selection、undo 和命中测试;关闭动画、系统 reduced-motion 或高负载时必须瞬时降级。

### 架构方向

- `tn-editor` 作为 headless 编辑核心,负责 `Document`、`CursorSet`、`Selection`、`EditTransaction`、`UndoStack`、`SearchState`、`LineLayout`、编码/换行符、dirty 状态和保存冲突检测。
- UI 层用共享 renderer 绘制文本、行号、选区、搜索命中、Diff decoration、IME preedit、caret 和 typing effects。
- Quick Look、Editor Pane、Diff Review 消费同一文档模型和 renderer;Quick Look 只是宿主形态之一。
- 文件访问必须保持命名空间边界:Host 直接读写;WSL 走 `\\wsl$` 映射;SSH/macOS 远端走 SFTP 后端,不得把远端路径伪装成本机 `PathBuf`。

## 退出条件或验证

- 普通文本文件即时可预览,进入编辑不改变原快捷键心智。
- 连续输入 200 字符无明显卡顿,真实 cursor 不落后文本状态。
- Markdown/txt/log 软换行不破坏滚动、选区、复制、查找;代码文件仍横滚。
- 4000 行文本滚动和编辑可用,无整面板每帧重建导致的 CPU 尖峰。
- 保存冲突、dirty close、外部修改检测有明确 UI,不静默覆盖或丢弃用户修改。
- File / Edit / Diff 三态交互无回归,`tn-editor` 核心逻辑有 headless 单测覆盖。
- 每个完成包从母索引或本文未完成区移出,并沉到对应修复专题。
