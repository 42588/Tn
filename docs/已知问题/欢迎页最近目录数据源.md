# 欢迎页最近目录数据源

## 定位

本文记录欢迎页“最近目录”功能缺少数据源的问题和退出条件。

## 背景

欢迎页需要展示最近使用的目录,但当前缺少可供 UI 消费的 recent-sessions 数据源。该问题不属于 Quick Look 编辑器,也不是远端 FS 实现缺陷,应作为独立数据源待办跟踪。

## 具体问题或计划

- 问题:欢迎页“最近目录”没有 agent 会话 cwd + mtime 数据源。
- 需要明确的数据:目录路径、命名空间 Host/WSL/SSH、来源会话、最后使用时间、是否仍可访问。
- 需要明确的边界:Host 路径、WSL Linux 路径和 SSH 远端路径不能混成一个本机 `PathBuf`。
- 可行计划:在会话生命周期或 workspace 状态变化处记录最近 cwd,落成 UI 可读取的 recent-sessions 模型,欢迎页只消费模型。

## 退出条件或验证

- 欢迎页能列出最近目录,按 mtime 排序。
- Host / WSL / SSH 目录显示和打开语义正确,不跨命名空间误用路径。
- 无可访问记录时有空状态,不会显示脏数据。
- 数据源更新有单测或可重复的集成验证路径。
