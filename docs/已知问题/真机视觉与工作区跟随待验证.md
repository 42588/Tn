# 真机视觉与工作区跟随待验证

## 定位

本文记录 headless 测试无法覆盖、必须由真实应用窗口确认的视觉、窗口系统和工作区跟随问题。

## 背景

GPUI 布局、窗口拖动、多显示器、高 DPI、启动首帧和动画顺滑度不能只靠单测判断。per-pane cwd 跟随虽然已有架构和部分修复,但仍需真实 shell / OSC 7 会话确认焦点切换时的文件树语义。

## 具体问题或计划

### 视觉与窗口系统待验

- 颜值微调是否符合当前 calm glass 方向。
- 标题栏拖动和控制点命中是否稳定。
- Quick Terminal 动画是否顺滑。
- 多显示器与高 DPI 定位是否正确。
- 首帧是否仍有空白或闪烁。

### per-pane cwd 经 OSC 7 实时跟随

- 问题:每个 pane 应拥有自己的工作区上下文,焦点切换时文件树、cwd 和打开文件夹行为不能串台。
- 背景:per-pane 工作区上下文实现细节已迁出到修复专题;本文只保留真机会话验证债。
- 验证路径:分屏两 pane,分别进入不同目录,切换焦点并观察文件树 root、展开态和选中文件;同一 pane 内 `cd` 进子目录时展开态应保留。

## 退出条件或验证

- `cargo run -p tn-app` 后,窗口首帧不空白,拖动标题栏和控制点命中可靠。
- 多显示器、高 DPI 环境下浮层、菜单和 Quick Terminal 定位正确。
- Quick Terminal 动画无明显卡顿或错位。
- 分屏 pane 各自维护 cwd、文件树展开态和选中项;切焦点不串台。
- 若发现视觉问题,必须在本文记录屏幕环境、DPI、显示器数量、复现路径和预期退出条件。
