# 快速预览编辑器执行包指导

## 定位

本文恢复并维护原 `未修复.md` 中「Tn Editor 用户体验升级:从 Quick Look 快编到可扩展编辑器」的具体执行指导。愿景、当前状态和剩余问题入口见 [快速预览编辑器用户体验升级](快速预览编辑器用户体验升级.md);已完成包的根因、修复和测试记录见 [快速预览编辑器近期变更](../修复与优化/快速预览编辑器近期变更.md)。

本文只承载执行包协议、依赖顺序、每包读入范围、改动范围、验收、验证和禁止项。已完成实现细节不在这里重复展开。

## 执行协议

- `TnE-*` 是后续实现的最小任务包。一次编码会话只认领一个包,完成验证和文档同步后再进入下一个包;不要顺手合并相邻包。
- 每包目标上下文预算不超过 180k token,至少预留 20k 处理意外。读入不超过 110k,实现推理不超过 45k,验证与收尾不超过 25k。
- 每包先读 `AGENTS.md`、`CLAUDE.md` 对应坑位、本文任务卡和任务卡列出的源码切片。`quick_look.rs` 只按 `rg` 定位切片读,不要整文件塞满上下文。
- 单包最多新增 1 个 crate 或 1 个 UI 子模块,最多实质修改 6 个源码文件和必要文档。若同时碰核心模型、renderer、workspace 生命周期,必须拆成两个包。
- 除任务卡明确允许的行为变化外,Quick Look 既有快捷键、鼠标选区、保存、Diff、IME、CJK 命中和滚动心智必须零回归。
- 每包至少跑任务卡列出的最小验证。涉及 headless 逻辑跑对应 crate 单测;涉及 UI 跑 `cargo test -p tn-ui --lib`;涉及新 crate 或依赖方向跑 `cargo test --workspace --lib` 和 `cargo build --workspace`。
- 每包完成后更新 [快速预览编辑器用户体验升级](快速预览编辑器用户体验升级.md) 的状态;若改变架构边界,同步 [编辑器与快速预览](../架构/编辑器与快速预览.md);若改变用户可见 UX,同步 [快速预览与编辑](../产品体验/快速预览与编辑.md);若清掉待修项,更新 [已知问题索引](../已知问题索引.md)。
- 遇到 IME、远端 FS、Diff apply、性能动画、workspace pane 生命周期任一未知坑,先停在当前包内记录发现,拆出新 `TnE-*` 包,不要在同一上下文里继续扩大半径。

## 现状锚点

动手前先确认这些锚点仍成立。若漂移,第一步只更新锚点和任务卡,不要带着过期行号编码。

- 实现时必读相关坑位:[图形界面与渲染坑位](../共享记忆/图形界面与渲染坑位.md)、[焦点与输入法坑位](../共享记忆/焦点与输入法坑位.md)、[终端与进程坑位](../共享记忆/终端与进程坑位.md)。

TnE-18 已完成 headless:Quick Look 编辑态接入轻量 typing effects 与降级 gate,并把光标运动调整为终端同款 chase 模型。验证记录见 [2026-06-09-TnE-18-typing-effects](../任务/2026-06-09-TnE-18-typing-effects.md)。仍需真机用 `TN_PERF` 覆盖连续输入、IME、4000 行和大文件。

### TnE-18 subtle typing effects + perf gate

- **目标**:实现可关闭、可降级、不会影响真实 cursor 的轻量打字反馈。
- **读入**:`tn-config` 的 editor motion policy,Quick Look renderer caret 绘制,`crates/tn-ui/src/perf`,`TN_PERF` 日志,动画卡顿相关坑位。
- **改动范围**:同一视觉行内 90ms cursor glide、块光标 squash/pop,运动使用终端同款 chase 模型;只在打字/删除触发,所有 snap 场景立即归位。
- **验收**:关闭动画、系统 reduced-motion、高负载降级时零行为差异;开启 subtle 时连续输入/删除不卡,真实 cursor 位置在输入事件提交时立即更新,selection/copy/鼠标/查找/滚动不受动画影响。
- **验证**:`cargo test -p tn-ui --lib`;真机用 `TN_PERF` 覆盖连续输入、IME、4000 行、大文件四类场景。
- **禁止**:不做字符 afterglow / settle;不积累 per-char 动画队列;不改变文本宽度、行高、selection、copy 或命中测试。

### TnE-19 文档会话共享与升级入口

- **目标**:Quick Look 可以把当前文档会话升级给正式 Editor Pane,状态不丢。
- **读入**:`workspace.rs` pane / tab / layout 管理,Quick Look open / close 生命周期,`tn-editor::Document` 所有权模型,dirty close 相关逻辑。
- **改动范围**:新增文档会话 registry / handle;Quick Look 增“打开为编辑器”入口;handoff 后共享 buffer、undo、dirty 和 cursor。
- **验收**:浮层升级后内容、光标、undo、dirty 连续;Quick Look 仍是默认入口;升级不改变终端 pane、Explorer per-pane cwd 或 agent 状态。
- **验证**:`cargo test -p tn-ui --lib`;真机 Quick Look 编辑几步 -> 打开为编辑器 -> undo / 保存连续。
- **禁止**:不做多文件标签、LSP、diagnostics、补全;不重构 workspace 分屏核心。

### TnE-20 Editor Pane UI / 状态栏 / 生命周期

- **目标**:提供可长期编辑的 pane / tab 形态。
- **读入**:TnE-19 session handle,workspace pane rendering,状态栏模式,[快速预览与编辑](../产品体验/快速预览与编辑.md),[设计真源与还原工作流](../界面样式/设计真源与还原工作流.md)。

- **验收**:Editor Pane 可编辑保存;关闭 dirty 守卫生效;状态栏准确;Quick Look 与 pane 之间不丢状态。
- **验证**:`cargo test -p tn-ui --lib`;真机浮层 -> pane -> 编辑保存 -> 关闭 / 取消 / 放弃路径。

## 产品形态验收参照

- **Quick Look 快编**:文件树 `Space` / `Enter`、agent 改动卡片、设置文件先开浮层;适合看文件、小改、配置和快速 Diff。
- **Diff Review**:agent 改动卡片进入审查视图;Diff 拖选 / 复制 / 跳转由 TnE-14 管理;hunk accept / reject 的远端实机验证看 [远端文件系统端到端验证](远端文件系统端到端验证.md)。
- **Editor Pane**:长时间 / 多文件编辑时 Quick Look 升级为正式 pane / tab,共享同一编辑核心,有更稳定的空间、状态栏和后续 LSP 落点。
- **命名空间铁律**:Host 直接读写;WSL 走 `\\wsl$` 映射;SSH / macOS 远端走 SFTP 远端 FS 后端,不得把远端路径伪装成本机 `PathBuf`。

## 全局通过标准

- 打开普通文本文件即时可预览;进入编辑不改变原有快捷键心智。
- 连续输入 200 字符无明显卡顿,真实 cursor 不落后文本状态。
- 关闭动画 / reduced-motion / 高负载自动降级时,体验回到瞬时反相块且无行为差异。
- 中文长行点击、拖选、横滚、IME candidate 位置不低于当前精度。
- Markdown/txt/log 软换行不破坏滚动、选区、复制和查找;代码文件仍横滚。
- 4000 行文本滚动和编辑可用,无整面板每帧重建导致的 CPU 尖峰。
- 保存冲突、dirty close、外部修改检测有明确 UI,不静默覆盖或丢弃用户修改。
- File / Edit / Diff 三态交互无回归;`tn-editor` 核心逻辑有 headless 单测覆盖。
- 每个完成包都更新本文状态和对应源文档;上下文压缩前按 `AGENTS.md` 记录目标、关键改动、验证结果和未完成事项。
