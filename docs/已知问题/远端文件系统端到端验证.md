# 远端文件系统端到端验证

## 定位

本文记录 SSH / SFTP / WSL 文件系统链路中仍需端到端真机验证的问题。已完成实现和 bug 复盘不在母索引展开。

## 背景

远端文件树和编辑首版已接 SFTP:`tn-pty::remote_fs` 提供 SFTP v3 `RemoteFileService`,`ExplorerRoot::Ssh` / `ExplorerPath::Remote` 用 `RemoteId` 表示远端路径,Quick Look 可通过 SFTP 做有界预览、编辑写回和保存冲突检测。远端 git 活动栏和 full diff 走 `tn-pty::remote_cmd` + `remote_git`,不跑本机 git 或 notify。

WSL 已走独立 Explorer root,通过 `ExplorerRoot::Wsl` 显式映射到 `\\wsl$\<distro>\...`,“打开文件夹”也走应用内 WSL picker。

这些已完成项不再作为未修问题展开;真正未闭环的是端到端真机验证。

### 仍需端到端验证

| 场景 | 验证路径 | 退出条件 |
|---|---|---|
| SSH 文件树 root | 连接 SSH pane 后观察文件树 | 文件树自动切到远端 cwd,不显示 Windows 主机目录 |
| 远端目录 picker | SSH pane 点“打开文件夹” | 弹应用内远端 picker,可进入子目录、返回上级、滚动长列表,确认后只该 pane `cd` |
| WSL 目录 picker | WSL pane 点“打开文件夹” | 弹应用内 WSL picker,可浏览 Linux 目录,不打开 Windows 原生选择器 |
| 远端文本编辑保存 | Quick Look 编辑远端文本并保存 | 写回远端;外部变化时弹冲突操作条,不静默覆盖 |
| 远端 Diff 拖选/复制 | 打开远端改动卡 Diff | 可拖选复制,CJK 和横滚命中正确 |
| 远端 hunk 跳转 | `PgUp/PgDn` 或点击 hunk | hunk 跳转正确;Diff->File 只在当前 Quick Look 内切 File tab,不落本机路径 |
| 远端 hunk accept/reject | 点击远端 Diff 的接受/拒绝按钮 | 真实修改远端 working tree 或 index,刷新后 hunk 消失;失败时显示可理解错误 |
| 并发 / 超时 / apply 拒绝 | 制造远端补丁冲突或网络慢 | 按钮 busy、防双击、超时和 `git apply` 拒绝都有失败文案 |

## 退出条件或验证

- 自动验证仍需保留:`cargo test -p tn-ui --lib`;涉及远端 git 解析时覆盖 `remote_git` 和 `parse_diff` 一致性。
- 真机验证必须使用真实 SSH/SFTP 目标和真实 WSL distro。
- 任何端到端失败都要记录目标环境、远端 shell、路径、stderr、是否代理/慢网、复现命令和下一步退出条件。
- 全部场景通过后,母索引删除“SSH / SFTP / WSL 远端文件系统仍缺端到端真机回归”条目。
