# 页面样式升级需求待澄清

## 定位

本文记录“对每一页面的样式重新设计,增加更多细节,尝试使用更高级的界面库或者自研界面库”这一开放需求的澄清项。它目前不是可直接实现的问题。

## 背景

该需求横跨页面视觉、组件体系、GPUI 实现方式和潜在 UI 库选型。若不先限定范围,容易变成无边界重做,也会与现有 `design/mockup.html`、`design/panels/`、`design/calm-glass.css` 视觉真源和界面样式规则冲突。

## 具体问题或计划

需要先由主控或产品侧明确:

- 页面范围:欢迎页、主工作区、Quick Look、命令面板、设置、远端 picker、Diff Review 中哪些优先。
- 成功标准:是视觉精修、交互密度提升、组件一致性,还是引入/自研 UI 基础设施。
- 技术边界:是否允许引入新 UI 库;若允许,需要评估与 GPUI、现有样式 token 和性能路径的兼容性。
- 迁移方式:整页重做、组件级替换,还是按问题驱动逐页修补。

## 退出条件或验证

- 形成独立设计/实现任务,列明页面范围、组件范围、视觉真源和验证方式。
- 若涉及 UI 库,必须有选型记录、替代方案和回退策略。
- 不在已知问题母索引中展开设计稿或实现细节;澄清完成后迁到产品体验、界面样式或任务文档。
