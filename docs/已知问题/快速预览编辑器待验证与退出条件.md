# 快速预览编辑器待验证与退出条件

## 定位

本文记录 Quick Look / Tn Editor 已实现但仍需真机复验的问题,包括复现方式、已知根因和退出条件。

## 背景

多项 Quick Look 编辑器改动已有 headless 单测或构建验证,但涉及 GUI、窗口关闭、IME、滚动、Diff 交互和大文件手感的行为必须由真实应用运行确认。母索引只标记“待验证”,本文承载具体验证清单。

## 具体问题或计划

| 项目 | 复现 / 验证路径 | 已知根因或背景 | 退出条件 |
|---|---|---|---|
| 本地保存冲突检测 | Quick Look 打开本地文件,外部编辑器修改同一文件后在 Tn 中保存 | `save_local_text` 已按打开时 `FileGuard` 对比当前磁盘,冲突应进入操作条 | 默认不覆盖;取消保留本地修改;覆盖后编码和换行风格保留 |
| dirty-close 守卫 | 编辑未保存时走关闭浮层、切文件、切 Diff、菜单退出、`Ctrl+Shift+Q`、标题栏关闭、Alt+F4 | 标题栏关闭和 Alt+F4 曾绕过 `request_quit`;已接 `window.on_window_should_close` | 所有离开路径弹保存/放弃/取消;取消不丢状态 |
| 编辑后退预览刷新 | Quick Look 编辑文件后按 Escape 回预览 | 旧预览读 `file_data`,编辑只改 `Document`;已加 `sync_preview_from_edit` | 回预览立即显示编辑内容,不需重开文件;dirty 状态不被清掉 |
| 4000 行连打性能 | 接近 `MAX_LINES` 的文本中连续输入和撤销 | `tn-editor::Document` 已用有界 transaction 和行区间镜像,headless 守卫已过 | 真实连打无明显输入延迟,undo/redo 粒度不回归 |
| 自绘 File / Edit renderer | 普通文本、CJK 长行、长文件、横滚、选区、复制、保存;Markdown/txt 长逻辑行内 `Shift+Up/Down` | File/Edit 默认已走自绘路径,`TN_QL_LEGACY=1` 仍是逃生口;软换行视觉行上下移动已有 headless 守卫 | 行号、着色、滚动、命中、选区和复制与旧路径一致或更好;软换行长行内键盘选择按视觉行扩展 |
| 查找框中文输入与候选框 | 打开查找,用微软拼音输入中文,执行 Enter / Shift+Enter 跳转 | 查找开时需要把可打印键放行给 IME handler,候选框贴查找框 bounds | 中文可合成,候选框贴查找框,命中高亮醒目,跳转纵横居中 |
| 正文编辑 IME 候选框与连续输入 | 打开长 Markdown/txt 文件,在软换行视觉行中用微软拼音连续输入;观察候选框和滚动 | 正文 `bounds_for_range` 已改用真实 caret paint rect;自绘路径文本提交不再每字强制居中 | 候选框贴当前 caret 底部;连续输入不明显卡顿、不每字跳到视口中心;代码文件长行仍横滚 |
| Quick Look 升级到 Editor Pane | Quick Look 编辑几步后点“打开为编辑器”或按 `Ctrl+Enter`,在 Editor Pane 继续输入、undo、保存、关闭 pane、关闭 tab、标题栏关闭或 Alt+F4 | TnE-19/20 已通过共享 `DocumentSession` 和 Workspace `editor_panes` 接入,headless 守卫覆盖 session 连续、pane 插入和 dirty close prompt 状态 | 内容、光标、undo、dirty 连续;保存后 dirty 清除;未保存关闭显示“保存 / 放弃 / 取消”;取消保留编辑器和未保存内容;终端 pane、Explorer cwd 和 agent 状态不被改写 |
| Diff renderer 对照 | agent 改动卡片打开本地和远端 Diff,滚动、拖选、复制、hunk 跳转 | Diff tab 已接自绘 renderer,metadata 过滤和 hunk 映射已补守卫 | new-file metadata 不显示为 `0` 行;本地 Diff->File 第 6 项通过;远端不落本机路径 |

## 退出条件或验证

- 自动验证底线:`cargo test -p tn-ui --lib`;涉及 `tn-editor` 时加 `cargo test -p tn-editor`;阶段收尾跑 `cargo test --workspace --lib` 和 `cargo build --workspace`。
- 真机验证底线:`cargo run -p tn-app`;IME 场景加 `RUST_LOG=tn::ime=debug`;性能场景可加 `TN_PERF`。
- 任一待验证项通过后,从本文移到修复专题或在修复专题中追加验证结果;母索引对应状态改为“已迁出”或删除。
- 任一复验失败时,本文新增失败现象、复现条件和新的退出条件,不要只在母索引写长段分析。
