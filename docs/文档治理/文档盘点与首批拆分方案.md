# 文档盘点与首批拆分方案

## 定位

本文记录当前文档系统的最小结构和后续扩展规则。

## 背景

项目已经清理为重新设计起点。当前 Markdown 文档只应承担两个职责:

- 提供协作入口和任务记录。
- 保留踩坑记录,避免新设计重复触发已知工程问题。

## 具体内容

### 当前必需入口

| 文件 | 职责 |
|---|---|
| `AGENTS.md` | Codex 启动入口,只保留索引和最小规则 |
| `CLAUDE.md` | Claude 启动入口,与 `AGENTS.md` 同级 |
| `TODO.md` | 当前任务母索引 |
| `docs/文档治理索引.md` | 文档治理母索引 |
| `docs/共享记忆索引.md` | 共享记忆母索引 |
| `docs/共享记忆/踩坑记录.md` | 坑位记录母索引 |

### 父索引规则

- 父索引只负责定位、入口表、状态摘要和新增信息落点。
- 父索引不写长篇方案、不写完整复盘、不承载实现细节。
- 父索引中的状态必须能区分“已记录”“规划”“待验证”“已验证”。

### 子文档规则

- 子文档承载具体内容、背景、过程、验证和反向链接。
- 每个子文档只服务一个主题。
- 不复制其它子文档的长期事实,需要时用链接引用。

### 当前状态词

| 场景 | 可用状态 |
|---|---|
| TODO | `进行中`、`待办`、`已完成`、`阻塞` |
| 文档记录 | `已建立`、`已记录`、`规划`、`待验证`、`已验证` |

## 验证 / 状态

- 当前文档系统按最小入口重建。
- 旧产品方案、旧任务、旧架构和旧修复记录不再保留。
