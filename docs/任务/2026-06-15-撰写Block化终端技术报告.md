# 撰写 Block 化终端技术报告

## 背景
用户在阅读了《终端项目互联网调研报告》后，对 Warp 终端的 Block-based 架构以及其深度的 AI 编排能力产生了浓厚兴趣，并询问是否能将其借鉴进 Tn 项目中。

## 方案与实现
- 分析了基于 Rust、GPUI 结合 Alacritty/ConPTY 解析器的可行性。
- 起草并输出了详细的《Tn-Block化终端重构可行性与技术报告》。
- 报告涵盖了：语义提示 (OSC 133) 的 Shell 注入方案、PTY 解析器状态机、GPUI UI 渲染层重构（从单一画布到列表流），以及与现有 Tn 生态（AI Agent 纠错、宠物情绪联动）的化学反应。
- 提出了包含数据采集、UI 解耦、生态赋能三个阶段的渐进式实施路线图。

## 验证结果
- 技术报告已成功归档于 `docs/参考资料/Tn-Block化终端重构技术报告.md` 中，为后续的架构级迭代提供了理论支撑与路线图规划。
