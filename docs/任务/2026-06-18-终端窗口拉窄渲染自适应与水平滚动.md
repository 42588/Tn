# 任务: 终端窗口拉窄渲染自适应与水平滚动

## 背景与目的
当用户把终端窗口拉得非常窄，或者进行多分屏时，由于终端可用宽度过小，PTY resize 到极小的列数导致终端输出文字被疯狂硬折行。这会导致输出结果极其杂乱无章，严重破坏可读性。
本任务旨在引入终端最小列数限制自适应，并在宽度不足时提供水平滚动功能（水平滚动条、鼠标滚轮/触摸板水平滑动、以及光标自适应追随），以保证在极窄窗口下输出排版依然工整清晰，且用户能够流畅地阅读和输入。

## 任务详情
1. **最小列数限制**
   - 设定最小 PTY 限制列数 `MIN_TERMINAL_COLS = 80`。
   - 修改 `fit_grid_size_from_bounds`，当实际列数小于 80 且视口有效时，强行限制其最小列数为 80，避免 PTY resize 到极小值。
2. **水平滚动机制与状态管理**
   - `TerminalView` 新增 `horizontal_scroll_offset`（水平像素偏移）和 `horizontal_scrollbar_drag`（水平滚动条拖拽起点 X 坐标缓存）。
   - 初始化时设置为默认值。
3. **渲染横向偏移**
   - 渲染终端主体网格时，在其 `left` 偏移中减去 `horizontal_scroll_offset`。
   - 渲染光标 `cursor_el` 和 IME 合成预览 `ime_preedit` 时，其 `left` 对应减去 `horizontal_scroll_offset`，确保在水平滚动时它们与网格的相对位置正确。
4. **鼠标与滚轮交互**
   - 更新 `on_scroll`：捕获 `ScrollDelta` 的水平分量（`p.x`），并根据水平滚动值更新 `horizontal_scroll_offset`。
   - 实现水平滚动条 `horizontal_scrollbar`：当网格总宽度大于可见宽度时渲染；支持鼠标按下并拖拽滚动。
   - 更新 `on_mouse_move` 和 `on_mouse_up`，正确响应水平滚动条的拖拽计算与释放。
   - 更新 `cell_at`，在计算单元格坐标时，鼠标物理 X 坐标需加上 `horizontal_scroll_offset`，以正确完成选区拖拽与点击交互。
5. **光标自适应追随**
   - 每次渲染时，检测当前光标的列物理 X 坐标。若光标溢出右侧可见视口，自动向右滚动使得光标立即可见；若光标移动到左侧可见视口左边，则向左滚动追随光标。
