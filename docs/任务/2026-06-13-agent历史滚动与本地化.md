# Agent 历史滚动 & 「本地化」澄清

## 背景

BUG 发现清单(③):agent 面板历史「能滚一部分,太长就看不到」;用户记得有个
「历史太多就本地存储、查看时再从本地拉起」的功能,怀疑它坏了。

## 调研结论

1. **滚动本身是好的**。Claude Code 是主屏(main-screen,Ink 渲染),对话进终端回滚缓冲。
   `on_scroll` 里 `scroll_wheel_drives_terminal_app(agent, alt_screen)` 对主屏 agent 返回
   false → 滚轮直接滚终端回滚(`Terminal::scroll`),alacritty 把偏移 clamp 到 history 内。
2. **真因 = 回滚缓冲上限**。`config.general.scrollback_lines` 此前默认 **5000**,在
   `terminal_view/mod.rs` 的 `Terminal::with_scrollback` 处生效(配置注释「not yet wired」是
   过期的,其实已接线)。长 Claude 会话输出轻松超 5000 行,更早的行被 alacritty 环形缓冲淘汰
   → 滚到顶就到头,「太长就看不到」。
3. **「本地化」功能确实存在,但不是历史扩展器**。`tn-core` 的 `swap_out_async` /
   `restore_if_swapped` 把**后台 tab** 的整块网格序列化到磁盘(`%TEMP%/tn/scrollback_{id}.bin`)
   以释放内存,任何交互(滚动/输出/resize)再从盘拉回。它是**省内存**机制,拉回的是同一块
   (同样 5000 行上限)的网格,**不会**让你看到更早的历史。用户记忆把这个省内存的 swap
   误当成了「历史扩展」。

## 实现

`crates/tn-config/src/config.rs`:

- `General::scrollback_lines` 默认 **5_000 → 50_000**。
- 修正过期注释「(not yet wired)」:说明它已接线、是回滚行数上限、惰性分配(内存随实际输出
  增长而非按上限预分配)、后台 tab 会 swap 到磁盘,所以放大默认是可负担的。

为什么是 50k 而非更大:alacritty 0.26 的 `max_scroll_limit = scrolling_history`,即此值就是
上限、无额外引擎封顶;但它在内存里,marathon 会话产 50k 行满宽输出会吃不少 RAM。50k 覆盖
绝大多数长会话,且惰性分配让常规会话成本不变,后台 tab 又落盘,取了平衡。可在配置里再调大。

## 验证

- `cargo test -p tn-config --lib`:44 绿。
- 真机:跑一个超 5000 行的长 agent 会话,确认能向上滚到更早内容(待用户确认)。

## 补记(2026-06-13):codex 滚不动 = 另一类 agent,真因不同

用户复测「修过一次没卵用」,截图是 **codex**(`codex GPT 5.4`),不是 Claude Code。两者渲染方式不同:

- **Claude Code**:Ink/主屏,对话进终端回滚缓冲 → 上面 50k 那条对它有效。
- **codex**:ratatui/crossterm **全屏 TUI**,跑在**备用屏(alt screen)**并**接管鼠标**
  (`EnableMouseCapture` → DECSET 1000/1002/1003 + SGR 1006)。**备用屏没有回滚缓冲**,
  那 500 行历史只存在 codex 自己的 transcript 里 → 放大 scrollback 对它**完全无效**
  (这就是「修过一次没卵用」)。滚动只能让 codex 自己滚。

**真因 = Tn 从来不向 PTY 转发鼠标事件**。`tn-core` 里没有任何 mouse-report 编码;`on_scroll`
对 alt-screen 只会发**方向键**(`ESC[A/B`),而 codex 接管了鼠标、按方向键不滚 transcript →
滚轮石沉大海,滚动条(本就因 alt-screen 无 history 而不渲染)自然也拖不动。

### 修复:滚轮 → 鼠标上报转发

- `tn-core` `InputMode` 暴露三个新位:`mouse_report`(任一 1000/1002/1003)、`sgr_mouse`(1006)、
  `alternate_scroll`(1007),来自 alacritty `TermMode`(`MOUSE_MODE`/`SGR_MOUSE`/`ALTERNATE_SCROLL`)。
- `terminal_view/mod.rs` 加纯函数 `encode_wheel_report(up, col, row, sgr, count)`:
  SGR → `ESC[<64/65;col;rowM`;legacy → `ESC[M` + 三个 `+32` 字节(坐标封顶 223)。
- `on_scroll` 改为三级优先:**①** 应用开了鼠标上报(codex)→ 按指针所在 cell 发滚轮上报(优先级
  最高,盖过 alt-screen 方向键路径);**②** alt-screen 无鼠标捕获的分页器(less/vim)→ 方向键;
  **③** 主屏 → 滚自家回滚缓冲(Claude Code 走这条)。
- 鼠标**点击/拖拽**的转发本次未做(只修滚动,避免动到本地选区语义),列为后续。

### 验证

- `cargo test -p tn-core`(34 绿,含新 `tracks_mouse_report_and_sgr_modes`)、
  `cargo test -p tn-ui`(192 绿,含 `wheel_report_sgr_*` / `wheel_report_legacy_*`)、
  `cargo check --workspace` 通过。
- 真机:在 codex 面板向上滚,应能滚回更早 transcript(待用户确认)。

## 再修(2026-06-13):旧配置模板冻结 5k

用户反馈多次修复后仍无法滚出历史输出。重新排查发现第一轮只改了
`General::default().scrollback_lines = 50_000`,但首次运行写入用户目录的
`DEFAULT_CONFIG_TOML` 来自 `config/config.toml`,其中仍写着 `scrollback_lines = 5000`。
磁盘配置优先级高于内置默认,所以已经运行过旧版本的真机仍会把 pane 构造成 5k scrollback,
表现为「改了默认但滚到顶还是没历史」。

本轮修复:

- `config/config.toml` 模板同步改为 `scrollback_lines = 50000`,新安装不再冻结旧上限。
- `tn-config::load_from` 对正好等于旧模板默认值 `5000` 的配置做内存迁移,加载后抬到
  `50_000`,覆盖存量旧默认配置;不改写用户文件。
- 新增回归测试:
  - `bundled_default_config_parses`:确保模板解析后就是 50k。
  - `legacy_default_scrollback_is_lifted_on_load`:确保旧 5k 配置加载后按 bugfix 生效。

验证:

- `cargo test -p tn-config legacy_default_scrollback_is_lifted_on_load`:先红后绿。
- `cargo test -p tn-config bundled_default_config_parses`:先红后绿。

## 再修(2026-06-13):agent 主屏清掉 Tn scrollback

用户截图显示 `config.toml` 已配置 `scrollback_lines = 10000`,但 codex 面板没有 Tn 滚动条,
滚轮也无反应;用户补充 Claude 也有同样现象。复查 `%APPDATA%\Tn\logs\tn.log` 后确认滚轮事件
走到:

```text
agent=Some("codex") alt_screen=false mouse_report=false history=0 offset=0 branch="scrollback"
```

第一轮尝试把 `agent && history == 0` 的滚轮转成 PageUp/PageDown (`CSI 5~` / `CSI 6~`) 发给
agent,但真机截图与日志证明 Codex/Claude 主屏并不把这当成可滚动 transcript;滚轮进入
`agent_pages` 后仍是 `history=0 offset=0`,所以该方向撤回。

最终根因收敛为:这类主屏 agent 在重绘时会发 `CSI 3 J`(Erase Saved Lines),alacritty 会按
终端语义清空 scrollback。于是用户看到的 500 行普通输出虽然短暂进入过终端,随后被 agent 的
清历史序列抹掉,Tn 滚动条自然消失;这和 `scrollback_lines` 配多大无关。

本轮修复:

- `tn-core` 新增 `ScrollbackClearFilter`,流式过滤 `CSI 3 J` / C1 `CSI 3 J`,且支持跨 reader
  chunk 拆开的序列;`CSI 2 J` 等可见区清屏仍原样交给 VT parser。
- `terminal_view::io::spawn_reader` 在 agent pane 中启用该过滤器,使 Codex/Claude 能清可见屏,
  但不能删除 Tn 保留的 scrollback。
- 普通 shell 保留标准终端语义:非 agent 时 `CSI 3 J` 仍会清本地 scrollback。
- shell 内手动运行 agent 时,reader 通过 shell integration 的 `CommandLine` 与
  `AgentRegistry::match_command` 提前打开过滤,尽量赶上第一批 agent 输出。
- 撤回无效的 `AgentPages` 路由;主屏滚轮仍滚 Tn scrollback,鼠标上报 TUI 和 alt-screen
  fallback 逻辑保持不变。

验证:

- 新增 `csi_3j_clears_scrollback`:证明原始 `CSI 3 J` 会清空 alacritty scrollback。
- 新增 `scrollback_clear_filter_strips_csi_3j_across_chunks`:先红后绿,覆盖跨 chunk 过滤。
- 新增 `command_line_agent_detection_uses_first_token_only`:保护 reader 提前识别 typed agent。
- `cargo test -p tn-core --lib`:36 绿。
- `cargo test -p tn-ui --lib`:194 绿。
- `cargo check --workspace`:通过(仅保留既有 `local_dir_picker::open_selected` dead_code 警告)。

## 再修(2026-06-13):主屏 agent 只重绘可见 transcript,不产生全量 scrollback

用户继续复测后补充了一个关键现象:正常大小的 agent 窗口无法滚动;把终端最大化后,可见的历史数据变多;
再缩小窗口后只能滚动到“最大化时曾经显示过”的那一段,不能再往前。这说明 Codex/Claude 并不是把 500 行
逐行追加进 Tn/alacritty 的全局 scrollback,而是由 agent 自己维护 transcript,按当前 PTY 高度重绘可见窗口。
窗口变大时 agent 只是把更多 transcript 画进当前屏;之后缩小时,Tn 最多只能把这些曾经可见的行挤入本地
scrollback,所以历史长度正好等于最大化时多看到的行数。

因此,单靠 50k scrollback、鼠标上报、PageUp/PageDown 或过滤 `CSI 3 J` 都只能修一部分场景;要让终端本地历史
真正可滚,需要让 agent 自身进入“非全屏 / inline / 保留原生 scrollback”的输出模式。

本轮修复:

- 本机 `codex --help` 确认 Codex 官方提供 `--no-alt-screen`,说明其 inline 模式会保留终端 scrollback。
- 本机 `~/.claude/cache/changelog.md` 确认 Claude Code 提供
  `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1`,用于关闭 fullscreen alternate-screen renderer 并保留原生
  terminal scrollback。
- `tn-agent::AgentDescriptor` 新增 `default_env`,与既有 `default_args` 一起作为 agent 身份能力的一部分。
- 内置 Claude descriptor 注入 `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1`;内置 Codex descriptor 注入
  `--no-alt-screen`。
- `tn-ai::builtin_adapter_for_manifest` 在用户通过 `[[agents]]` 重新声明 Claude/Codex telemetry 时,把内置
  descriptor 的 `default_args/default_env` 合并到用户 descriptor,避免真实 config-backed 路径丢失修复。
- `LaunchSpec` 新增 `env`,启动 agent profile 时合并 descriptor 默认值;Codex 默认参数插在 profile args
  之前,确保 `codex --no-alt-screen resume ...` 这种顶层参数顺序正确,并对用户已显式写入的默认参数做精确去重。
- `TerminalView::new` 将 `LaunchSpec.env` 传入 `tn_pty::SpawnSpec`,使环境变量真正作用到 PTY 子进程。
- `layout::LayoutPane` 持久化 `env`,避免保存/恢复布局后 Claude 的 inline 环境变量丢失。

## 后续 / 注记

- **真·无限历史 = 磁盘分页回滚**:alacritty 不支持把超出上限的历史分页到磁盘按需拉回,
  现有 swap 只搬「上限内的整块」。要做到 overnight 会话也不丢,需要自建 disk-paged scrollback,
  属大功能,本次未做。50k 是当前实际可达上限。
- 配置键 `[general] scrollback_lines` 用户可自行调更大(代价是活动 pane 的内存)。
