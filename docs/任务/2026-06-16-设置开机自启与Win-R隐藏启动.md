# 设置开机自启与 Win+R 隐藏启动

## 背景
用户期望 Tn 能够开机自启，且不弹出主终端界面，而是让幽灵终端（QuickTerminal）从电脑启动就常驻后台进程，由用户在需要时再手动唤起主终端界面。同时，程序需要注册到 Windows 中，向 cmd 一样能够通过 Win+R 输入 `tn` 快速打开。

## 方案与实现
- **打包器模板定制**：将 `cargo-packager` 的默认 NSIS 模板复制为本地的 `crates/tn-app/installer.nsi`，修改 `Section Install` 增加 `App Paths`（用于 Win+R）与 `Run`（开机自启，携带 `--startup` 参数）注册表项的写入，在 `Section Uninstall` 增加相应项的卸载删除。
- **命令行参数处理**：检查启动参数中的 `--startup` / `--silent` 标志。如果是开机启动，则不打开主窗口，但主进程、托盘图标和幽灵终端热键监听继续初始化。
- **单实例 IPC 通知**：实现 `notify_primary_instance` 逻辑。当用户再次启动程序（非 `is_primary`）时，通过 `FindWindowExW` 寻找后台实例的托盘消息窗口，并发送注册的窗口消息 `Tn.Terminal.ShowMainWindow.v1` 激活原进程的新窗口创建，随后本进程直接退出。
- **窗口生命周期重构**：将 AppState 的主窗口 ID 管理与托盘事件绑定进行闭包重构，使得后续唤醒能正确记录和管理重新生成的窗口。

## 验证结果
1. NSIS 脚本写入成功，支持在安装时正确配置注册表，卸载时干净清理。
2. 命令行参数 `--startup` 与 `--silent` 处理逻辑完善，启动时不弹窗，并成功驻留系统托盘。
3. IPC 通知机制测试通过，后续唤起可通过单实例通知创建新窗口。
4. Cargo Packager 构建无误，验证通过。
