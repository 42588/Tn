# 用量额度面板(5h/日/周 + 在线权威配额)

参考 `ai-limit`(github.com/zhuchenxi113/ai-limit)。用户选定**本地聚合 + 在线权威配额**全量方案。

## ai-limit 机制(调研结论)

- **消耗**(用了多少):本地 `~/.claude/projects/**/*.jsonl`(Tn 已在读)。
- **权威配额**(上限/剩余/重置):`claude.ai/api/organizations/{orgId}/usage`,用**浏览器
  session cookie** 认证(从 Chrome 经系统密钥库解密)。这是唯一拿到真·5h 窗口/周上限+重置的途径。
- Claude Code 本地**不存**配额/重置状态(`history.jsonl` 无 ratelimit 记录、`config.json` 空);
  只有 `stats-cache.json` 存按天聚合的 token(无上限、且滞后)。结论:权威配额本地拿不到。

## 分阶段

### Stage 1 — 窗口聚合后端(已完成)

`crates/tn-ai/src/usage_windows.rs`:
- 每个 assistant 行有顶层 ISO-8601 `timestamp` + `message.usage`。自写无依赖
  `parse_iso8601_utc`(Howard Hinnant days-from-civil → epoch)。
- `Turn`(at/model/4 桶 token)、`Window`(tokens/cost/turns/first_at)、
  `UsageWindows`(5h/day/week + `reset_5h` 估算 = 5h 窗内最早一条 + 5h)。
- `aggregate`(纯)、`turns_from_jsonl`(纯)、`collect_recent_turns`(按 mtime 只读近 7 天文件)、
  `current_windows`。cost 走 `pricing_for`。
- 测试 4 绿(epoch 解析、分窗聚合+重置估算、空、jsonl 解析)。

### Stage 2 — 面板 UI(已完成)

用户拍板:**合并进原有用量读数**(原信息本就不准),不另开独立面板。

- `TerminalView` 加 `quota_open` / `quota: Option<UsageWindows>` / `quota_gen`;
  `toggle_quota` + `refresh_quota`(后台线程算 `current_windows`,`quota_gen` 丢弃过期结果)。
- header 用量 pill 拆两块点击:**读数/环 → 开额度浮层**;**chip → 切 $/%/tokens**
  (`stop_propagation` 互不干扰)。
- `render_quota_panel`:浮层(L3+H2+R_PANEL+shadow_float),5 小时/今天/本周 行(成本/tok/条)
  + 5h 重置相对估算 +「官方配额:待接入(Stage 3)」占位 +「本地实算」标注。
- pane 外层加透明 click-catcher scrim:浮层外点击即关;浮层自身 `stop_propagation`。
- `clear_agent` 顺手关浮层。
- `cargo test -p tn-ui` 190 绿;`cargo build` 通过。

### Stage 3 — 在线权威配额(已搁置)

用户拍板浏览器 Edge 后,真机探测发现该 Edge 的 `Local State` 含 **`app_bound_encrypted_key`**
——即 Edge 127+ 的 **app-bound 加密**已启用,cookie 大概率是 v20。标准
DPAPI+AES-GCM(v10)解密 v20 直接失败;v20 要走 Edge 提权服务(COM `IElevator`),又难又脆、
大量 unsafe+新依赖(rusqlite/aes-gcm)、我侧无法验证、真机未必成。

**用户决定:先不做在线权威配额。** 浮层「官方配额」行显示「暂未接入」。

将来若要做,**优先「手动贴 sessionKey」**(用户从 Edge 复制一次 claude.ai 的 `sessionKey` →
Tn 调 `claude.ai/api/organizations/{orgId}/usage`),零原生解密、稳、可验证,绕开 app-bound;
而非硬啃 v20。已查清:`windows` 0.61 是工作区依赖(可加 crypto feature),Edge 已装。

## 结论

④ 以 Stage 1+2 交付:点用量读数/环 → 额度浮层(5h/日/周 token+$ + 5h 重置估算 +「本地实算」)。
