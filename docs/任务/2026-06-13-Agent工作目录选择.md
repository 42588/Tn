# Agent 工作目录选择

## 目标

修复 BUG #4:幽灵终端和主窗口欢迎页启动 Agent 前都要先选择本地主机工作目录,避免 Agent 只能进入默认目录。

## 范围

- 新增共享本地目录选择状态与最近工作目录持久化。
- 幽灵终端 Agent 磁贴点击后进入单层目录选择面板。
- 欢迎页 Agent 磁贴点击后打开临时目录选择浮层,不改变欢迎页布局。
- 两处统一键位:`Tab` 切焦点,`↑↓` 选择,`←` 返回上级,`→` 进入目录/打开系统目录选择器,`Enter` 用当前高亮工作目录启动 Agent。
- Windows 盘符通过 `This PC` 虚拟根统一挂载;普通目录列表只显示真实子目录,避免盘符混入每一层目录。
- 两处目录选择窗口使用固定高度;最近工作目录最多保留 5 条。
- 欢迎页 Explorer root 只作为初始目录 seed,不额外占用最近工作目录列表名额。
- Agent 面板进入后,鼠标滚轮应滚动终端 scrollback,不能把历史消息重新拉回输入框。
- 普通 shell、WSL、SSH 启动行为不改。

## 实现记录

- `crates/tn-ui/src/local_dir_picker.rs`:新增 `LocalDirPicker`、最近目录、目录读取、Windows 虚拟根盘符入口和键位状态测试。
- `crates/tn-ui/src/terminal_view/launch.rs`:新增 `LaunchSpec::with_cwd`。
- `crates/tn-ui/src/quick_terminal.rs`:Agent tile 先打开 Ghost 内联目录选择器,确认后用选中 cwd 启动 ephemeral Agent。
- `crates/tn-ui/src/workspace.rs`:欢迎页 Agent launch 由 Workspace 拦截,显示 overlay 目录选择器,确认后用选中 cwd 填充当前欢迎 tab。
- `crates/tn-ui/src/local_dir_picker.rs`:左右方向键只在当前目录焦点下导航;焦点在最近工作目录时只用于高亮和 `Enter` 启动,不会误进入或返回目录。
- `crates/tn-ui/src/terminal_view/mod.rs`:修正 Agent 主屏滚轮路由;此前 `scroll_wheel_drives_terminal_app(agent_active, alt_screen)` 把 Agent 主屏等同于 alt-screen,滚轮会被翻译为上下方向键发给 Agent 进程,导致历史输入被滚轮切换。现在只有真实 alt-screen 程序接管滚轮,Agent 主屏滚轮用于 scrollback。
