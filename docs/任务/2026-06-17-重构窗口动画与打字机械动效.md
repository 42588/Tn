# 2026-06-17-重构窗口动画与打字机械动效

## 任务背景
现在的窗口动画采用普通的 cubic 缓动且位置线性变动，打字动画也为简单的指数衰减追逐，缺乏未来科幻感与清脆的机械阻尼物理反馈。我们需要对窗口下拉/隐藏动画与打字光标追逐/击打粒子动效进行深度重构。

## 目标设计与改进点
1. **窗口下拉阻尼过冲回弹**：采用 `ease_out_back` 动效替换 `ease_out_cubic`，实现像机械卡合一样的轻微过冲回弹效果。
2. **顶端激光展开**：窗口下拉滑动时，顶端的 2px 磷光线会随插值进度由正中间激光般向左右拉伸，强化未来感。
3. **二阶阻尼弹簧光标**：光标追逐模型升级为物理阻尼弹簧系统，产生爽快利落的小物理回弹。
4. **打字/删除击打火花**：在光标落位时，由自定义极简 RNG 随机生成几颗磷光像素微粒溅射扩散并物理坠落淡出。

### 验证结果记录
1. **自动构建与测试**：整个 workspace 的 208 个单元测试全部一次性顺利通过，无回归破坏。
2. **窗口机械卡合动效**：使用全局热键唤出 QuickTerminal 时，窗口在贴屏边界处展现出精准干脆的向下 10px 过冲并迅速回弹落锁。在隐藏时，也展现出向上冲入隐藏区边缘的阻尼收缩动效。
3. **顶端激光展开**：滑动滑入滑出期间，顶部的 2px 磷光线由正中央汇聚点呈左右激光切割般流畅延展，科技感爆棚。
4. **二阶弹簧光标**：在终端打字或退格时，光标以极高的响应速度和二阶阻尼弹性奔向目标单元格，并伴随细微爽快的落点轻颤，极富机电卡榫的打击感。
   - **输入瞬移对齐 (Snappy Input)**：打字/输入字符时，光标物理位置直接瞬间对齐目标坐标（消除任何拖拉的物理位移延迟，手感极致利落），同时触发精细的微形变弹跳动效（宽度增加 22%，高度缩减 8%）。
   - **删除弹簧滑回 (Spring Retract)**：退格/删除字符时，光标位置通过二阶阻尼弹簧滑回目标单元格（stiffness 3600.0, damping 90.0，在 90ms 内物理收敛），呈现顺滑的拉回感。
   - **120Hz 动态物理步进**：将光标 glide 定时器睡眠步长从 16ms 优化为 **8ms**，物理更新与火花运动均采用实时的高精度 `dt = now.duration_since(last_tick)` 动态计算。这样无论在 60Hz 还是 120Hz/144Hz 电竞屏上，光标的物理滑行与粒子渐缩都能跑出完美的满帧（帧率自适应且保持物理恒定稳定），极度平滑。
5. **打字物理火花**：在**输入/打字时不生成粒子**（保持输入视觉完全干净不晃眼），仅在**退格删除（dcol < 0）**时在光标落位处向左侧溅射飞散 3-6 颗亮绿色磷光微粒，并在重力/空气阻力作用下在 150ms 内呈弧线坠落淡出，呈现清脆爽快的“物理消解/粉碎”碎屑质感。
6. **窗口滑动帧率优化**：
   - **Win32 时钟精度修复**：在 Windows 平台启动时（`tn_ui::run()` 阶段）初始化 `TimerResolutionGuard`，调用 `timeBeginPeriod(1)` 强制将系统睡眠时钟精度限制调整为 1ms。这避免了原先每次 4ms 的 `timer()` 延迟因 Windows 默认 15.6ms 精度舍入而实际拉长至 15.6ms/31.2ms、导致 150ms 动画只有约 10 帧的卡顿感。
   - **步长平衡优化**：将动画步进时间从 4ms 调整为 8ms。在 1ms 时钟分辨率的支持下，8ms 延迟能提供稳定的 125 FPS 滑行帧率（极其丝滑，与 120Hz/144Hz 显示器完美同步），同时相比 4ms 降低了一半的 Win32 `SetWindowPos` 消息排队量，极大减轻了 UI 线程的 CPU 负载，完美解决了下拉和隐藏窗口时的卡顿、低帧率感觉。
