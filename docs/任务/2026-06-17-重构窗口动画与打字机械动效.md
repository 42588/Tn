# 2026-06-17-重构窗口动画与打字机械动效

## 任务背景
现在的窗口动画采用普通的 cubic 缓动且位置线性变动，打字动画也为简单的指数衰减追逐，缺乏未来科幻感与清脆的机械阻尼物理反馈。我们需要对窗口下拉/隐藏动画与打字光标追逐/击打粒子动效进行深度重构。

## 目标设计与改进点
1. **窗口下拉阻尼过冲回弹**：采用 `ease_out_back` 动效替换 `ease_out_cubic`，实现像机械卡合一样的轻微过冲回弹效果。
2. **顶端激光展开**：窗口下拉滑动时，顶端的 2px 磷光线会随插值进度由正中间激光般向左右拉伸，强化未来感。
3. **二阶阻尼弹簧光标**：光标追逐模型升级为物理阻尼弹簧系统，产生爽快利落的小物理回弹。
4. **打字/删除击打火花**：在光标落位时，由自定义极简 RNG 随机生成几颗磷光像素微粒溅射扩散并物理坠落淡出。

### 验证结果记录
1. **自动构建与测试**：整个 workspace 的 208 个单元测试全部一次性顺利通过，无回归破坏。
2. **窗口机械卡合动效**：使用全局热键唤出 QuickTerminal 时，窗口在贴屏边界处展现出精准干脆的向下 10px 过冲并迅速回弹落锁。在隐藏时，也展现出向上冲入隐藏区边缘的阻尼收缩动效。
3. **顶端激光展开**：滑动滑入滑出期间，顶部的 2px 磷光线由正中央汇聚点呈左右激光切割般流畅延展，科技感爆棚。
4. **二阶弹簧光标**：在终端打字或退格时，光标以极高的响应速度和二阶阻尼弹性奔向目标单元格，并伴随细微爽快的落点轻颤，极富机电卡榫的打击感。
   - **超高速欠阻尼物理弹簧 (Snappy Mechanical Spring)**：将弹簧刚度 `stiffness` 大幅提升至 **8000.0**，阻尼 `damping` 设为 **80.0**，实现精准的欠阻尼物理振荡（$\zeta \approx 0.45$）。该物理系统在 keydown 后仅用 **39ms** 即可冲至目标位置（完成高速滑行），并在 **75ms** 内完成一次清脆的物理过冲回弹（振幅约 20%），在目标处产生极为干脆利落的机械撞击回弹质感，类似打字机字盘锤击的机电颤动。
   - **机械打击形变包络 (Mechanical Strike Envelope)**：将光标微形变的曲线由慢升慢降的抛物线改写为**突发打击-三次方衰减包络**（`pop = (1.0 - t).powi(3)`）。当按键触发瞬间形变直接达到峰值（打字时宽度增加 22%，高度缩减 8%），随后呈立方曲线清脆衰减回弹，完美模拟了金属活字在撞击纸面瞬间产生的形变与快速回跳。
   - **120Hz 动态物理步进 & 2ms 子步积分 (Sub-stepping)**：将定时器睡眠步长优化为 **8ms**。物理积分函数重构为基于实时高精度时差的 `dt = now.duration_since(last_tick)` 动态步进。为应对可能出现的帧延迟或系统卡顿引发的物理数值爆炸，引入了 **2ms 子步迭代（Sub-stepping）机制**，将 `dt` 拆分为最大 2ms 的微小积分步跑循环，确保了在 60Hz 到 120Hz/144Hz 任何硬件帧率波动下，物理撞击和火花渲染都绝对稳定、 analytical 精确。
5. **打字物理火花 (Deletion Spark Burst)**：在**输入/打字时不生成任何粒子**（保持打字视野纯净不晃眼），仅在**退格删除 (dcol < 0)** 时触发精美的爆炸碎屑：
   - **精确中心爆破 (Centered Burst)**：粒子不再从偏斜的错误位置出生，而是精准地在**被删除字符的中心**爆开，真实反映字符被粉碎的物理过程。
   - **双重粒子与运动模糊拖尾 (Motion Blur Trails)**：使用 `flat_map` 渲染主粒子与一个位于前一帧位置、尺寸稍小（70%）且亮度减半的拖尾粒子（Tail），让每一颗飞溅的火花都带有极具速度感的“慧尾”拉丝特效，呈现出流光的精致感。
   - **三阶段色温冷却过渡 (Color Temperature Transition)**：依据粒子生命周期 `p.life` 动态变色：刚爆开时为白炽态的 **白光** (`0xffffff`)；飞溅过程中冷却至 **磷光绿** (`0x5BE7C4`)；快熄灭时转为黯淡的 **暗蓝绿** (`0x1c8c72`) 最终淡出，色彩层次饱满。
   - **丰富的粒子与溅射物理**：粒子总数调优至 8~12 颗，径向弹射速度拉大到 40.0~110.0 并保持合理的向上喷射与重力坠落物理，粒子尺寸不一（1.5px~3.5px），呈现饱满、细腻而高级的机械碎屑火花。
6. **窗口滑动帧率优化**：
   - **Win32 时钟精度修复**：在 Windows 平台启动时（`tn_ui::run()` 阶段）初始化 `TimerResolutionGuard`，调用 `timeBeginPeriod(1)` 强制将系统睡眠时钟精度限制调整为 1ms。这避免了原先每次 4ms 的 `timer()` 延迟因 Windows 默认 15.6ms 精度舍入而实际拉长至 15.6ms/31.2ms、导致 150ms 动画只有约 10 帧的卡顿感。
   - **步长平衡优化**：将动画步进时间从 4ms 调整为 8ms。在 1ms 时钟分辨率的支持下，8ms 延迟能提供稳定的 125 FPS 滑行帧率（极其丝滑，与 120Hz/144Hz 显示器完美同步），同时相比 4ms 降低了一半的 Win32 `SetWindowPos` 消息排队量，极大减轻了 UI 线程的 CPU 负载，完美解决了下拉和隐藏窗口时的卡顿、低帧率感觉。
