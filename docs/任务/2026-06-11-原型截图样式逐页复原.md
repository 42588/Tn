# 原型截图样式逐页复原

## 定位

本文记录基于 `design/shots/` 下 5 张原型截图逐页复原 Tn 当前 GPUI 界面样式的任务拆分与实施状态。

## 背景

用户希望根据 `D:\coder\Tn\design\shots` 下的页面截图逐个页面复原样式。截图对应旧 HTML 原型的视觉目标,但项目已删除样式守卫,后续样式由新设计与实现自行决定。因此本任务以截图为视觉参考,以当前源码结构为实现边界,不恢复旧 token drift / spec 生成守卫。

用户补充约束:

- 本任务只做界面样式复原:颜色、间距、圆角、阴影、层级、hover/active 视觉、排版密度。
- 不允许丢弃或删除现有功能;不得移除 Quick Look 预览/编辑/保存/Diff、命令面板、Quick Terminal、窗口控制、快捷键、pane 管理、远端/本地文件逻辑。
- 不改变现有动作回调、数据结构、保存关闭保护、输入法和焦点归属,除非发现样式改动造成回归且必须修正。

## 具体内容

### 0. 共用视觉基线

- [x] 从 5 张截图提取共用视觉语言:深色背景层级、Calm Glass 透明面板、1px 边框、高光洗层、柔和阴影、radius、字体层级、状态色和 agent 色。
- [x] 对照 `design/calm-glass.css` 与当前 `crates/tn-ui/src/style.rs`,保留运行时共享 helper,不恢复旧 token drift / spec 守卫。
- [x] 建立页面复原验收口径:代码层面只改视觉属性,功能回归用 `tn-ui` lib 测试兜底;最终观感仍需真机截图复验。

### 1. 窗口外壳复原

- [x] 复原集成标题栏:品牌 mark、Tab 行、active tab 顶部 accent、hover/active 状态、窗口控制按钮。
- [x] 复原 App Menu:左上品牌菜单、分组线、图标、快捷键文本、hover 背景和菜单阴影。
- [x] 复原状态栏:分支、会话数、agent context、文件/编码/主题等分段信息。
- [x] 重点验收:标题栏高度、Tab 尺寸、菜单位置、状态栏对齐与窗口边缘留白。

### 2. 工作区窗格复原

- [x] 复原 pane 容器:玻璃面板、焦点描边、内层 specular wash、阴影和圆角。
- [x] 复原 Agent header:头像、名称、模型、副标题、用量 ring/pill、Codex/Claude 颜色差异。
- [x] 复原 Agent body 与 activity rail:保留真实 git diff 数据源,将右侧改动卡调回原型轻量卡片视觉。
- [x] 复原 Shell pane:cwd header、终端正文、block bar、提示符、成功状态芯片。
- [x] 重点验收:终端字体可读性、pane 内边距、右侧活动栏宽度、焦点状态不挤压正文。

### 3. 侧栏与 Quick Look 复原

- [x] 复原 Explorer 文件树:树缩进、目录/文件图标、选中行、git M/U 标签、侧栏玻璃容器。
- [x] 复原 Quick Look 浮层:Explorer + 右侧预览组合、标题栏、File/Diff segmented tab、底部提示条。
- [x] 复原 Diff 视图:行号、删除/新增背景、hunk 行、代码字体、右上编辑状态。
- [x] 复原 Quick Look 编辑态:File tab、光标、底部保存/退出提示、长行和中文命中测试保持现有逻辑。
- [x] 重点验收:不破坏 Quick Look 现有预览、编辑、保存、Diff、远端文件和冲突保护。

### 4. 浮层与 Quick Terminal 复原

- [x] 复原命令面板:居中玻璃浮层、模糊 scrim、输入行、结果行、agent 身份点、右侧路径/来源文本。
- [x] 复原 Quick Terminal 启动器:顶部下拉窗口、会话磁贴、selected 状态、底部 ephemeral 提示。
- [x] 重点验收:浮层打开时焦点不被 workspace 抢走,输入不破坏 IME,Quick Terminal popup 尺寸不受 DPI 误差影响。

### 5. 状态页与 Block 状态复原

- [x] 复原 Welcome launchpad:居中品牌图标、标题、副标题、启动磁贴、快捷键提示;最近目录不做假数据源。
- [x] 复原空状态:保持克制,不引入大面积营销式 hero 或无关装饰。
- [x] 复原 Block 三态:成功、运行中、失败的左侧状态条、右侧耗时/状态芯片、命令行背景。
- [x] 重点验收:欢迎页在默认窗口尺寸内居中但不过度放大,Block 状态色准确且不影响终端内容。

### 6. 页面级回归与收尾

- [x] 每完成一个页面后运行对应最小验证,必要时用截图检查视觉偏差。
- [x] 全部页面完成后运行 `cargo test -p tn-ui --lib`。
- [x] 根据 `target/ui-shots/` 真机截图回归修正面板底色、高亮和玻璃质感。
- [ ] 如启动 GUI 验收,优先检查 1100x720 默认窗口、宽屏、窄屏和 Quick Terminal popup。
- [x] 更新任务文档验证结果,必要时补充新的坑位到 `docs/共享记忆/`。

## 执行顺序建议

1. 先做共用视觉基线和 `style.rs` helper,避免 5 个页面重复造样式。
2. 再做窗口外壳和工作区窗格,因为它们定义全局框架与 pane 质感。
3. 再做侧栏/Quick Look 和浮层,因为它们依赖焦点、输入、overlay 层级。
4. 最后做 Welcome / Block 状态和全局回归,补齐状态色与空状态。

## 验证 / 状态

- 状态:已关闭 —— 多轮真机回归证实 Calm Glass 大面积玻璃渐变在 GPUI 下物理性不可达(无逐元素 blur/噪点,8-bit 必色带)。视觉目标由 [磷光设计语言与原型重做](2026-06-11-磷光设计语言与原型重做.md) 取代,旧原型与截图资产已删除。
- 2026-06-11(关闭记录):本任务沉淀的有效结论已固化 —— ① 大面积只能纯色;② 平铺投影会渗血,改接缝;③ specular wash 全幅透明层必须为空。这些已写入磷光渲染契约与 [图形界面与渲染坑位](../共享记忆/图形界面与渲染坑位.md)。
- 2026-06-11:用户已确认任务拆分;开始按页面实施样式复原。
- 2026-06-11:已按“只复原界面样式、不丢弃现有功能”约束完成视觉补丁;未新增样式守卫,未恢复旧 token drift 测试。
- 2026-06-11:用户提供 `target/ui-shots/` 真机截图后,确认偏差主要集中在材质层:常驻 pane / Explorer / Welcome / Quick Look 的底色被实现为单色中点,缺少原型纵向玻璃渐变、顶部高光、边缘导光和投影层次;Quick Terminal 运行态被终端实色背景铺满,缺少浮层玻璃外壳。
