# 终端删除光标手感优化

## 定位

优化终端 pane 内文本光标的打字与删除手感,重点解决 Backspace / DeleteForward 删除时视觉反馈偏软、跟手不明确的问题。

## 背景

终端 pane 当前在 `crates/tn-ui/src/terminal_view/mod.rs` 内自绘光标,按 cell 变化做同行动画。现有逻辑主要用 `dcol > 0` 判断前进或后退,没有显式区分输入、Backspace、DeleteForward 和跳转;删除反馈只是一段较软的反向 glide。

本轮不使用 Lottie 热路径,不改全局系统鼠标光标,不改变 PTY 输入编码。

## 具体内容

- 增加终端光标 motion 的小状态/决策函数 `classify_terminal_cursor_motion`。
- 区分 `InsertEcho`、`BackspaceEcho`、`DeleteForward`、`Snap`。
- Backspace 使用更短、更硬的压缩回弹;DeleteForward 在光标 cell 不移动时给原地短促反馈。
- 在 `on_key_down` 与 `replace_text_in_range` 中标记最近输入 intent,渲染时按真实光标回显位置分类。
- 保留 agent 自绘光标、大跳/换行时 snap 的性能边界;不引入 Lottie 热路径。
