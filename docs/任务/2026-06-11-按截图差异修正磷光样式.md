# 按截图差异修正磷光样式

## 定位

依据 [design/原型与真机截图差异总结.md](../../design/原型与真机截图差异总结.md) 的 P0/P1 清单,对现有 GPUI 实现做样式回归修正。本轮先收掉两个差异最大的 P0 集群:QuickLook 浮层语法与幽灵终端运行态比例,再把需要产品决策的结构性 P0 列为阻塞项交回用户。

## 本轮已改(P0 样式)

### QuickLook(`crates/tn-ui/src/quick_look.rs` + `workspace.rs`)

- 删除浮层左侧贯穿全高的 2px 磷光竖脊 `seam`(原 `quick_look.rs` 6604-6611 + `.child(seam)`)。左脊是 tile/row/命令块的「选中」语义,浮层借用会让磷光退化为装饰边框(总结 P0)。海拔改回 float 投影 + h2 边表达。
- 给 `workspace.rs` 的点击关闭 scrim 补上纯色压暗背景 `bg(rgba(SCRIM))`,让底层终端被压暗,QuickLook 才像「工作区之上的临时速览浮层」。Explorer 仍在 scrim 之外保持可点(点树换预览)。
- 浮层尺寸从「高 88% 撑满 pane」收回到横向速览比例:宽 86%/上限 1080、高 80%/**上限 600**,避免被 pane 可用高度撑成近全屏编辑器(总结 P0:尺寸/外边距)。
- File/Diff 分段开关顺序回到 **File 左 / Diff 右**(原为 Diff/File)。

### 幽灵终端(`crates/tn-ui/src/quick_terminal.rs`)

- 运行态窗体不再用配置的下拉尺寸(默认满屏宽 Top 45%),改为与启动器同一「760 顶垂窗」规格:宽固定 `CARD_W`、水平居中、顶贴屏,仅沿用配置高度(总结 P0:运行态不按屏宽拉伸成横条)。`hidden_for` 统一为从屏顶滑入/滑出。
- 顶缘磷光从「透明→中央亮、右半段满铺」改为左右两段镜像渐变拼出**中央亮、两端渐隐**(gpui `linear_gradient` 只吃两停靠点,故拆两半;总结 P1)。

## 复核未改

- 主 pane 焦点边:`style::plate()` 已是 PH_DIM(·32 dim)边 + 四角角标,符合原型 `.plate.focus`,无需改。

## 验证

- `cargo build -p tn-ui` 通过(本轮全部改动)。
- 真机回归截图待补:QuickLook 居中浮层 + scrim 压暗、幽灵运行态 760 顶垂窗。

## 阻塞:需产品决策的结构性 P0

以下 P0 不是单纯样式,需用户先定权威,再实现或回写原型:

1. 欢迎页是否常驻 Explorer(原型为无 Explorer 整窗 Launchpad)。
2. Agent 是「终端内 CLI」还是「专门 Agent 板面」(决定是否重画 02-workspace-panes)。
3. Agent/Shell 是否恢复 Explorer + Agent + Shell 多 pane 仪表舱。
4. SSH 过程态/TOFU/认证失败/断线重连卡片是否实现(原型有,真机只有原始日志)。
5. WSL 二级启动器补图纸或并入启动器家族。
6. 欢迎页/分屏/Ghost 的 launch tile 是否统一为共享 tile 语法(尺寸/左脊/图标/副标题)。
