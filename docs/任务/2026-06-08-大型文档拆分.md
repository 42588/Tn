# 大型文档拆分为母文档索引 + 子文档详情

进行中。

## 目标

把项目内所有长期维护文档逐步改造成“母文档精准定位子文档、子文档展示具体信息”的一对多结构,降低单文件膨胀和长期记忆漂移。

## 当前记录

本任务从现在开始用 `TODO.md` 跟踪。`TODO.md` 只保留总任务状态和下一步;本文负责记录全量范围、阶段清单、验证结果和未完成事项。

## 文档盘点范围

- 根入口: `AGENTS.md`, `CLAUDE.md`, `TODO.md`。
- 项目介绍与变更: `README.md`, `CHANGELOG.md`;`README.zh-CN.md` 在当前工作区未发现,若后续补回再纳入。
- docs 目录: `docs/系统架构索引.md`, `docs/产品体验索引.md`, `docs/界面样式实现规则.md`, `docs/修复与优化记录索引.md`, `docs/新增模块索引.md`, `docs/已知问题索引.md`, `docs/外部参考资料索引.md`。
- 后续新增子文档目录: `docs/任务/` 已建立;其它主题目录待按首批拆分方案创建。
- 文档治理方案: `docs/文档治理/文档盘点与首批拆分方案.md` 已建立,负责记录盘点表、首批目录命名、母文档模板和拆分顺序。

## 初始拆分对象

- `CLAUDE.md` / `AGENTS.md`: 作为同级 agent 入口,只保留各自索引和必要差异;共享规则沉到 `docs/` 权威文档。
- `docs/修复与优化记录索引.md`: 保留日期和主题索引,把具体优化记录拆到 `docs/修复与优化/` 下的主题文件。
- `docs/已知问题索引.md`: 保留问题索引和优先级,把具体根因、复现、限制拆到 `docs/已知问题/` 下的专题文件。
- `docs/系统架构索引.md`: 保留架构导航,把 crate / 子系统细节拆到架构子文档。
- `docs/界面样式实现规则.md`: 保留样式入口和守卫规则,把组件级细节拆到样式子文档。
- `docs/新增模块索引.md`: 保留模块索引,把模块详情拆到功能/子系统文档。
- `CHANGELOG.md`: 保持发布向变更记录;不承载详细设计,只链接到对应任务或专题文档。

## 拆分原则

- 按主题拆,不按行数拆。
- 母文档只放摘要、状态、日期和链接。
- 子文档是具体信息的权威落点。
- 同一事实不在多个母文档重复铺开。
- 迁移时先保持语义不变,再做必要的标题和索引整理。

## 验收标准

- 每个被拆的大型母文档都能快速定位到对应子文档。
- 子文档标题能清楚说明自己负责的主题。
- 原有有效信息没有丢失。
- 新增信息规则已写入 `AGENTS.md` 和 `CLAUDE.md`。
- `TODO.md` 能准确显示当前阶段、下一步和任务详情入口。
