# 宠物规则 v2 陪伴版

## 背景

用户诉求:在现有基础上设定一套更有趣的规则**覆写**原规则,让宠物系统成为用户最喜欢的部分 ——「每天都期待使用 Tn,与宠物见面」。

## 设计立场(本轮最重要的决定)

1. **期待靠惊喜,绝不靠愧疚**:明确反规则 —— 无饥饿/心情/健康值、无等级/成就/图鉴/商城、无推送红点;缺席零惩罚。每日仪式(见面问候、一块小饼干)制造期待,负罪机制制造的是卸载。
2. **活着的错觉来自微小与随机**:加「活物引擎」(idle 20-60s 随机微动作:抓痒/舔爪/追尾转圈/竖耳/望屏外/伸懒腰),全部用既有像素变换词汇,零新原语。
3. **打扰预算硬约束**:微动作 ≤2/分钟、庆祝 ≤2/10 分钟、气泡 ≤4/小时;Typing/Running/浮层/alt-screen 全停;气泡 ≤6 字封闭词表。

## v2.1 修订(同日 · 用户四问的心理学定夺)

用户问:① 能否命名?② 唯一一只 / 随机刷新 / 升级解锁怎么选?③ 动画要不要更用心?④ 要不要"买玩具"机制?分析与定夺:

| 问 | 定夺 | 心理学依据 |
|---|---|---|
| 命名 | **做**(规则 0):领养时起名,随时可改,状态栏/菜单档案/欢迎页三处上屏,不进气泡 | 禀赋效应 + 宜家效应 + 自我延伸 —— 最便宜的归属感杠杆 |
| 拥有模式 | **唯一主宠 + 领养仪式**;随机刷新废除(随机只活在领养卡「交给缘分」);升级解锁**否决** | 依恋需要身份连续性与不可替代性,可替换 = 壁纸;解锁制触发过度合理化效应(外在奖励排挤内在喜爱)+ 解锁完毕即流失悬崖 |
| 动画 | **质不是量**:立「关键互动三帧律」(预备/主体/收尾 + 次级运动 + ease-out);被动克制、主动反馈往死里打磨 | 生命感知来自运动时机(Heider-Simmel);粘性来自主动触发的 juice 手感 |
| 玩具 | **摆不买**(远期岗台小物):自由挑选、无货币、不消耗,配「拍球追回」微动作 | 供养行为建立依恋(照料心理);经济系统把照料变成肝,是黑暗模式滑坡 |

- 新增反规则:不做失去/告别戏码(换形象零愧疚)、玩具只挑不买、随机无常驻入口。
- 「换品种」语义收紧为「换个形象」:名字/档案/天数全延续(主宠是同一个体)。
- 分期更新:P1 加既有互动补三帧;P2 加领养仪式与命名(领养卡版式进板 F 审核稿)。
