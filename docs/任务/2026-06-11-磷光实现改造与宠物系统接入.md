# 磷光实现改造与宠物系统接入

## 定位

本文记录把「磷光 Phosphor」设计语言(design/panels/ 七张图纸)落到 Rust/GPUI 实现、并同轮接入像素宠物系统(特色③)的任务过程与验证结果。承接 [磷光设计语言与原型重做](2026-06-11-磷光设计语言与原型重做.md) 的「实现改造」与 [宠物系统实现](2026-06-10-宠物系统实现.md)。

## 背景

用户确认磷光原型后,要求「对原本的 UI 进行改造,期望与原型一模一样,顺便完善宠物系统」。改造以图纸 SPEC 表为机械对照,设计 token 单源在 `crates/tn-ui/src/style.rs`,调色板单源在 `config/themes/tn-dark.toml`。

### 6. 宠物系统(SHEET 05,特色③)

- [x] 新模块 `crates/tn-ui/src/pet.rs`:
  - 七犬像素资产转写自 `docs/宠物/原型/0X-*.svg`(14×12 网格),GPUI canvas 逐 quad 直绘(≤96 quad/帧,远低于 168 上限),无 SVG tint 限制。
  - 上下文状态机:进程级原子信号(`signal_typing` 接 terminal_view::on_key;`signal_command_start/end` 接 io reader 的 OSC 133 OutputStart/CommandFinished)→ 优先级 Drag > Error/Success > Running(>1s)> Typing > Hover > Idle;240ms tick,变化才重绘。
  - 像素演出:眨眼(4–8s)、尾摆(idle 1px / running 2px)、呼吸沉浮、立耳(typing)、小跑步态(脚掌交替)、success 上跳 + 像素小心、error 垂耳 + 「- -」眯眼(眼格压扁 40%)、click 歪头杀 + 「汪!」气泡 2s、drag 拎起悬空。
  - 互动:hover 摸摸(闭眼)、左键单击吠、左键拖拽换窝(落点持久化)、右键菜单(随机刷新/七犬直选/仅欢迎页/隐藏/关闭系统)。
  - 栖位:右下 100×84 固定容器 + 1px 岗台(左 28px 磷光点睛);状态栏法定席位 `WESTIE · IDLE`(点击=显隐,关闭后显示 `PET · OFF` 可点回)。
  - 持久化:`%APPDATA%\Tn\pet_state.json`(固定品种/显隐/仅欢迎页/栖位),与 ssh_recents 同模式;品种初始化随机、固定优先、手动刷新随机。
  - reduced motion:`[editor] animations = "off"` → 全静帧。
  - 欢迎页 2× 形态:`sprite_block(breed, 2.0)` 入 welcome.rs 右下栖位(岗台 + 标签)。
- [x] welcome.rs:Launchpad 重做(hero-mark 44 + `TN_` mono 26 + 副标、`.ltile` 磁贴 150px、kbd 提示行、宠物 2× 栖位)。
