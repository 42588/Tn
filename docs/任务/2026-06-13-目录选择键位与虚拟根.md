# 目录选择键位与虚拟根

## 目标

修正本地 Agent 工作目录选择器和 SSH/WSL 打开文件夹远端目录选择器的键位规则,并把 Windows 盘符切换改为类似 Linux `/` 的虚拟根模型。

## 范围

- Agent 本地目录选择器保留 `Tab` / `Shift+Tab` 在最近工作目录、当前目录、浏览本地文件夹之间切换焦点。
- Agent 本地目录选择器和 SSH/WSL 远端目录选择器统一键位:`↑↓` 选择,`←` 返回上级,`→` 进入目录,`Enter` 确认/启动。
- SSH/WSL 远端目录选择器不再用 `Backspace` 返回上级,也不再需要 `Ctrl+Enter` 确认。
- Windows 本地目录选择器不再把盘符混入每个目录列表;盘符只出现在 `This PC` 虚拟根下。
- 从 `C:\`、`D:\` 等盘根按 `←` 返回 `This PC`;在 `This PC` 再按 `←` 不返回。
- 幽灵终端与欢迎页的 Agent 工作目录窗口固定面板高度,目录数量增加时不继续撑高窗口。
- 最近工作目录最多保留 5 条,旧的持久化记录加载时也会裁剪。

## 实现记录

- `crates/tn-ui/src/local_dir_picker.rs`:新增 Windows 虚拟根 sentinel、`current_label()`、盘根上级处理、最近目录 5 条上限和虚拟根目录读取测试。
- `crates/tn-ui/src/quick_terminal.rs`:幽灵终端 Agent 目录选择器显示 `This PC` 标签,复用虚拟根 fallback,并固定 Ghost 工作目录面板高度。
- `crates/tn-ui/src/workspace.rs`:欢迎页 Agent 目录选择器复用虚拟根并固定 overlay 面板高度;SSH/WSL 远端目录选择器抽出 `remote_dir_key_action` 并改为左右/Enter 新规则;页脚提示同步改为 `↑↓` / `←` / `→` / `Enter`。

## 验证

- `cargo test -p tn-ui remote_dir_picker_keymap_uses_arrows_for_navigation_and_enter_for_confirm` 通过。
