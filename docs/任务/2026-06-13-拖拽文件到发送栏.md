# 拖拽文件/目录到 agent 发送栏

## 背景

BUG 发现清单(②):希望从文件管理器(资源管理器)拖一个目录/文件到 agent 窗格,
路径直接落入 agent 的输入行;按回车即发送给 agent。普通 shell 窗格也支持。

代码此前无任何外部文件拖入处理(无 `ExternalPaths` / `on_drop`)。agent 没有独立
「发送栏」UI,输入直接进 PTY,所以「落入发送栏」= 把路径写进 PTY 当前输入行(不回车)。

## 规格(用户确认)

- 落点:整个 agent 窗格都算。
- 多文件:拼成一行,空格分隔。
- 路径形式:**agent 窗格 → 相对路径**(相对 agent 的工作目录;文件在工作目录外则回退绝对路径);
  **普通 shell → 绝对路径**。
- 不加引号(即使含空格/中文)。
- 行为:只填入输入行,**不自动回车**,用户审阅后自己按回车发送。

## 实现

`crates/tn-ui/src/terminal_view/mod.rs`:

- import `gpui::ExternalPaths`(gpui 0.2.2 Windows 经 `FileDropEvent::Submit` 投递)。
- 外层 pane div 挂 `.on_drop::<ExternalPaths>(cx.listener(... drop_external_paths ...))`,
  并 `.drag_over::<ExternalPaths>(|s,..| s.bg(PH_DIM))` 拖悬时整面磷光微亮提示。
- `drop_external_paths(paths, window, cx)`:
  - `ssh_input_blocked` / 空列表 → 跳过。
  - base = agent 时取 `rail_cwd`(launch / Open-Folder 目录),shell 时 `None`。
  - `render_dropped_paths` 拼一行;空 → 跳过。
  - 复用 paste 路径:`scroll_to_bottom` + 读 `bracketed_paste`,bracketed 时用 `\e[200~..\e[201~`
    包裹(当输入,不当按键),否则直写;`flush`。
  - 落下后 `focus_handle.focus(window)`,让用户的回车进到本窗格;`cx.notify()`。
- 纯函数(headless 测试):
  - `relative_under(path, base)`:按 `Path::components` 逐段 `eq_ignore_ascii_case` 比较
    (Windows 盘符/大小写不敏感),base 耗尽后剩余段即相对路径,保留原始大小写与分隔符;
    path 在 base 外或等于 base → `None`。
  - `render_drop_path(path, base)`:有 base 且在内 → 相对,否则绝对。
  - `render_dropped_paths(paths, base)`:逐个渲染、滤空、空格 join。

## 验证

- `cargo test -p tn-ui --lib`:189 passed(含新增 `dropped_path_is_relative_under_agent_cwd_else_absolute`
  与 `multi_drop_joins_with_spaces`,均 `#[cfg(windows)]`)。
- 真机拖放观感待用户确认(从资源管理器拖入、多选拖入、目录拖入、相对/绝对回退)。

## 后续 / 注记

- 当前不在路径前后补空格:同一次多文件拖入内部已空格分隔;若用户连续做两次独立拖放,
  会贴在一起——真机如不便再考虑补尾随空格。
- 远端(SSH)agent 拖入的是本地 Windows 路径,语义上对远端无意义,本次不特殊处理。
- BUG 清单其余三项(①本次改动数据 / ③历史滚动·本地化 / ④用量准确·周日额度)按用户排序后续单独推进。
