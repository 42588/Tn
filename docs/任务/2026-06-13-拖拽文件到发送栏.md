# 拖拽文件/目录到 agent 发送栏

## 背景

BUG 发现清单(②):希望从文件管理器(资源管理器)拖一个目录/文件到 agent 窗格,
路径直接落入 agent 的输入行;按回车即发送给 agent。普通 shell 窗格也支持。

代码此前无任何外部文件拖入处理(无 `ExternalPaths` / `on_drop`)。agent 没有独立
「发送栏」UI,输入直接进 PTY,所以「落入发送栏」= 把路径写进 PTY 当前输入行(不回车)。

## 规格(用户确认)

- 落点:整个 agent 窗格都算。
- 多文件:拼成一行,空格分隔。
- 路径形式:**agent 窗格 → 相对路径**(相对 agent 的工作目录;文件在工作目录外则回退绝对路径);
  **普通 shell → 绝对路径**。
- 不加引号(即使含空格/中文)。
- 行为:只填入输入行,**不自动回车**,用户审阅后自己按回车发送。

## 实现

`crates/tn-ui/src/terminal_view/mod.rs`:

- import `gpui::ExternalPaths`(gpui 0.2.2 Windows 经 `FileDropEvent::Submit` 投递)。
- 外层 pane div 挂 `.on_drop::<ExternalPaths>(cx.listener(... drop_external_paths ...))`,
  并 `.drag_over::<ExternalPaths>(|s,..| s.bg(PH_DIM))` 拖悬时整面磷光微亮提示。
- `drop_external_paths(paths, window, cx)`:
  - `ssh_input_blocked` / 空列表 → 跳过。
  - base = agent 时取 `rail_cwd`(launch / Open-Folder 目录),shell 时 `None`。
  - `render_dropped_paths` 拼一行;空 → 跳过。
  - 复用 paste 路径:`scroll_to_bottom` + 读 `bracketed_paste`,bracketed 时用 `\e[200~..\e[201~`
    包裹(当输入,不当按键),否则直写;`flush`。
  - 落下后 `focus_handle.focus(window)`,让用户的回车进到本窗格;`cx.notify()`。
- 纯函数(headless 测试):
  - `relative_under(path, base)`:按 `Path::components` 逐段 `eq_ignore_ascii_case` 比较
    (Windows 盘符/大小写不敏感),base 耗尽后剩余段即相对路径,保留原始大小写与分隔符;
    path 在 base 外或等于 base → `None`。
  - `render_drop_path(path, base)`:有 base 且在内 → 相对,否则绝对。
  - `render_dropped_paths(paths, base)`:逐个渲染、滤空、空格 join。

## 补记(2026-06-13):真需求是「Tn 自己的资源管理器」→ 窗格,不是 OS 文件管理器

真机复测:`ExternalPaths` 那套只接 **OS 文件管理器**(Windows 资源管理器)拖入的事件;而用户要的是
从 **Tn 左侧 EXPLORER 面板**拖文件进窗格。两者机制完全不同:前者是 OS→应用的外部拖放,后者是
GPUI **应用内**拖放,需要拖源 row 挂 `on_drag`、落点挂 `on_drop::<内部载荷>`。

更糟的是:explorer row 在 **`on_mouse_down`** 上就 `emit(OpenFile)` → 一按下立刻预览,且 row 上
**没有任何 `on_drag`** → 拖拽手势根本无从开始(这正是「一点就预览、拖不动」)。

### 决策:左键原生拖拽(用户拍板)

用户最初想用**右键**拖(怕左键碰预览)。但 GPUI 的拖放**硬编码只认左键**(源码 `event.button ==
MouseButton::Left`),右键得手写整套(鼠标跟踪+浮动幽灵+跨实体命中),又多又脆。而「怕碰预览」其实
GPUI 原生就能解:预览改挂**真·click(按下+抬起)**,拖拽挂**左键 on_drag**,框架自动区分点击与拖动
——单击照常预览,按住拖才进入拖拽。用户选了**左键原生拖拽**。

### 实现

- `explorer.rs`:
  - 新增 `pub struct DraggedFile { file: ExplorerFile, label: SharedString }`(GPUI 窗口级 `active_drag`
    载荷)+ `FileDragChip`(跟随光标的磷光小票渲染视图)+ `wsl_unc_to_linux`(`\\wsl$\d\a\b`→`/a/b`)。
  - row 由 `.on_mouse_down(Left, …emit OpenFile)` 改为 `.id(("explorer-row", i))` + **`.on_click`**(预览/
    展开/选中,逻辑不变)+ **`.on_drag(DraggedFile{…}, |p,..| cx.new(|_| FileDragChip::new(p.label)))`**。
    文件与目录都可拖。
- `terminal_view/mod.rs`:
  - 抽出 `insert_input_text`(bracketed-paste 包裹 + 不回车 + focus),`drop_external_paths` 复用之。
  - 新增 `drop_dragged_file` + `dropped_file_text`:按**本窗格 namespace** 渲染路径——WSL 窗格遇
    `\\wsl$\…` UNC → Linux 路径;SSH 文件 → `id.path` 远端路径;本地 → 同外部拖放的 agent 相对/shell 绝对。
  - 窗格根 div 在原 `ExternalPaths` 旁并挂 `.on_drop::<DraggedFile>` + `.drag_over::<DraggedFile>`(磷光提示)。
    GPUI 拖放是跨实体、窗口全局的 → 光标下的窗格直接收到 drop,**workspace 零改动**。

### 补记验证

- `cargo build -p tn-ui` 通过;`cargo test -p tn-ui --lib` 192 绿。
- 真机待确认:从 EXPLORER 左键拖文件/目录到 agent / shell / WSL / SSH 窗格,路径正确落入输入行、不回车。

## 后续 / 注记

- 当前不在路径前后补空格:同一次多文件拖入内部已空格分隔;若用户连续做两次独立拖放,
  会贴在一起——真机如不便再考虑补尾随空格。
- 内部拖拽目前单文件(拖光标所在 row);explorer 暂无多选,多选拖拽属后续。
- SSH 远端文件拖入插入的是远端绝对路径(`id.path`);跨 namespace(远端文件拖进本地窗格)语义上少见,不特殊处理。
