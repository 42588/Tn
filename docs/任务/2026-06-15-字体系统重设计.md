# 字体系统重设计(未来科技感 + 可读)

- 日期:2026-06-15
- 范围:`crates/tn-ui/assets/fonts/`、`crates/tn-ui/src/{lib,style,workspace,welcome,quick_look}.rs`、
  `crates/tn-config/src/config.rs`、`config/config.toml`
- 状态:已完成（真机验证通过 · 2026-06-19）

## 目标

按用户要求,对 UI / 编辑器 / 终端字体做精美设计,体现未来科技感、同时保证可读性、字号合理。

## 选型(经用户确认)

| 用途 | 字体 | 说明 |
| ---- | ---- | ---- |
| 终端 / 编辑器 / 代码(等宽) | **JetBrainsMono Nerd Font** | 几何、可读性顶级、含 Nerd 图标字形(替换 CaskaydiaCove,图标覆盖等价、零功能回退) |
| UI 正文无衬线 | **Inter**(Regular/Medium/SemiBold) | 现代 app 标准、极可读、几何中性(替换系统 Segoe UI) |
| 词标 / hero 展示字 | **Space Grotesk**(Medium/Bold) | 几何科技感;用于欢迎页 `TN_` hero、标题栏 `TN_` 词标 |
| 中文回退 | **思源黑体 SC**(Source Han Sans SC,OFL) | 打包进 exe,西文遇 CJK 字形自动落到此族,各机一致 |

全部 `include_bytes!` 进 exe,运行时零系统字体依赖。授权:JBM/Inter/Space Grotesk/思源黑体均 OFL,可安全入库。

## 实现要点

- **加载**([lib.rs](../../crates/tn-ui/src/lib.rs)):`add_fonts` 注册 10 个字体文件(JBM×4 + Inter×3 +
  Space Grotesk×2 + 思源黑体 SC×1);移除 CaskaydiaCove×4。OTF/CFF(思源)在 Windows
  DirectWrite 原生支持,实测 `TN_AUTOQUIT` 启动不 panic = 全部解析成功。
- **中文回退机制**([style.rs](../../crates/tn-ui/src/style.rs) `with_cjk`):GPUI 的 `font_family()` 只改字族、
  不动 `font_fallbacks`,且 `font_fallbacks` 沿父级继承。故在**workspace 根节点**设一次
  `.font(with_cjk(UI_SANS))` → 全树 Div 文本继承「思源黑体」回退;终端网格继承同链;
  少数直接构造整 `Font` 做 shaped run 的地方(编辑器自绘 / Markdown 的 `TextRun`)单独
  调 `with_cjk` 挂回退。DirectWrite 把该回退排在系统回退之前 → 优先用打包思源。
- **family 默认值**:`config.rs` / `config.toml` 等宽默认 → `JetBrainsMono Nerd Font`。
- **展示字**:欢迎页 hero `TN_`、标题栏 `TN_` 词标改用 `UI_DISPLAY`(Space Grotesk);
  `_` 仍 PH 磷光当光标,品牌的终端提示符隐喻保留。
- **命名字号层级**([style.rs](../../crates/tn-ui/src/style.rs)):新增 `FS_MICRO=11 / FS_CAPTION=12 / FS_BODY=13 /
  FS_LABEL=14 / FS_TITLE=18 / FS_HERO=26`。**最小档 10→11 微放大**(状态栏/芯片/kbd 更易读),
  9→10、8→9.5;并把散落的 11/12/13/14 字面量**等值替换**为 token(零视觉变化、消灭魔法数字)。

## 真机踩坑:gpui 0.2.2 打包字体当不了「回退」(已绕过)

首次真机 `cargo run` 刷屏 `ERROR ... No matching font found for Source Han Sans SC`。
根因:gpui 0.2.2 的回退构建 [`generate_font_fallbacks`](direct_write.rs:333) **只在
`system_font_collection` 里按族名查**,完全不看 `add_fonts` 注册的 `custom_font_collection`。
而**主字族**查找(`get_font_id_from_font_collection(is_system_font=false)`)会查自定义集 —— 这
正是 Inter/JBM/Space Grotesk 作为主字族生效、思源作为回退失效的原因。

结论:**打包的内存字体只能当主字族,当不了 `font_fallbacks` 回退**(除非 fork gpui)。
显示其实没坏(找不到就 `continue`,再 `AddMappings(系统回退)` 兜底 → 中文仍渲染成系统
雅黑),只是刷 ERROR 且拿不到打包思源。

**绕过**:移除打包的思源黑体 OTF(16.5MB,当不了回退即死重量),把 `CJK_FALLBACK` 指向
系统必装的 `Microsoft YaHei UI`(匹配得上 → 0 ERROR,中文 = 雅黑)。西文三族打包主字族
不受影响,未来感保留。装了更现代 CJK(MiSans/HarmonyOS Sans SC)可在 `CJK_FALLBACK` 前置
自动升级。若要打包思源真生效,唯一路径是在自绘 run 处按字符脚本拆分、把思源当**主字族**
喂给 CJK run(终端/编辑器/Markdown 各自改造,工作量大,暂未做)。

## 体积代价

新增字体净增 exe 约 +20MB(其中思源黑体 SC Regular 单权重 16.5MB 为大头;中文字体
本身体量如此,且需全字形覆盖动态文件内容,不能子集化)。已移除 CaskaydiaCove×4 抵消部分。
