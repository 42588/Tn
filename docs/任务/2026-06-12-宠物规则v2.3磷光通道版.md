# 宠物规则 v2.3 磷光通道版

## 定位

记录在 [宠物系统规则](../宠物/宠物系统规则.md) v2.2 基础上,将宠物与 Tn 关键 UI 转场绑定为标志性空间共演语言的文档任务。

## 背景

用户指出“终端联动”不只是 Running/Typing 等状态映射,而是宠物参与界面空间转场:打开幽灵终端时从主终端钻入地洞、从幽灵终端出现;打开 Quick Look 时由宠物推动或拉出预览窗口;切换时也应有空间响应。进一步定夺:这不是克制彩蛋,而应成为 Tn 与其他终端的分水岭。

最终气质选择为“魔法入口/空间钥匙”,而不是“搬运工/引路犬”。因此本轮规则命名为 **v2.3 · 磷光通道**。

## 具体内容

- [x] 在 [宠物系统规则](../宠物/宠物系统规则.md) 中新增规则 J「磷光通道(空间共演)」。
- [x] 定义三种空间动词:开门、穿梭、召回。
- [x] 细化标志性场景:幽灵终端打开/隐藏、Quick Look 打开/关闭/切换、命令面板/浮层、分屏、欢迎页进入工作区、SSH/远端重连。
- [x] 写入视觉纪律:只用磷光线、裂缝、光点、残影;必须依附真实 UI 接缝;宠物仍是像素小狗。
- [x] 写入节律与降级:完整共演、短版共演、静默换岗、禁止共演。
- [x] 新增实现模型 `PetSpatialCue`,明确它是 UI 转场提示,不是终端上下文状态。
- [x] 将实现分期扩展到 P5,并要求先出 SHEET 05 板 H 或「磷光通道」跨页图纸。
- [x] 同步 [宠物系统索引](../宠物/README.md) 与 [TODO](../../TODO.md)。

## v2.3 定夺

| 主题 | 规则 |
|---|---|
| 设计定位 | 宠物是 Tn 空间的钥匙,用磷光通道召唤关键 UI |
| 标志场景 | 幽灵终端、Quick Look、浮层、分屏、欢迎页/工作区、远端重连 |
| 动效材料 | 1px 磷光裂缝、光点、残影、真实 UI 接缝 |
| 禁止方向 | 烟雾、粒子风暴、大面积渐变、全屏遮罩、宠物变形、宠物分身 |
| 实现模型 | `PetSpatialCue` 由 UI 控制层显式发出,只携带 cue 类型和锚点 |
| 降级策略 | 高频操作短版/reduced motion 静态换岗/UI 失败时宠物安静退场 |
