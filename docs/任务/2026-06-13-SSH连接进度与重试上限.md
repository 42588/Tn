# SSH 连接进度与重试上限

## 目标

修复 SSH 初始连接过程中的两个问题:连接进度浮层内部元素超出面板并被截断;目标不可达时自动重试没有上限。

## 范围

- SSH 连接进度卡片、失败卡片和重连横幅的文本布局约束。
- SSH 初始连接失败时的自动重试预算。
- 不改变已经成功连接后的断线自动重连行为。
- 不改变认证失败、主机密钥变更和密码输入流程。

## 实现记录

- `crates/tn-ui/src/terminal_view/mod.rs`:SSH 浮层面板保持 460px 设计宽度并限制最大宽度为视口 92%;头部目标 chip、三段进度、当前步骤详情、认证提示和重连横幅都加 `min_w(0)` / `overflow_hidden` / `text_ellipsis`,避免长 `user@host:port` 或长错误信息撑出浮层。
- `crates/tn-ui/src/terminal_view/mod.rs`:进度步骤从带长连接线的横排改为三列等宽,保留图标状态和当前步骤详情,减少小窗口下的横向挤压。
- `crates/tn-ui/src/terminal_view/mod.rs`:网络失败和认证失败卡保留手动“重试”按钮;主机密钥错误仍只允许关闭,避免误操作。
- `crates/tn-pty/src/ssh.rs`:初始连接失败后最多自动重试 3 次;第 3 次重试仍失败时发送 `SshFailed(Network)` 并停止连接循环。
- `crates/tn-pty/src/lib.rs`:新增 `SshErrorKind::Network`,让 UI 区分网络连接失败、认证失败和主机密钥错误。
