# 任务: 小狗历史命令面板尺寸与边界定位修复

## 背景与目的

点击小狗打开历史命令面板时,历史命令过多会把面板撑得过长并被视口裁切。小狗处于窗口边界附近时,面板仍按固定方向展开,也会被左右或上下边界裁切。

本轮目标是让历史命令面板尺寸有确定上限、列表在面板内部滚动,并按小狗位置自动选择可见展开方向或夹紧到视口安全区。

## 范围

- 修改 `crates/tn-ui/src/workspace.rs` 中小狗历史命令面板的布局计算与渲染约束。
- 增加针对过多历史命令、左右边界和上下边界的回归测试。
- 不改变命令块数据来源、点击跳转、复制输出和时间戳逻辑。

## 进度记录

- [x] 定位当前实现与根因:弹层使用固定 `right/bottom` 锚点,并只设置 `max_h`,列表内容仍可能撑开布局;靠边时没有方向选择或视口夹紧。
- [x] 增加失败回归测试。
  - 首次运行失败: `PetHistoryLayoutInput` / `pet_history_popup_layout` 尚不存在。
  - 加入旧逻辑骨架后,测试失败于右边界弹层 `left < 12`,证明回归用例能捕获裁切问题。
- [x] 实现面板尺寸与定位修复。
  - 抽出 `pet_history_popup_layout`,统一计算 `left/bottom/w/h`。
  - 面板高度按内容、视口安全区和最大高度取交集;历史过多时固定面板高度并让内部列表滚动。
  - 水平方向优先选择能完整显示的一侧,否则按可用空间选择并夹紧。
  - 垂直方向按视口安全区夹紧,避免靠上/靠下被裁切。
  - 浮层和 scrim 吞滚轮,避免滚动历史面板时带动底层终端 scrollback。
- [x] 运行验证。
