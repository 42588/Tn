## P1 · 性格 + 活物引擎(规则 C/D)

- `Personality` 七犬常量表(表即实现):眨眼区间 / Idle 尾摆幅度 / 打盹阈值 /
  口癖 / 微动作权重 / 入场权重;替换原硬编码。德牧无口癖 → 单击/Play 点头代替。
- `Micro` 六动作活物引擎(抓痒 / 舔爪 / 追尾水平镜像 / 竖耳 / 望屏外 / 伸懒腰):
  纯 Idle 下 20–60s 随机加权触发、≤1.5s、不连续两次、≤2 次/分钟、离 Idle 弃帧、
  reduced motion 停用;全部复用既有 dx/dy/镜像,**零新网格**。

## P2 · 领养命名 / 每日见面 / 档案 / 共情(规则 0/A/E/F)

- **规则 0 身份**:首次启用 → 一次性领养卡(品种架 + 交给缘分 + 名字字段,
  ≤8 字、默认品种名、Esc 跳过);改名浮层;换个形象品种架;**随机刷新废除**。
  名字三处上屏(状态栏席位 / 档案行 / 欢迎页 2× 标签)。文本输入沿用命令面板
  同款 `track_focus + on_key_down` 累积(CJK 走 key_char)。
- **规则 A 每日见面**:本地自然日首现 → 入场四式(探头 / 小跑 / 天降 / 它先到了,
  性格加权、不重样)+ 时段问候(清晨/白天/傍晚/深夜)+ 第N天短显 5s;深夜问候
  补哈欠帧。本地日期/小时用 Win32 `GetLocalTime`(新增 Cargo 特性
  `Win32_System_SystemInformation`)。
- **规则 F 羁绊档案**:`pet_state.json` 增 first_met / days_together /
  last_seen_date / treats_fed;里程碑词 7/30/100/365 天。无奖励、无解锁。
- **规则 E 工作共情**:结构化事件派生(exit code + 运行时长 + `git commit/push`
  首词),零输出扫描。三连胜 / 大功告成(>60s)/ 翻盘 / 提交时刻装点 Success 演出
  (双 / 单爱心);≥3 连败 → 就近趴下陪着 +「呜…」一次(永不评判);共享冷却
  ≤2 次/10 分钟。信号经 `SessionRunGuard` + `io.rs` 的 `CommandLine` 接入。
- 把变化检测元组改为 `frame_fingerprint`(u64),绕过元组 12 元 PartialEq 上限。

## P3 · 小饼干(规则 B)

- 菜单首组「给小饼干 · 今日还有 1 块」→ 喂过置灰「今天吃过啦」;每自然日 1 块
  (`last_treat_date`),不囤积、不补偿、不提示;新的一天自动补货(每日见面)。
- 新增 `Feed` 上下文 + 四帧演出:饼干(4×4 #C99052 非磷光)抛物线入场 → 三帧律
  接住 → 双腮交替咀嚼 → 单爱心 + 快摆尾。深夜彩蛋:吃完哈欠 → 主动趴下。

## P4 · 主动互动 / 互动记忆 / 透明(规则 G/H/I)

- **规则 G**:摸头(长按 350ms 不移动 → 头低眯眼 + 爱心;拖拽改为需真正位移)·
  叫名字(菜单,10s 冷却)· 换窝安顿(松手记常用窝象限 + 落地回弹嗅地)·
  摆个玩具(球/骨头/小毯子/收起,岗台暖色非磷光)· 久别亲近(≥3 日 → 多一颗心,
  无愧疚文案)。
- **规则 H 互动记忆**:`pet_state` 增 fav_pat/feed/play/call(衰减计数)、
  favorite_perch(象限)、toy、quiet_hours_seen、last_affection_at;只记偏好不做画像
  (recency-weighted、固定上限)。
- **规则 I 透明/清除**:菜单显示「它记得:≤3 标签」/「还在认识你」;
  「重置互动记忆…」二次确认卡只清五项偏好,不动名字/品种/档案/窝坐标/开关。
