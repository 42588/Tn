# 窄面板活动栏自适应折叠

## 背景

工作中频繁分屏或手动拖小窗口。agent 面板正文右侧的活动栏「本次改动」(代码里的 activity rail)
设计稿固定 `width:248px; flex:none;`,而终端正文是 `flex_1 + min_w:0`。pane 一窄,正文被压到 0、
rail 仍死守 248px,视觉上「吃掉/遮挡」了 agent 主内容。设计稿和踩坑记录此前对窄面板最小宽度无约定。

所有 pane 都会遇到此问题(分屏越多、窗口越小越明显)。

## 方案

自适应折叠(用户确认):pane 太窄到正文不可用时,**直接收起 rail**,把整宽让给 agent 正文;
拉宽后自动恢复并排。正文为主、rail 为辅。

关键约束:门限必须基于「与 rail 存隐无关的稳定宽度」=整 pane 宽度,而非被 rail 切剩的正文宽度,
否则会逐帧抖动(隐藏→正文变宽→又达标→又变窄)。

## 实现

`crates/tn-ui/src/terminal_view/mod.rs`:

- `RAIL_MIN_PANE_W = 560.0`:pane 内容宽度低于此值就收起 rail(248px 外壳 + 舒适正文下限)。
- `rail_layout_for_width(base, pane_w)`:纯函数 gate,`base != None && pane_w < 阈值 → None`,否则 `base`。
- `TerminalView.pane_width: Rc<RefCell<f32>>`,默认 `f32::MAX`(未测量→保留 rail,首帧不闪)。
- pane 外层 div 加 `.relative()` + 背景 `canvas` 抓自身 `bounds.size.width` 写入 `pane_width`;
  外层本就经 `plate()` 处于 `relative` 链上、且 SSH/sidecar 浮层卡是 `absolute().size_full()`,
  改成锚定到同尺寸的外层 div,无视觉位移。
- render 里 `activity_rail_layout(...)` 外再包一层 `rail_layout_for_width(.., *pane_width)`;
  Flex(launch-intent)和 Overlay(shell-detected)两种 rail 都受门限约束。

## 验证

- `cargo test -p tn-ui --lib rail`:3 passed(含新增 `rail_collapses_on_narrow_pane_and_restores`
  覆盖 宽→保留 / 窄→None / 本就 None / 未测量 f32::MAX→保留)。
- 整 crate 编译通过。
- 真机分屏拖动观感待用户确认;阈值 560 可按手感再调。

## 后续
