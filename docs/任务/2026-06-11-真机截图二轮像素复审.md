# 真机截图二轮像素复审(差异总结全量重写)

## 背景

用户更新了 `design/真机截图/` 全套样张(26 张,按页面命名),并清空了 `design/原型与真机截图差异总结.md` 要求重写,重点为幽灵终端与 QuickLook。复审期间用户追加三问:① 幽灵终端下方"透明材质"是什么、为何隐藏时全暴露;② SSH/WSL/添加 Agent 子页风格完全不统一;③ 宠物系统规则与交互未落实到位。

## 做法

- 对照源:`design/phosphor.css` token + 各 SHEET SPEC;对全部样张做无损逐像素采样(PowerShell + System.Drawing,色带 RLE + 定点采样)。
- 几何按 ~147% DPI(×1.4737,由幽灵窗物理 1120 ÷ 逻辑 CARD_W 760 标定,与外壳各条带交叉验证)折算成逻辑 px 再比 SPEC。
- 代码回溯:`quick_terminal.rs`(残影/透明窗)、`welcome.rs`(子页)、`pet.rs`(规则核对)、`style.rs`/`tn-config`(token 与主题加载链)、真机用户目录主题文件。

## 关键结论(详见 design/原型与真机截图差异总结.md)

1. **根因**:真机加载的是用户目录残留的旧 Tokyo Night 主题(`%APPDATA%\tn\themes\tn-dark.toml`,实测 surface_1=#1F2335、accent=#7AA2F7 等),`tn-config/src/lib.rs:90` 的 `write_if_absent` 永不覆盖 → 磷光主题从未到达真机;style.rs 常量驱动的部件(幽灵窗/brand mark/焦点边/磷光读数等)全部正确 → 一屏两个时代。
2. **幽灵"透明材质"**:`WindowBackgroundAppearance::Transparent`(lib.rs:344)在真机未生效,残影区 16px 露出纯白 #FFFFFF surface;隐藏滑出时白 surface 整块暴露。
3. **QuickLook**:无 scrim、浮层挂焦点 pane 而非整工作区、头 #283457/File 正文 #10141F/Diff 正文 #1F2335 三面三色、编辑态块光标 #7AA2F7(应磷光反相)。
4. **几何全面达标**:DPI 折算后标题栏 42/状态栏 26/板头 34/浮层头 38/页脚 30/磁贴 92/接缝 2/命令面板 640 全部 ±1px 命中 SPEC。
5. **子页不统一**:三种交互模式均有原型出处,丑在视觉件失配(三处表面色/选中态/页脚/chip 各自为政);SSH 浮层是全系统唯一 token 全对的浮层(L3/L4 精确)。
6. **宠物**:工作区 1× 宠物规则覆盖率高(状态机/眨眼/小跑/蹦跳/委屈/拖拽/右键全有);缺口 = 欢迎页 2× 是静态贴图(welcome.rs:745 sprite_block,无任何交互)、标签硬编码 IDLE 与状态栏 RUNNING 同屏矛盾、双击设置与命令面板入口未做、NO SESSION 下 RUNNING 状态泄漏嫌疑。
