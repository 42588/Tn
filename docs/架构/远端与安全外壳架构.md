# 远端与安全外壳架构

## 定位

本文承接原 `docs/系统架构索引.md` 中 WSL、SSH backend、SSH UX、SFTP 远端文件、远端 git 和远端文件命名空间边界。实现细节优先链接 [远端文件服务与改动流](../新增模块/远端文件服务与改动流.md) 和 `docs/新增模块索引.md` 的 SSH 专题。

## 背景

Tn 的远端能力分两层:

- 终端会话层:本地 ConPTY、WSL 和 SSH 都实现为可读写的 PTY-like backend。
- 文件/命令层:Host、WSL、SSH 都有自己的文件命名空间;SSH 远端文件和 git 操作必须通过 SFTP/SSH exec,不能退回本机路径。

## 具体内容

### PTY backend 抽象

`PtyBackend` trait 提供统一同步接口:

- `resize`
- `take_reader`
- `writer`
- `killer`
- `wait`
- `try_wait`

TerminalView 驱动层只面对 trait,不关心后端是 ConPTY、WSL 进程还是 russh channel。

### 本地 ConPTY

`LocalPty` 经 `portable-pty`:

1. `openpty`
2. `spawn_command`
3. `try_clone_reader`
4. `take_writer`

关键约束:

- Windows 上必须保留 slave 句柄,过早 drop 会扰动伪控制台。
- 子进程退出用 `try_wait` 轮询,不能只等 EOF。
- `LocalPty::spawn` 清理 VS Code/Cursor 注入的 IDE 变量,并合并当前 Tn 进程 PATH 的额外 shim 目录,保证 `codex.ps1` 等 npm/cargo shim 在 Tn 子 shell 中可解析。

### WSL

WSL 不需要专属 backend。Tn 使用 ConPTY 托管:

```text
wsl.exe -d <distro> --cd ~
```

相关规则:

- `wsl.rs::parse_distros` 解码 `wsl -l -q` 的 UTF-16LE 输出,枚举发行版。
- WSL 文件命名空间独立于 Windows cwd。
- Explorer 通过 `ExplorerRoot::Wsl` 把 Linux cwd 显式映射到 `\\wsl$\<distro>\...`。
- 显示 Linux dotfiles,不把 `/home/...` 映射成宿主盘路径。

### SSH backend

`SshBackend` 使用 russh。russh 给 async channel 而非 fd,所以 Tn 用专属线程跑 current-thread tokio,再通过 `select!` 把 `ChannelMsg::Data` 与同步 `std::mpsc` 桥成 Read/Write。

连接链路:

1. connect。
2. 主机密钥检查。
3. `authenticate_none` 探测服务器开放认证方式。
4. key。
5. password。
6. keyboard-interactive,复用同一密码。
7. request pty。
8. request shell。
9. `window_change` 处理 resize。
10. keepalive / drop 断开处理。

已真机端到端验证:2026-06-05,密钥 `id_ed25519` + 交互 shell 登录 AlmaLinux 9.7(WSL)。仍需真机肉眼复验的路径包括密码框、TOFU 首连、断线重连横幅、进度/错误/指纹卡、ssh-config 行和拨号校验。

### SSH 安全与输入边界

SSH 覆盖卡是 PTY 输入闸门。指纹、密码、错误、进度等覆盖卡存在时:

- 键盘不写 writer。
- 粘贴不写 writer。
- 重跑命令不写 writer。
- alt-screen 滚轮输入不写 writer。
- 进度卡除 `Esc` 取消外吞键。

`SshBackend::run_session` 在 `request_shell` 成功后、发送 `Connected` 前会 drain pending input,清掉覆盖层期间可能排队的字节,避免连接后 replay 到远端 shell。

TOFU 规则:

- 未知主机发 `NeedHostKeyConfirm` 给 UI。
- 用户可 AcceptOnce 或 AcceptAndSave。
- AcceptAndSave 写 known_hosts 失败时拒绝连接并显示错误,不静默 best-effort。
- 主机密钥不匹配默认危险中止;「仍信任并更新」属于低优打磨。

### SSH UX 架构入口

SSH 连接 UX 从单行输入升级为三轴:

- A 交互方式:最近连接、收藏、中文昵称、ssh-config alias 枚举、实时解析。
- B 界面:连接进度卡、TOFU 指纹卡、密码框、断线横幅、连接态胶囊。
- C 提示:可操作错误卡、拨号前校验、首连引导和成功反馈。

详细逐项实现仍保留在 [新增模块索引](../新增模块索引.md) 的 SSH 章节;本文件只记录架构入口和边界。

### 远端文件服务

`tn-pty::remote_fs` 提供 SFTP v3 后端:

- `RemoteFileService`
- `SftpFileService`
- 列目录。
- 有界读文件。
- stat 元数据。
- guarded write-back。

SFTP 后端复用 SSH 配置、密钥和内存密码;后台探测不弹 host-key/password UI。

`ExplorerRoot::Ssh` / `ExplorerPath::Remote` 用 `RemoteId` 表示远端路径。焦点 SSH pane 的 cwd 可以驱动左侧远端文件树;Quick Look 打开远端文件时走 `open_remote`,最多读取远端读限制。文本保存前用远端 stat + 内容 hash guard 检测冲突。

完整实现见 [远端文件服务与改动流](../新增模块/远端文件服务与改动流.md)。

### 远端 git 与 hunk

远端 git 不使用本机 git watcher:

- `remote_git` 通过 SSH exec 执行 `git diff --numstat`。
- Agent 活动栏可显示 SSH workspace 改动。
- Quick Look 可打开远端 full diff。
- hunk accept/reject 通过远端 `git apply -` 或 `git apply --reverse -`,patch 走 stdin。
- 成功后刷新 diff、刷新每个 pane 的「本次改动」并标记 explorer stale。

仍需端到端回归:hunk 按钮真改远端 working tree、`git apply` 拒绝补丁失败文案、并发/超时。

### 目录 picker

- Host / 欢迎页使用本机 native picker。
- SSH pane 打开当前远端 root 的应用内 SFTP 目录 picker。
- WSL 通过 `\\wsl$\<distro>` 的本地 `std::fs` 列目录。
- 确认后只向目标 pane 发送 `cd`,不广播到其它 pane。
