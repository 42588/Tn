# 终端渲染与界面宿主

## 定位

本文承接原 `docs/系统架构索引.md` 中 `tn-ui`、TerminalView、GPUI 前端、渲染方式、布局、文件树、Quick Terminal 和 TerminalElement 后续方向。样式真源和实现规则见 [界面样式实现规则](../界面样式实现规则.md)。

## 背景

`tn-ui` 是唯一链接 GPUI 的库。它把 headless crate 产出的终端快照、配置、Agent 事件、远端文件和编辑器模型组织成 Tn 的窗口、分屏、侧栏、状态栏、Quick Look 和各种覆盖卡。

## 具体内容

### TerminalView

`TerminalView` 是核心渲染实体,主要持有:

- 共享 `tn_core::Terminal`
- PTY writer 和 `Box<dyn PtyBackend>`
- focus handle
- 当前 grid size
- 实测 `cell_width` / 行高
- scrollback、选择、鼠标、IME 和输入状态
- agent、SSH、远端文件、活动栏和 overlay 状态

启动路径:

1. `new` 根据 `LaunchSpec` 创建本地、WSL 或 SSH backend。
2. spawn reader 线程读取 PTY。
3. reader 把字节交给 `Terminal::advance`,并回写 `PtyWrite`。
4. reader 发 wake 到前台,由 GPUI 合并重绘。
5. `render` 做 resize、snapshot、row_runs 和前端布局。
6. `on_key` / IME / mouse 把输入事件写回 PTY 或交给 overlay。

`LaunchSpec::file_namespace` 区分 Host / WSL / SSH。只有 Host cwd 可作为 Windows 进程 cwd;WSL 和 SSH 的文件树必须走显式 namespace,不能把远端路径落成本机 `PathBuf`。

### 当前前端能力

已落地能力:

- 每格颜色、Palette、run 批处理样式盒。
- 选择、复制、粘贴、滚动历史、右缘滚动条和可见光标。
- 标签页、n-ary 平铺分屏、键盘改尺寸、分隔线拖拽。
- 命令面板、文件树侧栏、Quick Look 文件/Diff 速览。
- 自绘集成标题栏、多段状态栏、Quick Terminal。
- 默认启动入口收敛为通用 `Agent` profile + shell / WSL / SSH,不再内置 Claude/Codex 双磁贴。
- Agent 活动栏、真实 git diff 改动卡、shell 内敲 `claude`/`codex` 的 agent surface 切换。
- SSH 进度、错误、密码、指纹信任、断线横幅和连接态信息胶囊等覆盖卡。

对应用户可见体验见 [产品体验索引](../产品体验索引.md)。

### Workspace 与 per-pane 上下文

Workspace 负责:

- 标题栏、标签、分屏树和焦点收敛。
- app 菜单、命令面板、分屏启动器和布局管理器。
- SSH 连接器和远端目录 picker。
- Agent 编辑器、launchpad 和自定义 agent profile 刷新。
- 文件树显示哪个 pane 的 Explorer 状态。

per-pane 工作区上下文:

- `ExplorerView` 仍是单一渲染实例。
- Workspace 按 `PaneId` 保存 `ExplorerSnapshot`,并用 `explorer_pane` 记录当前展示 pane。
- 跨 pane 聚焦时保存旧 pane 展开态/选中项,恢复新 pane 状态。
- 同 pane `cd` 走 `follow_root`,保留子目录展开态。
- `打开文件夹` 只影响当前焦点 pane;agent pane 不被本机 picker `cd`;SSH pane 走远端目录 picker。

细节见 [每窗格工作区上下文](../修复与优化/每窗格工作区上下文.md)。

### Agent 活动栏和正文布局

Agent 体验由 TerminalView 和 Workspace 协作:

- launched agent 的正文与活动栏是 side-by-side 布局。
- shell 内敲 `claude` / `codex` 识别出的 agent 走 overlay 活动栏,避免命令运行中改变终端宽度。
- agent surface 拥有 viewport,隐藏 shell block bar;滚轮交给 agent/TUI 程序。
- 活动栏「本次改动」读取真实 `git diff HEAD --numstat`,不解析终端正文。
- 变化通过 `notify` watcher 防抖刷新,idle 零唤醒。

正文显示和 PATH 修复见 [智能体活动栏与正文显示](../修复与优化/智能体活动栏与正文显示.md)。

### Quick Terminal

Quick Terminal 是 `tn-ui` / `tn-app` 侧的窗口能力:

- 全局热键。
- 无边框置顶 PopUp 窗口。
- 边缘滑入和失焦自动隐藏。
- 启动器和紧凑 SSH 连接器。

当前状态:M5 已落地,headless 路径可测;真机肉眼体验仍需 owner 持续复验。

### 渲染方向

当前终端正文仍以 GPUI div/text + run 批处理为主。后续规划是自定义 `TerminalElement`:

- 使用 GPUI `Element` 的 layout / prepaint / paint。
- 使用 typed quad 和字形图集。
- 背景、光标、选区、连字、box-drawing 分离绘制。
- 按 run 整形,fg/bg 分离。
- 使用 GPUI 自管图集和 DirectWrite,不写裸 D3D。

该方向参考外部资料中的 Ghostty AtlasEngine 和 Windows Terminal 渲染经验。入口见 [外部参考资料索引](../外部参考资料索引.md)。
