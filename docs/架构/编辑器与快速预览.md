# 编辑器与快速预览

## 定位

本文承接原 `docs/系统架构索引.md` 中 `tn-editor`、Quick Look、File/Edit/Diff、保存守卫、自绘编辑器和原型同步轨道的架构信息。近期实现和修复细节优先维护在 [快速预览编辑器近期变更](../修复与优化/快速预览编辑器近期变更.md)。

## 背景

Quick Look 从文件/Diff 查看器演进为贴树浮层,并逐步把编辑核心从 `tn-ui` 下沉到 headless `tn-editor`。目标是让预览、编辑、Diff Review 共享可测模型,同时保持文件浮层不占分屏、不改变终端主体布局。

## 具体内容

### `tn-editor` headless 核心

`tn-editor` 是无 GPUI 依赖的编辑核心。已提供:

- `Cursor`
- `TextRange`
- `Document`
- `Selection`
- `CursorSet`
- `EditTransaction`
- `UndoStack`
- `SearchState`
- `line_layout`

迁移自 Quick Look 的纯函数包括:

- `char_to_byte`
- `line_chars`
- insert / newline / backspace / delete
- move / page
- delete range
- insert multiline
- all matches / selected text 等编辑与查找模型

契约:

- 文档模型保持 `Vec<String>` + char-column。
- Undo/redo 使用行区间 snapshot,不存整 buffer。
- 连续打字按行合并,移动光标后断开。
- Quick Look 和 Editor Pane 通过 `tn-ui::editor::session::DocumentSession` 共享 `Document`、行镜像、cursor、selection、undo/redo 和 dirty 状态。

### Quick Look 浮层

Quick Look 的产品形态:

- 文件树选中后弹贴树浮层。
- 覆盖终端,不占分屏。
- File / Diff tab 切换。
- 行号槽、语法/增删着色、左缘 accent 指向选中文件。
- 键帽提示在浮层底部。
- 关闭后焦点还给终端。

入口:

- 点击文件树。
- 文件树焦点下 `Space` / `Enter`。
- Agent 活动栏改动卡。

### 预览态

预览态能力:

- 上下切换文件并跟随 Explorer 或 activity rail 上下文。
- File/Diff tab 切换。
- Diff tab Enter / 双击跳回 File tab 对应行。
- PgUp/PgDn 跳上/下一个 hunk。
- Esc / Space 收起。
- 只读选区、复制、全选。
- CJK 双宽 hit-test 和横向滚动。

### 编辑态

编辑态能力:

- 打字、Backspace、Delete、Enter、Tab。
- 方向键、Home、End、PgUp、PgDn。
- Shift 选择。
- Ctrl+C/X/V。
- Ctrl+Z / Ctrl+Y / Ctrl+Shift+Z。
- Ctrl+A。
- Ctrl+F / Ctrl+H 查找替换。
- Ctrl+S 保存。
- 中文 IME 合成和候选框定位。

本地保存守卫:

- 打开时记录 `TextFormat` 和 `FileGuard(mtime/size/hash)`。
- 保存前对比当前磁盘 guard。
- 冲突进入「重新载入 / 取消 / 覆盖保存」。
- 正常保存保留原编码、换行风格和 final newline。

远端保存守卫:

- SFTP stat + 内容 hash guard。
- 冲突时显示远端版本的 reload/cancel/overwrite。
- 不把远端路径落成本机 `PathBuf`。

### Editor Pane

TnE-19/20 新增长期编辑 pane:

- Quick Look 通过 `EditorHandoff` 把当前 `DocumentSession` 交给 Editor Pane,不是复制文本。
- 预览态升级会从当前文本行创建 clean session;编辑态升级直接共享已有 session,因此 buffer、cursor、undo 和 dirty 连续。
- Workspace 仍保留 `panes: HashMap<PaneId, Entity<TerminalView>>` 作为终端/agent pane 主表,并新增 `editor_panes: HashMap<PaneId, Entity<EditorPane>>`。pane tree leaf id 可指向 terminal 或 editor;terminal 的 cwd、Explorer per-pane 状态、agent activity rail 逻辑继续只遍历 terminal panes。
- Editor Pane 初版提供基础输入、撤销/重做、复制/粘贴、状态栏、本地保存和 dirty close prompt。关闭 pane、关闭 tab、应用退出和标题栏 / Alt+F4 都通过 `EditorCloseIntent` 记录待处理动作,在 pane 内显示“保存 / 放弃 / 取消”,确认后再续跑关闭或退出流程。Quick Look 格式保持与冲突保存模块共用仍留后续收敛。

当前架构边界:

- Editor Pane 是实验性轻量编辑宿主,不是完整 IDE 实现。
- 当前职责是承载长驻 pane、基础本地文本编辑、状态栏和 dirty close 保护。
- 还未承担完整 Quick Look 体验等价职责:同源编辑表面、真实光标/selection/IME 手感、格式保持、外部变更状态机、保存冲突 guard、Markdown 预览和终端往返。
- Phase 1 的架构重点是把 Quick Look 与 Editor Pane 收敛到统一文档会话、统一保存 guard 和同源编辑表面;Phase 3/4 完成终端桥接和工作集前,不应把该架构描述为轻量 IDE 已落地。

### Diff Review

Diff tab 架构:

- Diff 行分类为 hunk header、addition、deletion、context、meta。
- `+++` / `---` 文件头与 hunk 内正文区别处理,避免行号漂移。
- 删除行跳回 File 时落到同 hunk 内最近存活 new-file 行。
- 只读选择和复制复用 File/Edit 的 CJK + 横滚 hit-test。
- 远端 diff 的 hunk accept/reject 通过远端 git apply 服务。

Diff 细节和修复见 [快速预览编辑器近期变更](../修复与优化/快速预览编辑器近期变更.md) 与 [远端文件服务与改动流](../新增模块/远端文件服务与改动流.md)。

### 自绘编辑器方向

自绘路径逐步替代旧 `uniform_list`:

- `tn-ui::editor::geometry` 提供列宽、光标、命中、横滚、visible rows 等纯几何模型。
- `tn-ui::editor::prepaint` 提供只读布局模型。
- `tn-ui::editor::diff` 提供 DiffRowKind 和 hunk 导航模型。
- File 预览和编辑器默认走自绘路径;`TN_QL_LEGACY=1` 可回退旧 renderer。
- Diff 仍保留部分旧路作为逃生口。

`tn-editor::line_layout` 提供 logical -> visual 映射和词换行模型。Quick Look File/Edit 自绘路径已按 `WrapMode` 接入:Markdown/txt/log/说明文档使用 visual line 软换行,代码文件继续使用横向滚动。

### 原型同步轨道

Quick Look、welcome launchpad、agent 活动栏、app 菜单、per-pane 工作区上下文等视觉原型来自:

- `design/panels/`
