# 智能体宿主平台

## 定位

本文承接原 `docs/系统架构索引.md` 中 `tn-agent`、`tn-ai`、Agent Host、sidecar 遥测、shell agent 识别、block chrome 让位和运行时边界。实现明细优先维护在 [智能体宿主平台化](../新增模块/智能体宿主平台化.md) 与 [应用内智能体编辑器](../新增模块/应用内智能体编辑器.md)。

## 背景

Tn 曾经把 Claude/Codex 当作闭合枚举和 UI 特判处理。Agent Host 平台化后,`tn-agent` 对具体 agent 零知识,Claude/Codex 降为 `tn-ai` 中的可移除内置 adapter。这样在接入 Gemini、Aider 或其它 CLI/sidecar agent 前,不需要再把全 UI 串上一遍 `match` 分支。

## 具体内容

### 平台核心

`tn-agent` 是 headless 平台 crate,核心类型:

- `AgentId`:开放字符串身份,替代闭合 `AgentKind` 枚举。
- `AgentDescriptor`:身份、命令、颜色、标签、能力、运行时和实时通道描述。
- `AgentCapabilities`:usage、git diff、cursor ownership 等能力插槽。
- `AgentRuntimeKind`:PTY、本地结构化、remote daemon、HTTP、WebSocket 等运行位置。
- `AgentEvent` / `AgentStatus`:UI 唯一输入契约。
- `AgentAdapter`:把私有格式或实时通道归约成统一事件。
- `GenericAdapter`:有身份、无遥测的配置级 agent。
- `ExternalEventAdapter` / `ExternalProcessAdapter`:JSONL/stdout 实时事件入口。
- `AgentRegistry`:按 id/命令解析 agent;空 registry 表示纯 shell 宿主。
- `AiUsage` + pricing:通用用量契约,从 `tn-ai` 上移。

层次铁律:

```text
tn-config <- tn-agent <- tn-ai
```

平台层不认识 Claude/Codex;认 Claude/Codex 的代码只准在 `tn-ai`。

### 内置 adapter

`tn-ai` 当前保留:

- `claude.rs` / `codex.rs`:解析本地会话 JSONL。
- pricing 和上下文窗口解析。
- `detect.rs::resolve_pane_session(&dyn AgentAdapter)`:按 launch 后 stale -> fresh 绑定 pane 会话。
- `builtin_registry()`:构建 Claude/Codex 种子 provider,但默认 app 不自动注册内置 agent。
- `builtin_adapter_for_manifest`:用户 config `[[agents]]` 命令命中 claude/codex 时,可用用户的名称/颜色 + 内置解析能力。

这样用户可以通过应用内 Agent 编辑器或 TOML manifest 添加 `claude` agent,同时获得真实用量环,但平台本身仍保持 agent-agnostic。

### 配置 manifest 与应用内编辑器

用户通过 `[[agents]]` manifest 声明 agent:

- 名称、命令、颜色和 profile。
- `runtime_support`。
- `allow_network`。
- `sidecar = "cmd"`。

应用内 Agent 编辑器把「加 agent」从手编 `config.toml` 升级为 UI 交互:

1. 欢迎页 launchpad 的「+ 添加 Agent」磁贴打开居中浮层。
2. 用户填写名称、命令、颜色和高级 sidecar 字段。
3. `tn_config::append_agent` / `remove_agent` 写 `[[agents]]` + `[[profiles]]`,块级保注释。
4. `reload_agents` 重读配置,重建 `AgentRegistry` global 和 launchpad。
5. 无需重启即可生效。

架构意义是编辑器只产出 config 数据,不把具体 agent 知识写回 `tn-agent` / `tn-ai`。

### sidecar 实时层

`[[agents]] sidecar = "cmd"` 映射到 `AgentDescriptor.realtime_command`。launched agent pane 根据 `AgentDescriptor::sidecar_launch()` 决定是否启动 `ExternalProcessAdapter`:

- 本地 stdio sidecar 可以直接启动。
- networked sidecar 默认拒绝。
- `allow_network = true` 只进入 `AgentNetworkPolicy::Ask`,需要宿主层用户确认。
- 每个 pane 拉起自己的 adapter,由 `TerminalView.realtime_adapter` 持有;clear agent 或 drop 时杀子进程。
- stdout JSONL 转 `AgentEvent`,进入同一个 `reduce_agent_event` 漏斗。

这使配置声明的 agent 也能有真实状态、transcript、权限和用量遥测,无需内置 adapter。

当前原则性不做纯 terminal-less pane:没有本地 PTY 的 HTTP-only agent 需要重写 TerminalView 内核,且缺少真实 agent 可测,暂属投机。

### UI 归约与能力 gate

`tn-ui` 只消费 `AgentEvent` 和 descriptor:

- `StatusChanged`
- `ModelChanged`
- `TranscriptAppended`
- `PermissionRequested`
- `ErrorReported`
- `UsageUpdated`
- `CwdChanged`

这些事件进入同一个 reducer。agent 头部、活动栏、用量环、权限 chip、错误提示和 transcript 摘要都由 descriptor/capability gate 控制,而不是具体 agent 分支。

`CwdChanged` 只更新目标 pane 上下文,不碰全局 Explorer。

### 运行时与文件命名空间边界

必须区分:

- `AgentRuntimeKind`:进程或协议运行位置。
- `FileNamespace`:文件 I/O 命名空间。

SSH runtime 的文件 I/O 只能经 SFTP-backed `ExplorerRoot::Ssh` / `RemoteFileService`,不得落回本机 `PathBuf` 或本机 git watcher。WSL 文件命名空间也不能把 `/home/...` 当宿主盘路径。

### shell 内 agent 识别

用户在普通 shell 中敲 `claude` / `codex` 时,Tn 通过 shell integration 运行命令首 token 识别 agent:

- 命令运行时切 agent 态。
- 命令结束后恢复 shell。
- 不做脆弱进程树轮询。
- shell-detected agent 的 activity rail 走 overlay,避免运行中改变终端正文宽度。
- overlay 宿主必须保持 flex-column,否则终端正文会只剩 agent 外壳。

该修复见 [智能体活动栏与正文显示](../修复与优化/智能体活动栏与正文显示.md)。

### block chrome 让位

`tn-blocks` 仍负责普通 shell 的 Warp block 模型。但 alt-screen 或 agent surface 拥有 viewport 时,block chrome 必须隐藏并让程序全幅渲染。Codex/Claude 这类 TUI 不一定进入 alt-screen,只要 pane 被识别为 agent,也不能让 shell block bar 或 scrollback chrome 抢 viewport。
