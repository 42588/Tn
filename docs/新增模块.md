# 新增模块与功能记录

## SSH 远程连接支持 (M2b)

### 1. 已实现功能
- **`~/.ssh/config` 解析支持**：使用 `ssh2-config` crate 读取本地 SSH 配置文件，自动提取指定主机的 `HostName`、`Port`、`User`、`IdentityFile` 等配置，并与用户的 Tn 配置相融合。
- **TOFU (Trust On First Use) 与公钥安全验证**：通过 `russh::keys::check_known_hosts_path` 验证公钥。首次连接未知主机时自动将其写入 `~/.ssh/known_hosts`。如果密钥发生变动（潜在的中间人攻击），主动拦截并渲染巨型的红色警告框终止连接。
- **动态密码输入交互 (Calm Glass 风格)**：当秘钥验证失败且配置中未提供密码时，采用 GPUI 框架渲染极具现代感和主题化（冷灰渐变质感、完美居中、毛玻璃阴影）的 Calm Glass 悬浮密码输入框。输入框能够安全地拦截用户的终端按键并捕获密码，实现遮罩显示。
- **自动断线重连与超时控制**：实现了网络状态感知的会话重连机制。为 TCP 连接增加了 15 秒超时封装，彻底避免死锁挂起。连接断开时倒数并在指定时间内（5秒）循环自动重连。
- **终端本地化中文彩色反馈**：彻底淘汰了后端的英文黑盒日志，所有的前置消息（如尝试连接、验证密钥、请求密码等）已被中文化，并辅以青、绿、黄、红四色 ANSI 终端高亮，直接在终端视图上流式输出，直观清晰。
- **认证方式探测 + 密码双通道（password / keyboard-interactive）**：认证前先用 `authenticate_none` 探测服务器实际开放的认证方式并在终端打印（等价 `ssh -v` 的 "Authentications that can continue"），盲调时一眼看清。拿到密码后**同时尝试 `password` 与 `keyboard-interactive` 两种方法**（同一密码）——大量 Linux/PAM 服务器（含 WSL 自带 sshd，尤其 root）**只接受 `keyboard-interactive`**，仅发 `password` 会"密码正确仍认证失败"（OpenSSH 客户端自动回退掩盖了差异）。若服务器根本未开放任何密码类方法，错误信息会直接点名"仅支持 X，请检查 sshd_config 的 PasswordAuthentication / PermitRootLogin"。

### 2. 技术细节
- **前后端事件通信桥梁**：
  在后端的 `tn-pty/src/ssh.rs` (`PtyBackend` 的 `SshBackend`) 与前端的 `tn-ui/src/terminal_view/mod.rs` (`TerminalView`) 之间，建立了基于 `std::sync::mpsc` 与异步 waker 结合的事件通讯通道（Event Channel）。
  定义了 `PtyEvent::NeedPassword`，让异步的 SSH 协程可以中断并挂起，请求 UI 线程获取用户输入密码。UI 线程中利用了 GPUI 的 `cx.notify()` 与 `update` 打破线程壁垒，并通过 `PtyEvent` 的返回通道（reply channel）回传用户的明文密码，最终被 `russh` 进行 `authenticate_password`。
- **渲染循环与按键拦截**：
  在 `TerminalView::render` 注入悬浮输入框结构。当存在挂起的密码请求时，修改 `TerminalView::on_key`，拦截所有的终端按键流转（阻止冒泡），捕获用户的密码明文，同时支持退格和回车提交。
- **会话协程重构**：
  将 `run_session` 进行了循环包裹改造（loop）。利用 `tokio::select!` 监听来自 PTY `out_rx`、`kill_rx` 以及 `channel.wait()` 等数据流，明确区分 "网络层面断开"（自动进入 5s 延时重连）和 "应用主动要求关闭/Eof" (顺利结束线程)。

### 4. 待实现/未实现功能

> ✅ **更新（2026-06-05）**：SSH 已**首次真机端到端验证**——Tn 以**密钥认证（id_ed25519）+ 交互 shell** 成功登录 AlmaLinux 9.7（WSL）的 sshd。认证前 `authenticate_none` 打印的"服务器支持的认证方式"与 `ssh -v` 完全吻合，正是它帮我们定位到失败是服务器侧 `PermitRootLogin no`（drop-in 覆盖主配置），而非 Tn 缺陷。以下"待实现"项是**真机仍未走过的具体路径**。

- **真机仍未走过的路径**：密码**提示框 UI**（本次走的是密钥，未触发 `NeedPassword` 浮层）；`keyboard-interactive`（测试服务器未开放该方法）；**断线重连 UX**；**TOFU 首连警告**（本次主机已在 `known_hosts`，`check_server_key` 的写入/不匹配分支未被触发）。

## SSH 快速连接交互面板 (M2b)

### 1. 已实现功能
- **全局动态输入面板 (Quick Connect)**：在欢迎页（Welcome）、命令面板（Palette）以及分屏操作（Split）中去除了“即将支持”的占位符，落地了真实的 `LaunchRow::SshPrompt` 交互条目。
- **极致的 GPUI 视觉体验与动态居中**：面板复用了原生命令面板的“极简玻璃态（Calm Glass）”设计。更为精细的是，抛弃了单纯的全屏居中，引入了**工作区感知（Workspace-Aware）动态偏移居中**。通过读取左侧文件树（Explorer）的宽度与工作区缝隙，精确计算 `padding-left`，使得弹窗能够像磁吸一样，绝对对齐于用户当前主编辑区（欢迎页本体）的视觉正中心。
- **独立安全的焦点接管与交互闭环**：支持键盘无缝捕获，ESC 退出与 Enter 提交。在交互层面，实现了点击外围遮罩安全关闭，而点击面板本身则通过严格的事件阻断器（`stop_propagation`）配合主动焦点续签（`focus(w)` + `cx.notify()`），彻底解决了“按键穿透无反应”、“鼠标点不到面板”的焦点剥离问题。

### 2. 技术细节
- **UI 动态位置偏移计算**：
  在 `render_ssh_prompt` 渲染时，读取 `self.explorer_open` 状态和 `self.explorer_width`。当文件树展开时，通过设定外层容器的 `pl(px(ew + 23.))` 和 `pr(px(12.))`，利用 Flex 布局剩余空间划分机制，自动算出工作区的真正物理中点并垂直、水平双居中面板（代替原有的 `110px` 固定顶部悬浮）。
- **事件流与 Focus 强制保活**：
  在 `Workspace` 中保留 `ssh_prompt_focus`。为了防止 GPUI 焦点穿透至底层终端或因拦截事件导致丢焦，给输入面板本体增加独立的 `on_mouse_down`：
  ```rust
  // 核心阻断与保活逻辑
  cx.stop_propagation();
  this.ssh_prompt_focus.focus(w);
  cx.notify();
  ```
  这保证了面板对鼠标点击的百分百独占响应并维持打字光标活跃。
- **Intent 解析路由 (SshPromptIntent)**：
  利用 `SshPromptIntent` 状态机记录用户的操作来源（新建标签页 `Welcome`/`Palette` 还是分屏 `Split`）。用户按下 `Enter` 提交 `user@host:port` 时，直接实例化动态 `LaunchSpec` 交给 `spawn_pane_with`，全生命周期无缝连接 SSH 终端后台逻辑。

### A2 实现明细（保存为连接 → 写 `[[profiles]]`）

**目标**:把「最近」里反复用的主机**命名**留存为永久连接(写 `config.toml` 的 `[[profiles]]`),它比 A1 的 ⭐收藏更进一层——有人类可读的**名字**、属用户自己的配置文件(可手编、可携带、`清空最近`也不丢)。

- **A1 vs A2 的分工**:A1 收藏 = 把一条最近**置顶**(轻量、仅连接器内、存 json);A2 保存 = 升级为**命名 profile**(写 `config.toml`、永久、可手编)。两者并存:连接器列表 = **「已保存」段(命名,来自 `config.toml`)在上 + 「最近」段(自动,来自 json)在下**,同端点去重(最近被对应的已保存折叠)。

### B1 进度卡 + C1 可操作错误卡 实现明细（连接生命周期可视化）

**目标**:把 SSH 的「连接过程」与「出错」从终端里一串 `[SSH] 正在连接…` 的 ANSI 文字,升级成 Calm Glass 覆盖卡片(对照 mockup ②⑥)。连接器(A1–A3)管「连之前」,这两张卡管「连接中 / 失败」。

- **后端结构化事件**(`crates/tn-pty`):`ssh.rs` 原本只往终端写 ANSI 文字。新增
  - `PtyEvent::SshProgress { phase: SshPhase, detail }` —— `SshPhase = Connecting | Authenticating | OpeningShell`(`ordinal()` 定步序)。在 `run_session` 拨号前发 `Connecting`、`authenticate` 探测后发 `Authenticating`、`request_shell` 前发 `OpeningShell`;`Connected` 收尾。**诚实 3 步**:TCP 握手 + 主机指纹校验是 russh 一次原子 `connect()`,折进「建立连接」,不伪造 mockup 的 5 步。
  - `PtyEvent::SshFailed { kind: SshErrorKind, detail, offered }` —— `kind = Auth | HostKeyMismatch`。`authenticate` 全部方式失败时发 `Auth`(`offered` = 服务器开放方式串,`detail` = 服务端配置提示如 PermitRootLogin);主机密钥被拒时发 `HostKeyMismatch`。
  - 小工具 `emit_event(event_tx, waker, ev)` = 发事件 + 唤前台重绘(同 `Connected`/`NeedPassword` 范式)。ANSI 文字保留(卡片盖在终端上,文字在后面无妨)。
- **每窗格覆盖卡**(`terminal_view/mod.rs`,沿用密码框范式):新状态 `ssh_target`(`user@host:port`)、`ssh_progress: Option<(SshPhase,detail)>`、`ssh_error: Option<SshErrorInfo>`。`handle_pty_event`:`SshProgress`→置进度卡 + 清错误;`SshFailed`→清进度 + 置错误卡;`Connected`→清两者。渲染优先级 **密码框 > 错误卡 > 进度卡**(scrim + Calm Glass 面板,`card_chrome`/`card_header` 复用闭包)。
  - **进度卡(B1)**:3 步 done=✓绿 / active=accent 实心点 / wait=灰点(**无帧时钟 → active 用点不用转圈,诚实**)+ 取消按钮。
  - **错误卡(C1)**:alert 红头 + 原因(认证失败时附「服务器开放的方式:…」)+ **黄色上下文提示框**(后端 `detail` 的 PermitRootLogin 提示,或通用)+ 按钮:认证失败 = [重试][关闭],主机密钥变更 = [关闭]。键盘:错误卡 `↵`=重试 `Esc`=关闭;进度卡 `Esc`=取消(`on_key` 前置拦截,不漏给终端)。
- **重试/关闭动作**(`workspace.rs`):terminal_view 发 `SshRetryRequested` / `SshCloseRequested` 事件给 workspace。
  - 把 `spawn_pane_with` 的「建视图 + 接所有订阅 + 入表」抽成 `install_pane(id, spec, cx)`;**重试** = 用同一 `id` 再次 `install_pane`(`panes.insert` 替换旧视图 → 旧 backend 经 `Drop` 被杀 → 新视图用同 spec 重连;tab 树引用 id 不变,位置不动)。
  - **取消/关闭** = `close_pane_id(id, cx)`:在所属 tab 里 `prune` 掉该叶子(镜像 `close_pane`,但按任意 id、且不碰 `Window`——事件回调无 window,焦点交给 render 的焦点收敛);整窗仅剩它时保留不动。

### B3 密码框增强 实现明细（显隐 + 记住 + 原地重试）

**目标**:把基础密码框补成 mockup ④——显隐切换、记住密码(仅本会话)、**输错原地重试不杀会话**。

### B2 主机指纹确认(TOFU)+ B4 断线重连/连接态 实现明细

**B2 目标**:首连未知主机不再「静默自动接受」,而是弹**信任面板**(显示 SHA256 指纹,确认才接受 + 写 known_hosts);指纹不匹配 = 危险中止。

- **后端**(`ssh.rs::check_server_key`):原本未知主机(无 known_hosts / 不在表)**自动接受 + 写盘**。改为:`known` 直接过、`Mismatch` 发 `SshFailed{HostKeyMismatch, detail=新指纹}` + 中止、**`Unknown` 调 `confirm_host_key` 发 `PtyEvent::NeedHostKeyConfirm{host, fingerprint, reply}` 阻塞等 `HostKeyVerdict`**(`AcceptAndSave`=写 known_hosts+接受 / `AcceptOnce`=仅本次 / `Reject`=中止)。`ClientHandler` 加 `event_tx`/`waker`;指纹 = `key.fingerprint(Sha256)`。
- **UI**(`terminal_view`):`ssh_hostkey: Option<SshHostKeyPrompt>` + `ssh_hostkey_remember`。`NeedHostKeyConfirm` → 信任卡(shield 头 + 指纹框 + 「记住写 known_hosts」勾选 + [信任并连接][取消]);键盘 `↵`=信任 `Esc`=拒。优先级**最高**(指纹 > 密码 > 错误 > 进度)。`trust_host_key`/`reject_host_key` 回 `HostKeyVerdict`。错误卡的 `HostKeyMismatch` 分支显「服务器本次指纹」。
- **未做**:mockup ③b 的「仍要连接并更新」(删 known_hosts 旧条目 + 接受)——安全默认中止,留作打磨。

**B4 目标**:断线时给非模态横幅 + 头部连接态四相点,不再无声。

- **状态**(`terminal_view`):`ssh_conn: Option<SshConnState>`(`None`=非 SSH 窗格;`Connecting/Connected/Reconnecting/Disconnected`)。由事件驱动:首个 `SshProgress`=Connecting、之后的 `SshProgress`=Reconnecting、`Connected`=Connected、`Disconnected`=Disconnected(后端随后自动 5s 重连 → `SshProgress` → Reconnecting)。
- **UI**:shell 头(`.phead`,SSH 窗格也是普通 shell)右侧加**四相点**(连接中=accent / 已连=绿 / 重连=黄 / 断开=红,色取 `palette.ansi`)+ 文案;断线/重连时窗格**顶部非模态横幅**(黄,「连接已断开,自动重连中…」+ [取消]=`SshCloseRequested` 关窗格)。重连走 slim 横幅——**进度大卡 gate 掉 `Reconnecting`**,避免重连时弹大卡。
- **未做**:倒计时 + 「立即重连」(后端自动 5s 重连、无 reconnect-now 通道)· tab 上的小连接态点(需 workspace 知 pane 连接态)。
- **图标**:`assets.rs` 补 `shield`(TOFU)。
- **验证**:`cargo test -p tn-pty --lib` 9 / `-p tn-ui --lib` 69(四守卫含,连接态色取 `palette.ansi` 非硬编码)· build 仅余既有 `SplitDir` warning · **`TN_AUTOQUIT` exit 0**(全链路起窗)。**真机肉眼**(首连信任卡 / 断线横幅 / 四相点)待 owner。

### A4 ssh-config 枚举 + C2 拨号前校验 + C3 首连引导/成功反馈 实现明细（A/B/C 收官）

**A4 目标**:把 `~/.ssh/config` 里已有的 `Host` 别名直接做成连接器可选项——OpenSSH 早知道的主机不该让用户在 Tn 里重敲。

- **后端**(`ssh.rs`):`pub fn list_ssh_config_hosts() -> Vec<SshHostEntry>` 读 `~/.ssh/config`,先 `parse_host_aliases`(**纯函数**:逐行扫 `Host` 指令、大小写不敏感、跳通配 `*`/`?` 与否定 `!`、一行多别名、按文件序去重),再用 ssh2_config `query(alias)` 解析每别名的 `HostName`/`User`/`Port`(回退别名本身 / 默认 22)。`SshHostEntry { alias, host, user, port }` 导出。**2 个 headless 单测**(跳通配+去重 / 无 Host 行为空)。
- **连接器**(`workspace.rs`):`ssh_config_hosts` 字段缓存(`new` + 每次开连接器刷新,文件小、会话内基本静态);`ssh_conn_rows` 加**第三段** `SshConnRow::Config{alias,target}`,在 已保存 + 最近 之后、**对同端点去重**(统一 `saved_eps` 集),受输入过滤(匹配 alias/host/user/target)。渲染:**空心点**标识(区别已保存的实心 accent 点)+ `ssh-config` 徽标。**点别名经别名连接**(`ssh_connect(alias)` → `SshConfig::parse(alias)` 重查 config,应用 IdentityFile/User/Port),而非已解析的 host(否则丢别名级密钥)。

**C2 目标**:拨号前把明显连不上的目标当场标红,省一次失败往返。

- **`validate_ssh_target(typed) -> Result<(), &str>`**(纯函数):空输入 = Ok(占位态不报错);`@` 前为空 / host 为空 / 悬空 `:` / 端口越界(应 1–65535)→ Err(中文原因)。**非数字 `:后缀`不报错**——它当主机名一部分,与 `SshConfig::parse` 一致(避免把 IPv6-ish/笔误误判)。**4 个分支单测**(合法集 / 非法集)。
- **UI**:`render_ssh_prompt` 算 `validate_ssh_target`,非法时输入框尾**红色错误 chip**(alert 图标 + 原因)顶替解析 chips;`on_ssh_prompt_key` 的 Enter 分支对**所输入**目标(无选中行时)校验失败即拦截不拨号(已选行恒合法、不校验)。

**C3 目标**:首次用 Tn 连 SSH 的人有引导;连上后明确告知用了哪种认证。

- **首连引导**:连接器空态(无 已保存/最近/config)且输入空时,显**三步上手卡**(1 输入地址 → 2 自动认证[密钥优先、否则密码框] → 3 记住连接[最近 + ⊕ 命名]),每步 accent 图标 + 标题 + 副说明。非空但无匹配仍显原「Enter 连接所输入地址」提示。
- **成功反馈**:`TerminalView` 加 `ssh_conn_method: Option<AuthKind>`,`Connected{method}` 时置;header 的「已连接」标签据此带后缀(`· 密钥`/`· 密码`/`· 交互`)——诚实复用既有连接态点,不另起浮层/不伪造动画。
- **验证**:`cargo test -p tn-pty --lib` 11 / `-p tn-ui --lib` 71(四守卫含,新增 C2 2 测 + A4 后端 2 测)· build 仅余既有 `SplitDir` warning · **`TN_AUTOQUIT` exit 0**。**真机肉眼**(ssh-config 行 / 校验红 chip / 引导卡 / 已连接认证方式)待 owner。
