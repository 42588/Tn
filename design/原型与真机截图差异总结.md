## 0. 结论速览

1. **根因只有一个大头:真机加载的主题是旧的。** `C:\Users\Gua\AppData\Roaming\tn\themes\tn-dark.toml` 仍是磷光改造**之前**的 Tokyo Night 调色(`surface_1=#1F2335`、`surface_2=#24283B`、`palette_selected=#283457`、`accent=#7AA2F7`、`accent_alt=#BB9AF7`、`chrome_bg=#0B0D16`、`claude=#F0916D`、`codex=#73DACA`、终端前景 `#C0CAF5`)。仓库里的 `config/themes/tn-dark.toml` 已是正确的磷光版,但 `tn-config/src/lib.rs:90`(`write_if_absent`)只在首启写入、永不覆盖 → **磷光主题从未到达真机**。凡走 theme 的表面全是旧世界,凡走 `style.rs` 常量的部件全是新世界,一屏两个时代 —— 这正是「风格不统一、挺丑」观感的系统性来源。
2. **幽灵终端底部的"透明材质" = 透明窗失效后的白色 surface**(实测纯 `#FFFFFF`),残影磷光描边整个淹没在白底里;隐藏滑出时这块白 surface 随窗移动、整块暴露。详见 §6 专题 A。
3. **QuickLook 三宗罪**:无 scrim、浮层挂在焦点 pane 内而非整工作区、海拔体系崩(头 #283457 / File 正文 #10141F / Diff 正文 #1F2335 三个面三个值,应统一 L3/L4);编辑态块光标是旧 accent 蓝 #7AA2F7,踩了「磷光是唯一生命色」的红线。详见 §4.3。
4. **几何是好消息**:DPI 折算后,标题栏 42 / 状态栏 26 / 板头 34 / 浮层头 38 / 页脚 30 / 磁贴 92 / 接缝 2 / 命令面板 640×(44+36×n+30) 全部对上 SPEC(±1px)。**本轮差异 95% 是颜色与部件,不是尺寸。**
5. 三个用户追加问题的结论:白材质见 §6;SSH/WSL/添加 Agent 子页风格不统一见 §7;宠物规则落实核对见 §8(状态机和工作区交互其实已齐,缺口集中在**欢迎页 2× 宠物是死的静态贴图**与两个入口)。

**现象**(像素证据):两态窗底均有全宽圆角白条,launcher `y=427..447`、运行态 `y=658..681` 实测 `#FFFFFF`/`#F5FDFB`,中间 2px 暗缝把它分成两条 —— 形状正好是**两道残影的轮廓间隙**;隐藏(上滑)时它随窗体移动,在桌面上整块可见。

**代码链**:
- `quick_terminal.rs:899` `ghost_frame`:卡片底部预留 `ECHO_PAD=16px` 透明区,画两道 `echo(10,8,·14)` / `echo(20,15,·6)` —— 1px 磷光描边、无填充,**意图完全符合原型**;
- `quick_terminal.rs:762`:幽灵窗不走 Win32 圆角 region,依赖窗体透明;
- `lib.rs:344`:窗口以 `WindowKind::PopUp + WindowBackgroundAppearance::Transparent` 创建。

**判定**:在该真机上 GPUI 的 Transparent 背景没有生效,窗 surface 以**白色**清屏。于是:卡片画过的地方正常(L1 不透明),没画的 16px 残影区直接露出白 surface;两道 1px 磷光描边画在白底上完全不可见,只剩描边之间的缝隙透出暗色,形成「白色材质」。隐藏动画 = 整窗 set_bounds 上移(`slide`),白 surface 跟着滑过桌面 → 「全暴露」。

**修复方向(P0-2)**:
1. 治本:排查 gpui Windows 后端 PopUp 透明窗支持(DirectComposition / `WS_EX_NOREDIRECTIONBITMAP`);确认可用后残影即自然显形。
2. 兜底(若短期不可达):残影不依赖窗体透明 —— 取消 `ECHO_PAD`,窗高=卡高,残影改画进卡片内部底缘(L1 底上画 1px ph-dim 圆弧),或残影暂撤、保留顶缘召唤线作为唯一签名。
3. 无论取哪条,隐藏动画期间窗内不得存在未绘制区。
