# 原型与真机截图差异总结

## 定位

本文对比 `design/panels/*.html` 原型图纸与 `design/真机截图/` 下真机复刻截图,用于定位当前实现和「磷光 Phosphor」原型之间的视觉与结构差异。本文只记录差异和修正优先级,不替代 [design/README.md](README.md) 与 [docs/设计索引.md](../docs/设计索引.md) 的权威设计入口。

## 总体结论

当前真机已经接近「磷光」的基础色系:暗色 L0/L1 底盘、磷光强调色、Tab 身份棒、底部状态栏和像素宠物都已出现。差异主要不在单个颜色值,而在原型的结构语法没有完整复刻:多板面仪表舱、浮层海拔、幽灵终端残影、Agent 专属板面、QuickLook 纯浮层和启动器磁贴语法在真机里被简化为更平铺、更全屏、更终端化的界面。

如果继续以原型为权威,优先修正「布局结构、浮层层级、特色组件签名」,再微调颜色和字体。单纯调深浅、描边或字体不会解决主要偏差。

| 浮层语义 | 浮层使用 L3 面、h2 边、投影、纯色 scrim,与常驻板面明显分层 | QuickLook、SSH 连接器、幽灵终端有浮起感,但尺寸和 scrim/投影节奏不一致 | 浮层家族不像同一套系统 | P0 |

## 页面级差异

### 欢迎页 / Launchpad

- 原型欢迎页是无 Explorer 的整窗 Launchpad,真机欢迎页左侧常驻 Explorer,导致主视觉从「启动台」变成「文件浏览器旁的空页」。如果 Explorer 常驻是产品决策,需要回写原型;否则应在欢迎页隐藏或弱化 Explorer。
- 原型磁贴为一行 `pwsh / Ubuntu / SSH 连接 / Claude / Codex / 添加 Agent`,真机拆成上方 Codex + 添加 Agent、下方 PowerShell / WSL / SSH,入口层级与图纸不一致。
- 原型中选中 tile 使用左侧 2px 磷光脊和 L4 面;真机卡片更接近普通描边卡,选中态的语义不够统一。
- 原型欢迎页宠物为 2x 大形态并与 Launchpad 同屏构图;真机使用 Westie 且位置更靠角落,视觉更像状态角标而不是欢迎页陪伴角色。
- 真机状态栏显示已有 session 和工作分支,原型欢迎页为 `NO SESSION`。如果真机欢迎页允许会话已存在,需要定义「欢迎 tab」和「空应用启动」两种状态。

### Agent 页面

- 原型 Agent 是专门板面:头部有 Agent 身份、模型、用量环、费用/上下文读数,正文是工具流和消息区,右侧有本次改动 rail。真机是 Codex CLI 原始终端内容,缺少 Agent 专属 header、activity feed、usage ring 和 diff rail。
- 原型工作区是 Explorer + Agent + Shell 三栏;真机为 Explorer + 单个全宽 Agent 终端,缺少右侧 Shell 或活动栏,导致页面密度和任务可审计性下降。
- 真机 tab 身份色和 `codex` 名称基本正确,但页面内部没有延续 Codex 身份色,只剩终端文字。
- 原型要求运行语义只用磷光贯穿 update 状态点、RUN 块、prompt、状态栏;真机当前没有工具流状态点和命令块,运行语义只体现在终端光标/选中行。

### Shell 页面

- 原型 Shell 位于多板面工作区中,并展示命令块三态、运行块条和滚动条。真机 Shell 是单个主 pane,当前空命令状态下没有 blockbar/命令块可见。
- 真机 Shell 的 header、cwd、profile chip 和底部状态栏已接近图纸语法,但主 pane 过宽,空白区比例过高。
- 真机焦点边框贯穿整个主 pane,比原型四角角标更重。建议降低长边强调,让四角角标成为主要焦点信号。

### QuickLook

QuickLook 是当前差异最明显的一组。原型不是普通文件编辑面板,而是「工作区之上的临时速览浮层」:固定 `1080x520`,居中,背后有纯色 scrim,并且保留 File / Edit / Diff / CJK 单元格四套状态语法。真机已经做出了文件预览壳,但整体行为和视觉更像一个嵌入式全尺寸编辑器。

| 维度 | 原型目标 | 真机表现 | 差异判断 |
|---|---|---|---|
| 尺寸比例 | File 预览态 `1080x520`,比例约 `2.08:1`,横向速览感强 | `核心页面.png` 为 `1619x1266`,比例约 `1.28:1`,高度接近原型 2.4 倍 | 由速览浮层变成高耸编辑区 |
| 所在层级 | 浮层居中覆盖在 stage 上,底层工作区被 `rgba(5,7,12,.62)` scrim 压暗但仍可辨 | `quick整体.png` 中底层终端和 Explorer 仍然明亮,QuickLook 像压在主 pane 内部 | modal 海拔不足,不像系统级浮层 |
| 外边距 | 浮层上下左右都有明显舞台留白,视觉焦点在中央 | 真机 QuickLook 几乎填满主 pane 的纵向空间,下方直接贴近终端 blockbar/status 区 | 缺少“临时查看”的呼吸空间 |
| 投影与边界 | L3 面 + h2 边 + float 投影,边界从 scrim 中分离 | 真机边界主要靠深色框和局部磷光竖线,投影不明显 | 海拔从“浮起”变成“嵌入” |
| Header 高度 | 38px,紧凑仪表头 | 真机 header 明显更高,接近 56px,且顶部蓝色区域更厚 | 头部比例过重 |
| Header 信息 | 左侧磷光点 + 路径屑 `crates › tn-ui › src › element.rs`,右侧 `RS / UTF-8 · LF / 218 L / File Diff / close` | 真机主要是文件图标 + `rust-toolchain.toml` + `File/Diff`,缺少路径屑、类型、编码、行数和关闭入口 | 速览上下文信息缺失 |
| Header 色阶 | 原型 header 是 L4,但不抢正文;正文 code 区是 L3 | 真机 header 更偏蓝、更饱和,正文区接近终端深底 | Header 与正文色阶跨度过大,且不完全遵守 L3/L4 关系 |
| 正文区域 | `font:13px/1.75 mono`,行号槽 52px,内容区只承担预览,空白受 520px 高度控制 | 真机字体和行距更大,正文区高度巨大,5 行文件内容下方留下整屏空白 | 信息密度失衡,更像打开了编辑器 |
| Footer 高度 | 30px,低调快捷键条 | 真机 footer 更高、更像固定工具栏 | 底部工具条喧宾夺主 |
| Footer 语义 | `↑↓ 改动文件 / E 编辑 / ⇥ Diff / Esc 关闭 / RAIL · 2/3` | 真机为 `换文件 / 切 File / Enter 编辑 / Diff 只读审阅 / Esc 关闭`,缺少 RAIL 位置和从活动栏进入的语义 | 与 Agent activity rail / diff 文件流没有接上 |
| 编辑态 | 原型有 `EDIT` chip、未保存 warn chip、当前行、选区、块光标、外部修改警告和保存/强制保存动作 | 真机截图只展示 File 预览态,没有看到编辑态视觉语法 | 编辑能力可能存在,但状态视觉未按图纸验收 |
| Diff 态 | 原型有 hunk 行、add/del soft 行底、`+12/-4` chip、接受/拒绝按钮和远端 apply 提示 | 真机截图只看到 File/Diff 分段按钮,没有 hunk 级样式证据 | Diff 不应只是一枚切换按钮,需要完整状态页 |
| CJK 规范 | 原型明确固定单元格、宽字双格、块光标反相重绘、选区逐格平铺 | 真机截图没有 CJK 样例,无法验证中文漂移问题是否按图纸解决 | 需要单独补含中文的 QuickLook 回归截图 |

更具体地说,真机 QuickLook 当前有三层偏移:

- 第一层是 **容器定位偏移**:原型是整窗 modal,真机更像依附在主终端 pane 上的 child surface。它没有把 Explorer、终端和状态区一起压暗,所以用户会误认为这是当前 pane 的嵌入内容,不是临时速览。
- 第二层是 **尺寸策略偏移**:原型的 `1080x520` 是刻意压扁的速览比例,只让用户快速看、切、进 Diff 或编辑;真机 `1619x1266` 把它做成近全屏编辑器,导致空白面积和操作重量都变大。
- 第三层是 **状态机语法缺失**:图纸把 QuickLook 分成 File 预览、编辑态、Diff 态和 CJK 单元格规范;真机截图目前只能证明 File 预览壳存在,不能证明编辑态、Diff hunk 操作和中文单元格验收已经复刻。

QuickLook 的修正建议应单独列为 P0:

1. 先把外层恢复成真正浮层:整窗 scrim、居中、可见投影、固定最大宽高,不要由主 pane 高度决定。
2. 把尺寸从当前接近 `1619x1266` 收回到接近 `1080x520`;在大屏上可以允许轻微放大,但应维持横向速览比例,不要超过主工作区高度的约 65%。
3. Header 回到 38px 级别,补回路径屑、语言/编码/行数 chip、关闭入口;File/Diff 分段保留但不要成为唯一上下文信息。
4. Footer 回到 30px 级别,恢复 `改动文件 / 编辑 / Diff / Esc / RAIL` 语义;如果真实快捷键改为 Enter,也要保留“RAIL 第几项”的状态读数。
5. 分别补三张验收截图:File 预览、Edit 未保存/冲突、Diff hunk 操作;再补一张含中文宽字符和选区/光标的 CJK 验收图。

### 幽灵终端

- 原型幽灵窗宽度为 `760px`,从屏顶垂下,上直角下圆角。真机启动器裁剪图约 `1120x428`,运行态几乎满屏宽,明显大于图纸规格。
- 原型启动器有 5 张 tile:`pwsh / Ubuntu / prod-01 / Claude / Codex`;真机启动器有 4 张:`codex / PowerShell / WSL / SSH`,缺少按 profile 展开的 Ubuntu/SSH 具体目标和 Claude。
- 原型专属签名是顶缘磷光 + 两道 1px 残影轮廓。真机有顶缘磷光,但残影很弱或缺失,运行态尤其不明显。
- 真机整体截图中弹窗外部为大面积白色背景,与「任意应用之上」的真实桌面/底层应用语义不一致。需要确认这是截图工具背景、透明窗口未捕获,还是 PopUp 外层真的绘制了白底。
- 运行态原型保留 `GHOST_` header、session alive footer 和 760px 窗体身份;真机运行态更像一条超宽终端窗口,幽灵终端的品牌感被稀释。

### SSH / WSL / 连接器

- SSH 连接器真机已经有 scrim、居中卡片、输入行、最近连接、密码/密钥标签和 footer 快捷键,方向是正确的。
- 原型选中行是单行 L4 + 左 2px 磷光脊;真机连接器中列表区域出现整块蓝色高亮,两条记录被包进同一色块,选择边界不够清楚。
- 原型 SSH 过程态包含三步进度、认证失败卡、断线横幅和 TOFU 指纹卡;真机 `ssh界面.png` 主要展示终端日志和右上角已连接 chip,过程态 UI 还没有按图纸展开。
- WSL 真机是欢迎页内二级卡片页,有返回/Ubuntu/AlmaLinux-9;原型未单独定义该页面。建议补一张 WSL 二级启动器图纸,或者把该页纳入 `06-overlays` 的启动器家族。

## 修正优先级建议

### P0: 先定结构权威

1. 决定欢迎页是否常驻 Explorer。如果常驻,回写原型;如果不常驻,实现应在欢迎 tab 隐藏 Explorer。
2. 决定 Agent 是「终端内 agent CLI」还是「专门 Agent 板面」。若以前者为准,需要重画 `02-workspace-panes`;若以后者为准,实现需补 Agent header、工具流和 activity rail。
3. 将 QuickLook 恢复为固定上限的浮层语法:居中、受 max width/height 约束、纯色 scrim、可见投影,避免覆盖成全屏面板。
4. 将 Ghost 运行态和启动器统一到 `760px` 顶垂窗规格,补足两道残影轮廓和 `GHOST_` 身份头。

### P1: 统一组件语法

1. 抽一套共享 launch tile 规则,让欢迎页、幽灵终端、WSL 子页和 SSH 入口共用尺寸、左脊、图标和副标题层级。
2. 调整焦点边框:长边只做 dim 边,四角角标承担主要焦点信号。
3. 补齐 Shell 命令块、运行块条和状态脊在真实命令执行后的可见状态。
4. 将 SSH 连接器列表从整块蓝色高亮改为逐行选中态,并补 TOFU / 认证失败 / 断线重连卡片。

### P2: 最后做精修

1. 对齐宠物欢迎页品种、2x 尺寸、label 文案和 shelf 位置。
2. 对齐 footer 快捷键文案,避免同一能力在不同页面使用不同命名。
3. 复查微标签、chip、kbd 的字体大小和灰度,把真实数据密度拉回原型节奏。

## 验收建议

后续修正不建议一开始追逐像素级颜色。先按以下顺序做真机回归:

1. 结构:欢迎页是否隐藏 Explorer、Agent 是否多板面、QuickLook/Ghost 是否有正确尺寸上限。
2. 层级:浮层是否能从底层 pane 中一眼分离,焦点是否不抢内容。
3. 特色:Ghost 残影、QuickLook 纯浮层、Agent activity rail、统一 tile 是否全部出现。
4. 微调:颜色、字重、间距、footer 文案。
