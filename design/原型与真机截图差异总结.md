# 原型与真机截图差异汇总

## 定位

本文汇总 `design/panels/*.html` 原型图纸与 `design/真机截图/` 下真机复刻截图之间的差异,用于后续按像素级标准回归「磷光 Phosphor」设计语言。本文合并了第一版宏观差异总结和第二轮像素级复审清单,保留页面级结论、逐项尺寸差异、边线/层级/控件语法差异和修正优先级。

本文只记录差异和修正优先级,不替代 [design/README.md](README.md) 与 [docs/设计索引.md](../docs/设计索引.md) 的权威设计入口。最终权威仍以 `design/panels/*.html` 与 `design/phosphor.css` 为准。

## 总体结论

当前真机已经接近「磷光」的基础色系:暗色 L0/L1 底盘、磷光强调色、Tab 身份棒、底部状态栏和像素宠物都已出现。主要差异不在单个颜色值,而在原型的结构语法没有完整复刻:多板面仪表舱、浮层海拔、幽灵终端残影、Agent 专属板面、QuickLook 纯浮层和启动器磁贴语法在真机里被简化为更平铺、更全屏、更终端化的界面。

如果继续以原型为权威,优先修正「布局结构、浮层层级、特色组件签名」,再微调颜色和字体。单纯调深浅、描边或字体不会解决主要偏差。

## QuickLook

QuickLook 是当前差异最明显的一组。原型不是普通文件编辑面板,而是「工作区之上的临时速览浮层」:固定 `1080x520`,居中,背后有纯色 scrim,并且保留 File / Edit / Diff / CJK 单元格四套状态语法。真机已经做出了文件预览壳,但整体行为和视觉更像一个嵌入式全尺寸编辑器。

| 区域 / 维度 | 原型要求 | 真机偏差 | 修正要点 | 优先级 |
|---|---|---|---|---|
| 左侧边界 | 浮层左侧只有普通边界,没有磷光左脊 | 三张 QuickLook 真机图左侧都有贯穿高度的磷光竖线 | 删除 QuickLook 容器左脊;保留 header 小磷光点即可 | P0 |
| 所在层级 | 整窗 modal,底层工作区被 `rgba(5,7,12,.62)` scrim 压暗但仍可辨 | `quick整体.png` 中底层终端和 Explorer 仍然明亮,QuickLook 像压在主 pane 内部 | 以 window/stage 层居中绘制,不要挂在当前 terminal pane 内部 | P0 |
| 尺寸比例 | `1080x520`,比例约 `2.08:1`,横向速览感强 | `核心页面.png` 为 `1619x1266`,比例约 `1.28:1`,高度约为原型 2.4 倍 | 收回到横向速览比例,高度不应接近主 pane | P0 |
| 外边距 | 浮层上下左右都有明显舞台留白,视觉焦点在中央 | 真机 QuickLook 几乎填满主 pane 纵向空间,下方贴近终端 blockbar/status 区 | 保留 modal 呼吸空间,不要按 pane 可用高度撑开 | P0 |
| 投影与边界 | L3 面 + h2 边 + float 投影,边界从 scrim 中分离 | 真机边界主要靠深色框和局部磷光竖线,投影不明显 | 恢复浮层投影和普通边界,不要用左磷光线代替海拔 | P0 |
| Header 高度 | 38px,紧凑仪表头 | 真机 header 约 `55-60px`,顶部蓝色区域更厚 | 收回到 38px 级别,减少上下 padding | P0 |

| File/Diff 顺序 | 原型为 `File` 在左、`Diff` 在右,当前态高亮 | 真机显示为 `Diff` 在左、`File` 在右 | 顺序回到原型;如果产品决策改顺序,需同步改图纸 | P1 |
| Header 色阶 | header 为 L4,正文 code 区为 L3,层级相近 | 真机 header 更偏蓝、更饱和,正文区接近普通终端底色 | 使用 QuickLook 专属 `L3/L4` 色阶,不要复用 terminal pane 色阶 | P1 |
| 正文区域 | `font:13px/1.75 mono`,行号槽 `52px`,空白受 `520px` 高度约束 | 真机字体和行距更大,正文区高度巨大,5 行文件内容下方留下整屏空白 | 按浮层高度裁切,不要按 pane 可用高度撑开 | P0 |
| Footer 高度 | 30px,低调快捷键条 | 真机 footer 更高、更像固定工具栏 | 高度回到 30px 左右,避免底部工具条喧宾夺主 | P1 |
| Footer 语义 | `↑↓ 改动文件 / E 编辑 / ⇥ Diff / Esc 关闭 / RAIL · 2/3` | 真机为 `换文件 / 切 File / Enter 编辑 / Diff 只读审阅 / Esc 关闭`,缺少 RAIL 位置和从活动栏进入的语义 | 恢复 RAIL 来源和序号;若快捷键改为 Enter,也要保留 RAIL 状态读数 | P1 |
| 编辑态标识 | `EDIT` 绿色 chip + `未保存` warn chip,并显示当前行/列 | 真机用橙色“编辑中”文字,没有原型里的 `EDIT` chip 和 warn chip 组合 | 用同一 chip 系统表达编辑/未保存/冲突,不要另造文案样式 | P0 |
| 编辑态正文 | 当前行 L4、选区 `ph-soft 12%`、块光标反相,有冲突警告带 | 真机编辑态有正文和 footer,但未呈现原型要求的选区、冲突警告、块光标规范 | 补编辑验收状态图:选区、光标、未保存、外部修改冲突 | P0 |
| Diff 态 | hunk 行、add/del soft 行底、`+12/-4` chip、接受/拒绝按钮和远端 apply 提示 | 真机截图只证明 File/Diff 切换存在,未证明 hunk 操作状态 | 补 Diff 真机截图并实现 hunk 级 UI | P0 |
| CJK 规范 | 固定单元格、宽字双格、块光标反相重绘、选区逐格平铺 | 真机截图没有 CJK 样例,无法验证中文漂移是否按图纸解决 | 补含中文、选区、光标、横滚的 QuickLook 截图 | P1 |
| 宠物与 modal | QuickLook 图纸中没有宠物抢占 modal 区域 | `quick整体.png` 中宠物仍在右下明亮显示 | modal 打开时宠物应按图纸隐藏、压暗或另行定义层级 | P2 |

更具体地说,真机 QuickLook 当前有三层偏移:

- 第一层是 **容器定位偏移**:原型是整窗 modal,真机更像依附在主终端 pane 上的 child surface。它没有把 Explorer、终端和状态区一起压暗,所以用户会误认为这是当前 pane 的嵌入内容,不是临时速览。
- 第二层是 **尺寸策略偏移**:原型的 `1080x520` 是刻意压扁的速览比例,只让用户快速看、切、进 Diff 或编辑;真机 `1619x1266` 把它做成近全屏编辑器,导致空白面积和操作重量都变大。
- 第三层是 **状态机语法缺失**:图纸把 QuickLook 分成 File 预览、编辑态、Diff 态和 CJK 单元格规范;真机截图目前只能证明 File 预览壳和简单编辑壳存在,不能证明编辑态、Diff hunk 操作和中文单元格验收已经复刻。

QuickLook 的修正建议应单独列为 P0:

1. 先把外层恢复成真正浮层:整窗 scrim、居中、可见投影、固定最大宽高,不要由主 pane 高度决定。
2. 把尺寸从当前接近 `1619x1266` 收回到接近 `1080x520`;在大屏上可以允许轻微放大,但应维持横向速览比例,不要超过主工作区高度的约 65%。
3. 删除真机现有左侧连续磷光竖线;QuickLook 原型没有这一道左脊线。
4. Header 回到 38px 级别,补回路径屑、语言/编码/行数 chip、关闭入口;File/Diff 分段保留但不要成为唯一上下文信息。
5. Footer 回到 30px 级别,恢复 `改动文件 / 编辑 / Diff / Esc / RAIL` 语义。
6. 分别补三张验收截图:File 预览、Edit 未保存/冲突、Diff hunk 操作;再补一张含中文宽字符和选区/光标的 CJK 验收图。

幽灵终端不能只按“宽了一点”处理。当前真机启动器和运行态都已经偏离原型的核心签名:比例过宽、运行态铺成横条、外部白底破坏悬浮语义、残影签名缺失或过弱。修正顺序应为 `760px` 宽度上限、贴屏顶垂、白底排查、残影、`GHOST_` 身份头,最后才是 tile 文案和色值。

## 修正优先级建议

### P0: 先定结构权威

1. 决定欢迎页是否常驻 Explorer。如果常驻,回写原型;如果不常驻,实现应在欢迎 tab 隐藏 Explorer。
2. 决定 Agent 是「终端内 agent CLI」还是「专门 Agent 板面」。若以前者为准,需要重画 `02-workspace-panes`;若以后者为准,实现需补 Agent header、工具流和 activity rail。
3. 将 QuickLook 恢复为固定上限的浮层语法:居中、受 max width/height 约束、纯色 scrim、可见投影,删除真机现有左侧连续磷光竖线。
4. 将 Ghost 运行态和启动器统一到 `760px` 顶垂窗规格,补足两道残影轮廓和 `GHOST_` 身份头,排查外部白底。
5. 为 SSH 补齐过程态、TOFU、认证失败和断线重连卡片,不要只展示原始终端日志。
6. 为 WSL 二级启动器补图纸,或把它并入 `06-overlays` / `07-states` 的启动器家族。

### P1: 统一组件语法

1. 抽一套共享 launch tile 规则,让欢迎页、幽灵终端、WSL 子页和 SSH 入口共用尺寸、左脊、图标和副标题层级。
2. 调整焦点边框:长边只做 dim 边,四角角标承担主要焦点信号。
3. 补齐 Shell 命令块、运行块条和状态脊在真实命令执行后的可见状态。
4. 将 SSH 连接器列表从整块蓝色高亮改为逐行选中态。
5. 对齐 QuickLook header/footer 高度、File/Diff 顺序、RAIL 状态读数和 edit/diff chip 语法。

### P2: 最后做精修

1. 对齐宠物欢迎页品种、2x 尺寸、label 文案和 shelf 位置。
2. 对齐 footer 快捷键文案,避免同一能力在不同页面使用不同命名。
3. 复查微标签、chip、kbd 的字体大小和灰度,把真实数据密度拉回原型节奏。
4. 回归截图时关闭外部头像等非 Tn 悬浮物;若它属于 Tn UI,补设计图纸。

## 验收建议

后续修正不建议一开始追逐像素级颜色。先按以下顺序做真机回归:

1. 结构:欢迎页是否隐藏 Explorer、Agent 是否多板面、QuickLook/Ghost 是否有正确尺寸上限。
2. 层级:浮层是否能从底层 pane 中一眼分离,焦点是否不抢内容。
3. 特色:Ghost 残影、QuickLook 纯浮层、Agent activity rail、统一 tile 是否全部出现。
4. 状态:QuickLook Edit/Diff/CJK、SSH 过程态/TOFU/失败态、Shell 命令块三态是否都有截图。
5. 微调:颜色、字重、间距、footer 文案。

只有 P0 修完后再做颜色微调才有意义;当前最大差异是比例、层级和组件语法,不是单个色值。
