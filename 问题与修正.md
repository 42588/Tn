# 问题与修正

## 1. WSL 中手动敲 `claude` / `codex` 不切 agent 面板

### 现象

在 **pwsh** 终端里手动敲 `claude` 或 `codex`,会自动切到 agent 头/活动栏(诚实地识别"你真敲了这命令")。
但在 **WSL** 发行版终端(bash/zsh)里敲同样的命令,**不会**切到 agent 面板,头部仍是普通 shell。

### 根因(已定位)

切 agent 态的逻辑是 `TerminalView::sync_shell_agent`([crates/tn-ui/src/terminal_view/mod.rs:572](crates/tn-ui/src/terminal_view/mod.rs#L572)),
它在前台 repaint loop 里跑,做的事是:

1. 读 `BlockModel::current()` 的**当前运行命令**(`b.command`);
2. 取命令首个 token(程序名)过 `tn_ai::agent_kind_for_command`;
3. 命中 `claude`/`codex` → 置 `agent` + `agent_from_shell = true`,起用量轮询/活动栏监听/重标签;
4. 命令块结束(`current()` 不再 running)→ `clear_agent` 还原。

而 `BlockModel::current().command` 的数据来源是 **shell 集成 OSC 633;E**(命令行)。这条序列**只有 pwsh 集成脚本会发**:

- 脚本来自 `tn_shell::Integration::powershell()`([crates/tn-shell/src/integration.rs:38](crates/tn-shell/src/integration.rs#L38)),
  经 `-EncodedCommand` 注入,**仅当 `integrate_pwsh == true`** 时([terminal_view/mod.rs:361](crates/tn-ui/src/terminal_view/mod.rs#L361))。
- WSL 走 `LaunchSpec::launch_wsl`([crates/tn-ui/src/terminal_view/launch.rs:98](crates/tn-ui/src/terminal_view/launch.rs#L98)),
  明确 `integrate_pwsh: false`(注释:"a distro runs bash/zsh, not pwsh")。

**结论**:WSL 会话里没有任何 shell 集成信号 → `BlockModel` 永远没有 `command` → `sync_shell_agent` 的 `running_agent` 恒为 `None` → 永不切 agent。
这不是 bug,是**功能缺口**:bash/zsh 的 shell 集成从未实现。

> 注:同理,WSL 里也**没有 block 底栏**(命令/退出码/cwd/时长),因为那同样依赖 OSC 133/633。补齐 bash/zsh 集成会一并解决。

---

### 方案 A(推荐):为 bash/zsh 实现 shell 集成 + 注入 WSL

与项目既有取向一致(**诚实地认 shell 集成的运行命令,不走进程树轮询/会话新鲜度猜测**)。一次性补齐后,
`sync_shell_agent` **代码无需改动**——它只读 `BlockModel`,信号一通就自动工作;且顺带让 WSL 也有 block 底栏。

#### A.1 新增 bash/zsh 集成脚本(`tn-shell`)

在 `Integration` 上加 `bash()` / `zsh()`,产出与 `powershell()` 等价的 OSC 133/633 序列:

- **OSC 133;A**(prompt 开始)、**;B**(命令开始)、**;D;<code>**(上条退出码)→ 挂到 bash 的 `PROMPT_COMMAND`
  / zsh 的 `precmd` hook;
- **OSC 633;E;<cmdline>**(命令行,即 `sync_shell_agent` 要读的那条)+ **OSC 133;C**(输出开始)→ bash 的
  `trap '...' DEBUG`(在每条命令执行前拿到 `$BASH_COMMAND`)/ zsh 的 `preexec` hook;
- 转义规则照 pwsh 脚本:`\ ; CR LF` → `\x5c \x3b \x0d \x0a`(`tn-shell` 解析侧已会 un-escape);
- 复用同一 `nonce` 防伪。
- 纯函数 + headless 单测(同 `powershell_script_has_markers_and_nonce`,断言含 A/B/C/D/E marker + nonce)。

> 退出码:bash 在 `PROMPT_COMMAND` 里第一句必须 `__tn_code=$?`(读任何东西都会重置 `$?`),对应 pwsh 脚本里
> "capture FIRST" 的坑。

#### A.2 把脚本注入 WSL 会话(难点在这)

bash/zsh 不能像 pwsh 那样 `-EncodedCommand`。可选注入法(按侵入性从低到高):

1. **`--rcfile` / `--init-file`(推荐,VSCode/WT 同款)**:把集成脚本写到一个 WSL 能读到的路径
   (最简单:写 Windows 临时文件,WSL 经 `/mnt/c/...` 读;或写进发行版 `\\wsl.localhost\<distro>\tmp\`),
   启动改成
   `wsl.exe -d <distro> -- bash --rcfile <wsl路径>`。
   **关键**:该 rcfile **第一件事先 `source ~/.bashrc`**(否则覆盖用户的 prompt/别名),随后再追加我们的 hook。
   zsh 用 `ZDOTDIR` 指向我们的目录,内部 `source` 用户原 `.zshrc`。
2. **`bash -c 'source <script>; exec bash -i'`**:简单但 `exec bash -i` 会重读 `.bashrc`、丢掉我们的 hook(除非
   hook 写进会被 interactive 读到的地方)→ 不优。
3. **环境变量**:bash 交互式无干净 env hook(`BASH_ENV` 只对非交互生效)→ 不可行。

落地点:`LaunchSpec::launch_wsl` 需要知道发行版默认 shell(bash 还是 zsh)。可先**统一按 bash 处理**
(绝大多数发行版默认 bash;`--rcfile` 对 bash 适用),zsh 留待二期;或探测 `$SHELL`。
注入开关沿用 `TN_NO_SHELL_INTEGRATION`。

#### A.3 验证

- headless:`tn-cli -- wsl.exe -d Ubuntu -- bash --rcfile <script>` 跑通,确认流里出现 `]633;E`;`tn-shell` 单测覆盖脚本 marker。
- 真机:WSL 里敲 `claude` → 切 agent 头/活动栏;敲 `cd claude-proj` / `cat codex.md` **不**误触(首 token 判定);
  退出 claude → 命令块结束 → 还原普通 shell。

#### 影响面

- `tn-shell`:+`bash()`/`zsh()` + 单测。
- `tn-ui/terminal_view/launch.rs`:`launch_wsl` 改写注入参数;可能给 `LaunchSpec` 加"集成脚本路径/类型"字段。
- `tn-ui/terminal_view/mod.rs`:`sync_shell_agent` **不改**;注入分支(目前 `if launch.integrate_pwsh`)需扩展成"按 shell 类型选 pwsh / bash 注入"。
- 临时文件生命周期(写脚本 → 会话存活期保留 → drop 清理)。

### 方案 B(轻量、过渡):标题/alt-screen 启发式

不补 shell 集成,改用更弱的信号在 WSL pane 上识别 agent:

- claude/codex 启动后会设终端**标题**(OSC 0/2)且进 **alt-screen** 全屏 TUI。可在 WSL pane 上检测
  "进 alt-screen 且标题含 claude/codex" → 切 agent。
- **缺点**:① 标题内容依赖 agent 版本、不保证;② alt-screen 也被 vim/less 触发,需叠标题判定才够特异;
  ③ 不解决 block 底栏缺失;④ 与项目"不解析终端正文/不猜"的取向相左(标题哨兵虽有先例 `AGENT_EXIT_SENTINEL`,
  但那是我们自己注入的确定字符串,agent 标题是外部不可控的)。
- 仅在不想动注入链时作过渡,**不推荐**作为最终方案。

---

### 反模式(明确不做)

- **进程树轮询找 grandchild**:CLAUDE.md 已记坑,脆弱,弃。
- **会话日志新鲜度反推**:会把同目录的 dev agent 误标(codex 默认在 `~` 跑),弃。

---

### 建议

采方案 **A**:实现 bash 集成 + `--rcfile` 注入(zsh 二期),`sync_shell_agent` 零改动即生效,并补齐 WSL 的 block 底栏。
方案 B 仅在需要快速过渡时考虑。
